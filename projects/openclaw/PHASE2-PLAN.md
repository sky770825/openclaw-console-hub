# OpenClaw Phase 2 戰略發展計畫

## 1. 核心願景
在 Phase 1 建立的基礎架構上，Phase 2 旨在實現 OpenClaw 的「實質進化」。從單純的指令執行器，轉變為具備自我優化能力的「智慧大腦」，並透過多設備聯網接軌現實生活，最終建立完善的權限審核體系以確保系統安全。

---

## 2. 三大發展支柱 (排除商業變現)

### 支柱一：智慧大腦 (AI Evolution)
*目標：實現記憶的自動沉澱與 SOP 的閉環優化。*
1.  **記憶深化 (Memory Consolidation)**：建立從 `Session Context` 到 `Vector DB` 再到 `SOP/Knowledge` 的自動提取與歸檔流程。
2.  **反思機制 (Reflection Loop)**：開發「任務後反思」腳本，自動分析任務成功/失敗原因，並產出優化建議。
3.  **SOP 自動更新**：當 L2/L4 修改程式碼或流程後，自動檢測並更新 `sop-知識庫/` 中的對應文件。

### 支柱二：實境接軌 (Nodes & Voice)
*目標：讓 OpenClaw 走出電腦，進入生活與移動端。*
1.  **多設備聯網 (Multi-Node Coordination)**：優化 `nodes` 工具，實現跨設備（Mac/Linux/Windows）的協作與資源共享。
2.  **手機控制 (Mobile Gateway)**：建立手機端專用指令集與 PWA/Web 介面，支援行動端遠端觸發任務。
3.  **語音交互 (Voice First)**：整合 TTS 與音頻回傳機制，支援老蔡透過語音查詢狀態或下達緊急指令。

### 支柱三：權限進化 (Permissions)
*目標：建立「安全可控」的代理人權限體系。*
1.  **六級權限體系 (Tiered Access)**：明確 L1~L4 的檔案讀寫、指令執行與 API 調用權限。
2.  **審核工作流 (Review Workflow)**：完善 `openclaw_reviews` 機制，將 🔴 紅燈動作（如刪除、Git Push）標準化為待審核任務。
3.  **資源限額 (Quota Management)**：針對各 Agent 設定 Token 使用上限與執行時間限制。

## 4. 戰略路徑：第一步啟動建議

**啟動點：P2-T1「優化向量資料庫索引與自動提取機制」**

*   **原因**：記憶是智慧的基礎。目前 OpenClaw 的記憶提取較為零散，優化向量資料庫能立即提升小蔡對歷史任務的理解能力，並為後續的「SOP 自動更新」提供精確的參考數據。
*   **具體執行腳本建議**：開發一個 `scripts/memory-vacuum.py`，負責：
    1.  掃描 `memory/` 目錄。
    2.  將高頻使用的 SOP 或知識點重新向量化。
    3.  刪除過時或重複的 Context 片段。
