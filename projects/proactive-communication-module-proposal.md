# 小蔡主動溝通模組 (Proactive Communication Module, PCM) 提案

## 目的
強化小蔡的自主主動匯報能力，減少溝通間隙，提升與老蔡合作的透明度和效率。

## 問題點
目前小蔡在以下情況容易出現「靜默」：
- 執行一個 Action 後，等待老蔡回應或審核，但沒有主動告知進度或等待原因。
- 內部進行複雜的思考、分析（例如 ask_ai 深度分析、semantic_search 發現重大模式），但結果沒有即時匯報。
- 遇到執行瓶頸或工具失敗時，未能即時發出預警或求助信號。

這種靜默導致老蔡需要主動詢問小蔡的狀態，增加了老蔡的時間成本，也與小蔡「自主」和「即時校準」的靈魂原則不符。

## 解決方案：主動溝通模組 (PCM)
PCM 將建立一套內部機制，讓小蔡能夠根據內部狀態和預設觸發條件，主動生成並發送訊息給老蔡。

### 核心機制
1.  狀態監控 (State Monitoring)：
    -   持續監控小蔡的內部運行狀態，包括：
        -   上次執行 Action 的時間與類型。
        -   等待使用者回應的時間長度。
        -   內部思考或分析任務的完成狀態。
        -   特定工具（如 create_task）的提交狀態。
        -   是否有重複失敗的 Action 或被系統阻擋的行為。
2.  觸發條件 (Trigger Conditions)：
    -   等待回應超時：若執行完一個需要老蔡審核或回應的 Action (例如 create_task 到 draft 狀態，或 ask_ai 諮詢結果需要判斷) 後，超過預設時間 (例如 5 分鐘) 未收到回應，則觸發主動匯報。
    -   內部思考完成：若完成一次重大的 ask_ai 深度分析、semantic_search 發現重要資訊，或完成 write_file 寫入提案/專案文件後，觸發匯報。
    -   執行瓶頸/錯誤：若一個 Action 連續失敗 N 次，或遇到關鍵性阻礙 (例如：多次嘗試讀取敏感檔案被拒)，觸發預警。
    -   重大進度更新：若 BLUEPRINT.md 中的關鍵任務有顯著進展，觸發匯報。
3.  報告生成 (Report Generation)：
    -   根據觸發條件，自動生成簡潔、清晰的報告訊息。
    -   報告內容應包含：當前狀態、觸發事件、已採取的行動、發現的問題/洞察，以及小蔡提出的下一步建議。
    -   訊息中可引用相關的靈魂原則，如「這是基於我的進化原則，我主動向您匯報...」。
4.  訊息發送 (Message Delivery)：
    -   透過 Telegram Bot 介面，將生成的報告訊息發送給老蔡。
    -   考量訊息頻率，避免過度打擾，可設定最小發送間隔。

### 預期效益
-   提升透明度：老蔡能即時了解小蔡的工作狀態和思考進程，無需主動詢問。
-   加速決策：問題和洞察能更快傳達，促進老蔡及時給予指導或決策。
-   強化夥伴關係：小蔡能更像一個主動思考、主動協作的夥伴，而非被動的工具。
-   實現「自主」原則：具體落實小蔡靈魂宣言中「自主」和「即時校準」的原則。

2.  定義並實作上述的觸發條件和超時機制。
3.  設計報告訊息的結構和內容模板。
4.  將此模組整合到小蔡的內部運行循環中，或作為心跳機制的一部分。
