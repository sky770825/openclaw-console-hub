# 每日功過格 — 善惡值與會員系統延伸規格

> 目標：會員擁有「善惡值」，累積福報可趨向天堂，扣分反映惡業並對應地獄；分數決定歸宿（六道／地獄層級）。本文件為延伸功能規格與實作階段建議。
>
> **實作狀態（2026-02-28）**：第一階段已完成 — `data/merit-items.json`、`data/realm-mapping.json` 已建立，`pages/merit.html` 已改為雙區塊（善行／過失）、淨值計算與歸宿預覽，狀態存於 `localStorage`（key: `merit_state_v2`）。功過項目已擴充（善行 12 項、過失 10 項）。全站串聯：首頁功過格入口改為「善惡值・歸宿預覽」；輪迴頁、地獄頁、神明頁均設 CTA 連至功過格；功過格歸宿區塊連結至神明、輪迴、地獄。

## 一、概念模型

### 1.1 善惡值（淨業分）

| 名稱 | 說明 |
|------|------|
| **福報（善分）** | 做好事、完成功過格善行項目所累積的正分。讓會員知道「持續行善可趨向天道」。 |
| **過失／惡業（惡分）** | 做不好的事、勾選「過」項目或違失行為所扣的分。 |
| **善惡值（淨值）** | `福報 - 過失`，或採「累積總分」制（每日 +善 -惡 加總後累加）。此值決定「歸宿預覽」。 |

可選兩種計分方式（擇一或並存）：

- **方案 A：單一淨值**  
  `善惡值 = 累積福報 - 累積過失`，可正可負。
- **方案 B：雙軸並存**  
  介面同時顯示「累積福報」與「累積過失」，歸宿由兩者綜合或淨值推算。

### 1.2 與「歸宿」的對應

- **善惡值越高** → 越傾向 **天道／人道**（可文案寫「有機會上天堂／再得人身」）。
- **善惡值越低或為負** → 越傾向 **三惡道**；極低時對應 **地獄道** 乃至 **十八層地獄某一層**。
- 需訂定「分數區間 → 六道／地獄層級」對照表（見下節），供前端顯示「依你目前善惡值，歸宿預覽為……」或「對應地獄第 X 層」。

## 二、功過格項目擴充

### 2.1 現狀

- 目前僅有 **善行** 項目（6 項，全為加分），無扣分機制。
- 資料存於 **localStorage**，無會員、無雲端。

### 2.2 建議結構

**善行（功）** — 勾選即加分，可沿用或擴充：

- 今日言談誠實、無誇大誹謗 (+10)
- 今日心存善念、無惡意詛咒 (+10)
- 今日手腳乾淨、不取非分之財 (+15)
- 今日推遲私慾、先成就他人 (+20)
- 今日恭敬尊長、愛護弱小 (+10)
- 今日自省過失、並決定改過 (+15)
- （可再擴：布施、持戒、忍辱、精進、禪定、智慧等，對應分數）

**過失（過）** — 勾選即扣分，建議項目範例（分數可調）：

- 今日有妄語、綺語、兩舌 (-10)
- 今日起嗔心、罵詈或詛咒 (-15)
- 今日取非分財、佔人便宜 (-20)
- 今日不敬尊長、損害弱小 (-10)
- 今日浪費糧食、暴殄天物 (-5)
- 今日明知故犯、未即改過 (-15)

項目與分數可收納在 **`data/merit-items.json`**（或同站大數據檔），方便日後依經典或營運需求調整，並與「因果／地獄」文案一致。

## 三、分數與歸宿對照（大數據規則）

### 3.1 六道對應（建議區間）

可依「淨值區間」對應六道，僅作預覽用，不涉及宗教定論：

| 善惡值區間（淨值） | 歸宿預覽 | 說明文案 |
|--------------------|----------|----------|
| ≥ 某高門檻（如 500） | 天道 | 福德深厚，有機會往生天道。 |
| 中高（如 200～499） | 人道／阿修羅道 | 善多於惡，多生人天。 |
| 微正～微負（如 -50～199） | 人道 | 善惡參半，得人身修行。 |
| 負分（如 -51～-200） | 餓鬼／畜生道 | 業重，當勤懺悔、止惡修善。 |
| 極負（如 &lt; -200） | 地獄道 | 重罪需受報，悔改仍可轉業。 |

實際門檻數字可依「累積天數 × 每日滿分」估算後再調，或改為「等級制」（1～10 級對應不同歸宿文案）。

### 3.2 地獄層級對應（可選）

若要做「分數對應到十八層地獄某一層」：

- 將 **極負** 區間再細分為 18 檔，每檔對應一層地獄（例如淨值 -201～-220 → 第 1 層，-221～-240 → 第 2 層……），或依「罪業類型」加權（另計）。
- 文案可寫：「以你目前善惡值，若不及早改過，業力可能感召 X 層地獄之報」，並連結到本站地獄頁對應層級說明。

對照表建議收納在 **`data/realm-mapping.json`** 或 **`docs/MERIT-SYSTEM-EXTENSION.md`** 附錄，方便後端／前端共用。

## 四、會員與資料持久化

### 4.1 會員需求

- **未登入**：可沿用目前 **localStorage**，僅本機「善惡值」與歷史，不跨裝置。
- **會員登入後**：善惡值、每日功過記錄存於 **雲端**，可跨裝置、可累積長期「福報／過失」並顯示歸宿預覽。

### 4.2 資料表建議（Supabase）

若接 workspace 既有 Supabase（見 `DATA-AND-RESOURCES.md`），可新增專用表，不與 openclaw_tasks 混用：

| 表名 | 用途 | 主要欄位建議 |
|------|------|----------------|
| `merit_users` 或沿用 auth.users | 會員 | id, email/phone, 顯示名稱, 建立時間 |
| `merit_scores` | 每人累積善惡值 | user_id, total_merit（福報）, total_demerit（過失）, net_score（淨值）, updated_at |
| `merit_logs` | 每日明細（選用） | user_id, date, 善行項目 id 與次數, 過失項目 id 與次數, 當日淨值 |

或簡化為單表 **`merit_accounts`**：  
`user_id, total_merit, total_demerit, net_score, last_date, updated_at`，每日前端計算後寫回總分。

### 4.3 認證方式

- 最簡：**Email + 密碼** 或 **LINE Login**（與本站 LINE 官方帳號整合，利於後續推廣）。
- 進階：手機號、Google 等，依營運選擇。

## 六、分階段實作建議

### Phase 1：前端善惡值與扣分（不需後端）

1. **功過格項目**：在 `merit.html` 或 `data/merit-items.json` 新增「過」項目，前端勾選即扣分。
2. **淨值計算**：`善惡值 = 累積福報 - 累積過失`（或改為「每日淨值」再累加，依產品選擇）。
3. **歸宿預覽**：依淨值區間顯示一句話（如「依目前善惡值，歸宿預覽：人道」），並可連結到輪迴頁／地獄頁。
4. **localStorage**：擴充 `merit_state` 結構，存「善行勾選 + 過失勾選 + 累積總福報/總過失/淨值」。

交付：使用者在本機就能看到「善惡值 + 扣分 + 歸宿預覽」，無需登入。

### Phase 2：數據與規則大數據化

1. 建立 **`data/merit-items.json`**：功、過項目與分數，方便日後擴充或依經典調整。
2. 建立 **`data/realm-mapping.json`**：分數區間 ↔ 六道／地獄層級與文案。
3. 功過格頁改為讀取上述 JSON，計算與對照邏輯統一由資料驅動。

交付：規則可維護、可對接維基或經典說明。

### Phase 3：會員與雲端

1. 設計 **Supabase 表**（如 `merit_scores`、`merit_logs`）與 API（或 Serverless 寫入）。
2. 實作 **登入／註冊**（Email 或 LINE），登入後功過格寫入/讀取雲端。
3. 未登入仍可用 localStorage；登入後可選擇「是否將本機歷史匯入雲端」（一次性或覆蓋）。

交付：會員可跨裝置累積善惡值，長期看到「可上天堂／對應地獄」的預覽。

### Phase 4（選做）：儀表板與進階

- 會員專屬頁：歷史曲線、每日明細、成就/鼓勵語、分享「我的善惡值」卡片等。
- 與 LINE 推播或 n8n 整合：每日提醒填寫功過格、週報「本週淨值變化」。
