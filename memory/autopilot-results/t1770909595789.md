#### 3. 優化方案

##### 方案 A: 智能調度

```javascript
// 根據 Context 使用率動態調度
const scheduleCheckpoint = async () => {
  const usage = await getContextUsage();
  
  if (usage > 0.7) {
    // 緊急檢查點
    await cron.update('checkpoint', { intervalMs: 60000 });  // 每分鐘
  } else if (usage > 0.5) {
    // 標準檢查點
    await cron.update('checkpoint', { intervalMs: 300000 }); // 每 5 分鐘
  } else {
    // 低優先級
    await cron.update('checkpoint', { intervalMs: 900000 }); // 每 15 分鐘
  }
};
```

**預期節省**: 80% 的低優先級任務

##### 方案 B: 合併相似任務

```javascript
// Before: 3 個分離的 cron job
cron.add('token-count', { expr: '0 0 * * *' });          // 00:00
cron.add('memory-backup', { expr: '0 1 * * *' });        // 01:00
cron.add('cache-cleanup', { expr: '0 2 * * *' });        // 02:00

// After: 1 個合併任務
cron.add('daily-maintenance', { 
  expr: '0 0 * * *',
  tasks: ['token-count', 'memory-backup', 'cache-cleanup']
});
```

**預期節省**: 減少 2 個任務啟動開銷

##### 方案 C: 批量非同步執行

```javascript
// 使用 BullMQ 隊列而非 cron
const queue = new Queue('daily-tasks');

// 每天 00:00 單次觸發，內部批量完成
cron.add('daily-batch-trigger', {
  expr: '0 0 * * *',
  payload: { 
    tasks: ['token', 'backup', 'cleanup', 'report']
  }
});

// 隊列內部並行執行（無額外開銷）
queue.process('batch', async (job) => {
  return Promise.all([
    tokenCount(),
    memoryBackup(),
    cacheCleanup(),
    generateReport()
  ]);
});
```
