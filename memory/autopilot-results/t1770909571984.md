## 1. 功能概述

通用 Webhook 處理 Skill，支援：
- 接收外部 Webhook 事件
- 驗證請求來源 (Signature/HMAC)
- 事件轉發與路由
- 重試機制與去重

## 2. 核心架構

### 2.1 Webhook 伺服器
```typescript
interface WebhookServer {
  port: number;
  endpoints: WebhookEndpoint[];
  middlewares: Middleware[];
}

interface WebhookEndpoint {
  path: string;
  method: 'POST' | 'GET' | 'PUT' | 'DELETE';
  handler: WebhookHandler;
  validation?: ValidationConfig;
}
```

### 2.2 驗證機制
```typescript
interface ValidationConfig {
  type: 'hmac' | 'signature' | 'token' | 'custom';
  secret: string;
  headerField?: string;
  algorithm?: 'sha256' | 'sha512';
}

class WebhookValidator {
  validateHMAC(payload: string, signature: string, secret: string): boolean
  validateSignature(payload: string, signature: string, publicKey: string): boolean
  validateToken(token: string, expectedToken: string): boolean
}
```

## 3. 事件處理流程

```
接收請求 → 驗證來源 → 解析 Payload → 事件去重 → 路由分發 → 執行處理 → 記錄日誌
```

### 3.1 事件去重
```typescript
interface DeduplicationConfig {
  enabled: boolean;
  keyField: string;      // 用於去重的欄位 (如: id, event_id)
  ttl: number;           // 快取時間 (秒)
  storage: 'memory' | 'redis';
}
```

### 3.2 重試機制
```typescript
interface RetryConfig {
  maxRetries: number;
  backoffStrategy: 'fixed' | 'exponential';
  backoffDelay: number;  // 毫秒
  retryCondition?: (error: Error) => boolean;
}
```

## 5. 路由與轉發

```typescript
interface WebhookRouter {
  // 基於事件類型路由
  route(eventType: string, handler: RouteHandler): void
  
  // 轉發到其他端點
  forward(targetUrl: string, transform?: PayloadTransform): Promise<void>
  
  // 條件路由
  when(condition: RouteCondition, handler: RouteHandler): void
}

interface RouteCondition {
  eventType?: string | string[];
  source?: string;
  payloadMatch?: object;
}
```
