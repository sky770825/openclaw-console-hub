### 性能瓶頸分析

#### 1. 已知問題

**渲染卡頓**
- 任務數 > 500 時，DOM 更新變慢
- 拖曳操作延遲 > 300ms
- 搜尋結果渲染時間 > 500ms

**記憶體洩漏**
- 長時間操作記憶體佔用持續增長
- 事件監聽器未清理

#### 2. 優化方案

##### 虛擬滾動 (Virtual Scrolling)
```typescript
// 前 (渲染所有 500+ 任務)
<div className="task-list">
  {tasks.map(task => <TaskCard key={task.id} {...task} />)}
</div>

// 後 (只渲染可見部分 30 個)
import { FixedSizeList as List } from 'react-window';

<List
  height={600}
  itemCount={tasks.length}
  itemSize={100}
  width="100%"
>
  {({ index, style }) => (
    <div style={style}>
      <TaskCard {...tasks[index]} />
    </div>
  )}
</List>
```

**預期收益**: 渲染時間 30 秒 → 300ms ⚡

##### React.memo + useMemo
```typescript
// TaskCard 避免不必要重新渲染
export const TaskCard = React.memo(({ task, onUpdate }) => {
  return <div>{task.name}</div>;
}, (prevProps, nextProps) => {
  return prevProps.task.id === nextProps.task.id;
});
```

**預期收益**: 重新渲染次數減少 60%

##### 分頁與無限滾動
```typescript
// 初始載入 50 個，滾到底部自動加載下 50 個
const [page, setPage] = useState(1);
const [hasMore, setHasMore] = useState(true);

const loadMore = useCallback(async () => {
  const newTasks = await fetch(`/api/tasks?page=${page + 1}`);
  setTasks([...tasks, ...newTasks]);

```

**預期收益**: 初始載入時間 80% 改善
