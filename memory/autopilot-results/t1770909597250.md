# [Ollama] 子代理並發測試

## 任務: 同時 3-5 個子代理穩定性測試

**執行時間**: 2026-02-13 00:11 UTC+8  
**分類**: 監控與測試

### 測試方案

#### 1. 並發負載測試配置

**測試環境**
- macOS M系列 (16GB+ RAM 推薦)
- Ollama 本地服務運行中
- 模型: `deepseek-r1:8b` / `phi-4`

**並發層級**
- Level 1: 3 個子代理同時請求
- Level 2: 5 個子代理同時請求  
- Level 3: 監控內存、CPU、GPU 使用率

#### 2. 測試指標

**穩定性指標**
- ✅ 所有請求完成率 (應達 100%)
- ✅ 平均回應時間增長率 (應 < 50%)
- ✅ 內存洩漏檢測 (長時間執行)
- ✅ 模型卸載/重載穩定性

**性能指標**
- 平均 TTFT (首令牌時間): 應 < 2秒
- 平均吞吐量: tokens/sec 穩定性
- P99 延遲: 應控制在 5 秒內

#### 3. 測試腳本框架

```bash
#!/bin/bash
# parallel-agent-test.sh

# 啟動 3 個並發代理測試
for i in {1..3}; do
  (
    echo "🔄 代理 $i 啟動..."
    time curl -s http://localhost:11434/api/generate \
      -d '{
        "model": "deepseek-r1:8b",
        "prompt": "解釋什麼是 OpenClaw AI Agent？",
        "stream": false
      }' | jq .
  ) &
done
wait

# 等待 5 秒，測試序列相鄰請求
sleep 5

# 啟動 5 個並發代理測試
for i in {1..5}; do
  (
    echo "🔄 代理 $i 啟動..."
    time curl -s http://localhost:11434/api/generate \
      -d '{
        "model": "phi-4",
        "prompt": "Write a brief summary of AI automation",
        "stream": false
      }' | jq .
  ) &
done
wait

echo "✅ 並發測試完成"
```

#### 4. 預期結果

**成功指標**
- [ ] 3 代理併發: 所有請求 < 10s 完成
- [ ] 5 代理併發: 所有請求 < 15s 完成  
- [ ] 內存峰值: 不超過系統 60%
- [ ] CPU 峰值: 不超過 80%

**風險識別**
- ❌ 超時 (某代理無回應)
- ❌ 記憶體溢出 (OOM)
- ❌ 模型鎖死 (無法切換)
- ❌ 連接池耗盡
