# ClawFlow 錯誤處理與重試技術規格

## 功能概述
指數退避、斷路器 (Circuit Breaker) 模式、自動重試。

## 錯誤分類

```typescript
enum ErrorType {
  NETWORK = 'network',           // 網絡錯誤 (可重試)
  TIMEOUT = 'timeout',           // 超時 (可重試)
  RATE_LIMITED = 'rate_limited', // 限流 (可重試)
  INVALID_INPUT = 'invalid',     // 輸入錯誤 (不重試)
  AUTH_FAILED = 'auth',          // 認證失敗 (不重試)
  UNKNOWN = 'unknown'            // 未知 (評估後決定)
}
```

## 重試策略

### 固定延遲
```typescript
const retry = {
  maxRetries: 3,
  delayMs: 1000
};
// 重試間隔: 1s, 1s, 1s
```

### 指數退避
```typescript
const retry = {
  maxRetries: 5,
  initialDelayMs: 100,
  backoffMultiplier: 2
};
// 重試間隔: 100ms, 200ms, 400ms, 800ms, 1600ms
```

### 添加隨機抖動
```typescript
const delayWithJitter = baseDelay * (1 + Math.random());
// 避免雷群效應 (thundering herd)
```

## 斷路器模式

```
CLOSED (正常) 
  ↓ (失敗次數 > 閾值)
OPEN (熔斷)
  ↓ (等待超時)
HALF_OPEN (測試)
  ↓ (若恢復正常 → CLOSED)
```

### 配置

```yaml
circuitBreaker:
  failureThreshold: 5      # 5 次失敗
  timeout: 30000           # 30 秒後進入 HALF_OPEN
  successThreshold: 2      # 2 次成功後回到 CLOSED
```
