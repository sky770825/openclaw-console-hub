# GitHub Skill Release 自動化技術規格

## 功能概述
自動化 GitHub Skill 的版本發佈流程，包括版本號遞增、自動生成 Changelog、建立 Release。

## 核心功能

### 版本管理
```typescript
// 自動遞增版本
$ clawflow release patch    // 1.0.0 → 1.0.1
$ clawflow release minor    // 1.0.0 → 1.1.0
$ clawflow release major    // 1.0.0 → 2.0.0

// 自動更新
- package.json 版本號
- manifest.json 版本號
- package-lock.json
- CHANGELOG.md (自動生成)
```

### Changelog 自動生成
```markdown
## [1.1.0] - 2026-02-13

### Added
- 新功能描述 (#123)
- 另一個新功能 (#124)

### Fixed
- 修復 Bug 描述 (#125)

### Changed
- 行為變更 (#126)

### Deprecated
- 棄用功能列表

### Removed
- 移除功能列表
```

### GitHub Release 發佈
```typescript
// 自動建立 Release
- Tag: v1.1.0
- Release 標題: Release 1.1.0
- Release 描述: 自動生成的 Changelog
- 資產: 打包的 .tar.gz / .zip 檔案
```

## 實作流程

```
git pull
  ↓
更新版本號 (package.json)
  ↓
執行測試 (npm test)
  ↓
生成 Changelog
  ↓
打包構建 (npm run build)
  ↓
建立 Git tag
  ↓
Push to GitHub
  ↓
建立 Release
  ↓
發佈到 npm registry
  ↓
通知 Slack/Telegram
```

## 配置

```json
{
  "release": {
    "branches": ["main"],
    "changelog": {
      "auto": true,
      "keepUnreleased": false
    },
    "github": {
      "createRelease": true,
      "uploadAssets": true
    },
    "npm": {
      "publish": true,
      "registry": "https://registry.npmjs.org"
    }
  }
}
```

## 技術棧
- **版本管理**: semantic-release / standard-version
- **Changelog**: conventional-changelog
- **Git**: simple-git
- **GitHub API**: @octokit/rest
- **構建**: npm scripts / webpack
