## 功能概述

**目標**：為 Cursor 開發者提供無縫的本地 Ollama 模型管理，自動化模型選擇和性能優化。

### 核心功能
1. **模型自動切換** — 根據任務複雜度自動選擇最優模型
2. **效能監控** — 實時追蹤延遲、記憶體、溫度
3. **智能決策** — 根據可用資源自動降級或升級
4. **成本優化** — 最小化 token 消耗

## API 設計

### 主要接口

```typescript
interface OllamaModelManager {
  // 1. 模型管理
  listModels(): Promise<Model[]>;
  switchModel(modelName: string): Promise<void>;
  pullModel(modelName: string, onProgress?: (percent) => void): Promise<void>;
  
  // 2. 性能監控
  getMetrics(): Promise<Metrics>;
  getSystemHealth(): Promise<SystemHealth>;
  
  // 3. 智能選擇
  recommendModel(task: TaskType): Promise<string>;
  autoSelect(context: TaskContext): Promise<string>;
  
  // 4. 推理優化
  inferWithFallback(prompt: string, options?: InferOptions): Promise<string>;
  parallelInfer(prompts: string[]): Promise<string[]>;
}

// 數據結構
interface Model {
  name: string;
  size: number;              // 單位 GB
  family: string;            // llama, qwen, deepseek etc.
  parameterCount: number;    // 7B, 13B, 70B etc.
  quantization: string;      // Q4, Q5, Q6
  context: number;           // Max context window
  pullTime?: Date;
  lastUsed?: Date;
}

interface Metrics {
  currentModel: string;
  latency: number;           // ms per token
  throughput: number;        // tokens per second
  memoryUsed: number;        // GB
  cpuUsage: number;          // %
  temperature?: number;      // 如果有溫度傳感器
  cached: number;            // 緩存大小
}

interface SystemHealth {
  totalMemory: number;       // GB
  availableMemory: number;
  canRunModel: (model: string) => boolean;
  suggestedMaxModel: string;
  warningLevel: 'healthy' | 'caution' | 'critical';
}

interface TaskType {
  type: 'simple' | 'moderate' | 'complex' | 'reasoning';
  expectedTokens?: number;
  priority?: 'speed' | 'quality' | 'balanced';
  maxLatency?: number;       // ms
}

interface TaskContext {
  taskType: TaskType;
  availableMemory: number;
  currentLoad: number;       // 0-1
  previousModel?: string;
}

interface InferOptions {
  temperature?: number;
  top_p?: number;
  top_k?: number;
  timeout?: number;
  retries?: number;
  fallbackModels?: string[]; // 備用模型列表
}
```

## 自動降級機制

```
記憶體不足時的降級策略：

需求 70B 模型，但只有 8GB 記憶體
  ↓
檢查 13B 模型（最接近的下級）
  ↓
如果能裝入 → 自動切換 + 警告
  ↓
如果還是不夠 → 改用 7B + 限制 context window
  ↓
如果全部失敗 → 拋出例外 + 建議用戶清理記憶體
```

## 使用示例

### 示例 1：自動模型選擇
```typescript
const manager = new OllamaModelManager();

const result = await manager.inferWithFallback(
  "設計一個微服務架構",
  {
    priority: 'quality',
    timeout: 5000,
    fallbackModels: ['qwen2.5:13b', 'llama3.2:13b']
  }
);
// 邏輯：
// 1. 嘗試推薦的最佳模型
// 2. 如果超時或出錯 → 自動切換到備用
// 3. 返回首個成功的結果
```

```typescript
const metrics = await manager.getMetrics();
console.log(`
```
