## 痛點

我在使用 Ollama 時發現：
- 有強大的本地模型，但缺乏自動化整合
- 想讓 LLM 定時處理任務（Email 分類、資料整理、報表生成）
- 現有 workflow 工具都不支援 Ollama

## 解決方案

ClawFlow — 原生支援 Ollama 的自動化框架：

```yaml
# workflow.yaml
name: daily-report
trigger:
  type: cron
  schedule: "0 9 * * *"

steps:
  - skill: email
    action: fetch
    filter: unread
    
  - skill: ollama
    model: deepseek-r1:8b
    prompt: |
      Summarize these emails and categorize by urgency:
      {{email.content}}
      
  - skill: notion
    action: create_page
    database: Daily Reports
    content: "{{ollama.output}}"
```

## Ollama 整合特色

🦙 **原生支援**
```typescript
// 自動偵測本機 Ollama
const ollama = new OllamaSkill({
  baseUrl: 'http://localhost:11434',
  defaultModel: 'llama3.2',
  timeout: 300000 // 5 min for reasoning models
});
```

⚡ **模型自動選擇**
```typescript
// 根據任務複雜度選擇模型
simpleTask → llama3.2:3b
complexAnalysis → deepseek-r1:14b
codingTask → qwen2.5-coder:7b
```

🔄 **串流輸出支援**
```typescript
// 即時看到 LLM 思考過程
ollama.stream({
  model: 'deepseek-r1:8b',
  prompt: 'Analyze this data...',
  onToken: (token) => console.log(token)
});
```

## 討論策略

### 預期問題

**Q: 為什麼不用 Python script 直接呼叫 Ollama API？**
> 可以，但 ClawFlow 提供：工作流視覺化、錯誤重試、任務佇列、多 Skill 整合。對於複雜流程，比純腳本更好維護。

**Q: 支援 GPU 加速嗎？**
> 完全支援！Ollama 會自動使用可用 GPU。我們也支援指定 CUDA 裝置。

**Q: 可以串接多個模型嗎？**
> 可以！workflow 裡可以不同步驟用不同模型，例如先用小模型過濾，再用大模型深入分析。

**Q: 有 WebUI 嗎？**
> 有基本的 workflow 編輯器，但主要還是 YAML 配置（開發者取向）。
