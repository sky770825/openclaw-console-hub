## 1. 產品概述

### 目標
為 ClawFlow 提供直覺的視覺化工作流設計介面，讓用戶透過拖曳方式建立 AI Agent 自動化流程，無需撰寫程式碼。

### 核心價值
- 🎯 **零程式碼**: 拖曳組件即可建立複雜流程
- ⚡ **即時預覽**: 設計同時看到執行邏輯
- 🔗 **無縫整合**: 與現有 OpenClaw Skill 生態完全相容

## 2. 功能規格

### 2.1 畫布 (Canvas)

| 功能 | 說明 |
|------|------|
| 無限畫布 | 支援縮放 (25%-200%)、平移、網格對齊 |
| 多選操作 | Shift+點選框選多個節點 |
| 快捷鍵 | Delete 刪除、Ctrl+Z 復原、Ctrl+Y 重做 |
| 縮圖導航 | 右下角迷你地圖，快速定位 |

### 2.2 節點類型

```
節點類型:
├── 觸發節點 (Trigger)
│   ├── 定時觸發 (Cron)
│   ├── Webhook 觸發
│   └── 手動觸發
├── 動作節點 (Action)
│   ├── AI Agent 呼叫
│   ├── API 請求
│   ├── 條件判斷
│   └── 資料轉換
└── 控制節點 (Control)
    ├── 平行執行
    ├── 順序執行
    ├── 錯誤處理
    └── 迴圈
```

### 2.3 連線系統

- **智能連線**: 自動吸附最近連接點
- **動態路徑**: 貝茲曲線，隨節點移動調整
- **條件分支**: 不同顏色代表不同條件分支
- **連線樣式**: 實線(正常)、虛線(條件)、紅線(錯誤)

### 2.4 屬性面板

| 區塊 | 內容 |
|------|------|
| 基本設定 | 節點名稱、描述、啟用/停用 |
| 參數配置 | 動態表單根據節點類型變化 |
| 輸入輸出 | 資料結構預覽與映射 |
| 錯誤處理 | 重試次數、逾時設定、降級策略 |

## 3. 技術架構

### 前端技術棧
```
視覺化編輯器/
├── React 18 + TypeScript
├── ReactFlow (畫布核心)
├── Zustand (狀態管理)
├── Tailwind CSS (樣式)
└── Monaco Editor (JSON 編輯)
```

### 核心依賴
- **@xyflow/react**: 節點圖形引擎
- **react-hook-form**: 表單處理
- **zod**: 資料驗證
- **lucide-react**: 圖示庫

### 資料流程
```
用戶操作 → Canvas State → Workflow JSON → Validation → Save to Backend
                ↓
         Real-time Sync (WebSocket)
                ↓
         Export as ClawFlow Config
```

### 4.2 節點設計規範

**視覺樣式**:
- 圓角: 8px
- 陰影: `0 2px 8px rgba(0,0,0,0.1)`
- 選中: 藍色邊框 + 發光效果
- 錯誤: 紅色邊框 + 警告圖示

**顏色編碼**:
| 類型 | 顏色 | 用途 |
|------|------|------|
| 觸發 | 綠色 #22c55e | 流程起點 |
| 動作 | 藍色 #3b82f6 | 執行操作 |
| 控制 | 紫色 #a855f7 | 流程控制 |
| 錯誤 | 紅色 #ef4444 | 錯誤處理 |

## 5. 匯出整合

### 匯出格式
```json
{
  "version": "1.0",
  "name": "我的工作流",
  "nodes": [
    {
      "id": "node-1",
      "type": "trigger",
      "position": { "x": 100, "y": 100 },
      "data": { "triggerType": "cron", "schedule": "0 9 * * *" }
    },
    {
      "id": "node-2",
      "type": "action",
      "position": { "x": 300, "y": 100 },
      "data": { "skill": "github", "action": "createIssue" }
    }
  ],
  "edges": [
    { "source": "node-1", "target": "node-2", "type": "default" }
  ]
}
```

### ClawFlow 配置轉換
視覺化編輯器可一鍵匯出為 `clawflow.config.js`，直接部署執行。

## 7. 使用方式

### 啟動編輯器
```bash
# 在任務板中開啟
cd task-board
npm run dev

# 訪問 http://localhost:3011/workflow-editor
```

### 建立第一個工作流
1. 從左側面板拖曳「定時觸發」到畫布
2. 拖曳「AI Agent」節點並連接到觸發點
3. 點選 AI Agent 節點，在右側配置參數
4. 點選「儲存」並「發布」
