## Supabase 查詢（query_supabase）

不需要 Key，直接用 query_supabase action：
```json
{"action":"query_supabase","table":"openclaw_tasks","select":"*","limit":10}
```

帶條件：
```json
{"action":"query_supabase","table":"openclaw_tasks","select":"*","filters":[{"column":"status","op":"eq","value":"queued"}],"limit":50}
```

可用 filter op：eq, neq, gt, gte, lt, lte, like, ilike, in, is
可查的表：openclaw_tasks, openclaw_reviews, openclaw_automations, openclaw_evolution_log, openclaw_runs, openclaw_audit_logs, fadp_members, fadp_attack_events, fadp_blocklist

### 重要：Supabase 真實欄位（跟 API 不一樣！）

**openclaw_tasks 真實欄位：** id, title, cat, status, progress, auto, thought, subs, from_review_id, created_at, updated_at
- 沒有 owner/agent/executor/result/priority 欄位！這些存在 thought 的 JSON metadata 裡
- 查任務時用 `select:"*"` 就好，不要指定不存在的欄位
- name → 用 title；description → 用 thought

**openclaw_audit_logs 真實欄位：** id, action, resource, resource_id, user_id, ip, diff, created_at
- 沒有 timestamp/level/message/event_type 欄位！
- timestamp → 用 created_at；type → 用 action

**openclaw_evolution_log 真實欄位：** id, t, x, c, tag, tc, created_at
- 這些都是縮寫：t=title, x=content, c=context, tag=type
