# 對話輪數智能判斷系統

> 目的：預測任務所需輪數，動態決定壓縮時機
> 位置：`memory/conversation-intelligence.md`

---

## 🧠 智能判斷邏輯

### Step 1: 任務分類（會話開始時判斷）

| 任務類型 | 特徵 | 預估輪數 | 壓縮時機 |
|---------|------|---------|---------|
| **查詢型** | 問資料、查狀態、找資訊 | 1-2 輪 | 結束即壓縮 |
| **設定型** | 改配置、設參數、開關功能 | 2-3 輪 | 完成即壓縮 |
| **開發型** | 寫程式、建功能、設計架構 | 3-5 輪 | 功能完成壓縮 |
| **複雜專案** | 多階段、多檔案、長期任務 | 5-8 輪/階段 | 每階段結束壓縮 |
| **探索型** | 討論、腦力激盪、未定方向 | 不定 | 主題切換時壓縮 |

### Step 2: 動態調整（進行中監控）

**正常情況**: 進度符合預估
```
預估 3 輪 → 第 3 輪完成 → 壓縮 ✓
```

**延續情況**: 任務比預期複雜
```
預估 3 輪 → 第 3 輪還沒完成 → 延長到 5 輪 → 第 5 輪完成 → 壓縮 ✓
```

**發散情況**: 話題開始分支
```
原本：做 A 功能
變成：做 A + 順便討論 B + 想到 C 也要改
→ 立即壓縮 A → 新會話處理 B → 新會話處理 C
```

### Step 3: 壓縮決策樹

```
對話進行中
    ↓
檢查：目前輪數 vs 預估輪數
    ↓
┌─────────────────────────────────────┐
│ 輪數 < 預估？                        │
├─────────────────────────────────────┤
│ 是 → 繼續對話                       │
│ 否 → 檢查：任務完成了嗎？            │
└─────────────────────────────────────┘
                ↓
        ┌───────────────┐
        │ 完成了？       │
        ├───────────────┤
        │ 是 → 壓縮結束 │
        │ 否 → 檢查發散？│
        └───────────────┘
                        ↓
                ┌───────────────┐
                │ 話題發散？     │
                ├───────────────┤
                │ 是 → 壓縮 →   │
                │     新會話繼續 │
                │ 否 → 延長預估  │
                │     (+2 輪)    │
                └───────────────┘
```

## 📝 實際應用範例

### 範例 1: 查詢型（1-2 輪）

```
[輪1] 主人：今天天氣怎樣？
[輪2] 達爾：查詢完成，今天晴 25°C
        ↓
        任務完成 (1-2 輪內)
        ↓
        壓縮 → 記憶：「2026-02-11 天氣晴 25°C」
        ↓
        結束或新話題
```

### 範例 2: 開發型（3-5 輪）

```
[輪1] 主人：寫個自動備份腳本
[輪2] 達爾：設計架構，預估 4 輪完成
[輪3] 達爾：寫核心邏輯
[輪4] 達爾：寫設定檔
        ↓
        完成！符合預估 (4 輪)
        ↓
        壓縮 → 記憶備份
```

### 範例 3: 延續情況（動態調整）

```
[輪1] 主人：修這個 bug
[輪2] 達爾：預估 3 輪，開始除錯
[輪3] 達爾：發現問題比想像複雜，需要重構
        ↓
        進度：2/3，但還沒完成
        ↓
        主動告知：「這個比預期複雜，估計還要 3 輪，要繼續還是壓縮分段？」
        ↓
主人選擇：
  A. 「繼續」→ 延長到 6 輪
  B. 「壓縮」→ 記錄進度 → 新會話繼續
```

### 範例 4: 發散情況（立即壓縮）

```
[輪1] 主人：做 GitHub Skill
[輪2-5] 達爾：開發完成
[輪6] 主人：對了，順便設定監控
[輪7] 達爾：設定 Ollama 監控
[輪8] 主人：還有那個模型要切換
        ↓
        注意！話題開始發散
        從「GitHub Skill」延伸到「監控系統」再到「模型設定」
        ↓
        智能判斷：主題已切換
        ↓
        壓縮 Skill 開發 → 記憶
        ↓
        新會話：「設定 Ollama 監控和模型」
```

## 🎯 關鍵字識別

### 延續信號（預估輪數 +2）
- 「還有...」
- 「順便...」
- 「對了，那個...」
- 「既然都做了，不如...」

### 發散信號（立即壓縮）
- 主題完全改變（從 A 跳到 B）
- 出現「先不管這個，我們來看...」
- 多個獨立任務同時進行

### 結束信號（可以壓縮）
- 「好」、「可以」、「沒問題」
- 「下一步做什麼？」
- 「先這樣」
- 「明天再說」

## 💡 給達爾的執行指南

### 會話開始時
1. 判斷任務類型
2. 給出預估輪數
3. 告知主人：「這個任務預估 3-4 輪，完成後壓縮」

### 進行中（每輪檢查）
- 輪數 < 預估：繼續
- 輪數 = 預估：檢查是否完成
- 輪數 > 預估：主動告知延遲原因，詢問是否壓縮

### 發現發散時
- 立即提醒：「我們從 A 聊到 B 了，要壓縮 A 開新會話講 B 嗎？」
