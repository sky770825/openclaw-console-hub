## 一、目標

建立 **Codex/Cursor Agent I/O 閉環模式**，實現：
1. 老蔡下指令 → 小蔡啟動 Agent → Agent 直接回報老蔡
2. 小蔡不當回覆窗口，只做調度/監控
3. 降低成本（省 30-40% tokens）
4. 確立安全規範（Token 管理、ACK 機制、防重複）
