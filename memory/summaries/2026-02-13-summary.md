## 💡 值得深入的概念

### 1. **Idempotency Key Pattern**
   - `task_id:run_id` 組合防止重複執行
   - 應用場景: 網路重試、異步任務、分佈式系統
   - 實施: 寫入 `processed_tasks.log`，檢查重複

### 2. **自動升級機制**
   - 根據連續失敗次數自動上報
   - Codex: 2x timeout / 3x failed
   - Cursor: 1x syntax error / 2x test failed / 2x timeout / 3x git conflict
   - 避免無限重試，節省成本

### 3. **代碼重複識別與統一框架**
