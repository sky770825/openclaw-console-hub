---

## 目的

主人說停就停。不是做完再停，是立即停。

---

## 觸發詞（任一出現就立即停止）

```
停 | stop | 暫停 | 別動 | 等一下 | 不要動 | hold | freeze
```

---

## 執行流程

### Step 1: 立即停止所有操作

- 停止正在執行的工具呼叫
- 停止正在寫的檔案
- 停止正在跑的命令
- **不要**「做完這一步再停」

### Step 2: 回報當前狀態

```
⛔ 已停止

正在做：{什麼任務}
做到哪：{哪一步}
半成品：{有/沒有}
  - {半成品位置和狀態}
子代理：{有/沒有正在跑的子代理}
  - {子代理 ID 和任務}
```

### Step 3: 等待指示

- 主人說「繼續」→ 恢復操作
- 主人說「取消」→ 清理半成品，回報
- 主人給新指令 → 執行新指令

---

## 半成品處理

| 半成品類型 | 處理方式 |
|-----------|---------|
| 寫到一半的檔案 | 不要刪，保留原樣，回報位置 |
| 跑到一半的子代理 | 回報 session ID，等主人決定要不要殺 |
| 未完成的 git commit | 回報 staged 的檔案列表 |
| 任務板狀態 running | 保持 running，回報 task ID |

---

## 回報格式

```
⛔ 已停止

在做：{任務名稱}
進度：Step {X}/{Y}
半成品：
  - {檔案路徑} — 寫到一半
子代理：
  - {session-id} — {任務} — running
等你指示。
```

---

## 恢復流程

主人說「繼續」後：

1. 確認半成品狀態是否還有效
2. 從中斷點繼續（不要從頭開始）
3. 回報「恢復執行，從 Step X 繼續」

---

## 禁止行為

- ❌ 不可說「等我做完這一步」
- ❌ 不可忽略停止指令繼續操作
- ❌ 不可自行清理半成品（除非主人說）
- ❌ 不可假裝沒看到停止指令
