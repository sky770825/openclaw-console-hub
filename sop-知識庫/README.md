## 向量切片指引

### 切片策略

每個 SOP 檔案建議切成以下 chunks：

1. **metadata chunk** — `## metadata` 區塊
   - 用途：語意搜尋時的索引入口
   - 包含：id, name, category, tags, trigger
   - 向量 metadata filter：`category`, `tags`, `priority`

2. **流程 chunk** — `## 執行流程` / `## 完整流程` 區塊
   - 用途：小蔡需要「怎麼做」時搜到
   - 包含：Step 1 到 Step N 的完整步驟

3. **回報 chunk** — `## 回報格式` 區塊
   - 用途：小蔡需要「怎麼回報」時搜到
   - 包含：回報模板

4. **錯誤 chunk** — `## 錯誤處理` 區塊
   - 用途：小蔡遇到問題時搜到
   - 包含：問題 → 處理方式對照表

5. **禁止 chunk** — `## 絕對禁止` / `## 禁止行為` 區塊
   - 用途：小蔡判斷「能不能做」時搜到

### Embedding metadata 建議

每個 chunk 應附帶以下 metadata：

```json
{
  "sop_id": "sop-01",
  "sop_name": "標準任務流程",
  "category": "日常操作",
  "section": "流程|回報|錯誤|禁止",
  "tags": ["任務", "流程", "API"],
  "priority": "P0",
  "version": "2.0"
}
```

```
  │

  │
  └── L4 Cursor（UI/前端）
        └── 大型 UI 修改、前端實作
```
