## 目的

多個 Agent 同時工作會撞車。規則：一個任務一個人負責，透過任務板協調，不要直接互相叫。

---

## 核心原則

1. **一個任務一個 Agent** — 不可兩個 Agent 同時做同一個任務
2. **透過任務板溝通** — 不直接對話，用任務板 + 檔案系統
3. **不搶別人的任務** — 看到別人 running 的任務，不要碰
4. **發現問題回報主人** — 不要自己修別人的問題

---

## 開始工作前的檢查

### Step 1: 檢查任務板

```bash
# 查相關任務有沒有別人在做
curl -s http://localhost:3011/api/tasks?status=running
```

- 有人在做相同的事 → **停止，回報主人**
- 沒人在做 → 繼續

### Step 2: 檢查 git 狀態

```bash
git status
git log --oneline -5
```

- 有未 commit 的變更 → 可能是別的 Agent 在改 → **不要覆蓋**
- 乾淨 → 繼續

### Step 3: 檢查目標檔案

```bash
# 確認沒人在同時改這個檔案
find . -name "{target_file}" -mmin -5 2>/dev/null
```

- 最近 5 分鐘被改過 → 可能有人在改 → **等一下或問主人**

---

## 檔案鎖定規則

| 檔案 | 同時修改 |
|------|---------|
| AGENTS.md | ❌ 每次只能一個 Agent 改 |
| MEMORY.md | ❌ 每次只能一個 Agent 改 |
| 知識庫文件 | ❌ 同一檔案不可兩人改 |
| 任務板 task | ❌ 一個 task 一個 Agent |
| scripts/ | ⚠️ 不同腳本可以，同一腳本不行 |
| logs/ | ✅ 各寫各的 log |

---

## 需要別人做事時

**不要直接叫別的 Agent。建任務讓主人分派。**

```
POST /api/tasks
Body: {
  "title": "需要 {Agent} 做 {什麼}",
  "description": "...",
  "projectPath": "projects/openclaw"
}
```

然後回報主人：
```
我需要 {Agent} 幫忙做 {什麼}，已建任務 {taskId}，請分派。
```

---

## 發現別人的問題時

**不要自己修。回報主人。**

```
⚠️ 發現問題

位置：{檔案路徑}
問題：{描述}
可能是 {Agent} 的操作造成的
建議：{修復方式}

等你決定由誰來修。
```

---

## 回報格式

```
👥 跨 Agent 協作狀態

我的任務：{task ID} — {名稱}
工作區域：{目標檔案/目錄}
其他 Agent 活動：{有/無}
衝突風險：{低/中/高}
```

---

## 錯誤處理

| 狀況 | 處理方式 |
|------|----------|
| 兩個 Agent 同時改同一檔案 | 立即停止，回報主人，用 git diff 看衝突 |
| 別的 Agent 覆蓋了我的產出 | 回報主人，用 git log 找回 |
| 任務板顯示兩人在做同一任務 | 回報主人，其中一個要退出 |
| 子代理沒回來 | 等 10 分鐘，沒回就回報主人 |

## 子代理管理

### 派工時（SOP-4 整合）

```
sessions_spawn(
  task="...",
  label="...",
  model="..."  # 見 11-本地模型子派 選模型
)
```

### 子代理超時

- 10 分鐘沒回應 → 回報主人
- 30 分鐘沒回應 → 建議殺掉重派

### 子代理回報後

1. 驗證產出（ls -la、wc -c、cat 抽查）
2. 不可只看「完成」就信
3. 品質不過 → 退回重做或換模型
