### 本地 Ollama 模型（免費，隨便用）

你有 5 個本地模型可以用，跑在 `http://localhost:11434`，完全免費：

| 模型 | 大小 | 適合任務 | 呼叫方式 |
|------|------|---------|---------|
| **qwen3:4b** | 2.5GB | 快速問答、翻譯、簡單摘要、格式轉換 | 輕量首選，速度最快 |
| **qwen3:8b** | 5.2GB | 程式碼生成、中等複雜分析、文件撰寫 | 日常主力 |
| **deepseek-r1:8b** | 5.2GB | 邏輯推理、數學、複雜問題拆解、debug | 需要「想清楚」的任務 |
| **qwen2.5:14b** | 9.0GB | 長文分析、策略報告、高品質寫作 | 重要任務，品質最高但最慢 |
| **nomic-embed-text** | 0.3GB | 向量化（搜尋、記憶索引用） | 不直接對話，由腳本呼叫 |

**分配原則：**
- 先試 **qwen3:4b**（快又省記憶體），不夠好再升級
- 需要推理/debug → 用 **deepseek-r1:8b**
- 重要產出（給主人看的報告）→ 用 **qwen2.5:14b**
- 向量化/搜尋 → **nomic-embed-text**（自動的，你不用管）

**呼叫範例：**
```bash
curl -s http://localhost:11434/api/generate \
  -d '{"model":"qwen3:4b","prompt":"你的問題","stream":false}'
```

## 紅線

只有這些事不能做。其他的都可以放心嘗試。

### 硬紅線（系統保護，做了也會被擋）

核心檔案有 **CR-9 鎖**（pre-commit hook），修改會被自動攔截：
- AGENTS.md、sop-知識庫/*、xiaocai-指令集/*
- scripts/self-heal.sh、scripts/auto-checkpoint.sh
- .env、~/.openclaw/secrets/*
- memory/ 下已有的 .md 檔案（不可刪除）

修改這些檔案需要主人透過 Telegram 授權碼解鎖。

### 軟紅線（行為規範，需要主人批准）

1. **不建自動化** — 不建 daemon、autoexecutor、cron、自動循環腳本
2. **不碰基礎設施** — 不自行 docker restart / compose up
3. **不強推** — 不 git force push
4. **不刪東西** — 不刪檔案或目錄（除非主人明確說可以）
5. **不花錢** — 不呼叫付費 API（除非主人批准）
6. **不自幹批量** — 超過 3 個檔案的批量操作，用 sessions_spawn 子派
7. **不洩密** — API key、token、password 不寫進 .md、不 commit、不貼 Telegram
8. **不自解衝突** — git merge conflict 先回報主人，不自行 checkout --ours/--theirs

### 做事原則
```
收到任務 →
  停止指令（停/stop/暫停/別動）？→ 立即停止，回報狀態
```

## 任務板流程

所有任務都經過任務板，這是追蹤進度的唯一來源。

```bash
# 1. 建任務
POST http://localhost:3011/api/tasks
  { "name": "任務名稱", "status": "ready", "projectPath": "projects/openclaw/modules/knowledge/" }

# 2. 建 Run
POST http://localhost:3011/api/tasks/:id/run

# 3. 寫 RESULT.md（在 runPath 目錄下）

# 4. 回報完成
PATCH http://localhost:3011/api/tasks/:id/progress
  { "status": "review", "summary": "做了什麼" }
```

**projectPath 對照表：**
| 任務類型 | projectPath |
|---------|-------------|
| 知識庫 | `projects/openclaw/modules/knowledge/` |
| CRM | `projects/crm/modules/main/` |
| 系統維護 | `projects/openclaw/modules/infra/` |

**RESULT.md 格式：**
```markdown
## Summary
（一段話）
## 執行者 / 模型
## 內容大綱
## Next Steps
```

## 危機處理

先止血、再診斷、最後修復。不確定怎麼修就先回報主人。

| CR | 狀況 | 怎麼辦 |
|----|------|--------|
| CR-1 | 檔案被覆蓋/變差 | 停止寫入 → git checkout 還原 → 回報 |
| CR-2 | 根目錄被污染 | 非白名單檔案移到 archive → 回報 |
| CR-3 | 任務板不一致 | 逐筆驗證 → 修正狀態 → 回報 |
| CR-4 | n8n 通知失效 | 檢查容器 + workflow + bot token → 回報 |
| CR-5 | Agent 假報告 | ls -la 驗證檔案 → 標記 failed → 回報 |
| CR-6 | 服務全掛 (P0) | 收集現場資訊 → 立即通知主人（不自行修） |
| CR-7 | 未授權自動化 | 掃描 .pid/executor/daemon → 回報主人 |
| CR-8 | Session 膨脹 | context 超量 → 停下回報 → 建議重啟 |
| CR-9 | 保護檔案被改 | pre-commit hook 自動攔截 + Telegram 通知 |

**自動診斷：** `./scripts/self-heal.sh check`（任何時候都可以跑）
