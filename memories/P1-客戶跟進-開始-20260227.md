## 分析現行跟進流程（從父親事業推論）

### 標準跟進時間軸
- **Day 0**：初次接觸（電話/帶看）
- **Day 1**：感謝簡訊 + 提供補充資料
- **Day 3**：詢問意向 + 推薦類似物件
- **Day 7**：價格調整通知或新物件推薦
- **Day 14**：最後跟進 + 長期維護轉換

### 阻塞點
1. **人工記憶易遺漏** - 客戶多時記不住誰該哪天跟進
2. **跟進內容重複** - 每次重新想文案，效率低
3. **無法量化** - 不知道跟進成功率、轉換率

## 自動化解決方案

### 系統架構
```
客戶資料輸入 → 自動排程 → 提醒通知 → 跟進內容建議 → 結果記錄
     ↑                                                      ↓
     └────────────── 數據分析與優化 ←──────────────────────┘
```

### 核心功能
1. **自動排程引擎** - 根據接觸日期自動計算下次跟進日
2. **提醒通知** - Telegram/簡訊提醒父親「今天有 3 位客戶需跟進」
3. **跟進內容模板** - 根據客戶階段自動生成建議話術
4. **簡易資料庫** - CSV/JSON 記錄客戶狀態，避免遺漏
