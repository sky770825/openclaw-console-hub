# Proactive Vulnerability Discovery (PVD) Module - RESULT.md

## 1. 專案概述 (Project Overview)
本專案開發了「主動式漏洞發現 (PVD)」模組，旨在透過多層次的掃描技術（靜態分析、依賴掃描、AI 增強識別）主動發現程式碼庫中的安全性弱點。

- **目錄結構**: `core/auto-security/pvd_module/`
- **執行者**: L2 Claude Code (as Subagent)

## 2. 設計方案 (Design Architecture)
PVD 模組採用模組化架構，包含三個核心引擎：

1.  **Static Analysis Engine (SAE)**:
    - 採用 AST (Abstract Syntax Tree) 遍歷與模式比對。
    - 識別 SQL 注入、指令注入、硬編碼金鑰等常見模式。
2.  **Dependency Scanning Engine (DSE)**:
    - 掃描 `requirements.txt` 並與已知漏洞庫 (模擬 NVD/OSV 數據) 進行比對。
    - 識別已知 CVE 的過時套件。
3.  **AI-Enhanced Recognition Engine (AIRE)**:
    - 利用本地 Ollama 模型 (`qwen3:8b`) 進行推理分析。
    - 識別邏輯漏洞、複雜的注入點或不安全的加密實踐。
4.  **Reporting System**:
    - 彙整各引擎發現的漏洞，生成 Markdown 格式的詳細報告。

## 3. 實作細節 (Implementation Details)
- **核心檔案**:
  - `main.py`: 主程式，負責協調整合與報告生成。
  - `engines/static_engine.py`: 靜態分析邏輯。
  - `engines/dependency_engine.py`: 依賴掃描與已知漏洞庫比對。
  - `engines/ai_engine.py`: 封裝 LLM 呼叫與提示詞工程。
- **環境要求**:
  - Python 3.9+
  - Ollama (跑本地 `qwen3:8b` 模型)
