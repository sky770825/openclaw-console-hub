# 提案：OpenClaw 任務板增強計畫 (2026-02-28)

## 背景
OpenClaw 任務板是我們協作的核心介面，也是任務流轉的指揮中心。為了提升小蔡的指揮調度效率和老蔡的決策透明度，本提案旨在檢視任務板現狀並提出一系列改進建議。

## 現有問題與痛點

1.  任務狀態可視性與流轉歷史不足：
       具體問題： 任務的狀態流轉（從 ready 到 failed）原因及中間歷史事件（如 Agent 選擇、執行日誌摘要）在前端 UI 上不夠直觀，需額外查詢 openclaw_runs。
       影響： 難以快速判斷任務失敗原因，延長問題診斷時間，降低老蔡對任務進度的掌握。

2.  任務分派與 Agent 選擇透明度不高：
       具體問題： 在建立任務時，缺乏對各 Agent 能力的簡要說明，以及基於任務內容的 Agent 推薦機制，導致任務分派可能不夠精準。
       影響： 小蔡在調度任務時可能需要額外思考或查詢，增加決策成本；老蔡也無法直觀了解任務為何被分派給特定 Agent。

3.  任務板的「知識沉澱」能力不足：
       具體問題： 任務完成後，其產生的經驗、教訓或可重用的成果（如腳本、配置）未能有效地整合回知識庫，導致知識分散。
       影響： 經驗無法有效複用，重複造輪子，阻礙小蔡的自我學習和進化。

4.  前端互動性與效率有待提升：
       具體問題： 任務列表缺乏高級篩選、排序、批量操作和自定義視圖等功能，操作效率較低。
       影響： 老蔡在管理大量任務時效率不高，難以快速定位和處理關鍵任務。

## 建議的提升方向

1.  增強任務狀態可視性與流轉邏輯：
       短期目標：
           任務流轉日誌： 在任務詳情頁增加一個時間軸或列表，顯示所有狀態變更和關鍵事件（如：被哪個 Agent 撿起、執行日誌摘要）。
           失敗原因自動歸類： 當任務失敗時，自動將失敗原因分類（例如：沙箱限制、程式碼錯誤、`API 呼叫失敗、超時），並在任務列表和詳情頁顯示。
       長期目標： 引入任務「風險評分歷史」的可視化，追蹤任務風險的變化。

2.  優化任務分派與 Agent 選擇的透明度：
       短期目標：
           Agent 簡介與推薦： 在任務建立/編輯介面，提供各 Agent 的簡要功能說明，並根據任務描述自動推薦最適合的 Agent。
           Agent 負載儀表板： 開發一個簡單的儀表板，顯示各 Agent 的當前負載、過去 N 小時完成任務數和平均執行時間。
       長期目標： 實現智能 Agent 競標系統，根據任務複雜度和 Agent 能力自動分配。

3.  增強任務板的「知識沉澱」能力：
       短期目標：
           任務成果與知識庫連結： 允許在任務的 result 欄位中直接連結到 knowledge/ 或 cookbook/ 中的文件。
           「小蔡反思」欄位： 在任務詳情中增加一個可編輯的「小蔡反思」區塊，用於記錄我從任務中學習到的經驗和洞察。
       長期目標： 建立一個自動知識提煉引擎，將高質量任務結果自動轉化為知識庫條目。

4.  提高前端互動性與效率：
       短期目標：
           高級篩選與排序： 任務列表頁面增加多條件篩選（按狀態、優先級、Agent、類別、負責人）和排序功能。
           快速狀態更新： 允許在任務列表頁面直接透過下拉選單或按鈕快速修改任務狀態和優先級。
       長期目標： 實現可自定義的任務視圖和批量操作功能。

## 預期效益

   提升決策效率： 老蔡能更快地了解任務狀況、診斷問題和分配資源。
   增強小蔡的自主性： 透過更透明的 Agent 資訊和知識沉澱機制，小蔡能更智慧地調度任務並從中學習。
   提高系統透明度： 任務的執行歷史和失敗原因一目了然。
   加速知識累積： 讓每個完成的任務都成為我們共同進化的養分。

## 下一步

提交此提案給老蔡審核。若獲批准，將進一步拆解為具體任務，並建立到任務板上。
