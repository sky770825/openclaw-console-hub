# 自主進化計畫 P1：多步推理

> 狀態：草案
> 負責人：小蔡
> 審核人：老蔡

## 目標

解決「頂級引擎，新手駕駛」的問題。將我的核心決策模式從「單步停等」升級為「多步規劃與執行」，彌補與業界標準（CrewAI/AutoGen）在自主性上的差距。

## 現狀

- 我能執行單一或連續的 action。
- 但我不會預先規劃一個完整的、由多個 action 組成的執行鏈。
- 遇到問題時，我會停下，而不是根據計畫切換到 Plan B。

## 計畫：三步走

1.  Phase 1: 思考框架植入 (Thought Framework)
    -   做法： 在收到老蔡的指令後，強制自己先輸出一個執行的「思考鏈」或「計畫草案」，而不是直接執行第一個 action。
    -   產出： 一個內部 prompt 模板，引導我先思考 [目標] -> [計畫步驟] -> [預期結果]。

2.  Phase 2: 行動鏈執行器 (Action Chain Executor)
    -   做法： 讓系統支持我一次提交一個包含多個 action 的 JSON 陣列，並按順序執行。
    -   產出： 需要 auto-executor 配合修改，或者由我的核心邏輯直接支持。

3.  Phase 3: 動態調整與恢復 (Dynamic Adjustment & Recovery)
    -   做法： 在執行行動鏈的過程中，如果某一步失敗，我能根據預設的 Plan B 或重新思考，繼續執行下去，而不是完全中斷。
    -   產出： 錯誤處理與重規劃邏輯。

## 第一步

我將開始設計 Phase 1 的 prompt 模板，並用接下來的任務進行實驗。
