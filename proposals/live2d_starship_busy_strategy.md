# Live2D (星艦忙活主題) 專案統籌策略 v1.0

## 1. 專案目標
打造具備「星艦忙活 (Starship Busy)」氛圍的 Live2D 交互體驗，將任務面版與星艦運維場景結合。

## 2. 核心階段與里程碑
### 階段一：視覺與原畫 (Asset Preparation)
- **主題定義**：星艦指揮官/維修工造型。
- **拆圖層**：確保呼吸、眨眼、手臂動作（打字/焊接）及表情切換的圖層完整。
- **輸出格式**：PSD (準備導入 Cubism)。

### 階段二：模型製作 (Modelling)
- **物理運算**：頭髮擺動、服飾配件物理。
- **動作組**：
    - `Idle`: 輕微呼吸，查看終端。
    - `Working`: 快速打字或修理動作。
    - `Warning`: 警報紅光映射，神情緊張。
    - `Success`: 點頭認可。

### 階段三：系統整合 (Integration)
- **前端載入**：使用 `pixi-live2d-display` 或官方 SDK。
- **狀態聯動**：
    - 任務處理中 -> 觸發 `Working` 動作。
    - 系統出錯 -> 觸發 `Warning` 動作。
    - 閒置 -> `Idle`。

## 3. 資源分配
- **技術開發**：阿策 (全程統籌)
- **環境模擬**：Sandbox 測試環境
- **部署目標**：整合至任務面版 UI 右側或背景。

## 4. 風險管理
- 模型性能開銷：限制解析度及物理細節。
- 交互延遲：預加載核心動作組。
