# n8n 自動化首批候選流程 (Quick Wins) 提案

## 背景
為了提升「openclaw任務面版」的營運效率，我們（阿策、阿商、阿研）識別出以下三個最能快速產生效益的自動化需求。

## 候選流程一：任務狀態異動即時通知 (Task Notification)
- **觸發條件**: 任務面版中的任務狀態（Status）從「進行中」變更為「待驗收」。
- **自動化邏輯**: 透過 Webhook 接收變更訊息，自動發送 Discord/Slack 通知給相關負責人。
- **預期效益**: 減少人為溝通成本，縮短驗收等待時間。

## 候選流程二：GitHub Issues 自動同步為任務 (GitHub Sync)
- **觸發條件**: 在指定的 GitHub Repository 產生新的 Issue。
- **自動化邏輯**: GitHub Webhook -> n8n -> 調用 OpenClaw Task API 創建新任務。
- **預期效益**: 研發端 (阿研) 習慣使用的工具與任務面版無縫接軌。

## 候選流程三：每日進度匯報 AI 總結 (Daily AI Summary)
- **觸發條件**: 每日下午 6:00 定時觸發 (Cron)。
- **自動化邏輯**: 查詢當日更新的任務 -> 送入 OpenAI 進行摘要 -> 發送 Email/報表至管理群組。
- **預期效益**: 提供透明的進度追蹤，便於阿商與阿策進行決策。

## 技術實作建議
- 使用 n8n 作為核心編排引擎。
- 開放 OpenClaw API 提供內部存取。
