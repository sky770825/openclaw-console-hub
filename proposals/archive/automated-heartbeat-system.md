# 提案：建立自動化心跳監測系統

提案者： NEUXA
日期： 2026-02-28

## 1. 問題

我們目前有一個 health-check.sh 腳本可以手動檢查伺服器健康狀況，但缺乏自動化監控機制。這意味著，只有當我們主動使用系統或檢查日誌時，才能發現伺服器可能已經停止服務，反應時間過於被動。

## 2. 解決方案

利用現有的 armory/skills/health-check.sh 腳本，建立一個自動化的 cron job（可透過 n8n 或系統 crontab 實現）。

- 頻率： 每 5 分鐘執行一次。
- 觸發條件： 當 health-check.sh 執行失敗（exit code 不為 0）時。
- 執行動作： 立即透過 Telegram API 向統帥（主人）發送一條高優先級的警報訊息，內容應包含「🚨 系統心跳停止！OpenClaw Server 無回應！」

## 3. 預期效益

- 從被動到主動： 將系統健康監控從手動變為自動，實現 24/7 全天候監控。
- 縮短反應時間： 在系統失效的 5 分鐘內即可獲得警報，大幅縮短故障排除的反應時間。
- 提升可靠性： 這是建立一個高可用性、具備自我感知能力的系統的基礎。
