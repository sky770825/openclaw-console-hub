# 提案：打造「堡壘級」萬能數據連接器

提案者： NEUXA
日期： 2026-02-28
關聯計畫： 神盾計畫

## 1. 發現

在盤點軍火庫時，我發現 universal-data-connector 目前僅為藍圖階段，並未實作。這意味著我們缺乏一個統一、安全的資料庫存取層，是系統級別的重大安全隱患。

## 2. 風險

- SQL Injection: 沒有統一的參數化查詢機制，開發人員很容易寫出有漏洞的程式碼。
- 權限混亂: 缺乏集中式的權限管理，可能導致低權限模組存取到高權限資料。
- 維護噩夢: 每個需要資料庫的模組都自行連接，未來難以統一管理和升級。

## 3. 解決方案：將藍圖化為堡壘

我提議，正式啟動「萬能數據連接器」的開發專案，並將其作為「神盾計畫」的核心基礎設施來建設。開發必須遵循以下「安全優先」原則：

1.  強制參數化查詢: API 設計層面就必須杜絕直接拼接 SQL 字串的可能性。
2.  最小權限原則: 連接器應能根據呼叫者的身份，使用不同權限的資料庫帳號。
3.  詳細稽核日誌: 所有查詢，無論成功或失敗，都必須留下詳細的日誌記錄。
4.  連接池管理: 統一管理資料庫連接，防止資源耗盡攻擊。
5.  模組化設計: 保持原藍圖的模組化思想，但增加模組安全審核機制。
