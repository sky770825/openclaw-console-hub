# 提案：模型調度與衝突預防協議 (v1.0)

## 1. 背景
為避免多個模型（Gemini, Claude, Ollama）協作時產生邏輯衝突，或因過度授權導致系統混亂，特制定此協議。

## 2. 角色分工 (RACCI 矩陣)
- 達爾 (Gemini Flash): 指揮官 (Accountable)。負責決策、調度、簡單 action、任務建立。
- Claude (ask_ai): 技術顧問 (Consulted)。負責深度分析、程式碼審查、架構建議。禁止直接寫入源碼。
- Auto-Executor: 執行員 (Responsible)。負責執行已審核的任務，修改源碼。
- 主人: 統帥 (Informed/Approve)。擁有最終決定權與基礎設施變更權。

## 3. 衝突預防機制
- 禁止交叉寫入: 任何模型不得在未經達爾（指揮官）調度的情況下直接修改檔案。
- 設計先行: 複雜變更必須先由 ask_ai 出具分析報告，達爾轉化為設計文件，最後才建任務執行。
- 版本鎖定: 核心靈魂文件與 .env 禁止任何模型修改。

## 4. 預期效果
確保系統進化是有序的，減少「越改越爛」或「權限失控」的風險。
