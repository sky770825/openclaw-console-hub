# 飲料訂單系統開發計畫

## 1. 專案目標

建立一個簡單的網頁應用程式，讓使用者可以瀏覽飲料菜單、客製化選項、下訂單，並將訂單資料儲存起來。

## 2. 功能需求

- **前端 (使用者介面)**
  - 顯示飲料菜單，包含品名、價格。
  - 提供客製化選項（例如：甜度、冰塊）。
  - 將選擇的飲料加入購物車。
  - 購物車可以計算總金額。
  - 填寫訂購人資訊並送出訂單。

- **後端 (伺服器)**
  - 提供 API 接收前端送來的訂單資料。
  - 將訂單資料儲存到一個檔案中（例如 `orders.json`）。

## 3. 技術選型

- **前端**: HTML, CSS, JavaScript (無框架，保持簡單)
- **後端**: Python + Flask (輕量級 Web 框架)
- **資料儲存**: JSON 檔案

## 4. 檔案結構

```
/workspace/
├── drink_order_system_plan.md  (本計畫檔案)
├── index.html                  (前端頁面)
├── style.css                   (前端樣式)
├── script.js                   (前端互動邏輯)
├── app.py                      (後端 Flask 應用程式)
└── orders.json                 (儲存訂單資料)
```

## 5. 開發步驟

1.  **建立專案計畫** (目前步驟)。
2.  **開發前端介面** (`index.html`, `style.css`)。
3.  **開發前端互動邏輯** (`script.js`)。
4.  **開發後端 API** (`app.py`)。
5.  **整合前後端並進行測試**。
