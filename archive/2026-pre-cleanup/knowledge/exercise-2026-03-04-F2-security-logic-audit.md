# 練習 F-2：security.ts 安全邏輯深度審計

### 1. 目標
審查 isScriptSafe 與路徑過濾邏輯，找出潛在的繞過風險。

### 2. 審計發現
- 路徑過濾：FORBIDDEN_PATH_PATTERNS 包含了 .env 和 key。但如果使用軟連結 (symlink) 或特定的編碼，可能有繞過風險。
- 指令比對：目前主要檢查指令是否以白名單開頭。若指令中包含分號 ; 或 &&，可能導致指令注入。

### 3. 改進建議
- 在 isScriptSafe 中加入對 ;, &, |, > 等特殊符號的檢測。
- 強化路徑檢查，使用 path.resolve 將所有路徑轉為絕對路徑後再比對，防止 ../ 繞過。

### 4. 下一步
預計在下個階段實作一個強化版的 isScriptSafe 並進行模擬測試。
