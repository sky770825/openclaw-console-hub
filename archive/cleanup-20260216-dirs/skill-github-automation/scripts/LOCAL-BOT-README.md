# SkillForge 本地離線客服機器人

## 🔒 完全離線、零網路連線

這個客服機器人**完全不連網**，所有資料都儲存在本地檔案，最安全的客服方案。

---

## 🎯 兩種客服機器人比較

| 功能 | Telegram 版 | 本地離線版 |
|------|-------------|-----------|
| **網路連線** | 需要 | ❌ 不需要 |
| **自動回覆** | ✅ 24小時自動 | ✅ 手動執行 |
| **記錄問題** | 線上記錄 | 本地檔案 |
| **安全性** | 高 | 🏆 最高 |
| **使用場景** | 自動客服 | 手動回覆、敏感環境 |

---

## 🚀 使用方法

### 啟動機器人
```bash
cd skill-github-automation
node scripts/local-support-bot.js
```

### 主選單功能

```
1️⃣  開始客服對話    → 手動輸入客戶問題，機器人回覆
2️⃣  查看待辦清單    → 顯示無法自動回覆的問題
3️⃣  查看統計報表    → 客服數據統計
4️⃣  清除待辦清單    → 清空待辦事項
5️⃣  查看對話記錄    → 歷史對話
0️⃣  離開
```

## 📁 檔案結構

```
logs/
├── pending-questions.txt    # 待人工回覆的問題
├── support-stats.json       # 客服統計數據
```

## 🛡️ 安全性特點

- ✅ **不連網** - 零網路連線，無駭客攻擊風險
- ✅ **本地儲存** - 所有資料在你的電腦上
- ✅ **開源透明** - 程式碼完全可見，無隱藏功能

## 📊 查看待辦清單

無法自動回覆的問題會記錄在 `logs/pending-questions.txt`：

```
[2026-02-12T05:15:00.000Z] 👤 Guest: 這個問題機器人聽不懂
[2026-02-12T05:16:00.000Z] 👤 Guest: 可以客製化功能嗎
```

## 🔄 工作流程

```
客戶發問
    ↓
你手動輸入問題到本地機器人
    ↓
機器人查找知識庫
    ↓
├─ 找到答案 → 顯示回覆 → 複製給客戶
└─ 找不到 → 記錄待辦 → 你人工回覆
```

---

## 📝 擴充知識庫

```javascript
const KNOWLEDGE_BASE = {

  // 加入更多...
};
```
