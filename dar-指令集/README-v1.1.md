### 方式 1: 直接貼文字（傳統）

## ⚠️ 重要提醒

### 絕對不可以
- 不可跳過任務板直接操作檔案
- 不可自行宣布「完成」或「定版」
- 不可在 workspace 根目錄放 RESULT 檔案
- 不可說完成但沒有 evidenceLinks

### 任務流程（每次都要走）
```
建任務（帶 projectPath）→ 建 Run → 做事 → 寫 RESULT.md → 回報 review → 主人驗收
```

### 燈號制度
- 🟢 **綠燈**（直接做）: 讀檔、跑 check、寫 RESULT
- 🟡 **黃燈**（先說）: 建任務、裝套件、改程式
- 🔴 **紅燈**（等批准）: 刪檔、push、改設定、restart
