## 模型概述

Qwen3（千問3）是阿里巴巴通義千問系列最新一代大型語言模型，於2025年4月29日正式開源發布。作為通義千問家族的第三代產品，Qwen3 在多個核心能力上實現了重大突破，特別是在推理能力、Agent 能力以及多模態理解方面。

### 主要特點

| 項目 | 說明 |
|------|------|
| **發布日期** | 2025年4月29日 |
| **開發團隊** | 阿里巴巴通義千問團隊 |
| **模型數量** | 共8款模型 |
| **架構類型** | Dense（密集）+ MoE（混合專家） |
| **全球下載量** | 超過3億次 |
| **開源授權** | 開源可商用 |

Qwen3 系列涵蓋了從輕量級到超大規模的多種模型規格，能夠滿足從邊緣設備到企業級部署的各種應用場景需求。

## 模型規格

Qwen3 系列共提供8款模型，其中6款為 Dense（密集）架構，2款為 MoE（混合專家）架構。Dense 模型適合一般應用，而 MoE 模型則以較低的激活參數實現高性能，兼顧效率與效果。

### 完整模型規格表

| 模型名稱 | 架構類型 | 總參數 | 激活參數 | 上下文長度 | 適用場景 |
|----------|----------|--------|----------|------------|----------|
| Qwen3-0.6B | Dense | 0.6B | 0.6B | 32K | 邊緣設備、嵌入式應用 |
| Qwen3-1.7B | Dense | 1.7B | 1.7B | 32K | 輕量級應用、移動端 |
| Qwen3-4B | Dense | 4B | 4B | 32K | 中小型應用開發 |
| Qwen3-8B | Dense | 8B | 8B | 128K | 中型推理伺服器 |
| Qwen3-14B | Dense | 14B | 14B | 128K | 一般企業任務 |
| Qwen3-32B | Dense | 32B | 32B | 128K | 複雜推理、企業應用 |
| Qwen3-30B-A3B | MoE | 30B | 3B | 128K | 高效能邊緣部署 |
| Qwen3-235B-A22B | MoE | 235B | 22B | 128K | 高精度推理、企業級 |

### 架構說明

**Dense（密集）架構**：
- 所有參數在推理時全部激活
- 適合需要穩定性能、對延遲敏感的應用
- 部署簡單，硬體需求明確

**MoE（混合專家）架構**：
- 採用專家混合技術，每次只激活部分參數
- 以較低的計算成本實現大模型性能
- Qwen3-235B-A22B 以 22B 激活參數達到 235B 模型的效果
- 適合對成本敏感但需要高性能的場景

### 上下文長度支援

| 模型規模 | 上下文長度 | 典型應用 |
|----------|------------|----------|
| 0.6B - 4B | 32K tokens | 一般對話、短文本處理 |
| 8B - 235B-A22B | 128K tokens | 長文檔分析、程式碼理解、多輪對話 |

## 核心能力

Qwen3 在多個維度實現了顯著提升，以下是其核心技術能力的詳細說明。

### 混合推理模式

Qwen3 引入了創新的「思考/非思考」雙模式切換機制：

| 模式 | 特點 | 適用場景 |
|------|------|----------|
| **思考模式** | 進行深度推理、逐步分析 | 數學問題、邏輯推理、程式編寫 |
| **非思考模式** | 快速回應、直接輸出 | 一般對話、資訊查詢、簡單任務 |

使用者可以根據任務需求自由切換模式，在效率與質量之間取得最佳平衡。

### Agent 能力顯著增強

Qwen3 在自主 Agent 能力方面實現了重大突破：

1. **工具呼叫（Tool Calling）**：
   - 支援複雜的工具調度與組合
   - 能夠理解工具描述並自主選擇適當工具
   - 支援多步驟工具鏈執行

2. **規劃能力（Planning）**：
   - 將複雜任務分解為可執行的子任務
   - 動態調整執行計劃
   - 處理任務依賴關係

3. **執行能力（Execution）**：
   - 自主執行多步驟任務
   - 錯誤處理與自我修正
   - 結果驗證與迭代優化

### 多模態支援

Qwen3-VL 版本提供強大的視覺理解能力：

| 能力 | 說明 |
|------|------|
| **圖像理解** | 理解圖片內容、場景、物體關係 |
| **文字辨識** | 高精度 OCR，支援多語言 |
| **視覺問答** | 基於圖片內容進行問答 |
| **圖表分析** | 理解並分析各類圖表數據 |
| **文檔理解** | 處理掃描文檔、PDF 等 |

### 長上下文處理

Qwen3 大模型（8B 及以上）支援 128K tokens 的超長上下文：

- **長文檔分析**：可一次性處理整本書籍或長篇報告
- **程式碼理解**：能夠理解大型程式碼庫的完整上下文
- **多輪對話**：支援長時間、多輪次的深度對話
- **知識檢索**：結合 RAG 技術實現高效長文檔檢索

### 數學與程式碼推理

Qwen3 在數學和程式碼推理方面達到了業界領先水平：

- **數學推理**：QwQ-32B 在數學推理任務上達到 DeepSeek-R1 滿血版水平
- **程式碼生成**：支援多種程式語言，生成高質量、可執行程式碼
- **程式碼解釋**：能夠深入理解複雜程式碼邏輯並進行解釋
- **Debug 能力**：協助定位並修復程式錯誤

## API 定價

Qwen3 系列模型可通過阿里雲百煉平台進行 API 調用，以下是參考定價資訊。

### 定價概覽

| 項目 | 價格範圍 | 說明 |
|------|----------|------|
| **輸入（Input）** | 約 $0.5 - $2 / 1M tokens | 依據模型規格而定 |
| **輸出（Output）** | 約 $1 - $4 / 1M tokens | 依據模型規格而定 |
| **思考模式** | 按實際輸出計費 | 思考過程中的 tokens 也計入費用 |

### 模型價格梯度

| 模型類別 | 輸入價格（約） | 輸出價格（約） | 備註 |
|----------|----------------|----------------|------|
| 輕量級（0.6B-4B） | $0.5 / 1M tokens | $1 / 1M tokens | 最經濟實惠 |
| 中型（8B-14B） | $0.8-1.2 / 1M tokens | $1.5-2.5 / 1M tokens | 性價比最佳 |
| 大型（32B） | $1.5 / 1M tokens | $3 / 1M tokens | 高性能 |
| MoE（30B-A3B, 235B-A22B） | $1-2 / 1M tokens | $2-4 / 1M tokens | 以低激活參數實現高效能 |

### 計費說明

1. **計費單位**：按實際使用的 tokens 數量計費
2. **思考模式**：啟用思考模式時，模型內部思考過程產生的 tokens 也會計入輸出費用
3. **長上下文**：128K 上下文模型可能產生較高的輸入費用
4. **價格更新**：阿里雲可能會根據市場情況調整價格，請以官方最新公告為準
