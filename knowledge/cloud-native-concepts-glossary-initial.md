# 雲原生架構核心概念與術語表 (初步匯總)

## 雲原生核心概念

   雲原生定義：雲原生是建構和執行可擴充應用程式的方法，充分運用雲端式服務和提供模型。它不是單一技術，而是一種設計與運營系統的方法論，強調讓應用程式在雲端環境中能夠快速部署、自動擴展、故障自我修復、容易更新與維護。
   與傳統架構的差異：傳統IT 系統採用單體設計和固定基礎設施，限制了可擴展性且維護複雜。相對地，雲原生透過 DevOps 和 CI/CD 充分利用容器化且模組化的架構，實現快速部署、持續改進和業務敏捷性。
   主要目標：讓系統能夠靈活應對變化，快速交付新功能，提升整體可靠性與效率。

## 雲原生關鍵技術與組成部分

雲原生架構包含多個核心組成部分和技術，常見的包括：

   微服務 (Microservices)：將單一應用程式分解為一系列小型、輕量級且鬆耦合的獨立服務，這些服務透過應用程式設計介面 (API) 輕鬆進行組合及連線。每個微服務可以獨立開發、部署和擴展，提高彈性與維護性。
   容器 (Container)：輕量級的可執行元件，包含應用程式原始碼及其執行所需的所有元素（包括依賴項）。容器提供工作負載可攜性，支援「建構一次即可隨處執行」程式碼，簡化開發和部署作業，並確保跨環境的一致性。
   容器編排 (Container Orchestration)：用於自動安排、部署、管理和擴展大量容器服務的系統，其中 Kubernetes 是最廣泛使用的工具。
