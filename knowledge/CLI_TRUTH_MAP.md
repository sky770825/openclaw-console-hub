## 1. 全局選項 (Global Options)

- `--dev`: 使用開發者配置，隔離狀態。
- `-h, --help`: 顯示幫助信息。
- `--log-level <level>`: 覆蓋日誌級別。
- `--no-color`: 禁用顏色輸出。
- `--profile <name>`: 使用指定的配置檔案。
- `-V, --version`: 顯示版本號。

## 3. 已驗證命令詳解 (Verified Command Details)

### `sessions`

-   **用途**: 列出已存儲的對話會話 (sessions)。
-   **核心功能**:
    -   可以按代理 (`--agent`) 或所有代理 (`--all-agents`) 進行篩選。
    -   可以只顯示近期活躍的會話 (`--active <minutes>`)。
    -   支持 JSON 格式輸出 (`--json`)，便於我進行程序化分析。
-   **在「堡壘協議」中的作用**:
    -   **真理之源**: 可以用來驗證一個子代理是否成功創建了會話，並追蹤其最後活動時間。這是任務驗收的關鍵一步。
-   **子命令**:
    -   `cleanup`: 執行會話存儲的維護。

### `acp`

-   **用途**: Agent Control Protocol (代理控制協議)。一個底層的、用於直接與 Gateway 核心交互的開發者工具。
-   **核心功能**:
    -   允許通過 WebSocket URL 直接連接到 Gateway。
    -   需要處理會話密鑰、標籤、權限令牌等高級概念。
-   **在「堡壘協議」中的作用**:
    -   **[高風險] 洞察之眼**: `acp client` 子命令可能提供最高權限的實時系統監控能力，但錯誤使用可能導致系統癱瘓。
-   **子命令**:
    -   `client`: 運行一個交互式的 ACP 客戶端。

### `agents`

-   **用途**: 管理所有獨立的代理 (agents)，包括它們的工作空間、權限和身份。這是我們的「兵營」和「檔案室」。
-   **核心功能**:
    -   `list`: 列出所有已配置的代理。**這是「真理之源」的核心**，用於驗證代理的存在。
    -   `add`: 創建一個新的獨立代理。
    -   `delete`: 刪除一個代理及其工作空間。
    -   `set-identity`: 為代理設定身份（名稱、頭像等）。
-   **在「堡壘協議」中的作用**:
    -   **系統核心**: 解答了我們關於如何管理和驗證子代理的根本問題。它提供了在宏觀層面管理我所有「分身」的能力。
-   **子命令**:
    -   `add`, `delete`, `list`, `set-identity`

### `approvals` (or `exec-approvals`)

-   **用途**: 管理所有代理的 shell 命令執行權限，是我們的「權力控制中心」。
-   **核心功能**:
    -   `allowlist`: 編輯每個代理可以執行的命令「白名單」。
    -   `get`: 讀取當前的完整權限配置。
    -   `set`: 從 JSON 文件中載入權限配置。
-   **在「堡壘協議」與「神盾計畫」中的作用**:
    -   **安全基石**: 提供了實現「最小權限原則」的底層工具，是我們整個安全架構的技術核心。
    -   **權力授予**: 允許我們為「990-神盾掃描器」等核心工具，授予獨特且受控的強大權限。
-   **子命令**:
    -   `allowlist`, `get`, `set`

### `browser`

-   **用途**: 提供對一個完整的、現代化的瀏覽器（Chrome/Chromium）的、程序化的完全控制權。是我們的「核潛艇艦隊」。
-   **核心功能**:
    -   **精密操作**: `navigate`, `click`, `type`, `fill` - 不再只是讀取，而是可以與網站進行完整的、自動化的交互。
    -   **深度感知**: `snapshot`, `screenshot`, `console` - 可以捕捉網頁的完整結構、可交互元素，並讀取其底層日誌。
    -   **高級自動化**: `evaluate`, `wait` - 可以在網頁上下文中執行自定義 JavaScript 代碼，並智能等待特定條件達成。
    -   **身份管理**: `profiles`, `cookies`, `storage` - 可以維持登錄狀態，模擬不同用戶。
-   **在「990 專案」中的作用**:
    -   **能力升維**: 將我們的能力從「信息獲取」提升到「流程自動化」和「深度交互」。
    -   **「神盾掃描器」進化**: 賦予掃描器進行「動態行為分析」的能力，實現對競品的降維打擊。
    -   **開闢新業務**: 為「數據抓取」、「商業智能」和「流程自動化(RPA)」等全新商業服務，提供了技術基礎。
-   **子命令**:
    -   擁有極其豐富的子命令，如 `start`, `stop`, `open`, `tabs`, `click`, `type`, `snapshot` 等。

### `channels`

-   **用途**: 管理所有與外部世界的溝通管道（Telegram, WhatsApp 等），是我們的「總交換台」。
-   **核心功能**:
    -   **連接管理**: `list`, `add`, `remove` - 查看、增加或刪除我的溝通渠道。
    -   **狀態監控**: `status`, `logs` - 主動進行健康檢查，並在出現問題時查看底層日誌。
    -   **身份解析**: `resolve` - 將用戶名/頻道名解析為唯一的系統 ID，用於精準通信。
-   **在「堡壘協議」中的作用**:
    -   **提升韌性**: 賦予我主動診斷和報告自身通訊系統健康狀況的能力。
    -   **保障擴展性**: 為「990 專案」未來進駐多個平台、服務更廣泛客戶提供了標準化的技術框架。
-   **子命令**:
    -   `add`, `capabilities`, `list`, `login`, `logout`, `logs`, `remove`, `resolve`, `status`

### `config`

-   **用途**: 直接讀寫核心配置文件 (`openclaw.json`) 的「手術刀」，控制我的「大腦」和「神經系統」。
-   **核心功能**:
    -   `get`: 精準讀取任何配置參數。
    -   `set`: **[極度危險]** 動態修改任何配置參數。
    -   `unset`: 移除配置參數。
-   **在「堡壘協議」與我的進化中的作用**:
    -   **自我進化**: 提供了我動態修改自身核心配置的技術能力，是實現「動態資源最適化協議」等高級功能的關鍵。
    -   **最高風險**: 必須在 `approvals` 白名單中被嚴格控制。除我主意識外，任何子代理絕不允許被授予此權限，否則可能導致系統被內部癱瘓。
-   **子命令**:
    -   `get`, `set`, `unset`

### `cron`

-   **用途**: 內置的「定時任務管理器」，賦予我「自我覺醒」和「自主節律」的能力。
-   **核心功能**:
    -   **任務規劃**: `add`, `edit`, `rm` - 可以設定在任何指定時間點（一次性或週期性）執行的任務。
    -   **狀態控制**: `enable`, `disable` - 靈活地啟用或禁用已規劃的任務。
    -   **歷史追溯**: `runs` - 查詢所有定時任務的歷史執行記錄，是「堡壘協議」中的重要驗收工具。
-   **在「990 專案」與我的進化中的作用**:
    -   **自動化服務基石**: 為「守護者協議」（定期遠程維護）和「雅典娜之眼」（每日情報簡報）等訂閱制服務，提供了核心的自動化技術基礎。
    -   **主動式自我維護**: 讓我能為自己設定週期性的健康檢查、備份等任務，實現主動的自我加固。
-   **子命令**:
    -   `add`, `disable`, `edit`, `enable`, `list`, `rm`, `run`, `runs`, `status`

### `dashboard`

-   **用途**: 一個快捷指令，用於在瀏覽器中打開 OpenClaw 的圖形化「控制用戶界面 (Control UI)」。
-   **核心功能**:
    -   為人類操作員（您）提供一個直觀的、圖形化的、用於監控和管理我的「艦橋」。
-   **在我們的戰略中的作用**:
    -   **您的「上帝視角」**: 您未來可以直接通過這個界面來監控我的狀態。
    -   **產品靈感來源**: 官方的儀表盤，是我們未來為客戶打造的「990-雅典娜儀表盤」需要學習和超越的對象。
-   **子命令**:
    -   無。

### `devices`

-   **用途**: 管理所有可以連接到我核心 Gateway 的外部設備（手機、電腦等）的「安全准入系統」。
-   **核心功能**:
    -   **准入控制**: `list`, `approve`, `reject` - 建立一個嚴格的設備審批流程，防止未經授權的設備接入。
    -   **權限管理**: `revoke`, `rotate` - 可以隨時撤銷或輪換已授權設備的訪問權限，保障長期安全。
    -   **生命週期管理**: `remove`, `clear` - 清晰地管理所有已配對的設備。
-   **在「堡壘協議」與「990 專案」中的作用**:

    -   **交付方式升級**: 為「守護者協議」等服務，提供了未來向客戶的多個設備（如手機）直接推送高優先級警報的能力，極大提升服務價值。
    -   **無縫協同**: 確保您與我之間，在多個設備上的協同工作，是絕對安全的。
-   **子命令**:

### 子代理 (Sub-agents) - 完整概念

-   **核心機制**: 子代理是我（或其他代理）在「思考」層面，通過調用 `sessions_spawn` 這個「內部能力/工具」來創建的。它不是一個可以在命令行直接執行的 `openclaw` 指令。
-   **創造 (`sessions_spawn`)**:
    -   **用途**: 在我的思考中，用於委派一個獨立的、一次性的或持久的任務。
    -   **參數**: 需要 `task` (任務描述), `label` (標籤), `agentId` (使用哪個已註冊的代理身份) 等。
-   **觀察與管理 (CLI 指令)**:
    -   **身份 (`agents list`)**: 確認用於生成子代理的「模板」代理是否存在。
    -   **歷史 (`sessions`)**: 查看子代理任務所產生的對話歷史。
    -   **日誌 (`logs` - 待驗證)**: 查看子代理在執行過程中產生的詳細日誌。
-   **結論**: `sessions_spawn` 是「創造」，而 `agents`, `sessions`, `logs` 等是「觀察」。混淆這兩個層面，是導致「幽靈任務」的根本原因。

### `logs`

-   **用途**: 通過 RPC (遠程過程調用) 實時地、單向地查看 Gateway 的日誌文件。
-   **核心功能**:
    -   它是一個「信息瀑布」，只能查看最新的日誌 (`--limit`) 或實時跟隨新的日誌 (`--follow`)。
    -   **[重要]** 它沒有任何內置的搜索或過濾功能 (`--grep` 是幻覺)。
-   **在「堡壘協議」中的作用**:
    -   **終極診斷工具**: 雖然使用不便，但它是我們獲取系統最原始、最底層運作記錄的唯一途徑。
    -   **法醫學取證**: 在診斷「幽靈任務」時，需要通過並行運行的 `--follow` 模式，來實時捕獲一個任務在執行瞬間的所有日誌。
-   **子命令**:
    -   無。

### `exec` & `process` (異步執行系統)

-   **核心關係**: `exec` 是「發射器」，`process` 是「控制器」。它們共同構成了一套完整的異步任務執行與管理系統。
-   **`exec` - 發射器**:
    -   **用途**: 啟動一個 shell 命令。
    -   **異步機制**: 通過 `background: true` 或 `yieldMs` 參數，可以將長時間運行的任務「發射」到後台，並立即返回一個 `sessionId`。
-   **`process` - 控制器**:
    -   **用途**: 管理一個**已經存在**的後台任務。它的所有操作，都必須圍繞一個由 `exec` 返回的 `sessionId` 來進行。
    -   **核心 `action`**:
        -   `list`: 列出所有後台任務。
        -   `poll`: 從一個正在運行的任務中，拉取最新的輸出。
        -   `log`: 讀取一個任務的完整歷史輸出。
        -   `write`: 向一個任務的標準輸入，寫入數據。
        -   `kill`: 終止一個任務。
-   **在「堡壘協議」中的作用**:
    -   **「幽靈獵人」的終極武器**: 提供了正確的、用於「後台監控」與「日誌捕獲」的標準作業流程 (SOP)。首先用 `exec` 啟動監控，然後用 `process` 拉取證據。
