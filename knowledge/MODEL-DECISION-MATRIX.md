## 🎯 場景化選擇指南

### 1. 「我需要寫一個複雜的跨檔案功能」
- **首選**：**Sonnet 4.5** (速度與質量的平衡)。
- **升級**：若涉及核心系統架構或高安全性要求，請動用 **Opus 4.6**。

### 2. 「我手頭有 50 個簡單的 JSON 需要轉換格式」
- **首選**：**Qwen3:4b** (本地運行，零延遲，零成本)。

### 3. 「代碼報錯了，但我看不出邏輯問題在哪」
- **首選**：**DeepSeek-R1** (它會「思考」並給出完整的推理鏈)。

### 4. 「我需要分析過去一個月的所有 Kubernetes 運行日誌」
- **首選**：**Gemini 3 Pro** (超大上下文窗口是唯一解)。

### 5. 「我要確保我的 Docker Image 沒有安全漏洞」
- **唯一推薦**：**Trivy** (搭配 **Opus 4.6** 進行漏洞解釋與自動修復)。

## 💡 決策建議 (Rule of Thumb)

1. **優先本地**：能用 Qwen3 或 DeepSeek-R1 解決的，不要呼叫付費 API。
2. **安全第一**：涉及 Shell 腳本、Docker 配置或 API Key 處理時，先過 Trivy 掃描。
3. **長短結合**：日常用 Sonnet 寫代碼，遇到卡關或大重構才升級到 Opus。
4. **Agent 協作**：使用子代理 (Sessions Spawn) 處理批量任務，並由 L1 小蔡進行彙整。
