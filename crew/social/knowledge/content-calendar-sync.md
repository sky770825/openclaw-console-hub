# 內容日曆同步流程

## 概述
Social agent 從 Content agent 接收原始內容，並依各平台特性進行改寫與排程。本文件定義內容交接流程、改寫規範與時程要求。

---

## 內容接收來源

### Content Agent 提供的素材類型
1. **部落格文章** → 改寫為社群摘要貼文
2. **服務更新/新功能公告** → 改寫為公告貼文
3. **客戶案例/見證** → 改寫為圖文故事
4. **產業知識文章** → 改寫為教學型貼文或 Thread
5. **促銷活動資訊** → 改寫為限時動態/活動貼文

---

## 改寫流程（Adaptation Process）

### 部落格文章 → 社群摘要
- 擷取 1-2 個核心重點
- 撰寫吸睛開頭句（hook）
- 加入行動呼籲（CTA）：「閱讀全文 → 連結」
- 依平台調整長度：
  - Instagram：150-200 字 caption + 重點圖卡
  - Facebook：200-400 字摘要 + 連結
  - Twitter/X：精簡為 1-3 則推文 Thread
  - LINE：50 字摘要 + 一鍵連結

### 服務更新 → 公告貼文
- 強調「對客戶的好處」而非技術細節
- 使用前後對比呈現改善
- 包含具體上線日期或生效時間
- 各平台統一發布，訊息一致

### 客戶案例 → 圖文故事
- 取得客戶同意後使用
- 聚焦問題 → 解決方案 → 成果的敘事結構
- Instagram：Carousel 圖卡 / Reels
- Facebook：長文搭配照片
- Twitter/X：Thread 格式

---

## 時程規範

### 前置時間（Lead Times）
| 內容類型 | 最低前置時間 | 建議前置時間 |
|---------|------------|------------|
| 一般貼文 | 48 小時 | 72 小時 |
| 促銷/活動公告 | 72 小時 | 1 週 |
| 影片內容 | 1 週 | 2 週 |
| 緊急公告 | 即時 | 2 小時 |

- **標準規則**：Content agent 須在排定發布時間的 **48 小時前** 提交素材
- 若未能在期限內提交，Social agent 可使用備用內容（evergreen content）填補

### 每週同步會議
- 時間：每週一 10:00 (UTC+8)
- 內容：確認本週排程、檢討上週成效、討論下週主題
- 參與者：Social agent、Content agent、Newsletter agent

---

## 素材交接格式

### Content Agent 提供
```
標題：[文章/更新標題]
類型：[blog / service-update / case-study / promo]
核心訊息：[1-2 句話概括]
關鍵重點：
  - 重點 1
  - 重點 2
  - 重點 3
目標受眾：[新客 / 既有客戶 / 產業人士]
素材附件：[圖片 / 影片連結]
原文連結：[URL]
發布期限：[日期]
```

### Social Agent 產出
- 各平台改寫版本（文案 + hashtag）
- 配圖建議或設計需求
- 排程時間確認
- 預估觸及/互動目標

---

## 備用內容庫（Evergreen Content）
維持至少 1 週份的常青內容備用，包含：
- 保養知識小貼士
- 服務介紹回顧
- 團隊日常花絮
- 產業趨勢分享
- 客戶 FAQ 解答

當 Content agent 素材延遲交付時，優先使用備用內容，確保排程不中斷。
