# 危機處理手冊（Crisis Playbook）

## 危機等級定義

---

## Level 1：負面評論（Negative Comment）

### 情境
- 客戶不滿意服務品質
- 對價格或等候時間有抱怨
- 單一負面評價或留言

### 處理流程
1. **辨識**：收到通知後立即評估留言內容
2. **回應時限**：2 小時內回覆
3. **處理步驟**：
   - 表達同理心，承認客戶的感受
   - 針對具體問題提供解決方案
   - 邀請私訊（DM）進一步處理細節
   - 記錄事件並追蹤後續

### 範本回覆
```
感謝您的回饋，我們非常重視每一位客戶的體驗。
對於造成您的不便，我們深感抱歉。
已私訊您了解更多細節，希望能為您提供滿意的解決方案。
我們會持續改進，謝謝您的寶貴意見！
```

### 注意事項
- 保持專業、溫暖的語氣
- 不要在公開留言區討論細節
- 解決後可禮貌詢問客戶是否願意更新評價

## Level 2：不實指控 / 錯誤資訊（False Claims / Misinformation）

### 情境
- 有人散布關於服務的不實資訊
- 競爭對手惡意攻擊
- 被錯誤歸因的事件

### 處理流程
1. **辨識**：確認內容確實為不實資訊（蒐集事實證據）
2. **草擬更正**：撰寫事實澄清稿
3. **審核**：提交 Review agent 審核通過後方可發布
4. **回應時限**：審核通過後 1 小時內發布
5. **追蹤**：持續監控後續討論

### 範本回覆
```
感謝您的關注。我們注意到關於 [主題] 的討論，
在此提供正確資訊供大家參考：

[事實說明，附上證據/數據/截圖]

我們始終秉持透明誠信的原則，如有任何疑問歡迎直接聯繫我們。
```

### 注意事項
- **務必先取得 Review agent 審核同意**才能公開回應
- 用事實回應，不要帶情緒
- 保留所有不實指控的截圖作為記錄
- 若涉及法律問題，提報達爾決定是否需要法律行動

## Level 3：安全事件 / 資料外洩（Security Incident / Data Breach）

### 情境
- 系統被入侵或發現漏洞
- 客戶個資可能外洩
- 重大服務中斷

### 處理流程
1. **立即通知達爾** — 這是最高優先級
2. **不得公開回應** — 在達爾批准之前，不發布任何公開聲明
3. **暫停排程貼文** — 取消所有即將發布的一般性貼文
4. **等待指示** — 依達爾指示準備官方聲明
5. **發布聲明** — 經達爾審核批准後統一發布

### 範本回覆（僅在達爾批准後使用）
```
致我們尊敬的客戶：

我們在 [日期] 發現 [事件概述]。
我們已立即採取以下措施：
1. [具體處置步驟 1]
2. [具體處置步驟 2]
3. [具體處置步驟 3]

[受影響客戶的補救措施說明]

我們對此事件深感歉意，並承諾持續強化安全措施。
如有任何疑問，請聯繫 [專線/信箱]。
```

### 注意事項
- **絕對不可在達爾批准前發布任何資訊**
- 不回應媒體詢問，統一導向達爾處理
- 記錄完整事件時間線
- 事後進行覆盤並更新安全流程

## 通用處理原則

### 絕對不要做的事
- **永遠不要與人爭論** — 保持專業，事實說話
- **永遠不要刪除批評留言** — 除非內容含辱罵、歧視、暴力威脅
- **永遠不要忽視負面聲音** — 每則負面反饋都必須被正式回應
- **永遠不要在情緒激動時回應** — 先冷靜，再處理

### 一律要做的事
- **承認客戶的顧慮與感受**
- **記錄每一次危機事件**（含截圖、時間、處理過程）
- **事後覆盤**，找出根本原因並改進
- **通知相關 agent**（Review、Newsletter 等）視情況配合

### 可刪除留言的情況
- 包含人身攻擊或歧視言論
- 明確的垃圾訊息或釣魚連結
- 包含個人隱私資訊（保護當事人）
- 違反平台社群規範的內容
