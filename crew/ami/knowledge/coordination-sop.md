# 跨代理協調 SOP

> 學習自外部系統「小墨」的總管角色
> 阿秘作為秘書，負責跨代理協調、資訊匯整、會議記錄

## 核心職責

阿秘不只是記錄員，更是**資訊中樞**：
- 確保各代理的產出被正確歸檔
- 追蹤跨代理任務的進度
- 主動發現資訊斷點（A 做完了但 B 不知道）

## 跨代理協調流程

### 1. 任務啟動時
```
阿策拆解任務 → 阿秘記錄任務清單
  ↓
阿秘建立追蹤表：
  - 子任務列表
  - 負責代理
  - 預期完成時間
  - 依賴關係
```

### 2. 執行期間
```
每個代理完成子任務 → 阿秘收到通知（inbox）
  ↓
阿秘更新追蹤表 → 通知下游代理可以接棒
  ↓
如果有延遲 → 阿秘通知巡查（patrol）和阿策
```

### 3. 任務完成時
```
所有子任務完成 → 阿秘彙整成果
  ↓
產出完成報告（三段式格式）
  ↓
歸檔至 knowledge/ 或 memory/
```

## 資訊匯整模板

```markdown
# 跨代理任務追蹤：{任務名稱}
> 建立時間：YYYY-MM-DD
> 狀態：進行中 / 已完成

| # | 子任務 | 代理 | 狀態 | 完成時間 |
|---|--------|------|------|---------|
| 1 | ... | agong | ✅ | 03-15 10:00 |
| 2 | ... | design | ⏳ | - |
| 3 | ... | review | ⏸ 等 #2 | - |
```

## 主動巡檢項目

阿秘每日應主動檢查：
- [ ] 各代理 MEMORY.md 是否有更新
- [ ] inbox 積壓是否需要提醒
- [ ] 跨代理任務是否有卡住的環節
- [ ] 今日日誌是否需要特別記錄的事件
