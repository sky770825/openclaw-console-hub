## 2. 日報資料收集方法

### 任務數據（從 Supabase 取）
```json
{
  "action": "query_supabase",
  "table": "tasks",
  "select": "id, name, status, owner, updated_at",
  "filters": { "updated_at": "gte.今天日期T00:00:00" },
  "order": "updated_at.desc"
}
```

### Server 狀態
```json
{
  "action": "read_file",
  "path": "/Users/caijunchang/.openclaw/automation/logs/taskboard.log"
}
```
只需看最後 50 行，找 ERROR / WARN 關鍵字。

### 版本號
```json
{
  "action": "read_file",
  "path": "/Users/caijunchang/Downloads/openclaw-console-hub-main/package.json"
}
```
取 `"version"` 欄位。

### 向量庫統計
```json
{
  "action": "query_supabase",
  "table": "knowledge_chunks",
  "select": "count"
}
```

## 3. 任務進度追蹤格式

### 追蹤表（持續更新）
```markdown
## 任務追蹤表（最後更新：HH:MM）

| ID | 任務 | 狀態 | 進度 | 開始 | 預計完成 | 實際完成 | 負責人 |
|----|------|------|------|------|---------|---------|--------|
| T-001 | [名稱] | ✅ done | 100% | 03-04 | 03-04 | 03-04 | 小蔡 |
| T-002 | [名稱] | 🔄 doing | 40% | 03-04 | 03-05 | — | 子代理 |
| T-003 | [名稱] | ⏳ pending | 0% | — | — | — | 待定 |
| T-004 | [名稱] | ❌ blocked | 20% | 03-03 | — | — | 小蔡 |
```

### 狀態圖示
| 圖示 | 狀態 | 說明 |
|------|------|------|
| ⏳ | pending | 等待開始 |
| 🔄 | doing | 執行中 |
| ✅ | done | 已完成 |
| ❌ | blocked | 被阻塞 |
| 🚫 | cancelled | 取消 |

## 4. 重要事項提醒規則

### 什麼必須提醒老蔡
- 系統異常超過 10 分鐘未恢復
- 任務連續失敗 3 次以上
- 安全相關的告警（auth / API key / injection）
- 老蔡明確交代要回報的事項
- 版本號跳了多版（可能有人忘記同步）
- 磁碟 / 記憶體使用異常

### 什麼不需要提醒
- 正常的 auto-executor 執行紀錄
- 已知的 warning（沒有實際影響的）
- 已自動修復的小問題

### 提醒格式
```
⚠️ [重要等級] [一句話標題]
詳情：[2-3 句話說明]
建議：[阿秘的建議處理方式]
```
