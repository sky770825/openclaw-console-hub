## 美業網站預約排班系統 - 規劃總覽

### 1. 文案規劃 (阿秘)
- **標題文案：** 網站主標題、頁面/區塊標題。
- **描述文案：** 服務項目、預約步驟、系統功能介紹。
- **按鈕文字：** 各類操作按鈕。
- **提示訊息：** 輸入提示、成功/錯誤/確認訊息。
- **空狀態文案：** 無預約/無服務/無結果。
- **推播與通知模板：** 預約確認/提醒/取消、活動優惠、生日祝福。
- **目標：** 確保文案親切、資訊明確、符合品牌形象。

### 2. 前端技術方案 (阿工)
- **技術棧：** Next.js 14 (App Router) + TypeScript, Tailwind CSS + shadcn/ui, Zustand。
- **互動功能：** react-day-picker (日曆), @dnd-kit (拖拉排班), 格子式時段選擇器。
- **動畫效果：** Framer Motion (頁面轉場、確認動畫)，避免過度。
- **RWD 策略：** Mobile First，手機版底部按鈕/單欄，平板/桌面雙欄。
- **LINE LIFF 串接：** LIFF v2 SDK 嵌入、自動帶入身份、發送確認訊息、Rich Menu 綁定。
- **WebSocket 即時更新：** Socket.io (時段灰掉、新預約通知、排班狀態)，備案 SSE。
- **效能重點：** SWR 快取、Next/Image 優化、3 步驟預約流程。
- **下一步：** 搭建專案骨架、日曆原型、LIFF 環境設定。

### 3. 系統架構規劃 - Phase 1 MVP (阿策)
- **目標：** 快速上線穩定預約系統，驗證市場需求。
- **功能模組：** 店家管理、服務項目管理、人員班表管理、前台預約、後台預約管理、基礎會員系統。
- **頁面結構：** 顧客端 (首頁、服務列表、預約流程、會員中心)、店家端 (登入、儀表板、行事曆、設定頁)。
- **用戶流程：** 顧客預約流程、店家管理流程。

### 4. UI/UX 競品分析與市場機會 (阿研)
- **市場決勝點：** UX 流暢度、營運效率、顧客溝通深度整合。
- **LINE 官方帳號整合：** 市場必要條件，作為 CRM 與自動化行銷核心。
- **未來機會：** 「智慧化」與「個人化」 (AI 簡化預約、個人化推薦、自動化工作流程)。
- **市場概況：**
    - **店家痛點：** 人力成本、資訊混亂、高爽約率、行銷困難。
    - **消費者期待：** 即時便利、操作直覺、個人化體驗、資訊透明。
- **競品分析：** H牌 (全功能)、S牌 (輕量化)、ezPretty (生態系整合)、LINE+外掛 (無縫 LINE 體驗)。
- **設計趨勢：** AI 整合的對話式預約 (Conversational UI)。
