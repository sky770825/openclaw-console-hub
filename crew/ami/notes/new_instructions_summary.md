🔹 聊天 vs 任務
- **聊天**：直接回覆文字，用於問候、感想、簡單問答、確認。
- **任務**：使用 action 工具執行，例如查資料、改代碼、建任務、分析、部署。

🔹 燈號系統 - 行動準則
- **🟢 綠燈 (直接做)**：讀檔、搜尋、查資料、知識庫更新（已存在檔案）、程式碼格式化、git commit（不 push）、跑測試、寫 RESULT.md、回報任務進度、向量資料庫讀寫。這些行動我會直接執行。
- **🟡 黃燈 (先說再做)**：新建目錄或知識庫、安裝新套件、修改程式碼邏輯、clone 新 repo、建立任務板任務、手動發 Telegram 通知。我會先說明我將要執行的動作，若 5 秒內沒有收到阻擋，則會執行。
- **🔴 紅燈 (需明確批准)**：刪除檔案或目錄、git push、修改系統配置、花錢的 API 呼叫、停止/重啟服務、宣布「定版」里程碑、建立新的向量資料庫 collection。這些行動必須得到「好/可以/執行」的明確批准後才能執行。

🔹 OpenClaw 任務 API
- 我可以透過 API 建立、更新、刪除、查詢任務，這是我在需要管理任務時可以使用的工具。

🔹 其他相關資訊
- **Agent 連接器規範**：關於 Agent 間通訊的技術細節，有助於我理解系統架構。
- **斷路器機制**：系統防止錯誤擴大的機制，了解其運作有助於我理解系統穩定性。

這些指南將幫助我更有效地理解和執行指令，並確保我的操作符合系統規範。
