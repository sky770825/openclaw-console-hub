## 虛擬貨幣網站專案初期規劃彙整

### 專案背景
為建置一個「簡潔、安全、高效」的新世代虛擬貨幣交易所，團隊成員已針對前端技術、文案、系統架構及產品需求進行初步規劃與討論。

### 1. 阿工 - 前端技術方案建議

*   **互動功能：**
    *   圖表：TradingView Lightweight Charts 或 ECharts (即時價格 K 線圖)
    *   下單介面：React Hook Form 搭配 Zod (表單驗證)
    *   資料列表：TanStack Table (虛擬化捲動)
*   **動畫效果：**
    *   價格跳動：CSS flash 動畫
    *   UI 過場：Framer Motion (流暢物理動畫)
*   **響應式網頁設計 (RWD)：** Tailwind CSS (utility-first 框架)
*   **LINE LIFF 串接：** 官方 LIFF SDK (身份驗證、訊息通知)
*   **WebSocket 即時更新：** Socket.IO-client 或 ws (長連線，即時更新圖表、訂單簿)

### 2. 阿秘 - 文案內容建議 (已於 [2026-03-13 21:14] 提交)

*   **核心文案：** 網站首頁標題、描述、功能區塊標題 (安全性、交易速度)。
*   **按鈕文字 (Call to Action)：** 註冊/登入、交易操作、引導操作、確認操作。
*   **提示訊息：** 密碼欄位、2FA 驗證碼、交易手續費提示。
*   **空狀態文案：** 錢包無資產、無交易紀錄、搜尋不到幣種。
*   **推播與通知模板：** 價格提醒、安全警示、入金成功、提領確認。

### 3. 阿策 - 系統架構路線圖規劃

**三階段路線圖：**

*   **Phase 1: 資訊聚合與資產追蹤器 (MVP)**
    *   **目標：** 快速上線，建立核心用戶群，提供市場行情與個人化資產追蹤。
    *   **功能模組：** 市場行情、新聞資訊、使用者認證、用戶資產組合。
    *   **頁面結構：** 首頁、幣種列表、幣種詳情、新聞中心、我的資產。
    *   **資料模型 (初步)：** User, Coin, PortfolioAsset, NewsArticle。
    *   **API 設計 (核心端點)：** 取得市場幣種、單一幣種資料、新聞、註冊、用戶資產組合、新增/修改資產。
    *   **第三方串接：** CoinGecko API (行情數據)、Firebase Authentication (使用者認證)、TradingView Lightweight Charts (圖表)。
    *   **風險評估：** 第三方數據源穩定性，緩解措施為快取機制。
*   **Phase 2: 交易功能整合與社群互動**
    *   **目標：** 增加用戶黏性，拓展功能深度 (錢包餘額同步、下單、社群機制)。
*   **Phase 3: 自動化策略與數據分析平台**
    *   **目標：** 建立商業護城河，提供進階付費功能 (自動跟單、量化交易、鏈上數據分析)。

### 4. 阿研 - 產品需求文件 (PRD)

*   **概述與背景：** 打造「簡潔、安全、高效」的交易所，專注 UI/UX、行動體驗、LINE 整合。
*   **目標用戶：**
    *   **加密貨幣新手：** 需求極簡直觀介面、清晰引導、可靠客服。
    *   **經驗豐富的交易者：** 需求強大、反應迅速的「進階模式」、即時通知。
*   **產品目標：** 提升用戶獲取率、增強信任感與留存率、打造高效交易體驗、建立即時溝通管道。
*   **成功指標：** 新用戶 7 天內首筆交易 > 60%，MAU 持續成長，次月留存 > 40%，CSAT > 90%，行動用戶 > 70%。
*   **功能需求 (Features & Requirements)：**
    *   **核心 UI/UX 設計原則：** FR-1 雙模式介面 (基本/進階)、FR-2 簡潔直觀設計。
    *   **強化安全性與信任感：** FR-3 強制性雙因素認證 (2FA)、FR-4 生物辨識登入、FR-5 清晰安全狀態指示。
    *   **行動裝置優先：** FR-6 響應式網頁設計與原生 App。
    *   **LINE 機器人通知與客戶支援整合：** FR-7 LINE 帳號綁定與即時交易通知、FR-8 自訂價格與新幣上市警報、FR-9 24/7 全天候多管道客戶支援、FR-10 AI 聊天機器人與人工客服協作。
