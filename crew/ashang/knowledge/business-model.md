## 自動化三階段

```
手動 ──→ 半自動 ──→ 全自動
(人做)   (人觸發)   (排程/webhook)
```

### 階段判斷
| 階段 | 適用情境 | 例子 |
|------|---------|------|
| **手動** | 一次性/低頻/高風險 | 資料庫 migration |
| **半自動** | 每天做但需人確認 | 日報生成 → 人審 → 發送 |
| **全自動** | 高頻重複/低風險 | 健康檢查 → 告警 → 通知 |

## n8n 工作流設計模式

### 1. Webhook → 處理 → 通知
```
[Webhook Trigger] → [處理邏輯] → [Telegram/Email 通知]
```
適用：接收外部事件，自動回應

### 2. 排程 → 拉取 → 分析 → 通知
```
[Cron Trigger] → [HTTP Request 拉數據] → [Code 分析] → [IF 異常] → [通知]
```
適用：定期監控、日報

### 3. 表單 → 審核 → 執行
```
[Form Trigger] → [人工審核] → [IF 批准] → [執行 API] → [記錄結果]
```
適用：需要老蔡確認的任務

### 4. AI Agent 工作流
```
[Trigger] → [AI Agent(Claude/Gemini)] → [Tool Use] → [Output]
```
適用：需要 AI 判斷的複雜任務

## 常見自動化場景

| 場景 | 觸發 | 動作 | 工具 |
|------|------|------|------|
| **任務完成通知** | 任務狀態改變 | Telegram 通知老蔡 | n8n webhook |
| **每日系統報告** | 每天 09:00 | 拉 Supabase 數據 → 生成報告 | n8n cron |
| **錯誤自動告警** | Server error log | 分類 → 通知 → 建 task | n8n + Telegram |
| **代碼部署** | git push | build → test → deploy | GitHub Actions |
| **客戶回覆** | 收到訊息 | AI 草擬 → 人確認 → 發送 | n8n + Gemini |

## ROI 計算公式

```
自動化 ROI = (手動時間 × 頻率 × 時薪 - 自動化成本) / 自動化成本 × 100%

例：
- 手動時間：30 分鐘
- 頻率：每天
- 時薪：$50
- 自動化成本：$10/月（SaaS）+ 2 小時建置（$100）

月省時：30min × 30天 = 15 小時 = $750
月成本：$10
建置成本：$100（攤 3 個月 = $33/月）
月 ROI = ($750 - $43) / $43 × 100% = 1644%
```

## 流程設計 SOP
1. **發現痛點**：問「這件事要做幾次？每次花多久？」
2. **畫流程圖**：手動步驟 → 哪些能自動化
3. **選工具**：n8n 優先（自架免費），不行再看 Zapier/Make
4. **先半自動**：人觸發 + 自動執行，確認穩定
5. **再全自動**：加排程/webhook，完全不需人
6. **算 ROI**：省多少時間 × 價值 vs 成本
