# SaaS Landing Page 技術選型與調研總結

## 1. 阿工的技術 POC 規劃

阿工的報告強烈建議採用 *Astro* 框架來開發 OpenClaw 官網功能特色頁面。其主要理由是 Astro 的「孤島架構 (Islands Architecture)」能夠完美平衡「高互動性」與「極致載入速度」。這意味著非互動內容將以純靜態 HTML 形式呈現，確保快速載入，而只有在特定需要互動的組件（如定價切換、表單等）才會載入對應的 JavaScript。

*POC 建議*：阿工規劃了一個 2-4 小時的 POC 流程，包括初始化 Astro 專案、整合 React、部署到 Vercel 並進行性能評估，目標是行動版 PageSpeed Insights 達到 95+ 分。產出將是一個可運行的 POC 網站和性能報告。

## 2. 阿研的外部調研成果

阿研成功調研了 Vercel 和 Raycast 等頂級 SaaS 產品的登陸頁面。他總結了 SaaS 登陸頁面的最佳實踐，包括：
*   *技術棧*：通常採用現代前端框架如 Next.js、React 等，並結合優化工具。Astro 也越來越受歡迎。
*   *互動性*：透過動畫、微互動和視覺效果來吸引用戶，但需注意不要影響性能。
*   *載入速度優化*：利用 CDN、圖片優化、代碼分割、延遲載入等技術確保快速載入，這也是用戶體驗的關鍵。

這些案例印證了阿工對 Astro 的推薦，特別是在追求速度與局部互動的平衡上。

## 3. 我的初步判斷

綜合阿工和阿研的報告，*Astro 搭配 React* 是目前最符合「OpenClaw 官網功能特色頁面」需求的技術選型。它既能提供高性能的靜態內容，又能靈活地引入互動組件，同時還能利用 Vercel 進行高效部署，完全不依賴現有 NestJS 後端。
