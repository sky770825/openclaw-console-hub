# 北極星指標定義

## OpenClaw 北極星指標

> **Weekly Active Automated Tasks Completed Successfully**
> （每週成功完成的自動化任務數）

## 為什麼是這個指標

此指標同時衡量兩個關鍵維度：

1. **用戶參與度**：用戶持續建立並運行自動化任務 → 代表產品有價值
2. **系統可靠性**：任務「成功完成」→ 代表系統運作穩定

如果只看「建立任務數」，可能有人建了不用；如果只看「系統正常運行時間」，可能沒人在用。此指標結合兩者。

## 支撐指標（Supporting Metrics）

| 指標 | 說明 | 與北極星的關係 |
|------|------|----------------|
| 任務建立率 | 每週新建任務數 | 上游輸入（input） |
| 任務完成率 | 成功 / 總執行數 | 品質指標 |
| Agent 使用率 | 活躍 Agent 數 / 總 Agent 數 | 資源效率 |
| 錯誤率 | 失敗任務 / 總執行數 | 反向品質指標 |

## 反指標（Anti-Metrics）

這些指標需要監控，確保北極星的增長不是以犧牲它們為代價：

| 反指標 | 警戒線 | 說明 |
|--------|--------|------|
| 每任務成本 | 不應持續上升 | 成功任務增加但成本失控 = 不可持續 |
| 虛假完成率 | < 2% | 任務標記「成功」但實際未達預期 |
| 用戶覆寫率 | 應逐漸下降 | 用戶頻繁手動介入 = 自動化品質不足 |

## 審視週期

- **每季度重新評估**北極星指標是否仍然合適
- 評估要點：
  - 指標是否仍與商業目標對齊？
  - 是否有被「遊戲化」（gaming）的風險？
  - 是否需要調整定義（例如從 weekly 改為 daily）？
- 記錄每次評估的結論與理由
