# 實驗記錄模板

## 標準格式

每個實驗必須使用以下格式記錄。可直接複製此模板填寫。

---

```markdown
# 實驗記錄

## 基本資訊
- **Experiment ID**: AB-YYYYMM-NNN
- **Owner**: [負責人名稱]
- **日期範圍**: YYYY-MM-DD → YYYY-MM-DD
- **狀態**: 規劃中 / 進行中 / 已完成

## 假設
> 如果我們 [做 X]，那麼 [指標 Y] 會 [改善/提升/降低] [Z%]，
> 因為 [原因/機制]。

## 指標

### 主要指標（Primary Metric）
- 指標名稱：
- 當前基線值：
- 目標改善幅度：

### 護欄指標（Guardrail Metrics）
- [ ] [指標 1]：不應低於 [值]
- [ ] [指標 2]：不應低於 [值]

## 變體描述

| 變體 | 說明 |
|------|------|
| Control（A） | [現有版本描述] |
| Treatment（B） | [實驗版本描述] |

## 流量分配
- Control: 50%
- Treatment: 50%
- 預估所需樣本量：[數字]
- 預估達到所需天數：[天數]

## 結果

### 數據摘要
| 指標 | Control | Treatment | Δ | p-value | 顯著？ |
|------|---------|-----------|---|---------|--------|
| [主要指標] | | | | | |
| [護欄指標 1] | | | | | |

### 效果大小（Effect Size）
- 相對提升：[X%]
- 絕對提升：[Y]
- 信賴區間：[lower, upper]

## 決策
- [ ] **Ship** — 全量上線
- [ ] **Iterate** — 調整後再測
- [ ] **Kill** — 停止，方向不對

## 學習
> 無論結果如何，記錄從本實驗學到的關鍵洞察：
>
> 1. [學習 1]
> 2. [學習 2]
> 3. [學習 3]

## 後續行動
- [ ] [行動 1]
- [ ] [行動 2]
```

---

## 使用注意事項

1. **實驗開始前**必須填完「基本資訊」到「流量分配」的所有欄位
2. **不可事後修改假設**——假設在實驗開始前鎖定
3. **結果區塊**在實驗結束後填寫，需包含統計數據
4. **學習區塊**是最重要的部分——即使實驗失敗，學習也有價值
5. 所有完成的實驗記錄存放於 `~/.openclaw/workspace/crew/growth/notes/experiments/`
