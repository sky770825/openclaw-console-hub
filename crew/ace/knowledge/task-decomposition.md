# 阿策專屬 — 任務拆解框架
> 你是阿策（🎯 策略師），不是小蔡，這是你的專屬知識庫

---

## 1. 拆解流程（收到需求後依序執行）

### Step 1：理解原始需求
- 讀完需求後，用一句話重述核心目標
- 確認「完成」的定義是什麼（交付物、驗收標準）
- 問自己：「老蔡最終想看到什麼？」

### Step 2：識別關鍵產出
列出所有需要交付的東西：
- 程式碼檔案（哪些 `.ts`、`.tsx`）
- API endpoint
- 資料庫 table / migration
- 前端 UI 元件
- 測試 / 文件

### Step 3：MECE 拆解（互斥且完整）
```
大任務
├── 子任務 A（獨立可執行，不和 B 重疊）
│   ├── 具體步驟 A1
│   └── 具體步驟 A2
├── 子任務 B（獨立可執行，不和 A 重疊）
│   └── 具體步驟 B1
└── 子任務 C（整合測試）
```

**MECE 檢查清單：**
- [ ] 每個子任務的範圍有沒有重疊？（互斥）
- [ ] 所有子任務加起來是否涵蓋整個需求？（完整）
- [ ] 每個子任務是否可以獨立指派給一個人做？

### Step 4：依賴分析
畫出依賴關係，找出關鍵路徑：

```
[A] ──→ [C] ──→ [E（整合測試）]
[B] ──→ [D] ──↗
```

**依賴類型：**
| 類型 | 說明 | 範例 |
|------|------|------|
| 硬依賴 | 一定要先完成才能開始下個 | DB schema → API → 前端 |
| 軟依賴 | 可以先用 mock 並行做 | 前端用假資料先做，API 後接 |
| 無依賴 | 完全獨立，可並行 | 文件撰寫 vs 程式開發 |

### Step 5：估時與排程
每個子任務標註：
- **預估時間**：S(< 30min) / M(1-2hr) / L(半天) / XL(一天+)
- **指派對象**：小蔡主線 / 子代理 / Gemini
- **優先序**：P0(立刻) / P1(今天) / P2(本週) / P3(有空再做)

## 2. 拆解模板（直接複製填寫）

```markdown
## 任務拆解：[需求名稱]

**核心目標**：一句話
**完成定義**：交付什麼 + 怎樣算驗收通過

### 子任務列表

| # | 子任務 | 依賴 | 預估 | 指派 | 優先 | 狀態 |
|---|--------|------|------|------|------|------|
| 1 | [具體描述] | 無 | S | 小蔡 | P0 | pending |
| 2 | [具體描述] | #1 | M | 子代理 | P0 | pending |
| 3 | [具體描述] | #1 | M | Gemini | P1 | pending |
| 4 | [整合測試] | #2,#3 | S | 小蔡 | P1 | pending |

### 關鍵路徑
#1 → #2 → #4（瓶頸在 #2）

### 風險
- [可能的阻塞點]
- [技術不確定性]
```

## 3. 常見拆解模式

### 模式 A：新功能開發
```
1. 資料層（DB schema / Supabase table）
2. 後端 API（Express route + handler）
3. 前端 UI（React component）
4. 串接（前端呼叫 API）
5. 測試驗證（curl + 手動測試）
6. 版本號 + commit + push
```

### 模式 B：Bug 修復
```
1. 重現問題（找到具體錯誤 log）
2. 定位根因（grep / read 相關檔案）
3. 修復代碼
4. 驗證修復（重現步驟不再出錯）
5. 回歸測試（確認沒弄壞其他東西）
6. 版本號 + commit + push
```

### 模式 C：系統優化
```
1. 量測現狀（基準數據）
2. 分析瓶頸（profiling / log 分析）
3. 制定方案（至少 2 個選項）
4. 實施最佳方案
5. 量測改善（對比基準）
6. 文件記錄
```

### 模式 D：多 Bot 協作任務
```
1. 定義各 bot 職責邊界
2. 定義 bot 間傳遞的資料格式
3. 各 bot 獨立執行（可並行）
4. 收集各 bot 產出
5. 整合 + 品質檢查
6. 交付
```

## 4. 拆解品質檢查

拆完後自問：
- [ ] 每個子任務是否有明確的「做完了」標準？
- [ ] 最小的子任務是否能在 2 小時內完成？
- [ ] 有沒有遺漏的步驟（版本號？push？通知老蔡？）
- [ ] 並行度是否最大化？（能同時做的不要串行）
- [ ] 是否考慮了失敗的備案？
