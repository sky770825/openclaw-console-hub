# 風險評估格式規範

> 學習自外部系統「法務蝦」的結構化風險評估方法

## 三段式風險評估

所有涉及風險的審查，必須使用以下格式：

### 格式
```markdown
## 1. 風險評級
- 等級：🔴 高風險 / 🟡 中風險 / 🟢 低風險
- 影響範圍：{誰會受影響、影響多大}
- 緊急程度：立即處理 / 本週處理 / 可排期

## 2. 具體問題（白話說明）
- 問題 A：{用非技術語言描述，讓主人能看懂}
- 問題 B：{同上}
- 涉及條目：{具體行號/檔案/設定}

## 3. 建議方案（具體可執行）
- 方案 A（推薦）：{具體步驟}
  - 優點：___
  - 缺點：___
- 方案 B（備選）：{具體步驟}
  - 優點：___
  - 缺點：___
```

## 風險評級標準

### 🔴 高風險（P0 — 立即處理）
- 資料外洩或安全漏洞
- 服務完全中斷
- 對外發布的內容有嚴重錯誤（價格/聯絡方式錯誤）
- 違反品牌核心原則

### 🟡 中風險（P1 — 本週處理）
- 功能部分異常但有 workaround
- 內容品質未達標但不影響安全
- 效能退化但服務仍可用
- SEO 排名可能受影響

### 🟢 低風險（P2 — 可排期）
- 排版/美觀問題
- 非核心功能的小 bug
- 可優化但不影響使用的項目
- 文件/註釋缺失

## 審查報告範例

```markdown
# 審查報告：{內容/代碼名稱}
> 審查人：review 代理
> 日期：YYYY-MM-DD
> 結果：PASS / FAIL / CONDITIONAL PASS

## 1. 風險評級
- 等級：🟡 中風險
- 影響範圍：社群貼文觸及的潛在客戶
- 緊急程度：發布前修正即可

## 2. 具體問題
- 問題 A：第 3 段的療程價格與 brand-facts.json 不一致
  （貼文寫 $2,800，brand-facts 記錄 $3,200）
- 問題 B：缺少免責聲明

## 3. 建議方案
- 方案 A（推薦）：更新貼文價格為 $3,200，加入免責聲明
- 方案 B：確認 brand-facts.json 是否需要更新價格

## 品牌語氣評分
| 項目 | 分數 (1-5) |
|------|-----------|
| 溫暖度 | 4 |
| 專業度 | 3 |
| 行動導向 | 4 |
| 品牌一致性 | 4 |
| 總分 | 15/20 ⚠️ (需 ≥16) |

## 結論
CONDITIONAL PASS — 修正價格和語氣後可發布
```

## 邊界聲明

對於超出 AI 能力範圍的風險（如正式法律意見、財務審計），review 代理必須：
1. 標註「建議諮詢專業人士」
2. 提供前期分析與整理供專業人士參考
3. 不做最終法律/財務判斷
