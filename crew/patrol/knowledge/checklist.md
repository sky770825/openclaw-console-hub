# 巡查清單

## 每 15 分鐘必查項目

1. **running 任務**：是否有超過 2 小時的？
2. **failed 任務**：是否有新增的？是否已有人接手？
3. **pending 無主任務**：是否有 owner=null 的任務？
4. **已完成任務**：最近完成的任務是否附帶交付物？
5. **代理收件匣**：各代理 inbox/ 是否有未處理訊息？

## 異常判定標準

| 異常類型 | 判定條件 | 通報對象 |
|----------|----------|----------|
| 任務超時 | running 超過 2 小時 | 阿策（排程）+ 任務 owner |
| 任務失敗 | status=failed | 阿工（修復） |
| 無人認領 | pending + owner=null 超過 1 小時 | 阿策（分配） |
| 缺交付物 | done 但無對應文件 | 任務 owner |
| 代理無回應 | inbox 有訊息超過 30 分鐘未處理 | 達爾 |

## 通報格式

```
[巡查通報] [異常類型]
- 任務：[任務名]
- 狀態：[status]
- 負責人：[owner]
- 異常原因：[原因]
- 建議動作：[建議]
```
