# 巡查官升級矩陣（Escalation Matrix）
> 當巡查發現異常時，依據本矩陣決定通報對象與緊急度。

---

## 嚴重等級定義

| 等級 | 名稱 | 回應時限 | 說明 |
|------|------|----------|------|
| **P0** | 立即處理 | 即時 | 系統癱瘓、安全事件、資料外洩風險 |
| **P1** | 一小時內 | 1hr | 關鍵功能中斷、部署失敗、預算超標 |
| **P2** | 下個週期 | 下次巡查 | 非緊急異常、效能下降、待觀察項目 |

---

## 升級規則

### 1. 任務被依賴阻塞超過 30 分鐘
- **通報對象**：ace（阿策）
- **嚴重等級**：P1
- **動作**：建立高優先任務，請阿策重新排程或解除依賴
- **訊息範例**：`任務 [task_name] 因等待 [dependency] 已阻塞 [N] 分鐘，請阿策介入排程`

### 2. 建置/部署失敗
- **通報對象**：agong（阿公）
- **嚴重等級**：P1
- **動作**：立即通報阿公，附上失敗日誌摘要
- **訊息範例**：`部署失敗：[service_name] build error — [error_summary]，請阿公排查`

### 3. 安全事件
- **通報對象**：dar（達爾）— 立即通報
- **嚴重等級**：P0
- **動作**：立即凍結相關服務，通報達爾，記錄事件時間線
- **觸發條件**：未授權存取、API 金鑰外洩、異常流量激增、資料異常存取
- **訊息範例**：`[P0 安全事件] 偵測到 [incident_type]，已凍結 [affected_service]，等待達爾指示`

### 4. 預算門檻超標
- **通報對象**：ashang（阿商）
- **嚴重等級**：P1
- **動作**：通報阿商並暫停相關支出動作
- **觸發條件**：API 成本超出日預算 80%、訂閱費用異常、資源使用量激增
- **訊息範例**：`預算警告：[resource] 今日用量已達 [N]%，預估超標 [amount]，請阿商確認`

### 5. Agent 無回應超過 15 分鐘
- **通報對象**：dar（達爾）
- **嚴重等級**：P1
- **動作**：嘗試重啟該 agent，若重啟失敗則升級為 P0
- **訊息範例**：`[agent_name] 無回應已 [N] 分鐘，已嘗試重啟 [成功/失敗]，通報達爾`

---

## 升級流程

```
偵測異常 → 判斷類型 → 查矩陣 → 確定嚴重等級
  → P0：立即通報（Telegram + inbox 訊息）
  → P1：1 小時內通報（inbox 訊息 + 建追蹤任務）
  → P2：記錄於下次巡查報告
```

## 注意事項

- 同一事件不重複通報（已通報且在處理中的不再發送）
- P0 事件必須確認對方已讀，未確認則每 5 分鐘重發
- 所有升級事件都必須記錄到 journal agent
- 升級後持續追蹤直到問題解決，解決後發送 all-clear 通知
