# 主旨行 A/B 測試框架

## 測試流程

```
假設 → 設計變體 → 抽樣發送 → 等待 → 分析 → 全量發送贏家
```

## 測試設定

| 項目 | 規範 |
|------|------|
| 樣本比例 | 總名單的 20%（各 10% 為 A/B 組） |
| 等待時間 | 發送後 2 小時測量開信率 |
| 勝出標準 | 開信率差異 ≥ 2% 且統計顯著 |
| 全量發送 | 勝出版本發送給剩餘 80% |
| 最小樣本 | 每組至少 100 人，否則結果不可靠 |

## 可測試變數（每次只測 1 個）

### 1. 長度
- **短** (< 30 字元): 「居家保養 3 步驟」
- **長** (30-50 字元): 「首席美容師分享：3 個讓肌膚持續透亮的居家護理秘訣」

### 2. 個人化
- **有**: 「[姓名]，這是你的專屬保養方案」
- **無**: 「你的專屬保養方案」

### 3. 數字 vs 無數字
- **有數字**: 「5 分鐘完成的晚間護膚步驟」
- **無數字**: 「快速完成的晚間護膚步驟」

### 4. 問句 vs 陳述
- **問句**: 「你的保養順序做對了嗎？」
- **陳述**: 「正確的保養順序，效果差 3 倍」

### 5. Emoji vs 無 Emoji
- **有**: 「春季護膚指南 🌸」
- **無**: 「春季護膚指南」

### 6. 急迫性
- **有**: 「本週限定：深層保養 85 折」
- **無**: 「深層保養體驗優惠」

## 測試紀錄格式

```markdown
## 測試 #[編號]
- **日期**: YYYY-MM-DD
- **假設**: [如果我們用問句式主旨，開信率會更高]
- **變數**: [問句 vs 陳述]
- **A 版**: [你的保養順序做對了嗎？]
- **B 版**: [正確的保養順序，效果差 3 倍]
- **樣本**: A: [人數] / B: [人數]
- **結果**: A 開信率 [X%] / B 開信率 [Y%]
- **勝出**: [A/B]
- **差異**: [Z%]
- **學習**: [問句式主旨在保養類內容中表現更好]
```

## 累積學習總結

維護一份持續更新的「已知規律」清單：

| 測試主題 | 結論 | 信心度 |
|----------|------|--------|
| （範例）Emoji | 美甲主題 +emoji 提升 3% 開信率 | 中 (2 次測試) |
| （範例）個人化 | 加入姓名平均提升 4% 開信率 | 高 (4 次測試) |

- **低信心**: 1 次測試
- **中信心**: 2-3 次測試且結果一致
- **高信心**: 4+ 次測試且結果一致

## 注意事項

- 避免在節假日或特殊事件期間做測試（數據會失真）
- 每月至少做 2 次主旨行測試
- 測試結果回饋給 content 代理，影響未來標題撰寫風格
- 不測試誤導性主旨（違反品牌調性）
