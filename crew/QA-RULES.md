# Crew Bots 驗收規則（QA Rules）

> 所有 crew bot 交接後的成果，必須經過驗收。
> 默認立場：NEEDS_WORK。需要壓倒性證據才給 PASS。

---

## 判定標準

### PASS（通過）— 需要全部滿足

1. **有具體產出物** — 不是純文字報告，而是實際的檔案、代碼修改、數據表、API 回應等
2. **有數據支撐** — 結論必須有數字、log、截圖、或可驗證的事實作為依據
3. **Action 成功執行** — 呼叫的 action（patch_file、query_supabase、web_search 等）確實回傳了結果
4. **結論有依據** — 推論鏈完整，從證據到結論有邏輯連結，不是憑空斷言
5. **符合原始需求** — 做的事情確實是交接時「期望」欄位要求的

### NEEDS_WORK（打回）— 出現任何一項即打回

1. **只有文字沒有 action** — 光說不練，沒有實際呼叫任何工具
2. **結論無依據** — 「我認為」「應該是」「可能」但沒有查證
3. **幻覺內容** — 引用不存在的檔案、虛構的 API 回應、編造的數據
4. **答非所問** — 做的事情和交接要求不符
5. **遺漏關鍵步驟** — 只做了一半，缺少驗證或確認步驟
6. **錯誤未修正** — action 執行失敗但沒有重試或處理
7. **複製貼上** — 直接複製知識庫內容當回答，沒有針對問題分析

---

## 驗收流程

```
第 1 次提交
  ├── PASS → 任務完成，阿秘記錄
  └── NEEDS_WORK → 回饋具體問題，退回執行方
        │
第 2 次提交
  ├── PASS → 任務完成，阿秘記錄
  └── NEEDS_WORK → 回饋問題，再退回
        │
第 3 次提交
  ├── PASS → 任務完成，阿秘記錄
  └── NEEDS_WORK → 升級處理
        │
升級：通知達爾或主人介入
```

## 各角色驗收重點

### 阿研 ayan（研究員）的產出驗收
- **PASS 條件**：引用了實際來源（URL、檔案路徑、知識庫條目）、比較了多個方案、有明確推薦理由
- **常見打回原因**：來源不可驗證、只列方案不比較、結論和證據脫節

### 阿工 agong（工程師）的產出驗收
- **PASS 條件**：代碼能通過 TypeScript 檢查、修改有對應的測試或驗證步驟、patch_file 成功執行
- **常見打回原因**：patch 格式錯誤、沒有驗證修改後的效果、引入新 bug

### 阿策 ace（策略師）的產出驗收
- **PASS 條件**：任務拆解粒度合理（每個子任務可在一個 session 完成）、有明確優先級排序、指定了負責人
- **常見打回原因**：任務太大沒拆細、沒指定負責人、優先級不明確

### 阿秘 ami（秘書）的產出驗收
- **PASS 條件**：文件格式正確、內容完整、已存入正確路徑、知識庫已索引
- **常見打回原因**：格式混亂、遺漏重要資訊、沒有 index_file

### 阿商 ashang（商業分析）的產出驗收
- **PASS 條件**：有市場數據支撐、競品分析有具體對象、商業模式有數字驗證
- **常見打回原因**：泛泛而談沒有具體數據、忽略競品、營收假設不合理

### 阿數 ashu（分析師）的產出驗收
- **PASS 條件**：數據來源明確（Supabase 查詢或 log 分析）、計算過程可覆核、有視覺化或表格呈現
- **常見打回原因**：數字來源不明、計算邏輯有誤、缺少關鍵指標

## 升級條件

以下情況必須升級給達爾或主人：

1. **3 次驗收未通過** — crew bot 之間已盡力但無法達標
2. **需要 git push** — crew bot 無權限推代碼
3. **需要修改敏感檔案** — .env、auth、靈魂文件等
4. **跨系統操作** — 需要動到 server、launchd、n8n 等基礎設施
5. **意見分歧** — 兩個 bot 對結論有重大分歧，無法自行裁決
6. **緊急故障** — 影響到生產環境的嚴重問題，不能慢慢 QA

### 升級格式

```
[升級] crew → 達爾/主人
問題：[什麼問題解決不了]
已嘗試：[crew 做了什麼、失敗了幾次]
建議：[crew 的建議方向]
相關檔案：[涉及的路徑]
```

## 品質分級

| 等級 | 說明 | 對應判定 |
|------|------|---------|
| A | 超出預期，有額外洞見 | PASS + 加分 |
| B | 完整達標，無遺漏 | PASS |
| C | 勉強及格，有小瑕疵但不影響結論 | PASS（附註改善建議） |
| D | 不及格，缺少關鍵內容 | NEEDS_WORK |
| F | 嚴重問題，幻覺或答非所問 | NEEDS_WORK + 警告 |
