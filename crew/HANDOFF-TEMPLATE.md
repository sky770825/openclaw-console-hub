# Crew Bots 任務交接模板

> 適用於所有 crew bot 之間的任務轉交。統一格式，減少資訊遺漏。
> 搭配 QA-RULES.md 使用，確保交接後有驗收閉環。

---

## 交接格式（標準）

每次轉交任務時，轉交方必須在回覆中包含以下結構：

```
[轉交方] → [接收方]
任務：[一句話描述要做什麼]
背景：[已經做了什麼、目前狀態]
期望：[接手後需要完成什麼]
證據：[相關數據、檔案路徑、API 回應、錯誤 log]
優先級：[高/中/低]
```

### 範例

```
阿研 → 阿工
任務：修復 /api/tasks 回傳 500 錯誤
背景：已查 log，定位到 server/src/routes/tasks.ts 第 87 行 null reference
期望：patch_file 修復該 null check，確認 API 回傳正常
證據：log 截取 "TypeError: Cannot read properties of null (reading 'id')" at tasks.ts:87
優先級：高
```

```
阿策 → 阿研
任務：調研 WebSocket 即時通知方案
背景：主人要求任務面版支援即時更新，目前是 polling
期望：找出 2-3 個可行方案，比較優缺點，推薦一個
證據：主人原話見群組訊息 #1234
優先級：中
```

```
阿商 → 阿數
任務：試算 990 專案第一季營收預估
背景：已完成市場調研，目標客群 500 家中小企業，轉換率假設 3-5%
期望：用三種情境（樂觀/中性/保守）算出營收區間
證據：調研報告在 ~/.openclaw/workspace/crew/ashang/990-research.md
優先級：中
```

## 交接觸發方式

### 自動觸發（detectAndHandoff）
在回覆中**直接提到對方的名字**，系統會自動偵測並轉交：
- 提到「阿工」→ 自動轉交給阿工
- 提到「阿研」→ 自動轉交給阿研
- 一次只轉交一個 bot，避免連鎖反應

### 手動觸發
主人或達爾在群組中直接 @ 指定 bot，該 bot 優先接手。

## 驗收循環（QA Loop）

每次交接完成後，進入驗收循環：

```
步驟 1：執行方完成任務 → 提交結果（含具體產出物）
步驟 2：驗收方檢查結果（默認判定 NEEDS_WORK）
步驟 3：需要壓倒性證據才給 PASS
步驟 4：NEEDS_WORK → 執行方修正後重新提交
步驟 5：最多 3 次重試
步驟 6：3 次仍未通過 → 升級給達爾或主人介入
```

### 驗收方是誰？

| 情境 | 驗收方 |
|------|--------|
| 代碼修復 | 阿研（確認 log 乾淨）或 阿數（確認 metrics 正常） |
| 研究調研 | 阿策（確認結論可執行）或 阿商（確認商業邏輯通） |
| 數據分析 | 阿商（確認數字合理）或 阿策（確認可落地） |
| 文件整理 | 阿研（確認內容正確）或 阿策（確認架構清晰） |
| 任務規劃 | 阿工（確認技術可行）或 阿數（確認資源足夠） |
| 商業方案 | 阿策（確認策略對齊）或 阿數（確認數字支撐） |

## 協作矩陣（誰遇到什麼問題轉給誰）

| 遇到的問題 | 轉給 | 原因 |
|-----------|------|------|
| 代碼 bug / 需要改程式 | 阿工 agong | 唯一能寫代碼的 |
| 需要查資料 / 找方案 | 阿研 ayan | 研究能力最強 |
| 需要做決策 / 排優先 | 阿策 ace | 策略全局觀 |
| 需要記錄 / 整理文件 | 阿秘 ami | 文件管理專家 |
| 需要算數字 / 看數據 | 阿數 ashu | 數據分析能力 |
| 需要商業判斷 / 市場資訊 | 阿商 ashang | 商業嗅覺 |
| 超出 crew 能力 / 需要權限 | 達爾 dar | 副指揮官，可調度全局 |
| 重大決策 / 安全敏感 | 主人 | 最終決策者 |

### 常見轉交鏈

```
故障排查：阿研(查log) → 阿工(修code) → 阿數(驗metrics) → 阿秘(寫報告)
新功能：  阿策(拆任務) → 阿研(調研) → 阿工(實作) → 阿數(追數據) → 阿秘(歸檔)
商業分析：阿商(調研) → 阿研(蒐數據) → 阿數(算數字) → 阿策(出計畫) → 阿秘(整報告)
日常巡邏：阿研(掃log) → 阿數(查指標) → [有問題]阿工(修) → 阿秘(記錄)
```

## 交接禁忌

1. **禁止空轉交** — 沒有具體背景和證據就轉，等於甩鍋
2. **禁止循環轉交** — A→B→A 無限迴圈，第二次轉回就升級
3. **禁止越級轉交** — crew bot 之間先協作，解決不了才升級達爾/主人
4. **禁止無產出轉交** — 轉交前必須有自己做過的事，不能只寫「我覺得應該找阿工」
