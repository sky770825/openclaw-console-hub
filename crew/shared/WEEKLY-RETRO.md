# 週回顧模板 v1.0

> 每週一次，回顧上週表現，找出改進點
> 阿秘負責收集資料，阿策負責分析和制定改進計畫

---

## 回顧排程

- **時間**：每週日 20:00（由 journal 觸發）
- **產出**：週回顧報告，存入 `memory/weekly/`
- **參與**：所有代理的產出數據 + 巡查記錄

---

## 資料收集指引

### 阿秘收集項目
1. 本週所有代理的 inbox 處理數量（`find inbox/ -name "*.md" -newer ...`）
2. QA 退回記錄
3. 升級記錄
4. 巡查異常記錄

### 阿數提供項目
1. 系統可用率
2. API 錯誤率趨勢
3. 資源使用趨勢

### 日誌提供項目
1. 本週重要事件摘要
2. 知識庫變更記錄

---

## 月度深度回顧（每月最後一個週日）

在常規週回顧基礎上，額外加入：

- **知識庫健康度**：過期檔案數、覆蓋率
- **流程有效性**：哪些 SOP 被實際使用、哪些形同虛設
- **代理能力評估**：是否需要調整模型或增減代理
- **主人反饋彙整**：本月主人的指正和偏好變化
