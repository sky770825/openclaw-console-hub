# 交付物流水線規範

## 強制流程

所有對外內容（部落格、電子報、社群貼文、Landing Page）必須按以下順序通過每一關，**不可跳關**。

```
content 撰寫初稿
  ↓ 交付物 + brand-facts.json 引用標記
seo 關鍵字審查
  ↓ 修改建議回傳 content
content 修改後重新提交
  ↓
review 品質審查（品牌語氣 + 事實查核 + 安全）
  ↓ 通過 / 退回（附具體修改事項）
design 視覺審查（如涉及 HTML/CSS/圖片）
  ↓
newsletter/social 格式適配（依發佈管道）
  ↓
dar 最終批准
  ↓
發佈
```

## 每一關的通過條件

### Stage 1: content 初稿
- [ ] 所有價格、時長、聯絡方式引用自 `shared/brand-facts.json`
- [ ] 所有統計數據標註來源（論文/報告/官方數據）
- [ ] 無「研究顯示」「據統計」等無來源措辭
- [ ] 符合 `content-types-guide.md` 的格式規範

### Stage 2: seo 審查
- [ ] 標題含核心關鍵字
- [ ] 關鍵字密度 1-3%
- [ ] H2 標題含長尾關鍵字
- [ ] Meta description 已撰寫（150-160 字元）
- [ ] 至少 2 個內部連結
- [ ] 圖片有 alt 標籤描述

### Stage 3: review 審查
- [ ] 品牌語氣評分 ≥ 16/20（且單項 ≥ 3）
- [ ] 快速判定 ≥ 4 個 Yes
- [ ] 事實查核通過（所有聲明有來源）
- [ ] 無安全漏洞（HTML 內容適用）
- [ ] 跨文件一致性確認（同一活動的不同素材資料一致）

### Stage 4: design 視覺審查（HTML/圖片相關）
- [ ] CSS class 與 HTML 對應（無孤立 class）
- [ ] Email HTML 使用 inline style（非 `<style>` 區塊）
- [ ] gradient 有純色 fallback
- [ ] 行動裝置排版正常
- [ ] 字體 ≥ 14px、按鈕 ≥ 44px

### Stage 5: 管道適配
- [ ] 社群貼文：hashtag 15-20 個（含地域型+利基型）
- [ ] 電子報：主旨行 30-50 字元、有預覽文字、有 `[姓名]` 個人化
- [ ] 退訂連結為有效 URL

## 退回規則

- 任一關卡不通過 → 退回上一關修改 → 重新提交
- 退回必須附具體修改事項清單
- 同一交付物退回超過 3 次 → 升級至 ace 重新評估需求

## 跨文件一致性規則

同一活動（如「三月限定優惠」）的所有素材（部落格、電子報、社群貼文）：
- 價格必須一致（引用同一 brand-facts.json 條目）
- 療程時長必須一致
- 優惠截止日必須一致
- 服務描述用語必須一致

**檢查方式**：review 代理在審查時，必須同時讀取同一活動的所有素材進行交叉比對。
