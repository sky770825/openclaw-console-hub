# 結構化交付格式規範 v1.0

> 學習自龍蝦系統的「做了什麼 → 結論 → 下一步」交付模板
> 所有代理完成任務時，必須使用以下格式回報

---

## 標準交付格式

每個任務完成時，回報必須包含以下三個區塊：

```markdown
## 執行摘要（做了什麼）
- 列出具體執行的動作（2-5 項）
- 每項動作需包含「對象 + 動作 + 結果」
- 例：讀取 server/.env，確認 WORKSPACE 路徑已修正為正確值

## 結論與成果（結論）
- 任務是否成功完成（明確的 PASS / FAIL / PARTIAL）
- 產出的交付物清單（檔案路徑、報告、代碼等）
- 關鍵數據或指標

## 下一步建議（下一步）
- 後續需要執行的動作（如有）
- 需要其他代理接手的事項
- 潛在風險或注意事項
```

## 各類任務的交付格式

### 研究類任務（ayan 阿研）

```markdown
## 執行摘要
- 調研主題：___
- 搜索來源：___（web_search / semantic_search / 知識庫）
- 資料筆數：___

## 結論與成果
- 核心發現：___
- 交付物：`knowledge/研究報告名.md`
- 可信度：高 / 中 / 低（依來源品質）

## 下一步建議
- 建議由 ___ 代理進一步處理
- 需要追蹤的後續變化：___
```

### 工程類任務（agong 阿工）

```markdown
## 執行摘要
- 修改檔案：___（列出路徑和行號）
- 修改類型：新增 / 修復 / 重構 / 優化
- 測試狀態：通過 / 失敗 / 未測試

## 結論與成果
- 狀態：PASS / FAIL
- 影響範圍：___
- 交付物：代碼差異 + 測試結果

## 下一步建議
- 是否需要部署 / 重啟服務
- 是否需要 review 代理審查
```

### 策略類任務（ace 阿策）

```markdown
## 執行摘要
- 分析主題：___
- 拆解子任務：___ 項
- 已分配給：___

## 結論與成果
- 優先級排序：P0 > P1 > P2
- 風險評估：高 / 中 / 低
- 交付物：任務清單 + 時間估算

## 下一步建議
- 最優先執行：___
- 阻塞項：___
- 決策門檻：超過 ___ 需主人確認
```

### 巡查類任務（patrol）

```markdown
## 執行摘要
- 巡查時間：___
- 檢查項目：___ 項
- 服務狀態：___/6 正常

## 結論與成果
- 狀態：正常 / 發現異常
- 異常項目：___
- 嚴重程度：P0 緊急 / P1 重要 / P2 一般

## 下一步建議
- 需要 ___ 代理處理
- 預計影響：___
```

### 內容類任務（content / social / newsletter）

```markdown
## 執行摘要
- 內容類型：部落格 / 社群貼文 / 電子報 / 腳本
- 字數：___
- 引用 brand-facts.json：是 / 否

## 結論與成果
- 狀態：初稿完成 / 審查通過 / 已發布
- 交付物：`檔案路徑`
- SEO 標記：已加入 / 不適用

## 下一步建議
- 下一關：seo 審查 / review 品質審查 / design 視覺審查
- 發布管道：___
- 排程時間：___
```

## 決策門檻制度

學習自龍蝦系統的決策門檻，以下操作需要主人確認：

| 操作類型 | 門檻 | 說明 |
|---------|------|------|
| 對外發布內容 | 全部 | 所有對外發布必須主人或達爾最終審批 |
| 系統架構變更 | 全部 | 增減服務、修改核心配置 |
| 刪除檔案/數據 | 全部 | 任何永久性刪除操作 |
| 安裝新套件 | 大型套件 | pip/npm install 大型依賴 |
| 修改 LaunchAgent | 全部 | 開機服務的增減 |

以下操作可自主執行：

| 操作類型 | 說明 |
|---------|------|
| 讀取/搜索檔案 | 知識庫、日誌、代碼 |
| 寫入 inbox | 代理間任務派發 |
| 更新記憶 | MEMORY.md、daily log |
| 修復已知 bug | 明確的錯誤修復 |
| 巡查報告 | patrol 定時巡查 |

## 回報品質檢查清單

每次交付前，代理應自行檢查：

- [ ] 三個區塊都有填寫（摘要 / 結論 / 下一步）
- [ ] 沒有「研究顯示」等無來源措辭
- [ ] 有具體數據而非模糊描述（「修改了 3 個檔案」而非「修改了一些檔案」）
- [ ] 下一步建議是可執行的（有具體代理名稱和動作）
- [ ] 涉及品牌資訊時引用了 brand-facts.json
