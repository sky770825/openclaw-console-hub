# Live2D 開發溝通與風險控管協議

## 目標：解決設計與實作間的溝通不良 (阿工主導)

### 1. 輸出標準化 (The Deliverable)
- 設計師提供的 `.moc3` 檔案必須包含 `physics3.json`。
- 所有的參數命名必須遵循 SDK 標準 (如 `ParamAngleX`)。
- 自定義參數 (如 `ParamStarshipVibration`) 必須在開發文檔中註明範圍 (-1.0 to 1.0)。

### 2. 定期校準流程
- **每週二**: 視覺審查 (檢查動作平滑度)。
- **每週四**: 效能審查 (檢查頂點數與 Texture Atlas 大小，確保不影響網頁加載速度)。

### 3. 風險預案
- **風險**: 瀏覽器 WebGL 支援度不足。
- **對策**: 準備靜態 PNG 降級方案，透過 CSS 模擬簡單浮動。
