## 1. Abstract Factory Pattern（抽象工廠模式）

### 概念說明
Abstract Factory 提供一個建立一系列相關或相依物件的介面，而無需指定它們具體的類別。這讓系統可以根據配置或環境動態決定要建立哪種具體產品。

### TypeScript 範例

```typescript
type SkillType = 'github' | 'notion' | 'weather';

interface Skill {
  name: string;
  execute(): Promise<string>;
}

interface SkillConfig {
  token: string;
  endpoint?: string;
}

interface SkillFactory {
  createSkill(): Skill;
  createConfig(): SkillConfig;
}

class GitHubSkill implements Skill {
  name = 'GitHub Connector';
  async execute() { 
    return 'GitHub data fetched'; 
  }
}

class GitHubFactory implements SkillFactory {
  createSkill() { return new GitHubSkill(); }
  createConfig() { return { token: '' }; }
}

// 使用
const factory: SkillFactory = new GitHubFactory();
const skill = factory.createSkill();
```

### 應用場景（SkillForge）
- **第三方擴展**：讓開發者可以註冊自己的 Skill Factory
- **統一介面**：所有 Skill 建立方式一致
- **類型支援**：API、Scraper、Tool 等不同類型 Skill 可以共用介面

## 2. Builder Pattern（建造者模式）

### 概念說明
Builder 模式將複雜物件的建構過程與其表示分離，讓相同的建構過程能建立不同的表示。特別適合需要多個步驟設定的複雜物件。

### TypeScript 範例

```typescript
class SkillBuilder {
  private skill: Partial<SkillConfig> = {};
  
  setName(name: string) {
    this.skill.name = name;
    return this;  // 鏈式調用
  }
  
  setVersion(version: string) {
    this.skill.version = version;
    return this;
  }
  
  addEndpoint(url: string, method: string) {
    this.skill.endpoints = [...(this.skill.endpoints || []), { url, method }];
    return this;
  }
  
  build(): SkillConfig {
    return this.skill as SkillConfig;
  }
}

// 使用 - Fluent API
const skill = new SkillBuilder()
  .setName('Weather API')
  .setVersion('1.0.0')
  .addEndpoint('/api/weather', 'GET')
  .build();
```

### 應用場景（SkillForge）
- **簡化設定**：複雜 Skill 設定變得可讀
- **驗證機制**：build() 時驗證必填欄位
- **可選參數**：不用每次傳入所有參數

## 3. Strategy Pattern（策略模式）

### 概念說明
Strategy 定義一系列演算法，將每個演算法封裝起來，並且讓它們可以互相替換。策略模式讓演算法的變化獨立於使用演算法的客戶端。

### TypeScript 範例

```typescript
interface ExecutionStrategy {
  execute(task: Task): Promise<Result>;
}

class CursorStrategy implements ExecutionStrategy {
  async execute(task: Task) {
    // 使用 Cursor Agent
    return cursorAgent.run(task);
  }
}

class CoDEXStrategy implements ExecutionStrategy {
  async execute(task: Task) {
    // 使用 CoDEX Agent
    return codexAgent.run(task);
  }
}

class OpenClawStrategy implements ExecutionStrategy {
  async execute(task: Task) {
    // 使用 OpenClaw Agent
    return openclawAgent.run(task);
  }
}

// 使用
class TaskExecutor {
  constructor(private strategy: ExecutionStrategy) {}
  
  async run(task: Task) {
    return this.strategy.execute(task);
  }
  
  setStrategy(strategy: ExecutionStrategy) {
    this.strategy = strategy;
  }
}
```

### 應用場景（SkillForge）
- **引擎切換**：Cursor/CoDEX/OpenClaw 可互換
- **智能選擇**：根據任務類型自動選擇最佳策略
- **擴展性**：容易新增新的執行方式

## 4. Visitor Pattern（訪問者模式）

### 概念說明
Visitor 讓你可以在不修改物件結構的情況下，定義作用於這些物件元素的新操作。適合需要對一組物件執行多種不同操作的場景。

### TypeScript 範例

```typescript
interface ASTNode {
  accept(visitor: ASTVisitor): void;
}

class FunctionNode implements ASTNode {
  constructor(public name: string, public body: ASTNode[]) {}
  accept(visitor: ASTVisitor) {
    visitor.visitFunction(this);
    this.body.forEach(n => n.accept(visitor));
  }
}

class VariableNode implements ASTNode {
  constructor(public name: string, public value: string) {}
  accept(visitor: ASTVisitor) {
    visitor.visitVariable(this);
  }
}

interface ASTVisitor {
  visitFunction(node: FunctionNode): void;
  visitVariable(node: VariableNode): void;
}

class SecretDetector implements ASTVisitor {
  secrets: Array<{name: string, line: number}> = [];
  
  visitFunction(node: FunctionNode) {
    if (node.name.includes('secret') || node.name.includes('key')) {
      console.warn(`函數 ${node.name} 可能處理敏感資訊`);
    }
  }
  
  visitVariable(node: VariableNode) {
    if (/api[_-]?key|token|password/i.test(node.name)) {
      this.secrets.push({
        name: node.name,
        line: node.line || 0
      });
    }
  }
}
```

### 應用場景（EliteGuard）
- **AST 掃描**：遍歷程式碼樹尋找漏洞
- **規則擴展**：不同檢測規則實作不同 Visitor
- **解耦**：新增檢測邏輯不用修改 AST 結構

## 5. 靜態程式碼分析基礎

### AST 是什麼？
**AST（Abstract Syntax Tree，抽象語法樹）** 是原始碼的結構化表示。編譯器會將程式碼解析成樹狀結構，每個節點代表一個語法結構：

```
原始碼: function add(a, b) { return a + b; }
         ↓
AST: 
FunctionDeclaration
├── name: "add"
├── params: ["a", "b"]
└── body:
    └── ReturnStatement
        └── BinaryExpression
            ├── left: Identifier "a"
            ├── operator: "+"
            └── right: Identifier "b"
```

### AST 分析流程
1. **解析（Parse）**：原始碼 → AST（使用 @babel/parser、typescript 等）
2. **遍歷（Traverse）**：使用 Visitor 模式遍歷節點
3. **檢測（Detect）**：在特定節點應用檢測規則
4. **報告（Report）**：輸出發現的問題

### 常見安全漏洞檢測模式

| 漏洞類型 | 檢測模式 | 範例 |
|---------|---------|------|
| Secret Leak | API key 格式字串 | `/['"`]([a-zA-Z0-9]{32,})['"`]/g` |
| AWS Key | AKIA 開頭字串 | `/AKIA[0-9A-Z]{16}/g` |
| Private Key | PEM 格式 | `/-----BEGIN (RSA\|EC)? PRIVATE KEY-----/g` |
| XSS | innerHTML 使用 | 檢測 `innerHTML` + 外部輸入 |
| SQL Injection | 字串拼接 SQL | 檢測 `+` 或 template literal 組合 SQL |

### SkillForge Plugin 介面設計

```typescript
interface SkillPlugin {
  readonly name: string;
  readonly version: string;
  
  // 初始化時呼叫
  initialize(config: Record<string, unknown>): Promise<void>;
  
  // 執行技能
  execute(input: unknown): Promise<SkillResult>;
  
  // 取得輸入/輸出規格
  getSchema(): SkillSchema;
  
  // 清理資源
  dispose(): Promise<void>;
}

// 註冊外掛
type PluginFactory = () => SkillPlugin;

class SkillRegistry {
  private plugins = new Map<string, PluginFactory>();
  
  register(name: string, factory: PluginFactory) {
    this.plugins.set(name, factory);
  }
  
  create(name: string): SkillPlugin | undefined {
    const factory = this.plugins.get(name);
    return factory ? factory() : undefined;
  }
}
```

### EliteGuard Secret 檢測規則

```typescript
interface DetectionResult {
  type: string;
  line: number;
  severity: 'low' | 'medium' | 'high';
  message: string;
}

class SecretDetectionRule {
  private patterns = [
    { type: 'API Key', regex: /['"`]([a-zA-Z0-9]{32,})['"`]/g },
    { type: 'AWS Key', regex: /AKIA[0-9A-Z]{16}/g },
    { type: 'Private Key', regex: /-----BEGIN (RSA |EC )?PRIVATE KEY-----/g },
    { type: 'Password', regex: /password\s*=\s*['"`]([^'"`]+)['"`]/gi },
    { type: 'JWT Token', regex: /eyJ[a-zA-Z0-9_-]*\.eyJ[a-zA-Z0-9_-]*/g },
  ];
  
  scan(code: string, filename: string): DetectionResult[] {
    const results: DetectionResult[] = [];
    const lines = code.split('\n');
    
    for (const pattern of this.patterns) {
      for (let i = 0; i < lines.length; i++) {
        const line = lines[i];
        const matches = line.matchAll(pattern.regex);
        
        for (const match of matches) {
          // 排除註解和測試檔案
          if (this.isComment(line) || this.isTestFile(filename)) continue;
          
          results.push({
            type: pattern.type,
            line: i + 1,
            severity: this.getSeverity(pattern.type),
            message: `發現可能的 ${pattern.type} 洩漏: ${match[0].substring(0, 20)}...`,
          });
        }
      }
    }
    
    return results;
  }
  
  private isComment(line: string): boolean {
    return /^\s*(\/\/|\/\*|\*)/.test(line);
  }
  
  private isTestFile(filename: string): boolean {
    return /\.(test|spec)\.(ts|js)$/.test(filename);
  }
  
  private getSeverity(type: string): 'low' | 'medium' | 'high' {
    const highRisk = ['Private Key', 'AWS Key', 'JWT Token'];
    return highRisk.includes(type) ? 'high' : 'medium';
  }
}
```

## 總結：技術在 5 個商業專案的應用

### SkillForge（第一階段-養活自己）
| 技術 | 應用 |

| Abstract Factory | 統一 Skill 建立介面，第三方擴展 |

| Strategy | 可替換的執行引擎（Cursor/CoDEX/OpenClaw） |

### EliteGuard（第一階段）
| 技術 | 應用 |
|------|------|

| 技術 | 應用 |
|------|------|
| Multi-tenancy | 多租戶隔離架構 |

### AutoWealth（第三階段）
| 技術 | 應用 |
|------|------|
