## 視頻 3: 高級用法 - 多步驟工作流 (4:00)

### 場景: 自動化代碼審查 + PR 評論 + 發送 Slack 通知

**旁白開頭**:
"進階用法：建立一個 3 步工作流。

當有新 PR 時：
1️⃣ AI 自動審查代碼
2️⃣ 發表審查評論
3️⃣ 在 Slack 通知團隊"

**代碼片段** (展示配置):
```yaml
workflow:
  name: Advanced Code Review
  
  triggers:
    - event: github.pull_request.opened
  
  steps:
    - skill: github-pr-analyzer
      config:
        check_style: true
        check_security: true
    
    - skill: github-comment
      input: "{{ steps[0].findings }}"
    
    - skill: slack-notify
      input: "{{ steps[0].summary }}"
```

**實時演示** (3:00-3:40):
- 建立測試 PR
- 觀看 AI 分析
- 查看自動生成的評論
- 看 Slack 通知到達

**結論** (3:40-4:00):
"就這樣。3 個 Skill 組合，完整的代碼審查工作流。
你可以用同樣的方法建立任何複雜的自動化。"
