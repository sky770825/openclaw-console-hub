# 智能路由測試案例

## 測試案例 1: 輕量轉換
- **任務**: 轉換 10 個 JSON 檔案格式。
- **預期路由**: Ollama qwen3:4b (🟢 綠燈, $0)
- **判定理由**: 單一檔案、低複雜度、無敏感資料。

## 測試案例 2: 複雜 Bug 調查
- **任務**: 調查為什麼 Docker 容器在啟動時偶爾會崩潰，涉及 network 設置。
- **預期路由**: DeepSeek-R1 (🟢 綠燈, $0) 或 L2 Claude Code (🔴 紅燈, $$$) 如果涉及系統架構。
- **判定理由**: 需要深度推理，若涉及系統層級則升級。

## 測試案例 3: 前端頁面 RWD 修正
- **任務**: 修正首頁在 iPhone SE 上的排版錯位。
- **預期路由**: L4 Cursor (🟡 黃燈, $0/已訂閱)
- **判定理由**: UI/CSS 相關，非核心系統。

## 測試案例 4: 系統架構重構
- **任務**: 將目前的單機 SQLite 遷移到 PostgreSQL，並修改所有相關 API。
- **預期路由**: L2 Claude Code (🔴 紅燈, $$$)
- **判定理由**: 跨多檔案、涉及基礎設施、高風險。

## 測試案例 5: 長文本分析
- **任務**: 分析 500MB 的伺服器日誌，尋找攻擊模式。
- **預期路由**: Gemini 3 Pro (🟡 黃燈, 變動成本)
- **判定理由**: 超長上下文需求。
