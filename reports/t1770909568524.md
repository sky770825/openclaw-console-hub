# ClawFlow 配置管理技術規格

## 功能概述
統一的環境變數、密鑰、多環境配置管理。

## 核心功能

### 配置層級

```
1. 系統預設值 (內建)
2. .env.local (本地開發)
3. .env.{ENVIRONMENT} (環境特定)
4. 環境變數 (最高優先級)
```

### 配置文件結構

```
config/
├── .env.example          # 模板
├── .env                  # 預設值
├── .env.development      # 開發環境
├── .env.production       # 生產環境
├── .env.test            # 測試環境
└── default.config.json  # JSON 配置
```

### 密鑰管理

```typescript
interface SecretConfig {
  // 密鑰存儲位置
  storage: 'env' | 'vault' | 'aws-secrets' | 'azure-keyvault';
  
  // 密鑰前綴
  prefix: 'CLAWFLOW_';
  
  // 密鑰加密
  encryption: {
    enabled: boolean;
    algorithm: 'aes-256-gcm';
  };
}

// 環境變數載入
process.env.DATABASE_URL = '...'
process.env.API_KEY = '...'
process.env.JWT_SECRET = '...'
```

### 多環境配置

```yaml
environments:
  development:
    debug: true
    apiUrl: http://localhost:3000
    database:
      host: localhost
      port: 5432
      
  staging:
    debug: false
    apiUrl: https://staging.api.com
    database:
      host: staging-db.example.com
      
  production:
    debug: false
    apiUrl: https://api.example.com
    database:
      host: prod-db.example.com
      ssl: true
```

### 配置驗證

```typescript
import { z } from 'zod';

const ConfigSchema = z.object({
  DATABASE_URL: z.string().url(),
  API_KEY: z.string().min(32),
  LOG_LEVEL: z.enum(['debug', 'info', 'warn', 'error']),
  PORT: z.coerce.number().min(1).max(65535)
});

// 啟動時驗證
const config = ConfigSchema.parse(process.env);
```

### 動態重載

```typescript
// 不重啟應用，直接重載配置
const configManager = new ConfigManager();
await configManager.reload();

// 監聽配置變更
configManager.on('change', (newConfig) => {
  updateApplicationState(newConfig);
});
```

## 使用範例

```typescript
// 讀取配置
import { config } from '@clawflow/config';

const dbUrl = config.get('DATABASE_URL');
const apiKey = config.secret('API_KEY');
const port = config.getNumber('PORT', 3000);

// 檢查環境
if (config.isProduction()) {
  enableSecurityHeaders();
}

// 按條件讀取
const debugMode = config.isDevelopment();
```

## 配置優先級

```
1. 命令行參數           (最高)
   $ node app.js --port=4000
   
2. 環境變數
   $ export DATABASE_URL=postgres://...
   
3. .env.{ENV} 檔案
   .env.production
   
4. .env 檔案
   
5. 程式碼預設值        (最低)
```

## 安全最佳實踐

```yaml
security:
  # 不要在代碼中提交密鑰
  - .env.local
  - .env.*.local
  - secrets/
  
  # Git 忽略
  .gitignore:
    - .env.local
    - .env.*.local
    - config/secrets
    
  # 使用密鑰管理系統
  - HashiCorp Vault
  - AWS Secrets Manager
  - Azure Key Vault
```
