# Live2D 概念驗證報告：星艦忙活主題 (Starship Busy)

## 1. 視覺定義 (Visual Definition)
- **環境背景**: 星艦控制室、維修機庫、或是數據矩陣核心。
- **角色狀態 (忙活)**:
    - **動作 A (維修中)**: 手持雷射焊槍或扳手，對著虛擬浮空面板進行修理。
    - **動作 B (導航監控)**: 快速操作多個全息顯示屏，眼神隨螢幕移動。
    - **動作 C (資料傳輸)**: 身體被數碼流環繞，呈現處理海量數據的姿態。
- **星艦元素**: 
    - 低飽和度的金屬質感、發光的能量導管、偶爾閃爍的警報紅光。
    - UI 框架應具備 Sci-Fi 儀表板風格。

## 2. 互動定義 (Interaction Definition)
- **閒置 (Idle)**: 角色微浮空，呼吸帶有節律性的引擎嗡鳴視覺化。
- **點擊回應**:
    - 點擊頭部：角色露出「工作中，請勿打擾」的無奈表情，伴隨全息錯誤彈窗。
    - 點擊工具：角色加速「忙活」動作。
- **狀態連動**: 與任務面版的「任務進度」連動。進度越快，動作越頻繁。

## 3. 技術可行性驗證 (Technical Feasibility)
- **渲染引擎**: 建議使用 **PixiJS** + **pixi-live2d-display**。
- **整合點**: 
    - 透過 WebSocket 接收 server 端任務狀態。
    - 觸發 Live2D Expression (表情) 與 Motion (動作)。
- **效能考量**: 建議將 Live2D 渲染層獨立於 UI 層之外，利用 WebGL 硬件加速。
