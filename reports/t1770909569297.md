# GitHub Skill Issue 自動分類技術規格

## 功能概述
使用 AI 自動分析與分類 GitHub Issue，標籤標記、指派優先級、推薦審查者。

## 核心功能

### 自動分類

```typescript
// Issue 分類
enum IssueCategory {
  BUG = 'bug',
  FEATURE = 'feature',
  DOCUMENTATION = 'documentation',
  QUESTION = 'question',
  ENHANCEMENT = 'enhancement',
  CHORE = 'chore'
}

// 優先級
enum Priority {
  CRITICAL = 'priority-critical',
  HIGH = 'priority-high',
  MEDIUM = 'priority-medium',
  LOW = 'priority-low'
}

// 複雜度
enum Complexity {
  TRIVIAL = 'complexity-trivial',
  SIMPLE = 'complexity-simple',
  MODERATE = 'complexity-moderate',
  COMPLEX = 'complexity-complex'
}
```

### 標籤自動標記

```
📋 Issue: 如何在 OpenClaw 中使用自訂 Skill?
    ↓
分析標題 + 內容
    ↓
標籤: [question, documentation, skill]
優先級: medium
建議: 指派給文檔維護者
```

## 分類規則

### Bug 識別
```
關鍵詞:
- "crash", "error", "bug", "broken", "not working"
- "exception", "failed", "issue", "problem"
- 版本標記 (latest, main, v1.0)
```

### Feature Request 識別
```
關鍵詞:
- "feature", "enhancement", "new", "add", "would be nice"
- "let me", "can we", "should", "request"
```

### Documentation 識別
```
關鍵詞:
- "documentation", "doc", "readme", "guide", "tutorial"
- "confusing", "unclear", "missing"
```

## 工作流程

```yaml
on:
  issues:
    types: [opened]

steps:
  - name: 分析 Issue
    run: analyze-issue
    
  - name: 自動標籤
    run: add-labels
    
  - name: 指派優先級
    run: set-priority
    
  - name: 推薦審查者
    run: suggest-assignees
    
  - name: 發表評論
    run: post-comment
```

## 配置

```json
{
  "autoClassification": {
    "enabled": true,
    "labelsToAdd": ["auto-classified"],
    "rules": {
      "bug": {
        "keywords": ["crash", "error", "broken"],
        "priority": "high"
      },
      "feature": {
        "keywords": ["feature", "enhancement"],
        "priority": "medium"
      }
    },
    "assignees": {
      "suggestion": true,
      "autoAssign": false,
      "maxSuggestions": 3
    }
  }
}
```
