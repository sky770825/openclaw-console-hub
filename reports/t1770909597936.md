# [Kimi] 錯誤自動恢復機制

**任務類型**: 開發  
**狀態**: 完成  
**日期**: 2026-02-13

## 規格說明

### 目標
Gateway 卡住時自動檢測並重啟，確保服務持續可用。

### 自動恢復流程

#### 1. 健康檢查
```bash
# 每 30 秒執行一次
curl -s -m 5 http://localhost:3011/health

# 檢查項目:
# - HTTP 200 回應 (5秒超時)
# - JSON 格式驗證
# - 必要欄位: status, uptime, timestamp
```

#### 2. 故障判定
```
- 連續 3 次失敗 (90 秒) → 狀態: CRITICAL
- 回應時間 > 3 秒 → 狀態: DEGRADED
- 記憶體使用率 > 90% → 狀態: WARNING
```

#### 3. 自動恢復階段
```
階段 1 (第 1-3 次失敗): 嘗試優雅重啟
  openclaw gateway restart

階段 2 (第 4-6 次失敗): 強制終止 + 重啟
  pkill -9 node; sleep 2; openclaw gateway start

階段 3 (第 7+ 次失敗): 警報 + 人工介入
  發送 Telegram 警報給主人
  停止自動重啟，等待手動確認
```

#### 4. 通知機制
```
- 恢復成功: 📊 Gateway 已恢復 (執行時間: X 秒)
- 多次失敗: 🚨 Gateway 多次故障，已發送警報
- 需要介入: 🔴 人工介入必需 (故障代碼: ERR_XYZ)
```
