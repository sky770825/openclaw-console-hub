### 2026-02-16 00:00 - 腳本語法錯誤（Python heredoc）

**錯誤描述**：
執行 `compare-and-learn.sh` 時出現 Python 語法錯誤，原因是 heredoc 中的變數替換問題

**原因分析**：
- 複雜的 bash heredoc + Python 組合容易出錯
- 未在小範圍測試就直接執行完整腳本
- 對 bash heredoc 的引號規則不夠熟悉

**改進方法**：
- 建立簡化版腳本（simple-version-compare.sh）
- 避免過度複雜的 heredoc 嵌套
- 優先使用簡單可靠的工具（如 diff）

**避免重蹈覆轍**：
- 複雜腳本必須先在小範圍測試
- 優先使用簡單解決方案，避免過度工程
- 建立測試版本和生產版本

### 2026-02-15 23:58 - 腳本路徑錯誤

**錯誤描述**：
執行測試腳本時找不到檔案，錯誤訊息：`no such file or directory: ./scripts/simple-update-test.sh`

**原因分析**：
- 在不正確的工作目錄執行相對路徑
- 未確認當前工作目錄就執行腳本

**改進方法**：
- 使用 `cd` 切換到正確目錄後再執行
- 或使用絕對路徑執行

**避免重蹈覆轍**：
- 執行腳本前先 `pwd` 確認工作目錄
- 使用 `cd "$WORKSPACE" && ./scripts/xxx.sh` 模式

### 2026-02-15 23:50 - Ollama 模型不支援 embeddings

**錯誤描述**：
嘗試使用 `qwen3:8b` 生成 embeddings，API 返回錯誤：`this model does not support embeddings`

**原因分析**：
- 未先確認模型是否支援 embeddings
- 假設通用 LLM 都支援 embeddings（錯誤假設）
- 未閱讀模型文檔

**改進方法**：
- 改用專門的 embedding 模型（nomic-embed-text）
- 先測試 API 再寫完整腳本
- 查閱官方文檔確認功能支援

**避免重蹈覆轍**：
- 使用新 API 前先確認支援情況
- 不要假設，要驗證
- 優先使用專用工具而非通用工具

## 💡 常見錯誤模式（待累積）

### 模式 1：未先測試就全量執行

**特徵**：
- 寫了完整腳本直接執行
- 未在小範圍驗證可行性

**解決**：
- 先測試 1-3 個案例
- 確認無誤後再全量執行

### 模式 2：假設而非驗證

**特徵**：
- 假設 API/工具支援某功能
- 未查文檔就直接使用

**解決**：
- 使用前先查官方文檔
- 先測試最小案例
- 不要假設，要驗證

### 模式 3：複雜解決方案優先

**特徵**：
- 使用複雜的 heredoc/變數替換
- 過度工程化

**解決**：
- 優先使用簡單工具
- KISS 原則（Keep It Simple, Stupid）
- 複雜方案作為備選，不是首選
