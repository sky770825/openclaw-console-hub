## 二、瓶頸分析

### 🔴 高優先級問題

1. **Swap 使用過高**
   - Swap Out: 11,198,215 pages (~175 GB 累積)
   - 表示記憶體壓力持續存在
   - 可能原因：長時間運行未重啟、記憶體洩漏應用

2. **系統運行時間過長**
   - 已運行 7+ 天未重啟
   - 累積的記憶體碎片和背景進程影響效能

### 🟡 中優先級問題

3. **高 CPU 使用進程**
   - WindowServer: 50.5% CPU (圖形渲染)
   - Telegram: 21.8% CPU
   - Google Chrome Helper: 8.5% CPU
   - OpenClaw Gateway: 4.3% CPU, 3.4% MEM

4. **快取堆積**
   - 使用者快取達 16GB
   - 可能包含大量過期資料

### 🟢 低優先級問題

5. **背景服務過多**
   - launchctl 列出大量背景服務
   - 部分服務可能非必要
