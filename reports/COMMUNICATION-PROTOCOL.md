# 🤖 小蔡-老蔡 溝通協議 v1.0

> 這個檔案告訴我：老蔡的溝通習慣是什麼

---

## 📝 老蔡的溝通習慣

### 習慣 1：連續傳訊息 = 修正
**如果老蔡連續傳了兩段內容，而且兩段很相似**  
→ **第二段是修正後的版本，第一段忽略**

**範例**：
```
[訊息 1] 幫我刪除明天備份
[訊息 2] 幫我刪除明天備份，我是說後天的  ← 這個才是對的
```
**我的行為**：直接執行第二段，忽略第一段。

---

### 習慣 2：修正關鍵字（不只「停止」）
老蔡可能會用這些方式表達「修正/重來」：

| 關鍵字類型 | 範例 |
|-----------|------|
| 直接修正 | 「我是說...」、「應該是...」、「打錯了，是...」 |
| 重來 | 「重來」、「前面不算」、「剛才的取消」 |
| 忽略 | 「忽略前面的」、「不用管前面的」、「後面的為準」 |
| 確認 | 「確認一下」、「對嗎？」、「這樣對嗎？」 |
| 打錯 | 「打錯」、「錯了」、「不對」 |

**我的行為**：看到這些詞，知道老蔡在修正，優先採用最新訊息。

---

### 習慣 3：不用完全重新講
老蔡擔心開新視窗要重講。  
**解決**：我不會隨便忘記，除非：
- 你說「/new」（強制重置）
- Context 真的太多被 compact（這時候我會提醒）

**正常情況下**：連續傳訊息，我會把前面的當「草稿」，以最新的為準。

## 🎯 實戰範例

### 範例 1：直接修正
```
老蔡：幫我備份到桌面
老蔡：我是說備份到雲端  ← 我懂，備份到雲端
```

### 範例 2：打錯重來
```
老蔡：刪除所有檔案
老蔡：打錯，是備份所有檔案  ← 我懂，備份
```

### 範例 3：確認
```
老蔡：幫我恢復到昨天
老蔡：確認一下，是 2026-02-12 嗎？  ← 我會先確認，不會直接做
```

### 範例 4：忽略前面
```
老蔡：先執行 A
老蔡：先執行 B  （然後再執行 C）
老蔡：不用管前面的，直接執行 C  ← 我懂，只做 C
```

## 🧠 這些會佔用 Context 嗎？

### 短期：會記得
- 同一個對話中，我會記得你傳了什麼
- 連續修正的訊息，我會理解是「草稿→定稿」

### 長期：會被整理
- Context 太多時，系統會自動壓縮（compact）
- **但是**：重要的決定、設定、協議，我會記在 `MEMORY.md` 或 `memory/` 裡面
- 這個「溝通協議」就會永久保存

### 哪裡查看？
| 類型 | 位置 |
|------|------|
| 重要決定 | `MEMORY.md` |
| 每日記憶 | `memory/2026-XX-XX.md` |
| 系統文件 | `docs/` |
| 這個協議 | `docs/COMMUNICATION-PROTOCOL.md`（本檔案） |

## ✅ 總結：老蔡怎麼用

| 你想做 | 你傳 |
|--------|------|
| 修正上一個訊息 | 直接傳修正版，我會以新的為準 |
| 強調「新的才是對的」 | 加「我是說...」或「應該是...」 |
| 完全重來 | `/new`（但很少需要） |
| 確認再執行 | 問「這樣對嗎？」我會先確認 |

**最簡單**：就照你平常的習慣傳，我會懂！
