# Telegram 訊息發送失敗分析報告

**日期:** 2024-06-15

## 1. 問題摘要

系統日誌顯示，多則 Telegram 訊息因 `Bad Request: can't parse entities` 錯誤而發送失敗。

## 2. 根本原因分析

日誌中的關鍵錯誤訊息如下：

{
  "description":"Bad Request: can't parse entities: Unsupported start tag \"\\u8def\\u5f91\" at byte offset 1287"
}
```

- `\u8def\u5f91` 是「路徑」一詞的 Unicode 編碼。
- 此錯誤表示，發送給 Telegram API 的訊息 payload 中，包含了不被支援的 HTML 標籤。
- 具體來說，是程式在組合訊息時，錯誤地將「路徑」這個詞用角括號 `<` 和 `>` 包裹起來，形成了一個類似 `<路徑>` 的無效標籤。
- Telegram 的訊息解析器（在 `parse_mode=HTML` 模式下）只接受一組預定義的 HTML 標籤（如 `<b>`, `<i>`, `<a>`, `<code>` 等），任何非標準的標籤都會導致解析失敗。

## 3. 解決方案建議

此問題為應用程式層的程式碼錯誤，而非外部服務或環境問題。建議開發人員執行以下操作：

1.  **定位程式碼**：找到負責產生 Telegram 通知訊息內容的相關程式碼區塊。
2.  **修正格式化邏輯**：檢查字串組合或模板引擎的部分，移除將「路徑」或其他普通文本包裹在角括號 `<...>` 中的邏輯。
3.  **確保合規**：確保所有用於格式化的標籤都在 Telegram Bot API 官方文件所允許的範圍內。

## 4. 結論

此問題由訊息內容中包含無效的 HTML 標籤引起。修正程式碼中的字串格式化邏輯後，即可恢復正常的 Telegram 通知功能。
