# [Ollama] 記憶系統 v3 壓力測試

## 任務: 測試大量記憶寫入效能

**執行時間**: 2026-02-13 00:13 UTC+8  
**分類**: 系統穩定性測試

### 測試計劃

#### 1. 測試場景

**場景 1: 單個大型記憶寫入**
- 寫入 1MB+ 文件 (完整任務記錄)
- 目標: < 500ms 完成
- 驗證: 無資料損毀

**場景 2: 多併發寫入** 
- 同時 10 個記憶寫入
- 每個 50KB
- 目標: 總耗時 < 2s
- 風險: 檔案衝突/覆蓋

**場景 3: 記憶索引更新**
- 更新 MEMORY.md 100+ 條目
- 目標: 索引更新 < 100ms
- 風險: JSON 驗證超時

**場景 4: 長期運行穩定性**
- 連續寫入 1000+ 次
- 監控記憶體/磁碟
- 目標: 零洩漏

#### 2. 測試腳本

```bash
#!/bin/bash
# stress-test-memory.sh

MEMORY_DIR="$HOME/.openclaw/workspace/memory"
RESULTS_DIR="$MEMORY_DIR/stress-test-results"
mkdir -p "$RESULTS_DIR"

# 場景 1: 大型檔案寫入
echo "🔬 場景 1: 大型檔案寫入測試..."
for i in {1..5}; do
  # 生成 1MB 測試檔案
  dd if=/dev/urandom bs=1K count=1024 2>/dev/null | base64 > \
    "$RESULTS_DIR/large-file-$i.md"
  
  start=$(date +%s%N)
  # 寫入記憶系統
  cp "$RESULTS_DIR/large-file-$i.md" \
     "$MEMORY_DIR/test-large-$i.md"
  end=$(date +%s%N)
  
  duration=$(( (end - start) / 1000000 ))  # 轉換為毫秒
  echo "  ✅ 寫入 $i: ${duration}ms"
done

# 場景 2: 並發寫入
echo "🔬 場景 2: 並發寫入測試 (10 併發)..."
start=$(date +%s%N)
for i in {1..10}; do
  (
    for j in {1..10}; do
      echo "Task $i-$j content..." > \
        "$MEMORY_DIR/concurrent-$i-$j.md"
    done
  ) &
done
wait
end=$(date +%s%N)
duration=$(( (end - start) / 1000000 ))
echo "  ✅ 並發完成: ${duration}ms (100 個檔案)"

# 場景 3: 記憶索引更新
echo "🔬 場景 3: 記憶索引更新..."
start=$(date +%s%N)
# 生成 100 個索引條目
INDEX='{"items": ['
for i in {1..100}; do
  INDEX="$INDEX{\"id\":$i,\"date\":\"2026-02-$(printf '%02d' $((i % 28 + 1)))\",\"title\":\"Task $i\"},"
done
INDEX="${INDEX%,}]}"

echo "$INDEX" > "$MEMORY_DIR/.memory-index.json"
end=$(date +%s%N)
duration=$(( (end - start) / 1000000 ))
echo "  ✅ 索引更新: ${duration}ms"

# 場景 4: 長期穩定性 (1000 次反覆)
echo "🔬 場景 4: 長期穩定性測試 (1000 次)..."
start=$(date +%s%N)
for i in {1..1000}; do
  echo "Iteration $i - $(date)" >> "$MEMORY_DIR/stress-log.md"
  
  # 每 100 次檢查記憶體
  if [ $((i % 100)) -eq 0 ]; then
    MEM=$(ps aux | grep -i "openclaw\|memory" | grep -v grep | \
      awk '{sum+=$6} END {print sum}')
    echo "  📊 迭代 $i: 記憶體 ${MEM}KB"
  fi
done
end=$(date +%s%N)
duration=$(( (end - start) / 1000000 ))
echo "  ✅ 完成: ${duration}ms (1000 次寫入)"

# 驗證完整性
echo ""
echo "🔍 驗證檔案完整性..."
ERRORS=0
for file in "$MEMORY_DIR"/test-large-*.md; do
  if [ ! -s "$file" ]; then
    echo "  ❌ 檔案損毀: $file"
    ((ERRORS++))
  fi
done

if [ $ERRORS -eq 0 ]; then
  echo "  ✅ 所有檔案完整"
else
  echo "  ❌ 發現 $ERRORS 個檔案問題"
fi

# 生成報告
echo ""
echo "📈 測試報告："
echo "   檔案數: $(find "$MEMORY_DIR" -type f | wc -l)"
echo "   目錄大小: $(du -sh "$MEMORY_DIR" | cut -f1)"
echo "   完成時間: $duration ms"
```

#### 4. 故障排查

**如果寫入超時**
```bash
# 檢查磁碟空間
df -h $HOME/.openclaw/workspace

# 檢查檔案系統狀態
diskutil info /

# 清理舊測試檔案
rm -rf $MEMORY_DIR/stress-test-results
```

**如果出現並發衝突**
```bash
# 檢查是否有 zombie 進程
ps aux | grep defunct

# 使用檔案鎖防止競態條件
# 在腳本中加入: flock -x /tmp/memory.lock
```

**如果記憶體洩漏**
```bash
# 監控記憶體趨勢
watch -n 1 'ps aux | grep openclaw | grep -v grep'

# 強制垃圾回收 (如果支援)
# 或重啟 OpenClaw Gateway
```
