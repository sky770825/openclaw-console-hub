# Live2D 任務面板實作規劃

## 1. 數據聯動 (Data Linking)
- **模型定義**: 任務包含標題、內容、狀態、優先級以及  (觸發特定動作)。
- **狀態追蹤**: 任務完成時，透過 API 觸發 Live2D 的「慶祝」或「摸頭」動作。

## 2. API 整合
- **GET /api/tasks**: 獲取所有任務。
- **POST /api/tasks**: 新增任務，可指定 Live2D 動作類型。
- **PATCH /api/tasks/:id**: 更新狀態。

## 3. UI/UX 設計理念 (前端實作指引)
- **半透明面板**: 疊加在 Live2D 角色側邊。
- **互動反饋**: 點擊任務時，角色轉頭看向面板；完成任務時，角色進行專屬語音/動作。
