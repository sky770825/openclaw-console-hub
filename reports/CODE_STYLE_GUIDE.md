# ArchGuard - 程式碼風格指南 v1.0

> 本文件定義了 OpenClaw Agent 專案中所有腳本與技能的程式碼風格標準。所有新的貢獻都應遵循此指南。

---

## 1. 通用原則

-   **清晰性優於簡潔性**: 程式碼首先是寫給人看的，其次才是給機器執行的。避免使用過於晦澀的語法。
-   **保持一致性**: 遵循檔案中現有的風格。如果沒有，請遵循本指南。
-   **加入註解**: 對複雜的邏輯、演算法或意圖不明確的程式碼塊，添加有意義的註解。
-   **避免魔法數字**: 使用具名的常數代替硬編碼的數字或字串。

---

## 2. Bash 腳本 (.sh) 風格指南

### 2.1 檔案頭部
所有腳本都應該以 shebang (`#!/bin/bash` 或 `#!/usr/bin/env bash`) 開頭，並包含一個簡短的說明，描述其功能和用法。

```bash
#!/usr/bin/env bash
#
# script-name.sh - 簡短描述腳本的功能
#
# 用法: ./script-name.sh [參數]
#
```

### 2.2 錯誤處理
-   總是使用 `set -euo pipefail` 來確保腳本在錯誤發生時能可靠地退出。
    -   `set -e`: 當指令回傳非零時，立即退出。
    -   `set -u`: 當使用未定義的變數時，視為錯誤並退出。
    -   `set -o pipefail`: 當 pipeline 中任何一個指令失敗時，整個 pipeline 的回傳值為失敗。

### 2.3 變數
-   **命名**: 使用大寫蛇形命名法 (`SNAKE_CASE`) 代表全域常數，小寫蛇形命名法 (`snake_case`) 代表局部變數。
-   **引用**: 總是使用雙引號 (`"`) 來引用變數 (`"$my_var"`)，以防止 word splitting 和 globbing 問題。
-   **宣告**: 在函數內部，盡量使用 `local` 來宣告變數，避免污染全域命名空間。

### 2.4 函數
-   **命名**: 使用小寫蛇形命名法 (`my_function_name`)。
-   **結構**: 盡量將程式碼模組化為函數。一個函數只做一件事情。
-   **回傳值**: 使用 `return` 回傳狀態碼 (0 代表成功，非 0 代表失敗)，使用 `echo` 輸出結果。

### 2.5 條件判斷
-   優先使用 `[[ ... ]]` 而不是 `[ ... ]` 或 `test`，因為它更安全且功能更強大。
-   字串比較使用 `==`，例如 `[[ "$str1" == "$str2" ]]`。

## 3. JavaScript / TypeScript (.js, .ts) 風格指南

我們遵循業界廣泛採用的 [Airbnb JavaScript Style Guide](https://github.com/airbnb/javascript)，並在此基礎上做一些補充。

### 3.1 格式化
-   **工具**: 使用 `Prettier` 來自動化程式碼格式化。
-   **縮排**: 使用 2 個空格。
-   **行尾分號**: 必須使用分號。

### 3.2 變數
-   優先使用 `const`，只有在變數需要被重新賦值時才使用 `let`。
-   避免使用 `var`。

### 3.3 命名
-   **變數/函數**: 使用駝峰命名法 (`camelCase`)。
-   **類別/建構子**: 使用帕斯卡命名法 (`PascalCase`)。
-   **常數**: 使用大寫蛇形命名法 (`SNAKE_CASE`)。

### 3.4 模組
-   總是使用 ES6 的 `import` 和 `export` 語法。
-   避免使用 CommonJS 的 `require` 和 `module.exports`。

### 3.5 非同步操作
-   優先使用 `async/await` 語法來處理 Promises，它讓非同步程式碼更易讀。
-   總是對 `await` 的 Promise 進行錯誤處理 (例如使用 `try...catch` 或 `.catch()` 方法)。
