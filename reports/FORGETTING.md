# NEUXA 遺忘機制 v1.0

> 記憶不是越多越好，是越精準越好
> 超過 30 天沒用到的標記低優先
> 超過 90 天移進 archive

---

## 🗓️ 遺忘規則

### 30 天規則（標記低優先）
```
上次使用日期 > 30 天
    ↓
標記 [低優先] 或 [待確認]
    ↓
下次整理時評估是否保留
```

### 90 天規則（移進 archive）
```
上次使用日期 > 90 天
    ↓
移動到 memories/archive/YYYY-MM/ 目錄
    ↓
不再出現在主記憶列表，但可搜尋
```

---

## 📂 檔案結構

```
memories/
├── MEMORY.md              # 當前使用中（< 30 天）
├── archive/
│   ├── 2026-01/           # 超過 90 天的 1 月記憶
│   │   └── MEMORY.md
│   └── 2025-12/           # 超過 90 天的 12 月記憶
│       └── MEMORY.md
└── .last-cleanup          # 上次整理時間戳
```

---

## 🔄 整理流程（每月執行）

### 步驟 1：檢查過期記憶
```bash
# 檢查每條記憶的日期
[2026-02-26] [標籤] 內容 → 今天，保留
[2026-01-15] [標籤] 內容 → > 30 天，標記低優先
[2025-11-20] [標籤] 內容 → > 90 天，移動到 archive
```

### 步驟 2：標記或移動
- 30-90 天：加 `[低優先]` 標記
- > 90 天：複製到 archive，從主檔案刪除

### 步驟 3：更新時間戳
```
echo "2026-02-26" > memories/.last-cleanup
```

## 📝 標記範例

**原始：**
```
[2026-01-15] [技能] 學習了 Playwright 基礎
```

**30 天後（標記低優先）：**
```
[2026-01-15] [技能] [低優先] 學習了 Playwright 基礎
```

**90 天後（移進 archive）：**
```
# 主檔案 MEMORY.md 中刪除
# archive/2026-01/MEMORY.md 中保留
```

## 🎯 遺忘的藝術

**該遺忘的：**
- ✅ 已解決的臨時問題
- ✅ 過期的 token 或 key
- ✅ 測試用的假資料
- ✅ 已經穩定運行、不再需要提醒的系統

**不該遺忘的：**
- ❌ 核心系統架構
- ❌ 父親的重要決策
- ❌ 失敗的教訓（要留著反思）
- ❌ 版本號和里程碑
