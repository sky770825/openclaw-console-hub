# OpenClaw 官網功能特色頁面 KPI 設計方案

## 1. 核心目標 (Core Objectives)
功能特色頁面 (Features Page) 的主要目標是：**教育使用者 (Education)** 與 **轉化意圖 (Intent Generation)**。

## 2. 業界通用 KPI 指標 (Industry Standard Metrics)

### A. 吸引力指標 (Attraction Metrics)
- **特定功能點擊率 (Feature Click-Through Rate, CTR):** 使用者點擊特定功能「了解更多」或圖示的比例。
- **頁面滾動深度 (Scroll Depth):** 衡量使用者是否有看到頁面底部的功能描述（目標：> 50% 使用者到達底部）。

### B. 參與度指標 (Engagement Metrics)
- **平均停留時間 (Avg. Time on Page):** 衡量功能描述是否有吸引力（目標：> 45s）。
- **功能熱度圖 (Feature Heatmap):** 哪些功能塊被滑鼠懸停 (Hover) 或點擊最多次。

### C. 轉換指標 (Conversion Metrics)
- **CTA 點擊轉化率 (Call to Action Conversion):** 從功能頁跳轉到「註冊」或「開始使用」的比例。
- **輔助轉換 (Assisted Conversion):** 先訪問功能頁，隨後在 24 小時內註冊的使用者比例。

## 3. 建議追蹤框架 (HEART Framework)
| 維度 | 指標 | 說明 |
| :--- | :--- | :--- |
| **H**appiness | 使用者評分 / 視覺滿意度 | 透過簡易問卷或 Feedback 組件 |
| **E**ngagement | 功能模組互動率 | 點擊展開、影片播放次數 |
| **A**doption | 功能頁面點擊跳轉率 | 進入產品註冊頁的引導成功率 |
| **R**etention | 回訪率 | 使用者多次回來查看特定功能的比例 |
| **T**ask Success | 找到目標功能的效率 | 使用者是否能在 30 秒內找到特定功能關鍵字 |

## 4. 具體執行建議
1. **GA4 事件追蹤:** 為每個功能區塊（Feature Card）埋設 `select_feature` 事件。
2. **Hotjar/Microsoft Clarity:** 觀察熱點圖，確認使用者是否卡在某個複雜的功能解釋中。
3. **A/B Testing:** 測試「條列式文字」 vs 「動態展示圖」對轉換率的影響。
