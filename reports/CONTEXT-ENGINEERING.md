## 核心策略

參考 LangChain / Anthropic / Karpathy 的 4 大策略：**Write → Select → Compress → Isolate**

---

## 📝 Write — 寫出 Context（Scratchpad）

長任務中間結果不要留在對話裡，寫到外部：
- **任務計畫** → 寫入 `NOW.md` 或暫存檔
- **中間數據** → 寫入 `/tmp/` 或 workspace 檔案
- **重要發現** → 寫入 `memory/YYYY-MM-DD.md`

這樣即使開新會話也能接續。

---

## 🔍 Select — 按需載入（不要全塞）

- **不要一開始就讀所有記憶檔案**
- 只讀 `NOW.md`（<1k tokens），需要時才查 `MEMORY.md` 關鍵字索引
- 深度記憶用 `scripts/memory_recall.js` 向量檢索，不要整檔讀入
- 工具輸出只擷取需要的部分（用 `limit`/`offset`/`maxChars`）

### 📋 預載白名單（開機必讀）

| 優先級 | 檔案 | 說明 | 大小 |
|--------|------|------|------|
| P0 | `AGENTS.md` | 工作指南 | ~1k |
| P0 | `SOUL.md` | 人設與風格 | ~500 |
| P0 | `USER.md` | 老蔡檔案 | ~500 |
| P1 | `MEMORY.md` | 速查表段落 | ~2k |
| P2 | `HEARTBEAT.md` | 心跳任務（若空則跳過） | 可變 |

**非白名單**：`docs/` 其他文件、`memory/*.md`、session 歷史 → **按需才讀**

## 🗜️ Compress — 壓縮 Context

### 強制摘要門檻

| Tokens | 動作 |
|--------|------|
| **≥ 70k** | **強制執行對話摘要**，只保留關鍵結論 |
| ≥ 85k | 禁全量輸出 + 強制開新會話 |

**70k 摘要規範**：
- 保留：用戶核心需求、已確認的決策、待辦事項
- 捨棄：中間試錯過程、工具輸出原始碼、已完成的子任務細節
- 格式：`[摘要] 已完成 X，待處理 Y，關鍵決策 Z`

### 工具輸出壓縮

- `web_fetch` 設 `maxChars: 5000`（除非確實需要更多）
- `exec` 輸出超過 50 行 → 只取關鍵行
- `read` 大檔案 → 用 `limit`/`offset` 分段讀取

### 對話壓縮

- Context 達 **60%** → 主動建議老蔡 `/new`
- Context 達 **70%** → 執行 checkpoint（見下方）
- Context 達 **85%** → 強制建議開新會話

### 摘要策略

- 子 Agent 回傳的結果，只保留結論和關鍵數據
- 多輪研究的中間過程不需要保留在主對話中

## ⏰ 自動觸發規則

| Context 使用率 | Tokens (131k) | 動作 |
|---------------|---------------|------|
| < 50% | < 65k | 正常運作 |
| 50-60% | 65-78k | 新任務優先 spawn 子 Agent |
| 60-70% | 78-91k | 主動提醒老蔡可以 `/new` |
| **70%** | **~91k** | **執行 checkpoint，強烈建議 `/new`** |
| **≥ 85%** | **≥ 111k** | **禁全量輸出 + 強制 checkpoint + 警告** |

### >85% 禁全量輸出規範

當 Context ≥ 85% 時：
1. **禁止輸出完整對話歷史**（只給摘要）
2. **禁止長篇解釋**（只給結論）
3. **禁止多輪來回**（單輪完成或開 `/new`）
4. **工具輸出強制截斷**（只取前 20 行或關鍵部分）

## 📊 監控指標（5輪 p95）

每 5 輪對話記錄以下指標，用於優化策略：

| 指標 | 計算方式 | 健康門檻 |
|------|---------|---------|
| **p95 Tokens In** | 最近5輪輸入 tokens 第95百分位 | < 80k |
| **p95 Tokens Out** | 最近5輪輸出 tokens 第95百分位 | < 3k |
| **p95 Context %** | 最近5輪 context 使用率 p95 | < 70% |
| **p95 Latency** | 最近5輪回應延遲 p95 | < 30s |
| **Spawn Rate** | 子 Agent 佔比 | > 30%（高複雜度任務健康指標）|

**超標處理**：
- p95 Context > 70% → 下一輪強制摘要或建議 `/new`
- p95 Tokens In > 80k → 檢查是否有冗餘載入，調整白名單

## 🚫 Context 毒害防範

避免以下會污染 context 的行為：
- **Context Poisoning**：幻覺內容進入對話 → 發現錯誤立即更正
- **Context Distraction**：無關資訊干擾 → 嚴格控制工具輸出長度
- **Context Confusion**：多餘 context 影響回應 → 只注入相關資訊
- **Context Clash**：context 內部矛盾 → 保持資訊一致性

---

## 🤖 子 Agent 指派協議

spawn 子 Agent 時必須指定 `model` 和 `label`：
- **label 命名規範**：`{類型}-{描述}`（如 `research-openrouter`、`exec-setup`）
- **任務描述必須包含**：角色、具體任務、輸出格式要求
- **預設使用 Sonnet 4.5**，簡單任務用 Kimi K2.5

---

## 📦 回收摘要規範

- 子 Agent 回傳後，主會話**只保留結論**（不超過 500 字）
- 詳細內容寫入檔案，不留在 context
- 多個子 Agent 的結果合併彙整再回覆老蔡

---

## 🎯 模型選擇速查

- **簡單/格式化** → Kimi K2.5 或 Ollama
- **研究/分析** → Sonnet 4.5
- **大量文本** → Gemini Flash
- **重大決策** → Opus 4.6（需老蔡授權）
