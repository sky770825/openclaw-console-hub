## 2. 服務清單與用量查詢方式盤點

根據掃描結果與一般常見開發配置，整理以下服務用量查詢方式：

| 服務名稱 | 用途 | 額度/用量查詢方式 (API Endpoint / Web Console) | 備註 |
| :--- | :--- | :--- | :--- |
| **OpenAI** | LLM API | `https://api.openai.com/v1/usage` | 需要 API Key |
| **Anthropic** | LLM API | Console: `https://console.anthropic.com/settings/usage` | 目前 API 用量查詢較受限 |
| **Google Cloud / Gemini** | Cloud / AI | `https://console.cloud.google.com/billing` | API: Cloud Billing API |
| **Stripe** | 支付訂閱 | `https://api.stripe.com/v1/subscriptions` | 檢索活躍訂閱狀態 |
| **GitHub** | 程式碼託管 / Actions | `https://api.github.com/settings/billing/actions` | 查看 Actions 額度 |
| **AWS** | 雲端基礎設施 | `Cost Explorer API` / `Budgets API` | |
| **Vercel** | 前端部署 | `https://vercel.com/dashboard/usage` | |
| **Supabase** | 資料庫 / Auth | `https://supabase.com/dashboard/project/_/settings/billing` | |
