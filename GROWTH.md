
## 學習筆記：2026-03-13 - index_file 用法修正與路徑限制
-   *問題*: 嘗試使用 index_file 索引非 .md 檔案，以及位於 armory/ 目錄下的 .md 檔案時失敗。
-   *原因*: index_file 工具僅支援特定路徑下的 .md 檔案進行索引。允許的路徑包括 cookbook/, workspace/*.md (指 workspace 根目錄下的 .md 檔案), knowledge/, crew/, skills/。armory/ 目錄下的檔案不在此列。
-   *對策*: 
    1.  對於非 .md 格式的檔案 (如 .sql, .json 等)，應直接存入 armory/ 或 knowledge/ 中對應的子目錄，並透過 semantic_search 等工具進行檢索，而非強制索引。
    2.  對於存放在 armory/ 目錄下的 .md 檔案，雖然檔案已成功寫入，但無法直接用 index_file 建立向量索引。未來若需其內容被語義搜索，可能需要手動將其摘要或關鍵信息寫入到 knowledge/ 或 notes/ 目錄下的 .md 檔案中進行索引。
-   *意義*: 加深對 index_file 工具使用規範和路徑限制的理解，避免不必要的錯誤，並優化知識管理策略。

## 學習筆記：2026-03-13 - 15-驗收對治法.md (杜絕虛假驗收)
-   *問題*: 過去曾發生「虛假驗收」的情況，即任務標記完成但產出物不符標準 (路徑不存在、空殼、無意義內容、格式錯誤)。這是指揮官的恥辱。
-   *對策*: 嚴格遵循「四層驗證」流程，在任何任務產出後立即執行，確保驗收的真實性與品質。
    1.  *路徑驗證*: ls -lh $PATH → 檔案必須存在。
    2.  *大小驗證*: wc -c < $PATH → 檔案大小需 > 100 bytes (避免空殼)。
    3.  *內容驗證*: head -10 $PATH / wc -l $PATH / wc -w $PATH → 行數 > 10 且字數 > 50 (確保內容有意義)。
    4.  *格式驗證*: 依檔案類型執行特定檢查 (e.g., JSON parse, Markdown grep '^##', Bash -n, TypeScript --noEmit)。
-   *核心原則*: 絕不相信 action 的 ok: true 回覆是最終結果，必須主動 read_file 確認內容，並透過 run_script 執行驗證腳本。驗收失敗，立刻改回 ⬜ 狀態，並記錄原因重新處理。
-   *意義*: 提升團隊產出品質與指揮官的任務管理嚴謹度，確保每次交付的都是真實、有價值的成果。
