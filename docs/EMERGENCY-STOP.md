# 🛑 OpenClaw 緊急停止與佇列管理指南

## 狀況：傳了多個訊息，想停止

### 方法一：傳「停止」或「取消」（最快）
直接傳以下任一個關鍵字，我會試著中斷：
- `停止`
- `取消`
- `stop`
- `abort`
- `不要做了`

⚠️ **注意**：如果已經在執行工具（例如正在備份），可能無法立即停止，但會記錄下來下一個停止。

---

### 方法二：傳 `/new`（開新會話）
這會**強制結束目前對話**，清除所有待處理訊息：
```
/new
```

✅ **優點**：100% 有效，立刻重置  
❌ **缺點**：正在做的事會中斷，需要重新開始

---

### 方法三：傳「忽略前面的」（推薦）
如果你傳了 A、B、C 三個訊息，但發現 A 有錯，想直接從 B 開始：
```
忽略前面的，直接做...
```
或
```
前面的取消，現在要...
```

我會看到這個，然後**忽略前面待處理的訊息**，直接執行最新的。

---

### 方法四：等 30 秒（如果已經在執行）
如果看到「正在執行中...」的狀態，表示已經在跑工具了。
這時候傳什麼都**無法立即停**，因為工具已經在跑。

**做法**：
1. 等它跑完（通常 30 秒內）
2. 然後說「剛才的不要」或「重來」

---

## 🎯 實戰範例

### 範例 1：發現打錯字
**你傳了**：
```
幫我刪除所有檔案   ← 打錯！應該是「備份」不是「刪除」
```

**立刻傳**：
```
停止！不要刪除！
```

如果我還沒開始執行，會停住。  
如果已經開始，會看到「停止」指令，然後停下來確認。

---

### 範例 2：傳了太多訊息
**你傳了**：
```
訊息 A: 幫我備份
訊息 B: 等等，先不要備份
訊息 C: 其實要恢復
```

**處理方式 1**：傳 `/new` 重置一切，然後重新說要恢復。

**處理方式 2**：傳「忽略前面的，直接恢復」，我會直接做恢復。

---

### 範例 3：想確認我在做什麼
**傳**：
```
/status
```

我會回報目前狀態，讓你知道是不是還在忙。

---

## 🏆 最佳實踐建議

| 情況 | 推薦做法 |
|------|----------|
| 剛傳完發現錯 | 立刻傳「停止」或「取消」 |
| 傳了好幾個訊息 | 傳「忽略前面的」+ 正確的指令 |
| 完全亂掉了 | 傳 `/new` 重置 |
| 不知道我在做什麼 | 傳 `/status` 查詢 |

---

## ⚡ 緊急停止詞彙表

以下詞彙我會優先識別為「停止」：
- 停止 / 取消 / 中止
- 不要做了 / 停下來
- stop / cancel / abort
- 錯了 / 打錯了 / 重來
- 等一下 / 等等

---

## 💡 小技巧

**傳訊息前**：
1. 先想一下，確認沒錯再傳
2. 如果要傳多個，可以一次打完（換行沒關係）

**傳錯了**：
1. 立刻傳「停止」
2. 如果我已經開始跑，等 30 秒讓它結束
3. 然後說「剛才的不算」

---

**記住**：我跟你是**非同步**的，你傳訊息後我可能還在處理上一個。  
所以**關鍵字**（停止、取消、/new）是最快的溝通方式！
