## 1. 產品定義

### 1.1 產品概念

**AI Agent 自動化框架** 是一個讓「非技術用戶也能指揮 AI 完成複雜任務」的開發/運行平台。

與傳統自動化工具（如 n8n、Make）的差異在於：我們以 **AI Agent 為核心**，而非以 **流程節點為核心**。用戶用自然語言描述目標，框架自動規劃步驟、執行動作、處理異常。

**解決的問題**:
- 現有自動化工具需要手動連接節點，門檻高（不確定性：需用戶訪談驗證）
- AI 應用開發碎片化，缺乏統一框架（相對確定，市場反饋明確）
- 本地 AI 部署與自動化難以整合（我們有 Ollama 優勢，相對確定）

### 1.2 目標用戶

| 用戶群體 | 優先級 | 特徵 |
|---------|-------|------|
| 技術型個人開發者 | P0 | 想快速構建 AI 工具，厭倦重複造輪子 |
| 中小企業技術團隊 | P1 | 需要自動化業務流程，但預算有限 |
| 無代碼/低代碼用戶 | P2 | 想用自然語言創建自動化（長期目標） |

**不確定性**：P1/P2 用戶需求需進一步驗證，建議先聚焦 P0 開發者市場。

### 1.3 差異化定位

| 競品 | 核心模式 | 我們的差異 |
|-----|---------|-----------|
| LangChain | 開發者工具庫 | 更高層抽象，內建運行環境 |
| CrewAI | 多 Agent 協作框架 | 整合瀏覽器自動化 + 本地模型 |
| AutoGen | 微軟生態 Agent | 輕量、開源友好、不依賴 Azure |
| Dify | 低代碼 AI 平台 | 支援本地部署、開源核心 |
| n8n | 傳統自動化 | AI-first 設計，非節點拖拽 |

**核心差異化**：「開箱即用的本地 AI Agent 自動化」，結合 OpenClaw 的指揮經驗 + 瀏覽器自動化 + Ollama 本地模型支援。

## 2. 市場機會

### 2.1 TAM/SAM/SOM 估算

| 層級 | 估算 | 說明 |
|-----|------|------|
| **TAM** | $50B+ (2030) | AI Agent 市場整體預測（多家機構預估，不確定性高） |
| **SAM** | $2-5B | 開發者工具/自動化框架細分市場 |
| **SOM (Y1)** | $100K-500K | 開源社群 + 早期商業用戶 |
| **SOM (Y3)** | $2-5M | 商業版 SaaS + 企業支援服務 |

**不確定性**：TAM 數字差異極大（$5B-$150B），SAM 較為務實，SOM 需根據實際獲客成本調整。

### 2.2 競品分析速覽

| 競品 | 優勢 | 劣勢 | 定價 |
|-----|------|------|------|
| **LangChain** | 生態最大、文件完善 | 過於底層、學習曲線陡 | 開源免費 |
| **CrewAI** | 多 Agent 協作概念清晰 | 商業版貴、社群較小 | $0-$99/月 |
| **AutoGen** | 微軟背書、功能強大 | 依賴 Azure、設定複雜 | 開源 + Azure 費用 |
| **Dify** | UI 完善、易用 | 企業版貴、核心閉源 | $0-$199/月 |
| **n8n** | 節點豐富、社群大 | 非 AI-first、需手動連線 | $0-$50/月 |

### 2.3 SWOT 分析

**優勢 (S)**:
- OpenClaw 已累積實戰經驗（相對確定）
- 瀏覽器自動化能力現成（相對確定）
- Ollama 本地模型支援（差異化優勢）

**劣勢 (W)**:
- 品牌知名度低（高度確定）
- 資源有限（1-2 人開發）（高度確定）
- 缺乏企業客戶背書（需時間累積）

**機會 (O)**:
- 本地 AI / 隱私需求上升（趨勢明確，相對確定）
- 開發者厭倦複雜框架（需驗證）

**威脅 (T)**:
- 巨頭快速跟進（OpenAI、Anthropic）（高度確定風險）
- 競品已建立生態護城河（高度確定）

## 3. 技術架構設計

### 3.1 核心模組

```
┌─────────────────────────────────────────────────────────────┐
│                    Agent Orchestrator                       │
│              (任務規劃、狀態管理、異常處理)                    │
├─────────────────────────────────────────────────────────────┤
│  ┌──────────┐  ┌──────────┐  ┌──────────┐  ┌──────────┐   │
│  │  Skill   │  │ Browser  │  │   LLM    │  │ Memory   │   │
│  │  System  │  │  Agent   │  │ Adapter  │  │  Store   │   │
│  └──────────┘  └──────────┘  └──────────┘  └──────────┘   │
├─────────────────────────────────────────────────────────────┤
│                    Platform Abstraction                     │
│              (Docker / 本地 / 雲端適配層)                     │
└─────────────────────────────────────────────────────────────┘
```

### 3.2 技術棧選擇

| 層級 | 技術 | 理由 |
|-----|------|------|
| 運行時 | Node.js 20+ | 團隊熟悉、生態豐富 |
| Agent 核心 | TypeScript | 類型安全、可維護 |
| 瀏覽器自動化 | Playwright | 現有技能、跨瀏覽器 |
| LLM 介接 | 統一 Adapter | 支援 Ollama/OpenAI/Anthropic |
| 記憶存儲 | SQLite / Redis | 輕量、本地優先 |
| 任務調度 | 現有任務板系統 | 重用已有基礎設施 |

### 3.3 與現有工具整合

| 現有資產 | 整合方式 |
|---------|---------|
| OpenClaw Agent | 作為執行引擎核心 |
| skillforge-publisher | Skill 發布/安裝機制 |
| 任務板系統 | 背景任務調度與監控 |
| Ollama | 本地模型預設支援 |

## 5. MVP 設計

### 5.1 MVP 功能範圍

**核心功能 (必須)**:
1. Agent 定義：YAML/JSON 配置任務流程
2. Skill 系統：可外掛的動作模組（重用 skillforge-publisher）
3. 瀏覽器自動化：整合 Playwright
4. 本地 LLM：Ollama 整合（qwen3:8b 為預設）
5. 任務執行：CLI 驅動、任務板監控

**排除功能 (Y2)**:
- 視覺化流程編輯器
- 雲端多租戶
- 多 Agent 協作複雜場景

### 5.2 開發估算

| 階段 | 時間 | 人力 |
|-----|------|------|
| 核心框架 | 4-6 週 | 1 人 (Cursor/CoDEX) |
| Skill 系統 | 2-3 週 | 1 人 |
| 整合測試 | 2 週 | 1 人 + 小蔡驗收 |
| **總計** | **8-11 週** | **約 2 人月** |

**不確定性**：實際時間可能因 Cursor/CoDEX 效率而大幅縮短或延長。

### 5.3 驗證方法

| 指標 | 目標 | 驗證方式 |
|-----|------|---------|
| GitHub Stars | 500+ | 開源發布後 3 個月 |
| 活躍用戶 | 100+ | Discord/社群註冊 |
| NPS 評分 | >40 | 用戶問卷 |
| 商業意向 | 5+ | 企業諮詢數量 |

## 6. 執行路徑圖

### 6.1 Phase 1：核心驗證（2-3 個月）

**目標**：可運行的 MVP + 首批用戶反饋

| 里程碑 | 退出條件 |
|-------|---------|
| 框架核心可用 | 能完成「自動搜尋→摘要→發郵件」流程 |
| 3 個內部專案使用 | 小蔡團隊實際使用驗證 |
| GitHub 公開發布 | 獲得 100+ Stars |

**資源需求**：Cursor Pro + CoDEX + 小蔡 20% 時間

### 6.2 Phase 2：社群建設（3-6 個月）

**目標**：建立開源社群、驗證商業模式

| 里程碑 | 退出條件 |
|-------|---------|
| 社群 1000+ 成員 | Discord/Telegram |
| 10+ 外部貢獻者 | PR/Merge |
| 首個付費客戶 | 企業支援合約 |

**資源需求**：社群運營 + 文件完善

### 6.3 Phase 3：商業化（6-12 個月）

**目標**：穩定收入、產品市場契合

| 里程碑 | 退出條件 |
|-------|---------|
| MRR $5K+ | 可持續收入 |
| 產品市場契合 | 用戶留存率 >60% |
| 團隊擴張 | 招募第 2 位全職 |

### 6.4 退出機制

| 條件 | 行動 |
|-----|------|
| 3 個月無技術突破 | 暫停開發，評估轉向 |
| 6 個月無用戶增長 | 縮減範圍，專注單一場景 |
| 12 個月無收入 | 轉為純開源項目，尋求贊助 |
