# 還有什麼方式可以看到同步的任務板？

任務板資料存在 **Supabase**（`openclaw_tasks` 等表），後端 API 與各入口都是讀寫同一份資料，所以以下方式看到的都是**同一份同步任務板**。

---

## 1. 網頁中控台（最完整）

- **本機**：`npm run dev` 後開瀏覽器 → `http://localhost:3009` → 左側「任務板」
- **線上**：部署後的網址（例如 GitHub Pages、Vercel）→ 同一個任務板、任務列表、Runs

可篩選、拖拉、執行、看 Run 詳情，功能最完整。

---

## 2. Telegram（用講的）

裝好 **OpenClaw 任務板 Skill** 並設好 `OPENCLAW_TASKBOARD_URL` 後，在 Telegram 對 bot 說：

- 「任務列表」→ 回傳目前任務
- 「建立任務：修復登入 bug」
- 「執行下一個任務」

看到的任務就是同一份；用語音或打字都能查／建／觸發執行。  
詳見：`docs/openclaw-taskboard-skill/README.md`。

---

## 3. OpenClaw / Cursor 內嵌分頁

若你有用 **OpenClaw 板**（例如在 Cursor 裡開的 OpenClaw 介面），裡面的「任務」分頁（tasksTab）也是打同一個 API，顯示同一份任務板。  
適合一邊寫程式一邊看任務、點執行。

---

## 4. Supabase Dashboard（直接看資料庫）

- 登入 [Supabase](https://supabase.com/dashboard) → 選你的專案  
- 左側 **Table Editor** → 選 **openclaw_tasks**（或 openclaw_reviews、openclaw_evolution_log）

可直接看／改同一份資料，適合除錯或快速查表。  
注意：這裡改的是「原始資料」，不會經過後端邏輯。

---

## 5. 手機或另一台電腦

只要用瀏覽器打開**同一個中控台網址**（本機需同網段或已部署到線上），就能看到同一份任務板。  
例如：本機跑 `npm run dev`，手機連 `http://<你電腦的 IP>:3009`（或直接用 GitHub Pages / Vercel 網址）。

---

## 6. n8n / 其他工具（讀 API）

若你在 n8n 或其它系統裡用 **HTTP Request** 打任務板 API（例如 `GET /api/openclaw/tasks`），讀到的也是同一份任務；可再做成通知、報表或自動化。  
API 規格：`docs/openclaw-taskboard-skill/references/api.md`。

---

## 對照表

| 方式           | 哪裡看         | 適合情境                     |
|----------------|----------------|------------------------------|
| 網頁中控台     | 瀏覽器         | 完整操作、篩選、執行、Runs   |
| Telegram       | 手機/桌面 TG   | 用講的查任務、建任務、執行  |
| OpenClaw 分頁  | Cursor/OpenClaw| 寫程式時順便看／執行任務    |
| Supabase 後台  | 網頁           | 查表、除錯、直接改資料      |
| 手機/他機瀏覽器| 同上中控台網址 | 外出或第二台裝置看同一板    |
| n8n / API 整合 | 自建流程       | 通知、報表、自動化          |

以上任一方式改的任務，其它方式重新整理或重新查一次就會看到更新，因為都是同一份 Supabase 資料。
