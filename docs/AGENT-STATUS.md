# 🤖 Agent 狀態追蹤系統說明

## 這是什麼？

這個系統讓你可以隨時查詢「達爾現在在做什麼」，類似 Discord 的「正在玩...」狀態。

## 使用方法

### 查詢狀態
在對話中傳送：
```
/status
```
或
```
/狀態
```

會顯示：
- ✅ **待命** - 我準備好了，可以接收指令
- ⏳ **執行中** - 我正在執行工具/處理任務

### 執行中會看到什麼

```
🤖 達爾 狀態面板

   ⏳ 狀態：執行中

   📋 目前任務：建立系統恢復中心
   ⏱️  已執行：2分30秒
   🔧 使用工具：backup.sh, write, exec

   ⚠️  請稍候，完成後會回覆您
```

## ⚠️ 重要限制

### 1. 狀態更新是「手動」的
我（AI）需要在開始工作時**主動標記**狀態。這意味著：

- ✅ 長時間任務（備份、大量檔案操作）→ 會標記「執行中」
- ⚠️ 短時間任務（簡單查詢、回覆）→ 可能來不及標記就完成了
- ❌ 如果系統沒有標記 → /status 會顯示「待命」，即使我在思考

### 2. 無法強制中斷
顯示「執行中」不代表你可以中斷我。工具一旦開始執行，必須等它完成。
你可以傳「停止」，但我只能在**下一個檢查點**響應。

### 3. 排隊訊息看不到
/status 只顯示「我目前正在做的」，不顯示「還有多少訊息在排隊」。

## 🎯 實戰使用建議

### 情況 1：傳了訊息但沒回應
```
你：/status
系統：⏳ 執行中 - 正在備份系統 (已執行 1分鐘)
→ 你知道我在忙，稍等一下
```

### 情況 2：傳了訊息很久沒回應
```
你：/status
系統：✅ 待命
→ 可能訊息沒送到，或系統有問題，可以重傳
```

### 情況 3：想確認上一個做完了沒
```
你：/status
系統：✅ 待命 - 最後完成：建立備份 (17:30:15)
→ 上一個完成了，可以傳新指令
```

## 🔧 技術細節

### 狀態檔案位置
- 狀態檔案：`~/.openclaw/.agent-status.json`
- 鎖定檔案：`~/.openclaw/.agent-busy.lock`
- 活動日誌：`~/.openclaw/.agent-activity.log`

### 手動操作（如果你需要）
```bash
# 強制標記為忙碌
~/.openclaw/workspace/scripts/agent-state.sh start "手動任務"

# 強制標記為完成
~/.openclaw/workspace/scripts/agent-state.sh end

# 強制重置
~/.openclaw/workspace/scripts/agent-state.sh reset
```

## 📝 總結

| 功能 | 狀態 |
|------|------|
| 查詢我在做什麼 | ✅ 可用 (/status) |
| 精確追蹤所有動作 | ❌ 有限制（需要主動標記） |
| 看到排隊訊息 | ❌ 無法 |
| 強制中斷 | ❌ 無法（工具已執行） |

**最可靠的方式**：如果我幾分鐘沒回應，傳 `/status` 確認狀態。
