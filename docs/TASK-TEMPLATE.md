# 任務卡標準模板

> OpenClaw 以後全用這個。每張任務都填 6 欄，不亂解讀、驗收快、可自動化。

---

## 空白模板（複製貼上）

```
Goal:

Inputs:

Outputs:

Acceptance:

Owner:

Priority (1–5):
```

---

## 欄位說明

| 欄位 | 必填 | 說明 |
|------|------|------|
| **Goal** | ✓ | 這張任務要達成的目標（一句話） |
| **Inputs** | 選 | 需要的前置／輸入 |
| **Outputs** | ✓ | 產出（檔案、API、UI 等） |
| **Acceptance** | ✓ | 驗收條件（做到才算完成） |
| **Owner** | ✓ | 負責人 |
| **Priority (1–5)** | ✓ | 1 最高、5 最低 |

---

## 範例

```
Goal: 讓所有任務、檔案、資料結構有一致語言
Inputs: （無）
Outputs: docs/OPENCLAW-CONCEPT.md（名詞定義：Task / Run / Log / Alert）
Acceptance: 任務卡、UI、API 都用同一套名詞
Owner: OpenClaw
Priority: 5
```

---

*與 `docs/OPENCLAW-CONCEPT.md` 任務卡標準模板一致*
