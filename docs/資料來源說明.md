# 目前的數據是「真的」還是「假的」？

## 結論

| 情境 | 數據來源 |
|------|----------|
| **沒設 `VITE_API_BASE_URL`**，或後端沒跑 | **Mock（假的）**：來自瀏覽器 localStorage + 種子資料 |
| **有設 `VITE_API_BASE_URL` + `VITE_OPENCLAW_API_KEY`，且後端有跑** | **真實**：來自後端 API（後端若接 Supabase 則為資料庫裡的資料） |

---

## 怎麼判斷現在是 mock 還是真實？

- **開發時**：若只跑 `npm run dev`、沒跑 `cd server && npm run dev`，或 `.env` 裡沒填 `VITE_API_BASE_URL`，畫面上的任務／執行／警報都是 **mock**（種子 + localStorage）。
- **有接後端時**：在 `.env` 設好 `VITE_API_BASE_URL`（例如 `http://localhost:3011`）和 `VITE_OPENCLAW_API_KEY`，並啟動後端，儀表板、任務板等會打真實 API，數據就是 **真的**（依後端是否接 DB）。

---

## 想用「真實數據」要怎麼做？

1. 在專案根目錄 `.env` 加上：
   ```env
   VITE_API_BASE_URL=http://localhost:3011
   VITE_OPENCLAW_API_KEY=與後端OPENCLAW_API_KEY相同
   ```
2. 啟動後端：`cd server && npm run dev`（預設 port 3011）。
3. 後端若已設定 Supabase，任務／執行紀錄會存進資料庫，重開或換裝置也會是同一份資料。

若後端連線失敗（例如沒啟動），前端會自動 fallback 到 mock，避免畫面報錯；此時你看到的仍是 **假資料**。
