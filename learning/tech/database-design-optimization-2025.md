# 2025 資料庫設計與效能優化

> 來源：Gemini Flash 2.5 搜尋整理
> 日期：2026-02-16
> 適用：資料庫設計、效能調校

## 1. 資料庫正規化與反正規化

**正規化：**
- 目標：減少資料冗餘、提高一致性
- 遵循第三正規化 (3NF) 原則
- 注意過度正規化可能導致過多 JOIN
- 適用：交易型系統、需高度一致性

**反正規化：**
- 目標：提高查詢效能，減少 JOIN
- 適度引入冗餘資料，預先計算結果
- 透過應用程式或觸發器維護同步
- 適用：讀取密集型、分析型應用

**混合策略：**
- 彈性選擇正規化與反正規化
- 使用 Materialized View、Computed Columns
- HTAP (混合交易/分析處理) 實現即時分析

## 2. 索引設計策略

**索引類型：**
- B-tree：範圍查詢、排序
- Hash：精確匹配
- 全文索引：文本搜索
- 空間索引：地理位置查詢
- 向量索引：相似性搜索

**設計原則：**
- 選擇 WHERE、JOIN、ORDER BY 常用欄位
- 考慮欄位選擇性 (Cardinality)
- 複合索引將高選擇性欄位放前面
- 定期重建索引避免碎片化
- 監控並移除無效索引

## 3. 快取機制設計

| 層級 | 說明 |
|------|------|
| 瀏覽器快取 | HTTP 快取機制 |
| CDN 快取 | 加速靜態資源傳輸 |
| 應用程式快取 | Redis、Memcached |
| 資料庫快取 | Query Cache、Shared Buffer |

**快取策略：**
- Cache-Aside：先查快取再查資料庫
- Read-Through/Write-Through：快取負責與資料庫同步
- Write-Behind：異步更新資料庫

**淘汰策略：**
- LRU：淘汰最近最少使用
- LFU：淘汰使用頻率最低
- TTL：設定有效時間

## 4. 讀寫分離架構

**架構設計：**
- 主庫 (Master) 負責寫入
- 從庫 (Slave) 負責讀取
- 透過 Replication 同步資料

**考量點：**
- 選擇同步/異步/半同步複製模式
- 注意讀寫延遲，確保讀取最新資料
- 使用負載均衡器分配讀取請求
- 主庫故障時切換到從庫保持可用性
