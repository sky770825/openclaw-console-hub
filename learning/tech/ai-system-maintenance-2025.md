# AI 系統維護知識

> 來源：Gemini Flash 2.5 搜尋整理
> 日期：2026-02-16
> 適用：OpenClaw、n8n、Qdrant 維護

## 1. 常見故障排除

**模型崩潰/預測錯誤：**
- 檢查輸入數據品質，排除髒數據、缺失值、異常值
- 驗證模型版本與環境配置是否一致
- 檢視訓練數據分布是否與實際場景一致
- 調整超參數或重新訓練
- 排查硬體資源限制 (CPU, GPU, 記憶體)

**API 服務異常：**
- 檢查 API 端點可用性和網路連線
- 監控請求量和回應時間找瓶頸
- 查看日誌分析錯誤訊息
- 重啟 API 服務或相關組件

**資料管道故障：**
- 檢查資料來源可用性
- 驗證資料轉換流程
- 確保資料同步機制正常
- 排查儲存容量限制

## 2. 效能監控指標

| 類型 | 指標 |
|------|------|
| 模型效能 | 準確率、精確率、召回率、F1、預測延遲、資源消耗 |
| 系統效能 | RPS、回應時間、資源利用率、錯誤率 |
| 資料管道 | 處理速度、資料延遲 |

## 3. 安全檢查清單

**數據安全：**
- 數據加密、訪問控制、資料脫敏、備份與恢復

**模型安全：**
- 對抗性攻擊防禦、模型權重保護、漏洞掃描

**系統安全：**
- 身份驗證與授權、漏洞修補、入侵檢測、日誌審計、定期安全評估
