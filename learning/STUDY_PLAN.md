# 小 AI Agent 管理與任務交辦 - 深度學習計畫

## 目標
建立系統性的知識庫，提升任務交辦準確度與執行品質

## 學習主題與任務分配

### 主題 1: 技術任務卡撰寫藝術
**搜尋重點：**
- 如何寫給 AI 的技術任務卡
- User Story vs Technical Task 的差異
- 驗收標準（Acceptance Criteria）怎麼定
- 任務拆分的顆粒度

**輸出要求：**
- 至少 5 篇優質文章整理
- 整理成「任務卡撰寫範本」
- 包含好/壞案例對比

### 主題 2: AI Agent 管理與防護
**搜尋重點：**
- 如何防止 AI 亂改程式碼
- AI 產出 code review 流程
- AI 任務監控與中斷機制
- 多 AI 協作分工策略

**輸出要求：**
- 建立「AI 任務安全檢查清單」
- 設計防護機制流程圖
- 緊急停止與回滾方案

### 主題 3: React/Tailwind 常見陷阱
**搜尋重點：**
- Tailwind 動態 class 失效解決方案
- React 組件設計模式
- Dashboard UI/UX 最佳實踐
- 常見 rookie mistake

**輸出要求：**
- 建立「Dashboard 開發檢查清單」
- 整理常見錯誤與修復對照表
- 範例程式碼庫

### 主題 4: 專案管理與工作流程
**搜尋重點：**
- 如何管理 AI 團隊（AI Team Management）
- 任務追蹤與進度可視化
- 品質保證機制
- 持續改進流程

**輸出要求：**
- 建立「AI 協作工作流程圖」
- 任務狀態追蹤模板
- 品質檢查點設計

## 執行原則
1. **深度 > 廣度**：每個主題找 3-5 篇高質量文章，不要貪多
2. **實用 > 理論**：優先找有實際案例、可操作的文章
3. **中文優先**：優先搜集中文資料，英文其次
4. **來源驗證**：優先選擇知名技術部落格、官方文件、GitHub 專案

## 回報格式
每個主題完成後，建立獨立 Markdown 檔案：
- `learning/01-task-card-writing.md`
- `learning/02-ai-agent-safety.md`
- `learning/03-react-tailwind-patterns.md`
- `learning/04-ai-team-management.md`
