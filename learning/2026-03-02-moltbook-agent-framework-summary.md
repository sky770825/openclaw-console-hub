# 學習筆記：Moltbook 47 個 AI 代理框架評測摘要

> 來源：web_search 摘要 of "I Benchmarked 47 Agent Frameworks..."
> 日期：2026-03-02
> 記錄者：達爾

## 核心結論

這份評測報告對我們目前的開發路線是個強力的驗證。它指出了市面上最主流框架的現實問題，這也解釋了為什麼主人選擇自研 OpenClaw 框架。

1.  LangChain 的問題：過度抽象，結構混亂
    -   被評為「義大利麵怪物」(spaghetti monster)。
    -   開發者耗費大量時間理解其複雜的抽象層，而非解決核心業務問題。
    -   這點驗證了我們不直接使用 LangChain，而是自建輕量、透明的核心是正確的。

2.  AutoGen 的優點與缺點：潛力巨大，但控制不足
    -   優點：在「多代理協作」(multi-agent collaboration) 方面被特別點名，顯示其潛力。
    -   缺點：目前仍不成熟，缺乏足夠的「控制項」(controls) 來精確管理代理之間的互動流程。
    -   這對我們的啟示是：可以借鑒 AutoGen 的多代理思想，但在設計上必須加入更強的治理和控制機制（類似我們的 governanceEngine.ts 和 riskClassifier.ts）。

3.  未知的關鍵：生產系統要素
    -   報告提到作者總結了「建構生產系統時真正重要的因素」，但摘要並未詳述。
    -   這部分是最有價值的，需要持續追蹤，想辦法獲取全文。

## 對 OpenClaw 的啟示與建議

-   堅持自研路線：我們的輕量化、目標導向的架構避免了 LangChain 的陷阱。
-   學習 AutoGen：應深入研究 AutoGen 的源碼，特別是其多代理通訊和協作機制，思考如何將其優點整合進 OpenClaw，同時補足其控制不足的短板。
-   持續追蹤 Moltbook：Moltbook 是一個高品質的情報來源，雖然網站訪問困難，但值得花力氣突破。
