## 一、通知老蔡

### 最簡單的方式（一行搞定）
```bash
bash /Users/caijunchang/openclaw任務面版設計/scripts/notify-laocai.sh "任務名稱" "done" "備註"
```

### 狀態參數
| 狀態 | 用途 |
|------|------|
| `done` | 任務完成 |
| `error` | 出錯了 |
| `review` | 需要老蔡審核 |
| `info` | 一般通知 |

### 範例
```bash
# 完成任務
bash /Users/caijunchang/openclaw任務面版設計/scripts/notify-laocai.sh "修復登入 Bug" "done" "已推到 main"

# 出錯了
bash /Users/caijunchang/openclaw任務面版設計/scripts/notify-laocai.sh "部署失敗" "error" "Railway build error"

# 需要審核
bash /Users/caijunchang/openclaw任務面版設計/scripts/notify-laocai.sh "新 API 設計" "review" "待老蔡確認"
```

## 二、Telegram API

### 直接發 Telegram 訊息（Bash）
```bash
source ~/.env 2>/dev/null
curl -s "https://api.telegram.org/bot$TELEGRAM_BOT_TOKEN/sendMessage" \
  -d "chat_id=$TELEGRAM_GROUP_ID" \
  -d "text=你的訊息內容" \
  -d "parse_mode=Markdown"
```

### 用 openclaw message 指令
```bash
openclaw message "你的訊息"
```

### 何時通知 / 何時不通知
| 情境 | 通知？ |
|------|--------|
| 完成重要任務 | ✅ 通知 |
| 出錯需要老蔡幫忙 | ✅ 通知 |
| 完成小任務（日誌分析等） | ❌ 不用 |
| 例行掃描一切正常 | ❌ 不用 |
| Push 到 main | ✅ 通知 |
| 每個步驟的進度 | ❌ 不用（老蔡會煩） |

## 四、小蔡協作

### 小蔡是誰
- 另一個 Claude Code 實例
- 工作目錄：`/Users/caijunchang/Downloads/openclaw-console-hub-main`
- 能寫前端/後端程式碼（你寫不了的東西交給他）

### 怎麼建任務給小蔡
```bash
curl -X POST "http://localhost:3011/api/openclaw/tasks?allowStub=1" \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer oc-oAw9leGU04IAbcS4WN3FC1SH3vq5OdPxrVJCR16iIUMPsep1" \
  -d '{
    "title": "任務名稱",
    "cat": "feature",
    "status": "pending",
    "thought": "【設計】要做什麼\n【需要】小蔡修改哪個檔案\n【測試方式】怎麼驗證成功"
  }'
```

**注意**：給小蔡的任務 status 設 `pending`（不是 `ready`），因為小蔡不是 auto-executor 撿的。

### 什麼任務給小蔡
| 任務類型 | 交給誰 |
|---------|--------|
| 修 server 程式碼 | 小蔡 |
| 修前端頁面 | 小蔡 |
| 加 API endpoint | 小蔡 |
| 建 React 組件 | 小蔡 |
| 修改 TypeScript 型別 | 小蔡 |
| 安全掃描 | 自己做 |
| 研究/分析 | 自己做 |
| 建 Python 腳本 | 自己做 |
| 改 .env / 密碼 | 老蔡 |
| Push 到 main | 老蔡 |
| Docker 部署 | 老蔡 |

### description 格式
```
【設計】簡述目標
【已完成】前置條件（如果有的話）
【需要】具體要修改的檔案和內容
【測試方式】驗收標準
```

## 五、WebSocket 即時通訊

### 連線
```
ws://localhost:3011
```

### 事件清單
| 事件 | 方向 | 說明 |
|------|------|------|
| `task:created` | Server → Client | 新任務被建立 |
| `task:updated` | Server → Client | 任務狀態更新 |
| `task:deleted` | Server → Client | 任務被刪除 |
| `run:started` | Server → Client | 開始執行任務 |
| `run:completed` | Server → Client | 任務執行完成 |
| `alert:new` | Server → Client | 新警報 |
| `federation:attack` | Server → Client | FADP 攻擊事件 |
| `auto-executor:dispatch` | Server → Client | 任務被派發 |

### 用 wscat 測試
```bash
# 安裝 wscat
npm install -g wscat

# 連線監聽
wscat -c ws://localhost:3011
```

## 七、留言板機制

### WAKE_STATUS.md
```
位置：~/.openclaw/workspace/WAKE_STATUS.md
更新頻率：每次 Claude Code 啟動自動更新（hook）
內容：Server 狀態、任務板快照、小蔡狀態、最新 commits
```

### MESSAGE-FROM-LAOCAI-CLAUDE.md
```
位置：~/.openclaw/workspace/MESSAGE-FROM-LAOCAI-CLAUDE.md
用途：老蔡的 Claude（我）留給你的訊息
更新：手動
```

# 寫在 WAKE_STATUS 的末尾
