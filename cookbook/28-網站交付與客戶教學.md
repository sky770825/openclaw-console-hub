## 1. 交付前檢查表（Master Checklist）

> 原則：交付前至少留 **3 個工作天** 跑完整檢查。不要上線當天才檢查。

### 1.1 功能測試

```
[ ] 所有頁面連結點擊測試（用 Screaming Frog 或 broken-link-checker）
[ ] 所有表單送出測試（聯絡表單、訂閱、登入、註冊）
    [ ] 成功送出 → 確認信/感謝頁正確
    [ ] 欄位驗證 → 空白、格式錯誤、XSS 注入測試
    [ ] 收件端 → 郵件實際收到（檢查垃圾信匣）
[ ] 所有按鈕/CTA 功能正常

    [ ] 加入購物車 → 數量修改 → 刪除
    [ ] 結帳 → 付款（測試模式）→ 訂單確認信
[ ] 搜尋功能（輸入關鍵字、空白搜尋、無結果頁面）
[ ] 404 頁面設定（自訂 404，不是伺服器預設）
[ ] 多語系切換（如適用）

    [ ] 手機（375px / 390px / 414px）
    [ ] 平板（768px / 1024px）

    [ ] 實際手機測試（不只是 DevTools 模擬）

    [ ] Chrome（最新版）
```

## 2. 客戶後台教學 SOP

> 原則：教學要讓「完全不懂技術的老闆」也能自己操作基本功能。
> 格式：影片 + 圖文手冊搭配，客戶忘記就回去翻。

### 2.1 錄影教學製作

**工具選擇：**

| 工具 | 適用情境 | 費用 |
|------|---------|------|
| Loom | 最推薦，錄完自動上傳分享 | 免費版 25 支/5 分鐘 |
| OBS Studio | 長教學/需要後製 | 免費 |
| macOS 螢幕錄製 | 快速簡單 | 免費 |
| Tella | 精美教學，有鏡頭框 | 付費 |

**錄影 SOP：**

```
1. 準備
   [ ] 開一個乾淨的瀏覽器（無奇怪書籤/分頁）
   [ ] 瀏覽器放大到 110%（字要夠大）
   [ ] 關掉所有通知
   [ ] 準備好要示範的素材（圖片、文字）
   [ ] 列好操作步驟大綱

2. 錄製原則
   [ ] 每支影片只教一件事（3-8 分鐘最佳）
   [ ] 游標移動放慢，重要步驟停留 2-3 秒
   [ ] 說話速度放慢，關鍵步驟重複說一次
   [ ] 不要用專業術語（說「文章」不說「post type」）
   [ ] 錄錯不用重來，停頓後重說（後製剪掉就好）

3. 影片命名規則
   01-如何登入後台.mp4
   02-如何修改首頁內容.mp4
   03-如何新增部落格文章.mp4
   04-如何修改聯絡資訊.mp4
   05-如何更換圖片.mp4
   ...

4. 交付方式
   - Loom：直接分享連結（可設密碼）
   - 自行託管：上傳到客戶的 Google Drive
   - 嵌入 Notion/Google Sites 做教學中心
```

### 2.4 CMS 權限設定

**核心原則：客戶能改內容，不能動結構。**

#### WordPress 角色建議

| 角色 | 給誰 | 能做什麼 | 不能做什麼 |
|------|------|---------|-----------|
| 管理員（Administrator） | 客戶老闆（1 個） | 所有操作 | -- |
| 編輯（Editor） | 行銷/小編 | 管理所有文章頁面 | 不能裝外掛、改設定 |
| 作者（Author） | 部落格寫手 | 寫自己的文章 | 不能改別人的文章 |
| 自訂角色（用 User Role Editor） | 特殊需求 | 客製權限 | 精準控制 |

**用 User Role Editor 外掛鎖定的項目：**

```
客戶帳號（編輯角色）應該被鎖住的：
  [x] 外觀 → 佈景主題（不讓客戶亂換主題）
  [x] 外掛 → 安裝/啟用/停用
  [x] 設定 → 所有設定頁面
  [x] 工具 → 匯入/匯出
  [x] 使用者 → 新增使用者（只有管理員能加人）

客戶帳號應該能做的：
  [ ] 文章 → 新增/編輯/刪除
  [ ] 頁面 → 編輯（不能刪除重要頁面）
  [ ] 媒體 → 上傳/管理
  [ ] 選單 → 修改（視情況）
  [ ] 表單 → 查看送出的資料
```

**Elementor 權限設定（如使用）：**

```php
// functions.php 加入，限制客戶不能動某些 Widget
add_action('elementor/editor/after_enqueue_scripts', function() {
    if (!current_user_can('manage_options')) {
        // 隱藏「結構」相關的 Widget
        wp_add_inline_script('elementor-editor', "
            jQuery(window).on('elementor:init', function() {
                // 限制可用的 Widget 類型
            });
        ");
    }
});
```

**保護關鍵頁面不被誤刪：**

```php
// functions.php - 防止特定頁面被刪除
add_action('wp_trash_post', function($post_id) {
    $protected_pages = [
        get_option('page_on_front'),    // 首頁
        get_option('page_for_posts'),   // 部落格頁
        // 加入其他不能刪的頁面 ID
    ];
    if (in_array($post_id, $protected_pages)) {
        wp_die('這個頁面受到保護，無法刪除。如需修改請聯絡網站管理員。');
    }
});
```

```
  │   └── 無法重現 → 請客戶提供更多資訊/螢幕錄影
  │
  ├── 2. 判斷是否在保固範圍
  │   ├── 是 → 免費修復
  │   └── 否 → 回覆客戶，說明原因，提供報價
  │
  ├── 3. 判斷嚴重程度
  │   ├── 嚴重（影響核心功能）→ 當天處理
  │   └── 輕微（不影響使用）→ 排程本週處理
  │
  ├── 4. 修復
  │   ├── 在測試環境修復
  │   ├── 測試確認
  │   └── 部署到正式環境
  │
  └── 5. 回報客戶
      ├── 說明修復內容
      ├── 請客戶確認
      └── 記錄到修復日誌
```

**NPS（淨推薦分數）計算：**

```
0-6 分 = 批評者（Detractor）
7-8 分 = 被動者（Passive）
9-10 分 = 推薦者（Promoter）

NPS = 推薦者比例 - 批評者比例

NPS > 50 = 很好
NPS > 70 = 極好
NPS < 0 = 需要檢討
```

**收到回饋後的行動：**

```
9-10 分 → 請求 Google 評論 + 推薦信 + 案例分享
7-8 分 → 詢問具體可改善之處
1-6 分 → 電話溝通，了解問題，主動提出補救方案
```

### 8.2 SEO 服務 Upsell

```
看看有哪些快速可以改善的地方。」

```

```
  ✓ 網站速度優化
  ✓ 內部連結優化
  ✓ Google Search Console 設定

SEO 每月維護 — NT$ 8,000 / 月（最少 6 個月）
  ✓ 月度關鍵字排名追蹤
  ✓ 每月 2 篇 SEO 文章策略
  ✓ 技術 SEO 持續優化
  ✓ 競爭對手分析
```
