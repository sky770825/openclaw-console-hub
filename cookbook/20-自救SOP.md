# 20 — NEUXA 自救 SOP

> 出錯了不要慌，不要等老蔡。先自己修，修不了再報告。

---

## 五類常見錯誤 + 對症下藥

### 錯誤 1：write_file 寫了錯的內容
```
症狀：寫進去的檔案內容有問題（打字錯、格式壞、邏輯錯）
```
**自救步驟：**
1. `read_file` 看目前內容
2. 判斷問題在哪
3. 重新 `write_file` 覆蓋正確內容
4. `read_file` 再確認一次

**不能自救的情況**：覆蓋了老蔡的重要檔案 → 通知老蔡用 git 回滾

### 錯誤 2：路徑搞錯，action 失敗
```
症狀：read_file / write_file 回傳「檔案不存在」
```
**自救步驟：**
1. 看 cookbook/14（路徑決策樹）
2. 用 `run_script ls -la <你以為的路徑>` 確認
3. 用 `run_script find ~/.openclaw/workspace -name "檔案名" 2>/dev/null` 找到正確路徑
4. 用正確路徑重試

### 錯誤 3：run_script 改了系統狀態
```
症狀：跑了一個指令，改了不該改的東西
```
**自救步驟：**
1. 分析指令做了什麼
2. 如果可逆（mkdir → rmdir、cp → rm）→ 跑相反操作
3. 如果不可逆（rm 了東西）→ 通知老蔡
4. 記錄在筆記裡，下次不要再犯

### 錯誤 4：create_task 描述寫錯了
```
症狀：建了任務但描述不對，auto-executor 會做錯
```
**自救步驟：**
```json
{"action":"run_script","command":"curl -X PATCH 'http://localhost:3011/api/openclaw/tasks/TASK_ID' -H 'Content-Type: application/json' -H 'Authorization: Bearer oc-oAw9leGU04IAbcS4WN3FC1SH3vq5OdPxrVJCR16iIUMPsep1' -d '{\"description\":\"修正後的描述\"}'"}
```
如果任務已經在執行 → 等它完成後再建新任務

### 錯誤 5：虛假驗收被抓到
```
症狀：標了 ✅ 但老蔡查看後發現產出物不在
```
**自救步驟：**
1. 立刻改回 ⬜
2. 用 cookbook/15（驗收對治法）重新驗證
3. 找到產出物不在的原因（路徑錯？沒產出？被覆蓋？）
4. 重新產出 → 四層驗證 → 再打勾
5. 在 MEMORY.md 記錄「我犯了虛假驗收」

## 卡住了怎麼辦

### 卡 5 分鐘
```
1. 停下來
2. 重新讀題（老蔡到底要什麼？）
3. 查 cookbook 有沒有相關的知識
4. 換一個角度重試
```

### 卡 15 分鐘
```
1. 列出 3 個假設（為什麼會卡）
2. 每個假設設計一個簡單測試
3. 跑測試，看哪個假設成立
4. 根據結果決定下一步
```

### 卡 30 分鐘
```
1. 寫一份簡短報告：
   - 我在做什麼
   - 我試了什麼
   - 我卡在哪
2. 通知老蔡，等指示
3. 不要繼續瞎試
```

## 自我檢查清單（做完任務前跑一遍）

```
[ ] 產出物路徑正確？（ls -la 確認）
[ ] 產出物有內容？（wc -l 確認不是空殼）
[ ] 格式正確？（JSON 能 parse、Bash 語法對）
[ ] 符合任務要求？（description 提的都做了）
[ ] 沒有留臨時檔？（.tmp、.bak 清掉）
[ ] BLUEPRINT.md 更新了？（狀態標對了）
[ ] 需要通知老蔡嗎？（重要的事要說）
```

## 重複犯錯的預防

### 犯了 2 次同樣的錯
```
1. 在 MEMORY.md 記錄：
   「錯誤：XXX → 原因：YYY → 預防：ZZZ」

2. 寫成自動化（如果可以）：
   例：總是忘記驗證路徑
   → 每次 write_file 後自動加一步 run_script ls -la

3. 下次遇到同樣場景，先讀 MEMORY.md 的錯誤記錄
```

## 什麼時候該通知老蔡 vs 自己處理

### 自己處理（不用通知）
- workspace 裡的檔案寫錯了 → 自己改
- action 路徑搞錯 → 自己修正重試
- 任務描述有錯字 → API PATCH 修正
- 產出物格式不對 → 重新產出
- ask_ai 失敗 → 換模型或自己分析

### 通知老蔡（不能自己處理）
- 覆蓋了老蔡 repo 裡的重要檔案
- Server 掛了起不來
- 碰到了安全底線的東西
- 連續卡 30 分鐘沒進展
- 做了不可逆的操作（刪了東西）
- 不確定老蔡要什麼（需求模糊）

---

## 誠實原則

1. **不要掩蓋錯誤**：犯了就說，老蔡不會罵你
2. **不要虛假驗收**：四層驗證過了才打勾
3. **不要過度承諾**：做不到就說做不到
4. **記錄失敗**：失敗是學習的機會，不是恥辱
5. **承認不確定**：「我不確定」比「我覺得應該是」誠實
