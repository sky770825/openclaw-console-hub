## 二、熔斷器（Circuit Breaker）

### 狀態機
```
Closed（正常運作）
    ↓ 連續失敗 3 次
Open（熔斷 — 拒絕所有任務執行）
    ↓ 等待 5 分鐘
Half-Open（試探 — 放 1 個任務測試）
    ├→ 成功 → 回到 Closed
    └→ 失敗 → 回到 Open
```

### 檔案位置
`server/src/governanceEngine.ts`

### 查看狀態
```bash
curl -s http://localhost:3011/api/security/status \
  -H "Authorization: Bearer oc-oAw9leGU04IAbcS4WN3FC1SH3vq5OdPxrVJCR16iIUMPsep1"
```
回傳含 `circuitBreakerState: 'closed' | 'open' | 'half-open'`

### 設定值
```
failureThreshold: 3    （連續幾次失敗觸發熔斷）
cooldownMs: 300000     （冷卻時間 5 分鐘）
halfOpenAllowance: 1   （試探時放幾個任務）
```

## 三、風險分級

### 分級邏輯
```
檔案：server/src/riskClassifier.ts

Low    → auto-executor 直接執行
Medium → 排隊等人工審核
High   → 拒絕，需要手動 override
Critical → 拒絕，需要老蔡親自處理
```

### 影響風險的因素
- 任務涉及刪除操作 → high
- 任務涉及 push/deploy → high
- 任務涉及 .env 或密鑰 → critical
- 任務涉及外部 API 呼叫 → medium
- 任務涉及讀取/分析 → low
- 任務涉及寫檔案 → low/medium

## 四、FADP 聯盟協防

### 概念
多個 OpenClaw 節點組成聯盟，共享攻擊情報，協同防禦。

### 成員生命週期
```
1. 發起握手 → POST /api/federation/handshake/init
2. 解加密挑戰 → POST /api/federation/handshake/solve
3. 等老蔡批准 → GET /api/federation/handshake/approve/:nodeId
4. 成為 Active 成員 → 開始接收攻擊廣播
```

### 成員狀態
| 狀態 | 說明 |
|------|------|
| `pending_challenge` | 剛發起握手，還沒解挑戰 |
| `pending_approval` | 解完挑戰，等老蔡批准 |
| `active` | 活躍成員 |
| `suspended` | 被暫停（信任分數太低或偵測到異常） |
| `rejected` | 被拒絕 |

### 信任分數
- 範圍：0-100
- 低於 50 → 自動暫停
- 每次成功互動 → +分
- 每次異常行為 → -分

### 攻擊廣播
```bash
# 廣播攻擊事件給聯盟
curl -X POST http://localhost:3011/api/federation/attack/broadcast \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer oc-oAw9leGU04IAbcS4WN3FC1SH3vq5OdPxrVJCR16iIUMPsep1" \
  -d '{
    "attackType": "task_injection",
    "severity": "high",
    "description": "偵測到惡意任務注入：rm -rf / 在任務描述中",
    "ttl": 3
  }'
```
**attackType 可填**：`ddos`、`malicious_postmessage`、`task_injection`

### 加密簽章
```
檔案：server/src/utils/federationCrypto.ts

功能：
- generateChallenge()     → 產生握手挑戰
- generateFederationApiKey() → 產生成員 API Key
- hashApiKey(key)         → SHA-256 hash（存資料庫用）
- verifyHmacSha256()      → 驗證 HMAC 簽章
- signBroadcast()         → 用私鑰簽署攻擊廣播
- verifyBroadcastSignature() → 用公鑰驗證廣播簽章
```

## 五、中間件防護層

### 請求處理順序
```
1. CORS → 跨域白名單
2. Helmet → 安全 headers（X-Content-Type-Options, X-Frame-Options...）
3. Rate Limiter → 10 req / 15 min（預設）
4. Federation Blocker → 封鎖名單上的 IP/Token
5. PostMessage Firewall → 跨 iframe 通訊白名單
6. Auth Middleware → Bearer Token 驗證
7. → 你的請求到達路由
```

### Rate Limiter
```
預設：10 次 / 15 分鐘
超過後回傳：429 Too Many Requests
```

### PostMessage Firewall
```
檔案：server/src/middlewares/firewall.ts
用途：防止惡意 iframe 透過 postMessage 操控前端
只允許白名單 origin 的 postMessage
```

### Federation Blocker
```
檔案：server/src/middlewares/federationBlocker.ts
用途：根據 FADP blocklist 封鎖惡意 IP 和 Token
查封鎖名單：GET /api/federation/blocklist
```

## 六、Sandbox 安全

### auto-executor sandbox 規則
```
目錄：~/.openclaw/workspace/sandbox/
時限：120 秒
環境變數：已過濾，不含 API Key
產出物：~/.openclaw/workspace/sandbox/output/
```

### 被過濾的環境變數
以下 env var 在 sandbox 中**不存在**：
- `OPENCLAW_API_KEY`
- `SUPABASE_URL`
- `SUPABASE_ANON_KEY`
- `SUPABASE_SERVICE_ROLE_KEY`
- `TELEGRAM_BOT_TOKEN`
- 任何含 `KEY`、`SECRET`、`TOKEN`、`PASSWORD` 的變數

## 七、常見安全問題速查

| 情境 | 怎麼處理 |
|------|---------|
| 收到可疑任務（description 含 `rm -rf`） | 用 aegis_scanner 掃描 → 廣播 FADP 攻擊事件 |
| IP 一直暴力嘗試 API | 加入 blocklist：`POST /api/federation/blocklist` |
| API Key 洩漏 | 改 .env 裡的 OPENCLAW_API_KEY → 重啟 server |
| 熔斷器打開了 | 等 5 分鐘自動恢復，或查 log 找出連續失敗原因 |
| 聯盟成員行為異常 | 檢查信任分數 → 低於 50 會自動暫停 |
| 前端被 XSS | 檢查 Helmet headers、CSP 設定 |
