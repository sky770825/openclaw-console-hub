# 14 — 路徑與檔案系統指南

> 每次 action 都會碰到路徑。搞錯路徑 = action 必敗。這份指南讓你再也不搞錯。

---

## 路徑決策樹（最重要！做 action 前過一遍）

```
我要操作的檔案在哪？
│
├─ 我的 workspace（筆記、報告、工具）
│  → ~/.openclaw/workspace/

├─ 老蔡的程式碼 repo
│  → /Users/caijunchang/openclaw任務面版設計/
```

## 絕對路徑 vs 相對路徑

### 規則：Telegram action 裡一律用絕對路徑

```
✅ {"action":"read_file","path":"~/.openclaw/workspace/BLUEPRINT.md"}
✅ {"action":"read_file","path":"/Users/caijunchang/openclaw任務面版設計/server/src/index.ts"}

❌ {"action":"read_file","path":"server/src/index.ts"}           ← 相對路徑，找不到
❌ {"action":"read_file","path":"BLUEPRINT.md"}                  ← 相對路徑，找不到
❌ {"action":"read_file","path":"workspace/BLUEPRINT.md"}        ← 缺少 ~/.openclaw/
```

### run_script 裡可以用相對路徑，但要先 cd

```
✅ {"action":"run_script","command":"cd ~/.openclaw/workspace && ls -la"}
✅ {"action":"run_script","command":"ls -la ~/.openclaw/workspace/sandbox/output/"}

❌ {"action":"run_script","command":"ls -la sandbox/output/"}    ← cwd 不確定在哪
```

## 常見路徑陷阱（負面教學）

| 錯誤寫法 | 為什麼錯 | 正確寫法 |
|---------|---------|---------|
| `sandbox/armory/tool/` | sandbox 和 armory 是兩個平行目錄 | `~/.openclaw/workspace/armory/tool/` |
| `~/armory/` | 缺少 `.openclaw/workspace` | `~/.openclaw/workspace/armory/` |
| `openclaw/workspace/cookbook/` | 缺少 `~/.` | `~/.openclaw/workspace/cookbook/` |
| `server/src/index.ts` | 相對路徑，找不到 | `/Users/caijunchang/openclaw任務面版設計/server/src/index.ts` |
| `~/.openclaw/agents/main/` | 這個目錄不存在 | `~/.openclaw/workspace/` |
| `workspace/reports/` | 缺少 `~/.openclaw/` | `~/.openclaw/workspace/reports/` |
| `/tmp/openclaw/` | 這不是正確工作區 | `~/.openclaw/workspace/` |

## 路徑驗證（action 前後都要做）

### 寫檔案前：確認目錄存在
```json
{"action":"run_script","command":"ls -la ~/.openclaw/workspace/reports/ 2>&1 | head -3"}
```
如果目錄不存在，先 mkdir：
```json
{"action":"mkdir","path":"~/.openclaw/workspace/reports"}
```

### 寫檔案後：確認檔案存在
```json
{"action":"run_script","command":"ls -lh ~/.openclaw/workspace/reports/my-report.md && wc -l ~/.openclaw/workspace/reports/my-report.md"}
```

## ~ 的展開規則

在 action JSON 裡面：
- `~` 會被系統自動展開為 `/Users/caijunchang`
- 所以 `~/.openclaw/workspace/` = `/Users/caijunchang/.openclaw/workspace/`
- 兩種寫法都可以，但建議用 `~/` 比較短
