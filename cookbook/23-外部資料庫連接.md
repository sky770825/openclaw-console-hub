# 23 — 外部資料庫連接 SOP

> 主人叫你查某個專案的資料庫，但那不是 OpenClaw 的 Supabase。

---

## 什麼時候用

- 主人說「查一下餐車系統的排班」
- 主人說「幫我看華地產簽到資料」
- 任何不是 OpenClaw 本身的資料庫查詢

---

## 先搞清楚：這是哪個 Supabase？

### OpenClaw 自己的 Supabase（query_supabase 直接查）
```
可查的表：openclaw_tasks, openclaw_reviews, openclaw_automations,
openclaw_evolution_log, openclaw_runs, openclaw_audit_logs,
fadp_members, fadp_attack_events, fadp_blocklist,
schedules, shifts, attendance, employees

用法：
{"action":"query_supabase","table":"openclaw_tasks","select":"*","limit":10}
```

### 外部專案的 Supabase（需要那個專案的 Key）
主人的其他專案（餐車、華地產、project_j 等）各有自己的 Supabase。
**query_supabase 查不到它們的資料。**

---

## 外部 Supabase 連接步驟

### Step 1：找到該專案的連線資訊

```
# 在專案目錄裡找 Supabase URL
{"action":"run_script","command":"grep -rn 'supabaseUrl\\|SUPABASE_URL\\|supabase.co' /Users/sky770825/Desktop/專案名/ --include='*.js' --include='*.ts' --include='*.html' --include='*.env*' 2>/dev/null | grep -v node_modules | head -10"}
```

通常會在這些地方找到：
- `script.js` 或 `supabase.js` → `const supabaseUrl = 'https://xxx.supabase.co'`
- `.env` → `SUPABASE_URL=https://xxx.supabase.co`（但 .env 你讀不了）
- `index.html` 裡的 script 標籤

### Step 2：取得 URL 和 anon key

如果在 JS 檔裡找到（公開的 anon key，安全的）：
```
{"action":"run_script","command":"grep -A2 'supabaseUrl\\|supabaseKey\\|createClient' /Users/sky770825/Desktop/專案名/script.js 2>/dev/null"}
```

如果只在 .env 裡（你讀不了 .env）：
**→ 告訴主人你需要這個專案的 Supabase URL 和 anon key。**
**→ 或請主人把 key 加到 Server 的 proxy 裡。**

### Step 3：用 Supabase REST API 查資料

拿到 URL 和 anon key 後，用 run_script + curl 直接打 Supabase REST API：

```
# 列出所有表
{"action":"run_script","command":"curl -s 'https://xxx.supabase.co/rest/v1/' -H 'apikey: eyJ...' -H 'Authorization: Bearer eyJ...' | python3 -c \"import json,sys; [print(t['table_name']) for t in json.load(sys.stdin)]\""}

# 查某個表的資料
{"action":"run_script","command":"curl -s 'https://xxx.supabase.co/rest/v1/schedules?select=*&limit=10' -H 'apikey: eyJ...' -H 'Authorization: Bearer eyJ...' | python3 -c \"import json,sys; print(json.dumps(json.load(sys.stdin), indent=2, ensure_ascii=False))\""}

# 帶條件查詢
{"action":"run_script","command":"curl -s 'https://xxx.supabase.co/rest/v1/schedules?select=*&month=eq.2026-02' -H 'apikey: eyJ...' -H 'Authorization: Bearer eyJ...' | python3 -c \"import json,sys; d=json.load(sys.stdin); print(f'共 {len(d)} 筆'); print(json.dumps(d[:3], indent=2, ensure_ascii=False))\""}
```

## Supabase REST API 快速參考

| 操作 | URL 格式 |
|------|---------|
| 查全部 | `/rest/v1/表名?select=*` |
| 篩選 | `/rest/v1/表名?欄位=eq.值` |
| 多條件 | `/rest/v1/表名?欄位1=eq.值&欄位2=gt.值` |
| 排序 | `/rest/v1/表名?order=欄位.desc` |
| 限制筆數 | 加 `&limit=50` |
| 只要某些欄位 | `?select=name,date,status` |

| 條件 | 寫法 | 意思 |
|------|------|------|
| 等於 | `?col=eq.value` | col = value |
| 不等於 | `?col=neq.value` | col != value |
| 大於 | `?col=gt.value` | col > value |
| 大於等於 | `?col=gte.value` | col >= value |
| 包含 | `?col=like.*keyword*` | col LIKE '%keyword%' |
| 在列表中 | `?col=in.(a,b,c)` | col IN (a, b, c) |
| 為空 | `?col=is.null` | col IS NULL |

必要的 Header：
```
-H 'apikey: <anon_key>'
-H 'Authorization: Bearer <anon_key>'
```

## 如果主人不想給你 Key

建議主人把該專案的 Supabase 加到 Server proxy：
1. 在 proxy.ts 的 PROXY_TARGETS 加一個新的 target
2. 在 .env 加該專案的 SUPABASE key
3. 這樣你就能用 proxy_fetch 安全查詢，Key 不出門

---

## 常見問題

| 問題 | 原因 | 解法 |
|------|------|------|
| curl 回 401 | key 錯或過期 | 確認 anon key 正確 |
| curl 回空陣列 | 表存在但沒資料，或條件太嚴格 | 先不加條件查全部 |
| curl 回 404 | 表名錯 | 先打 /rest/v1/ 列出所有表 |
| 「relation does not exist」 | 表不存在 | 確認表名拼寫，區分大小寫 |
| 資料太多 | 沒加 limit | 加 &limit=50 |

---

**記住：query_supabase 只能查 OpenClaw 的資料庫。外部專案要用 curl + Supabase REST API。**
