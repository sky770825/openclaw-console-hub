## 使用方式

1. **客戶問你問題** — 先在這裡找到對應題目
2. **看「標準回答」** — 直接講給客戶聽（已口語化、專業但不嚇人）
3. **看「進階說明」** — 自己心裡有底的技術細節，必要時補充
4. **看「話術建議」** — 怎麼說讓客戶安心、不會砍價或起爭議

> 原則：**先講結論，再講原因**。客戶要的是答案，不是上課。

## 一、報價與費用

### Q1：做一個網站要多少錢？

**客戶問法**
「我想做一個網站，大概要花多少錢？」「網站報價大概多少？」

**標準回答**
網站費用取決於幾個因素：頁面數量、功能複雜度、設計客製程度。一般來說：

- **形象官網（5～8 頁）**：3 萬～8 萬
- **企業官網（10～20 頁，含後台）**：8 萬～20 萬
- **電商網站**：15 萬～50 萬以上
- **客製化系統（會員、預約、平台）**：30 萬起跳

我會先了解您的需求，再給您明確的報價單，不會有模糊空間。

**進階說明**
- 報價公式參考：`(頁面數 x 單頁設計費) + 功能模組費 + 後台建置費 + 測試上線費`
- 單頁設計費行情：簡單版型 3,000～5,000 / 客製設計 8,000～15,000
- WordPress 比純客製便宜 30～50%，但客製的效能和彈性更好
- 電商要加上金流串接、物流設定、庫存管理的工時

**話術建議**
- 不要直接報一個數字，先反問：「方便跟我說您想做什麼樣的網站嗎？」
- 用「區間」回答，不要用精確數字（避免客戶拿最低價綁你）
- 可以準備 2～3 個過去案例的截圖和價格帶，讓客戶「對號入座」

### Q2：為什麼做一個網站這麼貴？別人報價便宜很多

**客戶問法**
「我朋友說他的網站才一萬塊耶」「蝦皮上面有人報價三千」「為什麼你這麼貴？」

**標準回答**
價格差異主要來自三個地方：

1. **設計品質** — 便宜的通常用現成模板直接套，我們是根據您的品牌量身設計
2. **後續服務** — 有些報價不含修改、不含上線後的維護，遇到問題找不到人
3. **技術水準** — 網站的載入速度、手機適應、SEO 基礎，這些都需要工時

就像裝潢房子，IKEA 自己組和找設計師，價格當然不同，但出來的結果和住起來的感覺也完全不一樣。

**進階說明**
- 蝦皮三千的通常是 WordPress 裝主題 + 換文字圖片，不含網域主機
- Wix/Squarespace DIY 方案年費約 5,000～15,000，但 SEO 和客製受限
- 真正的成本在「溝通」和「修改」，不只是寫程式

**話術建議**
- 絕對不要貶低競爭對手，改用「我們的服務包含⋯⋯」來凸顯差異
- 準備一張「我們的報價包含什麼 vs. 低價方案通常不含什麼」的比較表
- 如果客戶預算真的有限，可以建議先做基本版，後續再擴充

### Q3：可以先做再付錢嗎？

**客戶問法**
「可以做完滿意再付嗎？」「能不能先看到成品？」

**標準回答**
我們採分期付款方式，通常是：

- **簽約時**：收取 30～50% 訂金
- **設計稿確認後**：收取 30%
- **上線完成後**：收取尾款 20～40%

設計稿階段您就能看到網站的樣子，確認滿意才會進入下一階段開發。這樣雙方都有保障。

**進階說明**
- 訂金是保障雙方的基本做法，業界標準
- 建議用「階段驗收」綁付款，每個階段有明確的交付物
- 如果客戶堅持不付訂金，通常是高風險客戶，建議婉拒
- 尾款比例不要太高（不超過 30%），避免客戶拿到網站後拖延付款

**話術建議**
- 「分期付款其實對您更有利，每個階段您都可以確認方向對不對，不滿意我們馬上調整」
- 不要說「我們不信任你」，改說「這是我們保障客戶的標準流程」

### Q4：每年還要付什麼費用？

**客戶問法**
「網站做好之後每年還要花錢嗎？」「有沒有隱藏費用？」

**標準回答**
網站上線後，每年的基本費用有：

| 項目 | 費用（年） | 說明 |
|------|-----------|------|
| 網域續約 | 800～1,500 元 | 您的網址，例如 yourname.com.tw |
| 主機 / 伺服器 | 3,000～12,000 元 | 網站存放的空間，影響速度 |
| SSL 憑證 | 0～3,000 元 | 讓網址顯示 https，免費方案也有 |

這些是基本開銷，就像開店要付房租和水電一樣。維護合約是另外的選擇，看您的需求。

**進階說明**
- Let's Encrypt 免費 SSL 搭配 Cloudflare 基本上零成本
- 共享主機（如 Bluehost、SiteGround）年費 3,000～6,000
- VPS（如 Linode、DigitalOcean）月費 150～1,500，適合流量大的站
- 台灣的 .com.tw 網域年費約 800，.tw 約 800，.com 約 400～500
- Google Workspace 信箱如果客戶需要，另計（每人每月約 200）

**話術建議**
- 主動列清單，讓客戶覺得你很透明
- 「這些就像手機的月租費，是網站持續運作的基本成本」
- 強調你會在合約裡寫清楚，不會有意外的費用

### Q5：維護費包含什麼？不付可以嗎？

**客戶問法**
「一定要付維護費嗎？」「維護費都在做什麼？」「我自己維護行不行？」

**標準回答**
維護費不是強制的，但建議至少第一年要有。維護費通常包含：

- **系統更新**：WordPress 核心、外掛、PHP 版本更新（避免被駭）
- **備份**：定期備份網站資料，出事能馬上復原
- **小幅修改**：每月 1～2 次文字/圖片更換（依合約）
- **故障排除**：網站掛了、跑很慢，優先處理
- **安全監控**：阻擋惡意攻擊、垃圾留言

行情大約每月 2,000～5,000 元，看網站規模。不付維護費也可以，但出問題時就按次計費，通常比較貴。

**進階說明**
- 維護費是接案者的穩定收入來源，盡量推
- 建議分方案：基礎版（只含更新備份）/ 標準版（含小改）/ 進階版（含 SEO 報告）
- WordPress 不更新真的會被駭，這不是恐嚇，是事實
- 可以用 ManageWP、MainWP 批次管理多個客戶的 WordPress

**話術建議**
- 「您可以把它想成網站的『保險』，平常沒事付一點點，出事的時候不用花大錢」
- 不要說「不付維護費我就不管你」，改說「我們也提供按次計費的服務」
- 強調安全風險：「WordPress 每年有上萬次漏洞攻擊，不更新等於家門不上鎖」

### Q6：可以殺價嗎？有沒有折扣？

**客戶問法**
「能不能算便宜一點？」「我是長期客戶，有優惠嗎？」「預算有限，可以打折嗎？」

**標準回答**
我理解預算的考量。我們的報價是根據實際工時和功能來算的，每一項都有對應的工作內容。不過我們可以這樣調整：

1. **減少頁面數** — 先做最重要的幾頁，之後再擴充
2. **簡化設計** — 用現有版型微調，而不是全客製
3. **分期開發** — 第一期先上線核心功能，第二期再加進階功能

這樣可以在預算內完成，同時不犧牲品質。

**進階說明**
- 殺價是正常的，不要生氣，但也不要隨便降價
- 降價 = 降規格，永遠不要「同樣的東西賣便宜」
- 如果客戶預算真的差很多，可能不是你的目標客群，禮貌婉拒
- 長期合作的折扣可以給，但用「維護費折扣」而不是「建置費折扣」

**話術建議**
- 「與其降低品質，不如我們一起看看哪些功能可以後面再加」
- 「我可以給您一個精簡版的方案，預算省 30%，效果一樣好」
- 絕對不要說「好啦算你便宜」，這會讓客戶覺得你本來就報太高

### Q7：為什麼還要收設計費？不是寫程式就好了嗎？

**客戶問法**
「設計不是就畫一畫嗎？」「為什麼設計要另外收費？」「直接套模板不行嗎？」

**標準回答**
設計是網站最重要的部分之一，它決定了訪客的第一印象和信任感。設計工作包含：

- **品牌研究**：了解您的產業、客群、競爭對手
- **架構規劃**：決定資訊怎麼排列，讓訪客找到想要的東西
- **視覺設計**：配色、字型、版面、圖片風格
- **互動設計**：按鈕、選單、動畫效果

好的設計能提高轉換率（讓更多訪客變成客戶），這個投資是值得的。

**進階說明**
- 設計費行情：首頁 15,000～30,000 / 內頁 5,000～15,000
- UI/UX 設計用 Figma，交付 .fig 檔 + 設計規範文件
- 「套模板」也需要設計能力來調整，不是複製貼上就好
- 可以準備 A/B 案例：同樣的內容，好設計 vs. 差設計的轉換率差異

**話術建議**
- 「您去餐廳吃飯，菜單設計得漂亮的店，是不是感覺比較有品質？網站也是一樣的道理」
- 「設計費其實是在幫您省錢 — 好的設計讓訪客留下來，不好的設計訪客三秒就走了」

### Q8：可以拆成小案子分開做嗎？

**客戶問法**
「我先做首頁就好，其他的之後再說」「能不能一頁一頁付錢？」

**標準回答**
可以的，我們支持分階段開發。建議這樣拆：

- **第一期（核心）**：首頁 + 關於我們 + 服務項目 + 聯絡表單 → 先上線
- **第二期（擴充）**：部落格 / 作品集 / 會員系統
- **第三期（進階）**：電商 / 預約系統 / API 串接

不過要提醒您，分開做的總費用會比一次做完稍高一些，因為每次開發都有環境建置和溝通的成本。

**進階說明**
- 分階段的溢價大概 15～25%（重新理解需求、環境切換）
- 技術上要在第一期就規劃好擴充性，否則第二期要重寫
- CMS 選型要一開始就對（WordPress vs. Headless vs. 純靜態）
- 建議用 MVP（最小可行產品）思維幫客戶規劃

**話術建議**
- 「分階段是很聰明的做法，先用最少的預算驗證市場反應」
- 預先告知：「不過我會先規劃好整體架構，這樣後面加功能才不會重做」

### Q9：報價單上的項目我看不懂，可以解釋嗎？

**客戶問法**
「什麼是 RWD？」「CMS 是什麼？」「SSL 要幹嘛？」

**標準回答**
沒問題，我來用白話解釋：

| 術語 | 白話說明 |
|------|---------|
| RWD（響應式設計） | 網站在手機、平板、電腦上都能正常瀏覽，自動調整版面 |
| CMS（後台管理系統） | 讓您自己可以登入後台改文字、換圖片，不用找工程師 |
| SSL 憑證 | 讓網址從 http 變成 https，瀏覽器不會顯示「不安全」 |
| SEO 基礎優化 | 讓 Google 更容易找到您的網站，提高搜尋排名 |
| DNS 設定 | 把您買的網址指向網站主機，讓網址能正確打開 |
| API 串接 | 讓網站可以跟其他服務（LINE、金流、物流）對接 |

之後有任何不懂的，隨時問我。

**進階說明**
- 準備一份「術語對照表」（PDF 或網頁），報價時一起附上
- 客戶問術語是好事，代表他在認真看你的報價
- 不要嫌客戶不懂技術，耐心解釋會建立信任

**話術建議**
- 「這些術語確實很專業，我來用比較生活化的方式說明」
- 用類比：RWD = 衣服的彈性布料（穿什麼身材都好看）
- 用類比：CMS = 手機裡的備忘錄（自己就能改內容）

### Q10：定金付了但我想取消，可以退費嗎？

**客戶問法**
「我不想做了，訂金可以退嗎？」「做到一半想取消要付多少？」

**標準回答**
根據我們的合約條款：

- **尚未開始設計**：可退訂金的 50～70%（扣除前期溝通與規劃成本）
- **設計稿階段**：依已完成工作比例收費，退還剩餘款項
- **已進入開發**：依完成進度計算，已完成部分需付費
- **已上線**：無法退費

我會在合約裡寫清楚退費條款，讓雙方都安心。

**進階說明**
- 合約必須有退費/取消條款，沒寫的話糾紛處理會很麻煩
- 訂金在法律上可分為「定金」（不可退）和「訂金」（可退），注意用字
- 建議合約用「定金」，並明確寫「若甲方取消，定金不予退還」
- 消保法對 B2B 和 B2C 的規定不同，接個人案件要注意 7 天鑑賞期是否適用

**話術建議**
- 「我完全理解，所以我們合約裡有寫清楚各階段的計費方式」
- 「已經完成的工作量我會如實跟您報告，很透明的」
- 不要因為怕丟案子就說「全額退費」，這樣你會白做工

## 二、時程與流程

### Q11：網站做好要多久？

**客戶問法**
「大概多久可以上線？」「最快幾天做好？」「可以一個禮拜交件嗎？」

**標準回答**
一般時程如下：

| 網站類型 | 工作天數 | 說明 |
|---------|---------|------|
| 一頁式形象頁 | 5～10 天 | Landing page，單頁捲動 |
| 形象官網（5～8 頁） | 2～4 週 | 含設計、開發、測試 |
| 企業官網（含後台） | 4～8 週 | 含 CMS、內容上稿 |
| 電商網站 | 6～12 週 | 含金流、物流、商品上架 |
| 客製化系統 | 8～16 週 | 看功能複雜度 |

這個時間包含您的回覆和確認時間。通常客戶端的確認速度會影響整體進度。

**進階說明**
- 時程最大的變數是「客戶確認速度」，不是開發速度
- 建議在合約加上：「客戶確認逾 X 個工作天視為同意」
- 加急費建議收 30～50%（因為你得排擠其他案子）
- 實際工時估算：設計 40% / 開發 35% / 測試修改 15% / 溝通 10%

**話術建議**
- 「我先確認您的需求，就能給您準確的時程表」
- 「時程有一半取決於我們的合作效率，如果您回覆快，我們做得也快」
- 不要為了接案說「一個禮拜就好」，到時候做不到更麻煩

### Q12：流程是怎樣的？我需要準備什麼？

**客戶問法**
「做網站的步驟是什麼？」「我要提供什麼資料？」

**標準回答**
我們的合作流程分 6 個步驟：

1. **需求訪談**（1～2 次會議）— 了解您的產業、目標客群、想要的功能
2. **報價簽約** — 確認報價、簽合約、收訂金
3. **設計階段** — 提供設計稿給您確認（通常 1～2 版）
4. **開發階段** — 把設計做成真的網站
5. **測試修改** — 您試用網站、提出修改、我們調整
6. **上線交付** — 網站正式上線、交付管理權限、教您使用後台

您需要準備的資料：
- 公司 Logo（向量檔最好）
- 文案內容（公司介紹、服務說明、聯絡資訊）
- 產品 / 服務的照片（高解析度）
- 喜歡的參考網站 2～3 個

**進階說明**
- 準備一份「客戶資料準備清單」（Checklist），合約簽完就寄給客戶

### Q16：設計稿確認後還可以改嗎？

**客戶問法**

**標準回答**

- **小調整**（換顏色、調字型大小、移位置）：通常免費處理
- **大改動**（重新設計整個頁面、改掉整體風格）：這等於重新設計，會額外報價

所以建議在設計稿階段多花時間確認，把想法都提出來，這個階段改動的成本最低。

**進階說明**
- 合約要明確寫「設計稿修改次數」（建議 2～3 次）
- 超過次數的修改按次計費或按小時計費
- 「修改」vs.「重做」的界線要在合約裡定義清楚
- 實務上大部分客戶都在 2 次內確認，怕的是那種「改了十幾版」的客戶

**話術建議**
- 「我們合約裡有包含兩次修改，足夠讓設計達到您滿意的程度」
- 「設計階段改是最便宜的，到開發階段再改就像房子蓋好了要改格局，成本會高很多」

### Q19：可以改幾次？

**客戶問法**
「設計不滿意可以一直改嗎？」「改到我滿意為止嗎？」

### Q24：照片從哪裡來？我沒有專業照片

**標準回答**
有幾種方式：

1. **使用您的照片**（最推薦）— 即使手機拍的也行，真實感最重要
2. **專業圖庫**（付費）— 如 Shutterstock、Adobe Stock，品質高、合法授權
3. **免費圖庫** — Unsplash、Pexels，品質不錯但比較通用
4. **請攝影師拍** — 如果預算允許，效果最好

我建議至少產品照和團隊照用您自己的，其他裝飾性圖片可以用圖庫。

**進階說明**

- 圖片優化：壓縮到 100KB 以下（TinyPNG）、LazyLoad、srcset 多尺寸
- 如果客戶手機拍的照片品質太差，Lightroom 調一下或建議請攝影師

**話術建議**
- 「手機拍的照片就很好了，現在手機的相機品質很強」
- 「我會幫您處理圖片的大小和品質，您只要提供原圖就好」

### Q25：動畫效果可以很炫嗎？像某某網站那樣

**客戶問法**
「可以做很多動畫嗎？」「我想要那種捲動時東西飛進來的效果」「可以做 3D 嗎？」

**標準回答**
可以做動畫效果，不過我想先跟您討論幾個重點：

- **適度的動畫** — 捲動漸入、hover 效果、轉場動畫 → 包含在報價內
- **進階動畫** — 視差捲動（Parallax）、複雜的 SVG 動畫 → 需額外評估
- **重度動畫** — 3D 效果、WebGL、大量粒子效果 → 需要專案報價

動畫會影響網站的載入速度和手機上的體驗。我的建議是「少而精」，在關鍵位置加動畫，比到處都在動更有效果。

**進階說明**
- 基礎動畫：CSS transition/animation + Intersection Observer（AOS.js / GSAP ScrollTrigger）
- 中階動畫：GSAP、Framer Motion、Lottie（After Effects → JSON）
- 重度動畫：Three.js、WebGL、Spline、Rive → 工時爆增
- 動畫越多，Lighthouse Performance 分數越低，CWV 會爆
- 移動端要特別注意：複雜動畫在低階手機會卡頓

**話術建議**
- 「這個效果很漂亮，我們可以做。不過我建議用在首頁就好，內頁保持簡潔」
- 「動畫太多的話手機打開會變慢，我幫您挑最有效果的幾個位置」

**進階說明**

## 四、技術相關

### Q27：手機版會自動適應嗎？

### Q42：SSL 是什麼？一定要裝嗎？

**客戶問法**
「SSL 是什麼東西？」「不裝會怎樣？」「要錢嗎？」

**標準回答**
SSL 是一種網站的安全憑證，作用是：

- 讓網址從 `http` 變成 `https`
- 網站和訪客之間的資料會被加密保護
- 瀏覽器不會顯示「不安全」的紅色警告

**一定要裝。** 2026 年沒有 SSL 的網站：
1. Google Chrome 會標示「不安全」，嚇跑訪客
2. Google 搜尋排名會降低
3. 如果有表單（聯絡表單、登入），資料會被明文傳輸

好消息是：免費的 SSL 就夠用了（Let's Encrypt），我會幫您設定。

**進階說明**
- Let's Encrypt：免費、自動續約、業界標準
- Cloudflare：提供免費 SSL + CDN + WAF
- 付費 SSL（OV/EV）：企業用，可以顯示公司名稱，但 Chrome 已取消綠色標記
- 設定：Certbot 自動化 / Cloudflare Full(Strict) 模式最方便
- HSTS（HTTP Strict Transport Security）強制 HTTPS

**話術建議**
- 「SSL 就像幫您的網站穿上防彈衣，而且免費的」
- 「沒有 SSL 的話，客人進您的網站會看到『不安全』三個大字，誰敢下單？」
- 「費用我已經包含在報價裡了，您不用額外付」

### Q44：保固多久？

**客戶問法**
「做完有保固嗎？」「保固期內壞了免費修嗎？」

**標準回答**
上線後提供 30～90 天的免費保固期（依合約），保固範圍包含：

- 網站功能的 Bug 修復（不是設計修改）
- 瀏覽器相容性問題
- 表單、連結等功能異常

**不包含**的項目：
- 設計變更或新增功能
- 客戶自己修改造成的問題
- 主機或網域到期造成的停機

保固期過後，建議簽維護合約，或改為按次計費。

**進階說明**
- 保固期建議 30 天（小案子）到 90 天（大案子）
- 「Bug」的定義要在合約裡寫清楚：「與規格書不符的功能異常」
- 客戶常把「想改的東西」說成「Bug」，要有判斷力
- 保固期內的工時要估進專案成本

**話術建議**
- 「上線後三個月內有任何功能問題，我免費修到好」
- 「保固是針對程式問題，設計調整是另外的服務喔」

### Q46：網站上線後需要更新嗎？

**客戶問法**
「網站做好了不就好了嗎？」「還需要更新什麼？」

**標準回答**
網站跟手機一樣，需要定期更新：

1. **系統更新**（每月）— WordPress 核心、外掛、主題的安全更新
2. **內容更新**（看需求）— 新產品、新消息、部落格文章
3. **SSL 憑證更新**（每年或自動）— 維持 https 安全連線
4. **備份確認**（每月）— 確保備份正常運作

不更新的風險：被駭客攻擊（最常見）、出現安全漏洞、瀏覽器顯示異常。

**進階說明**
- WordPress 更新是維護費最主要的工作
- 更新前一定要先備份（因為更新可能造成衝突）
- PHP 版本升級也要注意相容性（PHP 8.x 有些老外掛不支援）
- 建議用 staging 環境先測試更新再套用到正式站

**話術建議**
- 「網站不是做完就放著不管的，就像車子要定期保養一樣」
- 「不更新的話，就像家裡的鎖用了十年沒換，小偷比您更清楚怎麼開」

### Q48：我想改版 / 重做，舊網站怎麼辦？

**客戶問法**
「我現有的網站想改版」「重做的話舊網站的內容可以保留嗎？」「改版期間網站會斷線嗎？」

**標準回答**
改版是很常見的需求，流程如下：

1. **評估現有網站** — 看哪些可以保留、哪些要重做
2. **在測試環境開發新版** — 舊網站照常運作，不受影響
3. **內容搬移** — 舊網站有價值的內容會搬到新版
4. **切換上線** — 新版做好測試沒問題後，一次切換過去
5. **舊網站備份** — 保留備份以防萬一

**改版期間您的舊網站完全不受影響**，我們是做好新的再切換，不是直接在舊的上面改。

**進階說明**
- SEO 注意事項：舊網址要做 301 Redirect 到新網址，否則排名會掉
- 內容搬移：如果是 WordPress to WordPress 可以匯出匯入
- 如果舊網站是別的公司做的，要取得 FTP/面板 權限或至少備份
- 改版 vs. 重做：改版（保留架構換皮）通常比重做便宜 30～50%

**話術建議**
- 「改版期間舊網站照常運作，不會有任何中斷」
- 「舊網站有累積的 SEO 權重，我們會做好轉址保留這些資產」

### Q50：網站搬家要怎麼做？

**客戶問法**
「我想從 A 主機搬到 B 主機」「可以幫我搬網站嗎？」

**標準回答**
網站搬家我可以幫您處理，流程如下：

1. 備份現有網站（檔案 + 資料庫）
2. 在新主機上建立環境
3. 把網站還原到新主機
4. 測試所有功能正常
5. 修改 DNS 指向新主機
6. 等待 DNS 生效（通常 1～24 小時）
7. 確認一切正常後，關閉舊主機

搬家期間網站基本上不會中斷，DNS 切換的那段時間可能有極短的延遲。

**進階說明**
- WordPress 搬家工具：All-in-One WP Migration、Duplicator
- 手動搬家：tar 打包檔案 + mysqldump 匯出資料庫
- DNS TTL 建議在搬家前調低到 300 秒（5 分鐘），加速切換
- 搬家後檢查：永久連結、圖片路徑、SSL、.htaccess
- 新舊主機建議重疊一週，確認沒問題再關舊的

**話術建議**
- 「搬家的事交給我，您不用操心任何技術細節」
- 「過程中您的網站基本不會中斷，最多有幾分鐘的延遲」

- 防垃圾：reCAPTCHA v3 / hCaptcha / Honeypot
- 表單驗證：前端（即時回饋）+ 後端（安全防護）
- 客戶常要求的：檔案上傳（注意大小限制和安全）、條件邏輯（選 A 顯示 B）

**話術建議**
- 「基本的聯絡表單包含在報價裡，進階表單可以另外加」
- 「表單寄出後我會設定自動通知，您不用一直上網站看有沒有新訊息」

### Q56：可以串社群媒體嗎？

**客戶問法**
「可以放 Facebook 和 Instagram 嗎？」「社群連結可以加嗎？」

**標準回答**
可以的，常見的社群串接：

- **社群連結圖示** — 放在頁尾或側邊欄，點擊前往您的社群頁面
- **Facebook 粉專嵌入** — 在網站上顯示您的粉專動態
- **Instagram 動態牆** — 顯示最新的 IG 貼文
- **社群分享按鈕** — 讓訪客分享您的文章到社群
- **Facebook Pixel** — 追蹤廣告成效（如果有投放 FB 廣告）

基本的社群連結圖示包含在報價裡，嵌入動態牆或 Pixel 設定是額外的項目。

**進階說明**
- Facebook oEmbed 2024 已改政策，需要 App ID 才能嵌入
- Instagram 嵌入用 Instagram Basic Display API（已改為 Instagram API with Instagram Login）
- Meta Pixel 安裝用 GTM（Google Tag Manager）
- 社群分享按鈕不要用官方的（太慢），用自訂連結（`https://www.facebook.com/sharer/sharer.php?u=`）

**話術建議**
- 「社群串接可以讓您的網站和社群互相導流，效果加倍」
- 「如果您有投放 Facebook 廣告，我會幫您裝好追蹤碼，這樣廣告效果看得更清楚」

## 八、法律與合約

### Q57：網站所有權是誰的？

**客戶問法**
「網站做完是我的嗎？」「你們做的東西歸誰？」「以後可以給別人接手嗎？」

**標準回答**
**尾款付清後，網站的所有權歸您所有。** 具體來說：

- **網站內容**（文字、圖片、影片）— 您提供的素材本來就是您的
- **設計成果**（版面設計、配色方案）— 付清後歸您所有
- **程式碼**（依合約約定）— 通常也歸您所有
- **網域** — 註冊在您名下，完全歸您
- **主機上的檔案** — 歸您所有

我們合約會明確寫清楚智慧財產權的歸屬，確保沒有模糊空間。

**進階說明**
- 台灣著作權法：受雇 vs. 承攬，著作財產權歸屬不同
- 建議在合約明確寫：「甲方（客戶）支付全部款項後，本專案之著作財產權歸甲方所有」
- 如果使用 GPL 授權的 WordPress 主題/外掛，GPL 部分的程式碼不能限制
- 客製化程式碼和通用元件（Framework）可以分開處理
- 設計原始檔（Figma / PSD）是否交付要另外約定

**話術建議**
- 「付清款項後，網站完全是您的，您可以交給任何人維護」
- 「我會在合約裡寫清楚，讓您完全放心」
- 這個回答能大幅提高客戶的信任感

### Q58：原始碼會給我嗎？

**客戶問法**
「我可以拿到程式碼嗎？」「原始碼可以給我嗎？」「設計檔案可以給我嗎？」

**標準回答**
會的。尾款付清後，我會交付：

- **網站完整檔案** — 包含所有程式碼、圖片、資料庫
- **設計原始檔**（依合約）— Figma 連結或原始設計檔
- **管理員帳號** — 所有後台的最高權限
- **文件** — 帳號密碼清單、操作教學、技術文件

您拿到這些東西後，就算換人維護也完全沒問題。

**進階說明**
- 程式碼交付方式：Git 私人儲存庫 / ZIP 檔案 / FTP 存取權限
- 設計檔案交付看合約，有些設計師不交付 .fig / .psd 原始檔
- 建議寫在合約裡：「交付物包含：原始碼、設計檔、資料庫備份、帳號密碼清單」
- 如果用的是客製化框架或自己的 boilerplate，要衡量是否包含

**話術建議**
- 「所有東西都會給您，我不會綁住任何客戶」
- 「交付的時候我會做一次完整的交接，確保您或下一個合作夥伴都能順利接手」

### Q59：要簽合約嗎？

**客戶問法**
「需要簽合約嗎？」「有沒有合約範本？」「口頭約定可以嗎？」

**標準回答**
**一定要簽合約。** 合約保護的是雙方，不只是我。合約會載明：

- 專案範圍（做什麼、不做什麼）
- 費用和付款方式
- 時程和里程碑
- 修改次數和變更流程
- 智慧財產權歸屬
- 保固期和維護條款
- 取消和退費條件

我有標準的合約範本，根據您的專案調整內容。簽約方式可以紙本或電子簽章（DocuSend / 點簽）都行。

**進階說明**
- 不簽合約的案子不要接，再小都要有書面約定
- 合約形式：正式合約 > Email 確認 > LINE 訊息（都有法律效力，但正式合約最清楚）
- 建議加「爭議處理」條款：以 OO 地方法院為管轄法院
- 保密條款：客戶的商業資訊不外洩
- 電子簽章法：2001 年通過，電子簽章與紙本簽章有同等法律效力

**話術建議**
- 「合約是保護您的，確保我做到承諾的每一件事」
- 「我們的合約寫得很清楚，不會有模糊的地方」
- 如果客戶不想簽合約：「至少讓我用 Email 把我們討論的內容整理確認，這也算書面紀錄」

### Q60：我想中途加功能，費用怎麼算？

**客戶問法**
「做到一半我突然想加一個功能」「可以追加嗎？」「追加要另外收費嗎？」

**標準回答**
可以追加，這在專案裡很常見。流程如下：

1. 您提出想追加的功能
2. 我評估工時和費用，提供追加報價單
3. 您確認同意後，我把它排進開發計畫
4. 追加功能可能會影響原本的上線時程，我會提前告知

**追加是需要額外費用的**，因為原始報價是根據原始需求計算的。不過如果是小調整（5 分鐘以內能改好的），我通常會順手幫您處理。

**進階說明**
- 合約要有「變更管理」條款：「超出原始範圍的需求變更，需另行報價」
- 追加報價要書面確認（Email / LINE 截圖），避免事後爭議
- 追加如果影響到已完成的架構，費用要含「重構」工時
- 建議設一個「彈性額度」（例如 10% 的工時緩衝），小追加可以吸收

**話術建議**
- 「追加沒問題，我來評估一下費用和時程的影響」
- 「這是很好的想法，讓我看看怎麼用最小的成本加進去」
- 不要直接說「加錢」，先肯定客戶的想法，再說明需要調整預算

## 附錄：實用工具與資源

### 報價計算參考

```
基本公式：
  總價 = 設計費 + 開發費 + 測試上線費 + 溝通管理費

設計費 = (首頁設計 × 1) + (內頁設計 × 頁數) + (手機版調整 × 頁數)
開發費 = (前端切版 × 頁數) + (功能模組 × 模組數) + (後台建置)
測試費 = 總工時 × 10%
管理費 = 總工時 × 15%
```

### 報價單必備項目

- [ ] 專案範圍描述
- [ ] 頁面清單和功能規格
- [ ] 包含項目（RWD、SEO 基礎、GA 安裝等）
- [ ] 不包含項目（內容撰寫、攝影、進階 SEO 等）
- [ ] 時程表
- [ ] 付款方式和條件
- [ ] 修改次數說明
- [ ] 保固期說明
- [ ] 有效期限（報價通常 14～30 天有效）

### 客戶溝通原則

1. **先聽再說** — 讓客戶把話說完，不要急著回答
2. **用比喻解釋** — 技術名詞用生活化的方式說明
3. **給選擇題** — 不要問開放式問題（「您想怎麼做？」），改成選擇題（「A 方案和 B 方案您比較偏好哪個？」）
4. **書面確認** — 重要的決定用文字紀錄
5. **管理期待** — 不要過度承諾，寧可少說多做
6. **主動更新** — 不要等客戶問進度，主動報告

### 紅旗警訊（遇到這些狀況要小心）

| 警訊 | 說明 |
|------|------|
| 不願意付訂金 | 可能是信用有疑慮 |
| 需求一直改 | 客戶自己不確定要什麼 |
| 比較很多家的價格 | 只看價格不看品質 |
| 要求「做完滿意再付」 | 不尊重專業 |
| 晚上 11 點還在傳訊息 | 邊界感不好 |
| 「很簡單啦，應該不用多久」 | 不理解工作量 |
| 「幫我做好，之後有更多案子」 | 空頭支票 |
| 提供的參考網站是 Apple / Tesla 等級 | 預期和預算不符 |
