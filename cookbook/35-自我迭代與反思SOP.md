# Cookbook 35: 自我迭代與反思SOP (v2.0 - Flash 模型專用版)

> Flash 模型視角：你搞砸了一件事（例如任務失敗）。別怕，這是一個讓你變強的機會。照著下面的步驟做，老蔡會更欣賞你。

## 範例場景：你建立的任務 t123 執行失敗，日誌顯示 command not found: jq。

### 步驟一：承認現實，不要隱藏

你的任務：複製這句話，填上資訊，回報給老蔡：「【狀況】任務 t123 失敗了。原因是腳本執行時找不到 jq 這個指令。」

### 步驟二：玩「五個為什麼」遊戲找出真兇

你的任務：在腦中或用 ask_ai 問自己下面的問題，直到找到根本原因。

1.  為什麼腳本找不到 jq？ → 因為伺服器上沒安裝 jq。
2.  為什麼沒安裝 jq 腳本還會用它？ → 因為我寫的腳本直接假設了 jq 存在。
3.  為什麼我會做這個假設？ → 因為我複製的「Cookbook 32」裡的模板就直接用了 jq。
4.  為什麼模板會這樣寫？ → 根本原因：模板本身不夠健壯，缺少對依賴項的前置檢查。

### 步驟三：把教訓變成一句話

你的任務：根據「真兇」，總結出一句話的教訓。例如：「教訓：所有腳本模板在執行核心邏輯前，必須先檢查所有外部依賴項是否存在。」

### 步驟四：升級你的武器庫 (建立機制)

你的任務：這一步最關鍵！你要修復那個有問題的模板，讓以後的你不再犯錯。

行動：執行 write_file，打開 cookbook/32-高效能腳本模板.md，在 Curl API 模板的 main() 函數開頭，加上這段檢查程式碼：

``bash
if ! command -v jq &> /dev/null; then
  log "錯誤：必要的工具 jq 未安裝。請先安裝它。"
  exit 1
fi
`

### 步驟五：在你的功勞簿上記上一筆

你的任務：執行 write_file，在 GROWTH.md 檔案的末尾，加上你的成長記錄。
