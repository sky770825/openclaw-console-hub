## openclaw_tasks（任務板）

### Schema（Supabase 真實欄位）

| 欄位 | 型別 | 說明 | 範例 |
|------|------|------|------|
| `id` | text PK | 任務 ID | `"task-20260227-001"` |
| `title` | text | 任務名稱 | `"修復登入 Bug"` |
| `cat` | text | 分類 | `feature` / `bugfix` / `learn` / `improve` |
| `status` | text | 狀態 | `queued` / `in_progress` / `done` |
| `progress` | int | 進度 0-100 | `75` |
| `auto` | boolean | 可自動執行？ | `true` |
| `from_review_id` | text | 來源審核 ID | `"r3"` |
| `subs` | jsonb | 子任務 | `[{"t":"步驟一","d":true}]` |
| `thought` | text | 說明 + metadata JSON | `"修改登入流程\n\n<!--meta{\"owner\":\"小蔡\"}meta-->"` |
| `updated_at` | timestamp | 最後更新 | 自動 |
| `created_at` | timestamp | 建立時間 | 自動 |

> **注意：沒有 `owner`、`agent`、`executor`、`result`、`priority` 欄位！**
> - `owner` 存在 `thought` 欄位的 JSON metadata 裡，無法直接 select 或 filter
> - `query_supabase` 時用 `select:"*"` 拿全部資料就好，不要指定這些欄位
> - 要查特定任務者的任務？用 `select:"*"` 取回後在回覆中自己判斷 thought 內容

### 查詢範例（curl）

```bash
# 查所有任務
curl -s http://localhost:3011/api/openclaw/tasks \
  -H "Authorization: Bearer oc-oAw9leGU04IAbcS4WN3FC1SH3vq5OdPxrVJCR16iIUMPsep1"

# 建新任務
curl -X POST "http://localhost:3011/api/openclaw/tasks?allowStub=1" \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer oc-oAw9leGU04IAbcS4WN3FC1SH3vq5OdPxrVJCR16iIUMPsep1" \
  -d '{
    "title": "掃描全部 Skill 安全",
    "cat": "feature",
    "status": "ready",
    "auto": true,
    "thought": "【設計】用 aegis_scanner 掃描 skills/ 下所有目錄\n【已完成】rules.json 3 條規則\n【需要】執行掃描並存報告\n【測試方式】確認 scan_report.json 存在"
  }'

# 更新任務狀態
curl -X PATCH "http://localhost:3011/api/openclaw/tasks/TASK_ID" \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer oc-oAw9leGU04IAbcS4WN3FC1SH3vq5OdPxrVJCR16iIUMPsep1" \
  -d '{"status": "done", "result": "掃描完成，0 個威脅", "progress": 100}'

# 刪除任務
curl -X DELETE "http://localhost:3011/api/openclaw/tasks/TASK_ID" \
  -H "Authorization: Bearer oc-oAw9leGU04IAbcS4WN3FC1SH3vq5OdPxrVJCR16iIUMPsep1"
```

### 狀態說明

| status | 誰看到 | 意思 |
|--------|--------|------|
| `queued` | auto-executor | 排隊等執行 |
| `in_progress` | auto-executor | 正在跑 |
| `done` | 老蔡 | 完成 |

### cat 分類映射

| 輸入 | 會被映射為 |
|------|-----------|
| `feature`、空白、未知 | `feature` |
| `bug`、`fix`、`錯誤`、`修復` | `bugfix` |
| `learn`、`study`、`research`、`學習` | `learn` |
| `perf`、`opt`、`stability`、`improve`、`優化` | `improve` |

## openclaw_runs（執行記錄）

### Schema

| 欄位 | 型別 | 說明 |
|------|------|------|
| `id` | uuid PK | 自動產生 |
| `task_id` | text | 對應任務 ID |
| `task_name` | text | 任務名稱 |
| `status` | text | `queued`/`running`/`success`/`failed`/`cancelled` |
| `started_at` | timestamp | 開始時間 |
| `ended_at` | timestamp | 結束時間 |
| `duration_ms` | int | 執行時間（毫秒） |
| `input_summary` | text | 輸入摘要 |
| `output_summary` | text | 輸出摘要 |
| `steps` | jsonb | 執行步驟 |
| `created_at` | timestamp | 建立時間 |

### 查詢

```bash
# 查最近 20 筆執行記錄
curl -s "http://localhost:3011/api/openclaw/runs?limit=20" \
  -H "Authorization: Bearer oc-oAw9leGU04IAbcS4WN3FC1SH3vq5OdPxrVJCR16iIUMPsep1"

# 查某任務的執行記錄
curl -s "http://localhost:3011/api/openclaw/runs?taskId=TASK_ID" \
  -H "Authorization: Bearer oc-oAw9leGU04IAbcS4WN3FC1SH3vq5OdPxrVJCR16iIUMPsep1"
```

### status 映射規則

| 內部狀態 | 寫入 DB 的值 |
|---------|-------------|
| `timeout` | `failed` |
| `retrying` | `running` |
| 其他不認識的 | `failed` |

## openclaw_memory（AI 記憶庫）

### Schema

| 欄位 | 型別 | 說明 |
|------|------|------|
| `id` | text PK | 記憶 ID |
| `ts` | timestamp | 時間戳 |
| `type` | text | `note`/`pattern`/`insight`/`decision`/`error` |
| `source` | text | `manual`/`auto`/`scan`/`neuxa` |
| `title` | text | 標題 |
| `content` | text | 內容 |
| `tags` | text[] | 標籤 |
| `meta` | jsonb | 額外資料 |
| `importance` | int | 重要度 1-10 |
| `related_ids` | text[] | 關聯記憶 ID |

### 查詢

```bash
# 查記憶（可篩選 type、source）
curl -s "http://localhost:3011/api/openclaw/memory?type=insight&limit=10" \
  -H "Authorization: Bearer oc-oAw9leGU04IAbcS4WN3FC1SH3vq5OdPxrVJCR16iIUMPsep1"

# 搜尋記憶
curl -s "http://localhost:3011/api/openclaw/memory/search?q=aegis+scanner" \
  -H "Authorization: Bearer oc-oAw9leGU04IAbcS4WN3FC1SH3vq5OdPxrVJCR16iIUMPsep1"

# 存記憶
curl -X POST http://localhost:3011/api/openclaw/memory \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer oc-oAw9leGU04IAbcS4WN3FC1SH3vq5OdPxrVJCR16iIUMPsep1" \
  -d '{
    "id": "mem-aegis-v1",
    "type": "pattern",
    "source": "neuxa",
    "title": "Aegis Scanner 掃描模式",
    "content": "使用 rules.json 定義偵測規則，支援單 pattern 和多 pattern match_all 模式",
    "tags": ["security", "aegis", "scanner"],
    "importance": 7
  }'

# 批次存記憶
curl -X POST http://localhost:3011/api/openclaw/memory/batch \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer oc-oAw9leGU04IAbcS4WN3FC1SH3vq5OdPxrVJCR16iIUMPsep1" \
  -d '{"memories": [...]}'
```
