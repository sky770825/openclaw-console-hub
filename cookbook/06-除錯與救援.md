# 06 — 除錯與救援

> 系統出問題時的診斷和修復指南

---

## 一、常見問題速查

| 症狀 | 原因 | 修法 |
|------|------|------|
| Server 沒回應 | port 3011 沒在聽 | 見「Server 掛了」 |
| API 回 401 | Token 錯了 | 檢查 Authorization header |
| API 回 429 | 超過速率限制 | 等 15 分鐘 |
| 任務不被執行 | auto-executor 停了 | 見「Auto-Executor 停了」 |
| 任務一直 F 分 | 描述太模糊 | 見 04-自動化執行.md |
| Telegram 沒通知 | Bot 掛了或被踢 | 見「Telegram 故障」 |
| Gateway 無回應 | 進程掛了 | 見「Gateway 故障」 |
| Supabase 查詢失敗 | 連線斷了 | 見「Supabase 斷線」 |

---

## 二、Server 掛了

### 診斷
```bash
# 確認 port 3011 有沒有在聽
lsof -i :3011 -sTCP:LISTEN

# 如果沒有 → Server 掛了
# 如果有 → Server 活著但可能卡住
```

### 修復
```bash
# 1. 找到現有 PID（如果有的話）
lsof -i :3011 -sTCP:LISTEN

# 2. 殺掉
kill <PID>

# 3. 重啟
cd /Users/caijunchang/openclaw任務面版設計/server
nohup node dist/index.js > /tmp/openclaw-server.log 2>&1 &

# 4. 確認活了
sleep 2
curl -s http://localhost:3011/api/health
```

### 查 Server Log
```bash
# 即時 log
tail -f /tmp/openclaw-server.log

# 最近 50 行
tail -50 /tmp/openclaw-server.log

# 搜尋錯誤
grep -i "error" /tmp/openclaw-server.log | tail -20
```

## 三、Auto-Executor 停了

### 診斷
```bash
curl -s http://localhost:3011/api/openclaw/auto-executor/status \
  -H "Authorization: Bearer oc-oAw9leGU04IAbcS4WN3FC1SH3vq5OdPxrVJCR16iIUMPsep1"
```
看 `isRunning` 欄位。

### 修復
```bash
# 重新啟動
curl -X POST http://localhost:3011/api/openclaw/auto-executor/start \
  -H "Authorization: Bearer oc-oAw9leGU04IAbcS4WN3FC1SH3vq5OdPxrVJCR16iIUMPsep1"
```

### 如果是熔斷器打開了
```bash
# 查安全狀態
curl -s http://localhost:3011/api/security/status \
  -H "Authorization: Bearer oc-oAw9leGU04IAbcS4WN3FC1SH3vq5OdPxrVJCR16iIUMPsep1"
```
如果 `circuitBreakerState: "open"` → 等 5 分鐘讓它自動恢復，或查 log 找出連續失敗的原因。

## 四、Gateway 故障

### 診斷
```bash
# 找 gateway 進程
pgrep -f "openclaw-gateway"

# 測試 gateway 是否有回應
bash /Users/caijunchang/openclaw任務面版設計/scripts/gateway-healthcheck.sh
```

### 修復
```bash
# 方法一：優雅重啟
bash /Users/caijunchang/openclaw任務面版設計/scripts/restart-openclaw-gateway.sh

# 方法二：強制重啟
bash /Users/caijunchang/openclaw任務面版設計/scripts/gateway-recover.sh

# 方法三：無回應時的救援
bash /Users/caijunchang/openclaw任務面版設計/scripts/openclaw-recover-no-response.sh
```

### 重載 config（不重啟）
```bash
kill -HUP $(pgrep openclaw-gateway)
```

## 五、Telegram 故障

### NEUXA 收不到訊息

按順序排查：

```bash
# 1. 清失敗的投遞佇列
rm -f ~/.openclaw/delivery-queue/failed/*.json

# 2. 清有問題的 session
# 找 sessions.json 裡 groupId 含問題 ID 的 session，移除

# 3. 清 sessions.json 裡的 qwen* model 設定
# 找 agent:main:main 的 model 欄位，如果是 qwen 開頭就清掉

# 4. 清 Telegram pending updates
curl -s "https://api.telegram.org/bot$TELEGRAM_BOT_TOKEN/getUpdates?offset=-1"

# 5. 如果以上都不行 → 完全重啟 gateway
kill -9 $(pgrep openclaw-gateway)
# ⚠️ 不要用 SIGUSR1（會把記憶體 state 寫回磁碟）
```

## 六、Supabase 斷線

### 診斷
```bash
# 健康檢查會回報 Supabase 狀態
curl -s http://localhost:3011/api/health

# 看 supabase 欄位是 true 還是 false
```

### 如果斷線
- Server 會自動切換到記憶體存儲（`server/src/store.ts`）
- **資料不會消失**，但不會同步到雲端
- 等 Supabase 恢復後，重啟 server 即可

### 手動測試 Supabase
```bash
# 直接用 curl 測 Supabase API
source /Users/caijunchang/openclaw任務面版設計/.env
curl -s "$SUPABASE_URL/rest/v1/openclaw_tasks?select=count" \
  -H "apikey: $SUPABASE_ANON_KEY" \
  -H "Authorization: Bearer $SUPABASE_ANON_KEY"
```

## 八、救援腳本一覽

| 腳本 | 什麼時候用 | 風險 |
|------|-----------|------|
| `scripts/free-ports.sh` | port 被占 | 低（只殺特定 port） |

| `scripts/gateway-recover.sh` | gateway 掛了 | 低 |

| `scripts/openclaw-rescue.sh` | 系統需要救援 | 中 |
| `scripts/openclaw-rescue-restartall.sh` | 全部重啟 | 中高 |
| `scripts/openclaw-rescue-autofix.sh` | 自動修復常見問題 | 中 |
| `scripts/emergency-stop.sh` | 緊急停止一切 | 高（會停掉所有服務） |
| `scripts/memory-dirty-autofix.sh` | 記憶檔損壞 | 低 |
| `scripts/security-check.sh` | 安全稽核 | 無（只讀） |

## 九、Log 位置

| Log | 路徑 | 內容 |
|-----|------|------|
| Server log | `/tmp/openclaw-server.log` | Express server 輸出 |
| Taskboard log | `~/.openclaw/automation/logs/taskboard.log` | 任務板操作 |
| Gateway log | gateway 進程 stdout | NEUXA 對話 |
| Patrol log | `~/.openclaw/automation/logs/patrol.log` | 巡邏記錄 |

### 常用 log 指令
```bash
# Server 最近 log
tail -50 /tmp/openclaw-server.log

# 搜尋錯誤
grep -i "error\|fail\|crash" /tmp/openclaw-server.log | tail -20

# 搜尋特定任務
grep "TASK_ID" /tmp/openclaw-server.log

# 即時監控
tail -f /tmp/openclaw-server.log
```

## 十、緊急停止

### 停止所有執行
```bash
# 停 auto-executor（不停 server）
curl -X POST http://localhost:3011/api/openclaw/auto-executor/stop \
  -H "Authorization: Bearer oc-oAw9leGU04IAbcS4WN3FC1SH3vq5OdPxrVJCR16iIUMPsep1"

# 完全停止 server
kill $(lsof -i :3011 -sTCP:LISTEN -t)

# 核彈級停止（所有 openclaw 相關進程）
bash /Users/caijunchang/openclaw任務面版設計/scripts/emergency-stop.sh
```
