## 一、寫程式前 — 先想清楚

### 1. 確認你要改什麼
- 先用 read_file 看完整個檔案，搞清楚現有結構
- 不要只看某一段就開始改 — 你可能漏掉上下文的依賴
- 問自己：這個檔案的其他部分會不會被我的改動影響？

### 2. 確認語言和規範
| 語言 | 我們的規範 |
|------|-----------|
| TypeScript | 嚴格模式、不用 any、用 interface 不用 type alias（除非聯合型別）|
| Bash | 開頭加 `#!/bin/bash` 和 `set -euo pipefail`、變數加雙引號 |
| Python | 用 f-string、加 type hints、用 argparse 處理參數 |
| JSON | 用工具生成（jq / JSON.stringify），不要手動拼字串 |

## 二、TypeScript / Node.js 常見錯誤

### 語法類
| 你寫的 | 問題 | 正確寫法 |
|--------|------|---------|
| `/ 註解 /` | 斜線不是註解 | `/* 註解 */` 或 `/** JSDoc */` |
| `console.log()` | 生產環境不該有 | `logger.info()` 用 pino |
| `var x = 1` | var 會提升作用域 | `const x = 1` 或 `let x = 1` |
| `x == y` | 隱式型別轉換 | `x === y` 嚴格比較 |
| `function(x) {}` | 沒有型別 | `function(x: string) {}` |

### 非同步類
| 你寫的 | 問題 | 正確寫法 |
|--------|------|---------|
| `promise.then().catch()` | callback hell | `await promise` + try/catch |
| `await` 在迴圈裡 | 序列執行，慢 | `Promise.all()` 並行 |
| 忘記 `await` | Promise 沒被等待，靜默失敗 | 加 await 或 `.catch()` |
| `async` 函數沒 try/catch | 未捕獲的 rejection | 包 try/catch 或 caller 處理 |

```typescript
// ❌ 慢：序列執行
for (const url of urls) {
  const data = await fetch(url);
}

// ✅ 快：並行執行
const results = await Promise.all(urls.map(url => fetch(url)));
```

### 型別安全類
| 你寫的 | 問題 | 正確寫法 |
|--------|------|---------|
| `(x as any).foo` | 繞過型別檢查 | 定義正確的 interface |
| `JSON.parse(str)` 直接用 | 沒有型別驗證 | 用 Zod 或手動檢查 |
| `obj!.prop` | 非空斷言可能爆 | 先 if (obj) 檢查 |
| `Record<string, any>` | 弱型別 | 定義明確的 interface |

```typescript
// ❌ 不安全
const data = JSON.parse(body) as UserData;

// ✅ 安全
const parsed = JSON.parse(body);
if (!parsed || typeof parsed.name !== 'string') {
  throw new Error('Invalid user data');
}
```

### 錯誤處理類
| 你寫的 | 問題 | 正確寫法 |
|--------|------|---------|
| `catch {}` 空的 | 吞掉錯誤，無法除錯 | `catch (e) { logger.error(e); }` |
| `catch (e) { throw e }` | 多餘的重新拋出 | 不 catch，讓上層處理 |
| 只 catch Error | 可能漏掉非 Error 物件 | `catch (e: unknown)` |

```typescript
// ❌ 吞掉錯誤
try { await doSomething(); } catch {}

// ✅ 至少記錄
try {
  await doSomething();
} catch (e) {
  logger.error({ err: e }, 'doSomething failed');
}
```

## 三、安全編碼 — 絕對不能犯的

### SQL 注入
```python
# ❌ 直接拼字串
cursor.execute(f"SELECT * FROM users WHERE name = '{name}'")

# ✅ 參數化查詢
cursor.execute("SELECT * FROM users WHERE name = ?", (name,))
```

### Shell 注入
```bash
# ❌ 變數沒引號
curl $URL

# ✅ 變數加雙引號
curl "$URL"
```

```bash
# ❌ 用戶輸入直接進指令
eval "$USER_INPUT"

# ✅ 永遠不要 eval 用戶輸入
# 用白名單驗證，或用 jq/python 處理
```

### 路徑穿越
```typescript
// ❌ 沒有驗證路徑
const filePath = userInput; // "../../../etc/passwd"
fs.readFileSync(filePath);

// ✅ 限制在安全目錄內
const safePath = path.resolve(SANDBOX_DIR, userInput);
if (!safePath.startsWith(SANDBOX_DIR)) {
  throw new Error('Path traversal blocked');
}
```

### 敏感資訊洩漏
```typescript
// ❌ 錯誤回應包含系統路徑
res.json({ error: err.stack });

// ✅ 只返回安全的錯誤訊息
logger.error({ err }, 'Request failed');
res.json({ error: 'Internal server error' });
```

## 四、寫完程式後 — 自我檢查清單

每次寫完程式，跑一遍這個清單：

### 必查（不過就不要提交）
- [ ] **能 build 嗎？** — 語法正確、型別正確、import 正確
- [ ] **有沒有 console.log？** — 換成 logger
- [ ] **有沒有 any？** — 盡量用具體型別
- [ ] **catch 是空的嗎？** — 至少 log error
- [ ] **有沒有用戶輸入直接進 SQL/Shell？** — 參數化或驗證
- [ ] **路徑有沒有驗證？** — 特別是 read_file/write_file 的目標

### 建議查（品質提升）
- [ ] **變數命名清楚嗎？** — `x` 不行，`taskCount` 才行
- [ ] **函數太長了嗎？** — 超過 50 行考慮拆分
- [ ] **重複的程式碼？** — 3 次以上一樣的邏輯就抽成函數
- [ ] **邊界情況？** — 空陣列、null、undefined、空字串
- [ ] **超時保護？** — 外部 API 呼叫要有 timeout

## 五、Bash 腳本品質

### 開頭模板（每個腳本必須有）
```bash
#!/bin/bash
set -euo pipefail

# 說明：這個腳本做什麼
# 用法：bash script.sh [參數]
```

### 常見錯誤速查
| 你寫的 | 問題 | 正確寫法 |
|--------|------|---------|
| 沒有 `set -e` | 錯誤繼續執行 | `set -euo pipefail` |
| `$VAR` | 空格會拆開 | `"$VAR"` |
| `rm -rf $DIR` | DIR 為空就刪根目錄 | `rm -rf "${DIR:?}"` |
| `cd dir && ...` | 相對路徑不可靠 | 用絕對路徑 |
| `sleep 2000` | 睡 33 分鐘 | `sleep 2`（秒為單位）|
| 沒有 `chmod +x` | 腳本不能執行 | `chmod +x script.sh` |

## 六、你改別人的程式碼時

1. **先讀完整個檔案** — 不要只看你要改的那段
2. **保持風格一致** — 原作者用 2 空格你就用 2 空格，用 tab 你就用 tab
3. **不要順便重構** — 你的任務是修這個 bug，不是重寫整個模組
4. **改完跑 build** — `npm run build` 或 `tsc --noEmit`
5. **改完跑測試** — 如果有的話

## 八、黃金法則

1. **能 build 才算寫完** — 寫完馬上 build 驗證
2. **改一個東西測一個** — 不要改十個一起測
3. **不確定就先看文件** — CODEBASE-MAP.md 有所有函數簽名
4. **安全永遠排第一** — 不確定安不安全，就當不安全
5. **簡單勝過複雜** — 3 行直白的代碼好過 1 行聰明的代碼
