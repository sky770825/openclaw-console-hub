## 1. 多語系架構選擇

### 四種主流方案比較

| 方案 | 範例 URL | SEO 效果 | 實作難度 | 適用場景 |
|------|----------|----------|----------|----------|
| **子目錄（推薦）** | `example.com/en/about` | 極佳 | 中 | 大多數專案首選 |
| **子網域** | `en.example.com/about` | 佳 | 高 | 各語言內容差異大、獨立團隊維護 |
| **URL 參數** | `example.com/about?lang=en` | 差 | 低 | 不建議，Google 不推薦 |
| **客戶端切換** | `example.com/about`（JS 切換） | 極差 | 低 | 僅限 SPA 內部工具，不適合公開網站 |

### 決策流程圖

```
需要多語系？
│
├─ 是否需要 SEO？
│  ├─ 需要 → 子目錄（/en/、/ja/）← 90% 的專案選這個
│  │         或子網域（en.example.com）← 各語言獨立營運
│  │
│  └─ 不需要（內部系統）→ 客戶端切換 + localStorage 記住偏好
│
├─ 各語言內容差異？
│  ├─ 差異大（不同產品線）→ 子網域或獨立網域
│  └─ 內容一致（純翻譯）→ 子目錄
│
└─ 維護團隊？
   ├─ 同一團隊 → 子目錄（同一 codebase）
   └─ 不同團隊 → 子網域（可獨立部署）
```

### 子目錄方案（最推薦）

**優點**：
- SEO 權重集中在同一網域
- 一個 codebase、一個部署
- Google Search Console 一次管理
- SSL 只需一張憑證

**URL 結構範例**：
```
example.com/          → 預設語言（繁體中文）
example.com/en/       → 英文
example.com/ja/       → 日文
example.com/en/about  → 英文的關於頁面
```

### 子網域方案

**何時適用**：
- 日本站有日本專屬產品
- 各地區有不同定價策略
- 各語系由不同團隊維護

**URL 結構範例**：
```
www.example.com       → 繁體中文
en.example.com        → 英文
ja.example.com        → 日文
```

**DNS 設定**：
```
; A Record
www    A    203.0.113.1
en     A    203.0.113.1
ja     A    203.0.113.1

; 或用 CNAME
en     CNAME    www.example.com.
ja     CNAME    www.example.com.
```

### 語言代碼標準（ISO 639-1 + 區域碼）

| 語言 | 代碼 | 用於 | 備註 |
|------|------|------|------|
| 繁體中文 | `zh-TW` | 台灣 | 不要用 `zh-Hant`（Google 較不認） |
| 簡體中文 | `zh-CN` | 中國 | |
| 英文 | `en` | 全球 | 美式用 `en-US`，英式用 `en-GB` |
| 日文 | `ja` | 日本 | |
| 韓文 | `ko` | 韓國 | |
| 法文 | `fr` | 法國 | 加拿大法文用 `fr-CA` |
| 德文 | `de` | 德國 | |
| 西班牙文 | `es` | 西班牙 | 拉美用 `es-419` |
| 阿拉伯文 | `ar` | 中東 | RTL 語言 |
| 希伯來文 | `he` | 以色列 | RTL 語言 |

### 2.4 路由設定

**src/i18n/routing.ts**
```typescript
import { defineRouting } from 'next-intl/routing';

export const routing = defineRouting({
  // 支援的語言列表
  locales: ['zh-TW', 'en', 'ja'],

  // 預設語言
  defaultLocale: 'zh-TW',

  // 預設語言是否顯示在 URL 中
  // 'as-needed': example.com/ → 中文, example.com/en/ → 英文
  // 'always': example.com/zh-TW/, example.com/en/
  // 'never': 不在 URL 中顯示預設語言
  localePrefix: 'as-needed',

  // 路徑翻譯（可選：讓 URL 也翻譯）
  pathnames: {
    '/about': {
      'zh-TW': '/about',
      en: '/about',
      ja: '/about',
    },
    '/services': {
      'zh-TW': '/services',
      en: '/services',
      ja: '/services',
    },
  },
});

// 匯出型別供其他模組使用
export type Locale = (typeof routing.locales)[number];
export type Pathnames = keyof typeof routing.pathnames;
```

**src/i18n/request.ts**
```typescript
import { getRequestConfig } from 'next-intl/server';
import { routing } from './routing';

export default getRequestConfig(async ({ requestLocale }) => {
  // 取得目前語言
  let locale = await requestLocale;

  // 驗證語言是否在支援清單中
  if (!locale || !routing.locales.includes(locale as any)) {
    locale = routing.defaultLocale;
  }

  return {
    locale,
    messages: (await import(`../../../messages/${locale}.json`)).default,
    // 時區設定（影響日期格式化）
    timeZone: locale === 'ja' ? 'Asia/Tokyo' : 'Asia/Taipei',
    // 數字、日期格式設定
    formats: {
      number: {
        currency: {
          style: 'currency',
          currency: locale === 'ja' ? 'JPY' : locale === 'en' ? 'USD' : 'TWD',
        },
      },
      dateTime: {
        short: {
          day: 'numeric',
          month: 'short',
          year: 'numeric',
        },
      },
    },
  };
});
```

### 2.6 Layout 與頁面

**src/app/[locale]/layout.tsx**
```tsx
import { NextIntlClientProvider } from 'next-intl';
import { getMessages, getTranslations } from 'next-intl/server';
import { notFound } from 'next/navigation';
import { routing, Locale } from '@/i18n/routing';

// 靜態產生所有語言的頁面
export function generateStaticParams() {
  return routing.locales.map((locale) => ({ locale }));
}

// 動態產生各語言的 metadata
export async function generateMetadata({
  params,
}: {
  params: Promise<{ locale: string }>;
}) {
  const { locale } = await params;
  const t = await getTranslations({ locale, namespace: 'metadata' });

  return {
    title: t('title'),
    description: t('description'),
    alternates: {
      canonical: `https://example.com/${locale === 'zh-TW' ? '' : locale}`,
      languages: {
        'zh-TW': 'https://example.com/',
        en: 'https://example.com/en',
        ja: 'https://example.com/ja',
      },
    },
    openGraph: {
      locale: locale.replace('-', '_'),
      alternateLocales: routing.locales
        .filter((l) => l !== locale)
        .map((l) => l.replace('-', '_')),
    },
  };
}

export default async function LocaleLayout({
  children,
  params,
}: {
  children: React.ReactNode;
  params: Promise<{ locale: string }>;
}) {
  const { locale } = await params;

  // 驗證語言
  if (!routing.locales.includes(locale as Locale)) {
    notFound();
  }

  // 取得翻譯訊息
  const messages = await getMessages();

  // 決定文字方向
  const dir = ['ar', 'he', 'fa'].includes(locale) ? 'rtl' : 'ltr';

  return (
    <html lang={locale} dir={dir}>
      <body>
        <NextIntlClientProvider messages={messages}>
          <Header />
          <main>{children}</main>
          <Footer />
        </NextIntlClientProvider>
      </body>
    </html>
  );
}
```

| 特性 | react-i18next | react-intl |

**建議**：大多數接案場景用 **react-i18next** 即可，生態系更完整、學習曲線低。

## 6. SEO 多語系設定

### 6.1 hreflang 標籤

**用途**：告訴 Google 這個頁面有哪些語言版本，避免重複內容問題。

**HTML Head 中設定**：
```html
<head>
  <!-- 目前頁面：繁體中文 -->
  <link rel="alternate" hreflang="zh-TW" href="https://example.com/about" />
  <link rel="alternate" hreflang="en" href="https://example.com/en/about" />
  <link rel="alternate" hreflang="ja" href="https://example.com/ja/about" />
  <!-- x-default：語言選擇頁或預設語言 -->
  <link rel="alternate" hreflang="x-default" href="https://example.com/about" />
</head>
```

**重要規則**：
1. 每個語言版本的頁面都要有完整的 hreflang 標籤組（包含自己）
2. hreflang 必須**雙向**指定（A 指向 B，B 也要指向 A）
3. `x-default` 指向語言選擇頁或預設語言
4. URL 必須是**完整的絕對路徑**

### 6.2 Next.js 自動產生 hreflang

```tsx
// src/app/[locale]/layout.tsx
import { routing } from '@/i18n/routing';

export async function generateMetadata({
  params,
}: {
  params: Promise<{ locale: string }>;
}) {
  const { locale } = await params;

  // 自動產生所有語言的 alternate links
  const languages: Record<string, string> = {};
  routing.locales.forEach((loc) => {
    const path = loc === routing.defaultLocale ? '' : `/${loc}`;
    languages[loc] = `https://example.com${path}`;
  });

  return {
    alternates: {
      canonical: languages[locale],
      languages,
    },
  };
}
```

### 6.3 多語系 Sitemap

**sitemap.xml（手動撰寫格式）**：
```xml

<urlset xmlns="http://www.sitemaps.org/schemas/sitemap/0.9"
        xmlns:xhtml="http://www.w3.org/1999/xhtml">

  <!-- 首頁 -->
  <url>
    <loc>https://example.com/</loc>
```

**Next.js 動態產生 sitemap**：
```typescript
// src/app/sitemap.ts
import { MetadataRoute } from 'next';

const BASE_URL = 'https://example.com';
const locales = ['zh-TW', 'en', 'ja'];
const defaultLocale = 'zh-TW';

// 網站所有頁面
const pages = [
  { path: '/', changeFrequency: 'weekly' as const, priority: 1.0 },
  { path: '/about', changeFrequency: 'monthly' as const, priority: 0.8 },
  { path: '/services', changeFrequency: 'monthly' as const, priority: 0.8 },
  { path: '/contact', changeFrequency: 'monthly' as const, priority: 0.7 },
];

function getUrl(path: string, locale: string): string {
  const prefix = locale === defaultLocale ? '' : `/${locale}`;
  return `${BASE_URL}${prefix}${path === '/' ? '' : path}`;
}

export default function sitemap(): MetadataRoute.Sitemap {
  const entries: MetadataRoute.Sitemap = [];

  for (const page of pages) {
    for (const locale of locales) {
      const alternates: Record<string, string> = {};
      for (const altLocale of locales) {
        alternates[altLocale] = getUrl(page.path, altLocale);
      }
      alternates['x-default'] = getUrl(page.path, defaultLocale);

      entries.push({
        url: getUrl(page.path, locale),
        lastModified: new Date(),
        changeFrequency: page.changeFrequency,
        priority: page.priority,
        alternates: {
          languages: alternates,
        },
      });
    }
  }

  return entries;
}
```

### 6.6 常見 SEO 多語系錯誤

| 錯誤 | 問題 | 修正 |
|------|------|------|
| 未設 hreflang | Google 不知道語言版本關係 | 每頁加上完整 hreflang 組 |
| hreflang 單向 | A→B 有但 B→A 沒有 | 必須雙向指定 |
| canonical 衝突 | 所有語言 canonical 指向同一頁 | 每語言的 canonical 指向自己 |
| 自動翻譯不標記 | Google 視為低品質重複內容 | 機器翻譯也要加 hreflang |
| 缺 x-default | Google 不知道預設語言 | 加上 hreflang="x-default" |
| 語言代碼錯誤 | 用了 `zh` 而非 `zh-TW` | 使用正確的 ISO 代碼 |

### 7.7 封裝成 React Hook

```typescript
// src/hooks/useIntlFormatters.ts
import { useLocale } from 'next-intl';  // 或從 i18next 取得

export function useIntlFormatters() {
  const locale = useLocale();

  return {
    formatNumber: (value: number, options?: Intl.NumberFormatOptions) =>
      new Intl.NumberFormat(locale, options).format(value),

    formatCurrency: (amount: number, currency?: string) =>
      new Intl.NumberFormat(locale, {
        style: 'currency',
        currency: currency || getCurrencyForLocale(locale),
        minimumFractionDigits: ['JPY', 'TWD'].includes(currency || '') ? 0 : 2,
      }).format(amount),

    formatDate: (date: Date | string, options?: Intl.DateTimeFormatOptions) =>
      new Intl.DateTimeFormat(locale, options).format(
        typeof date === 'string' ? new Date(date) : date
      ),

    formatRelativeTime: (value: number, unit: Intl.RelativeTimeFormatUnit) =>
      new Intl.RelativeTimeFormat(locale, { numeric: 'auto' }).format(value, unit),

    formatList: (items: string[], type: 'conjunction' | 'disjunction' = 'conjunction') =>
      new Intl.ListFormat(locale, { type }).format(items),

    formatCompact: (value: number) =>
      new Intl.NumberFormat(locale, { notation: 'compact' }).format(value),
  };
}

function getCurrencyForLocale(locale: string): string {
  const map: Record<string, string> = {
    'zh-TW': 'TWD',
    'zh-CN': 'CNY',
    en: 'USD',
    'en-US': 'USD',
    'en-GB': 'GBP',
    ja: 'JPY',
    'de-DE': 'EUR',
    'fr-FR': 'EUR',
    ko: 'KRW',
  };
  return map[locale] || 'USD';
}
```

## 8. RTL 語言支援

### 8.1 什麼是 RTL

**RTL (Right-to-Left)**：文字從右到左書寫的語言。

| 語言 | 代碼 | 文字方向 | 使用人口 |
|------|------|----------|----------|
| 阿拉伯文 | `ar` | RTL | ~4.2 億 |
| 希伯來文 | `he` | RTL | ~0.9 億 |
| 波斯文 | `fa` | RTL | ~1.1 億 |
| 烏爾都語 | `ur` | RTL | ~2.3 億 |

### 8.2 HTML dir 屬性

```html
<!-- 整頁設定 -->
<html lang="ar" dir="rtl">

<!-- 局部設定（混合文字方向） -->
<p dir="rtl">مرحبا بالعالم</p>
<p dir="ltr">Hello World</p>

<!-- 自動偵測（讓瀏覽器判斷） -->
<p dir="auto">مرحبا Hello</p>
```

### 8.3 CSS 邏輯屬性（核心概念）

**不要用 left/right，改用 start/end**，CSS 會自動處理 RTL。

```css
/* 舊寫法（不支援 RTL 自動翻轉） */
.card {
  margin-left: 16px;    /* LTR 時在左邊，RTL 時也在左邊（錯的） */
  padding-right: 8px;
  text-align: left;
  border-left: 3px solid blue;
}

/* 新寫法（RTL 自動翻轉） */
.card {
  margin-inline-start: 16px;   /* LTR 時在左，RTL 時自動變右 */
  padding-inline-end: 8px;     /* LTR 時在右，RTL 時自動變左 */
  text-align: start;           /* 跟隨文字方向 */
  border-inline-start: 3px solid blue;
}
```

**完整的邏輯屬性對照表**：

| 物理屬性 | 邏輯屬性 | LTR 效果 | RTL 效果 |
|----------|----------|----------|----------|
| `margin-left` | `margin-inline-start` | 左 | 右 |
| `margin-right` | `margin-inline-end` | 右 | 左 |
| `padding-left` | `padding-inline-start` | 左 | 右 |
| `padding-right` | `padding-inline-end` | 右 | 左 |
| `border-left` | `border-inline-start` | 左 | 右 |
| `border-right` | `border-inline-end` | 右 | 左 |
| `left` | `inset-inline-start` | 左 | 右 |
| `right` | `inset-inline-end` | 右 | 左 |
| `text-align: left` | `text-align: start` | 靠左 | 靠右 |
| `text-align: right` | `text-align: end` | 靠右 | 靠左 |
| `float: left` | `float: inline-start` | 靠左 | 靠右 |
| `width` | `inline-size` | 水平尺寸 | 水平尺寸 |
| `height` | `block-size` | 垂直尺寸 | 垂直尺寸 |

### 8.4 Tailwind CSS RTL 支援

```html
<!-- Tailwind v3.1+ 內建 RTL 支援 -->

<!-- 基本用法：加 rtl: 和 ltr: 前綴 -->
<div class="ml-4 rtl:mr-4 rtl:ml-0">
  <!-- LTR: margin-left 4 / RTL: margin-right 4 -->
</div>

<!-- 更好的方法：用邏輯屬性（Tailwind v3.3+） -->

    <span class="text-start">文字</span>  <!-- text-align: start -->
  </p>
</div>

<!-- Flexbox 自動翻轉 -->
<div class="flex flex-row">
  <!-- LTR: 左到右 / RTL: 自動變右到左 -->
  <div class="me-4">項目 1</div>  <!-- margin-inline-end -->
  <div class="me-4">項目 2</div>
  <div>項目 3</div>
</div>

<!-- 圖示翻轉（箭頭需要翻轉，logo 不需要） -->
<svg class="rtl:scale-x-[-1]"><!-- 箭頭圖示 --></svg>
```

```tsx
interface LocalizedImageProps extends Omit<ImageProps, 'src'> {
  /** 圖片的語言變體路徑，如 "hero-banner.webp" */
  name: string;
  /** 是否有語言變體（false 則使用 common 資料夾） */
  localized?: boolean;
  /** 備援語言（找不到對應語言圖片時用） */
  fallbackLocale?: string;
}

export default function LocalizedImage({
  name,
  localized = true,
  fallbackLocale = 'zh-TW',
  alt,
  ...props
}: LocalizedImageProps) {
  const locale = useLocale();

  const src = localized
    ? `/images/${locale}/${name}`
    : `/images/common/${name}`;

  return (
    <Image
      src={src}
      alt={alt}
      // 載入失敗時用備援語言的圖片
      onError={(e) => {
        const target = e.target as HTMLImageElement;
        if (localized && !target.src.includes(fallbackLocale)) {
          target.src = `/images/${fallbackLocale}/${name}`;
        }
      }}
      {...props}
    />
  );
}
```

```typescript
// scripts/translate.ts
import OpenAI from 'openai';
import * as fs from 'fs';

const openai = new OpenAI({ apiKey: process.env.OPENAI_API_KEY });

interface Messages {
  [key: string]: string | Messages;
}

async function translateJSON(
  source: Messages,
  targetLang: string,
  context: string = ''
): Promise<Messages> {
  // 展平巢狀 JSON
  const flat = flattenObject(source);
  const keys = Object.keys(flat);

  // 分批翻譯（每批 50 個 key，避免超過 token 限制）
  const batchSize = 50;
  const translated: Record<string, string> = {};

  for (let i = 0; i < keys.length; i += batchSize) {
    const batch = keys.slice(i, i + batchSize);
    const batchObj: Record<string, string> = {};
    batch.forEach((key) => {
      batchObj[key] = flat[key];
    });

    const response = await openai.chat.completions.create({
      model: 'gpt-4o',
      messages: [
        {
          role: 'system',
          content: `你是一位專業的網站翻譯師。請將以下 JSON 的值翻譯成${targetLang}。
規則：
1. 只翻譯 value，不要改 key
2. 保留 {{variable}} 和 {variable} 等佔位符不翻譯
3. 保留 HTML 標籤
4. 翻譯要自然、口語化，不要太文言
5. 按鈕文字要簡短有力
${context ? `6. 網站背景：${context}` : ''}
回傳純 JSON，不要加 markdown 格式。`,
        },
        {
          role: 'user',
          content: JSON.stringify(batchObj, null, 2),
        },
      ],
      temperature: 0.3,
      response_format: { type: 'json_object' },
    });

    const batchResult = JSON.parse(response.choices[0].message.content || '{}');
    Object.assign(translated, batchResult);

    // Rate limiting
    await new Promise((r) => setTimeout(r, 500));
  }

  // 還原巢狀結構
  return unflattenObject(translated);
}

function flattenObject(obj: Messages, prefix = ''): Record<string, string> {
  const result: Record<string, string> = {};
  for (const [key, value] of Object.entries(obj)) {
    const newKey = prefix ? `${prefix}.${key}` : key;
    if (typeof value === 'object' && value !== null) {
      Object.assign(result, flattenObject(value as Messages, newKey));
    } else {
      result[newKey] = value as string;
    }
  }
  return result;
}

function unflattenObject(obj: Record<string, string>): Messages {
  const result: Messages = {};
  for (const [key, value] of Object.entries(obj)) {
    const parts = key.split('.');
    let current: Messages = result;
    for (let i = 0; i < parts.length - 1; i++) {
      if (!(parts[i] in current)) {
        current[parts[i]] = {};
      }
      current = current[parts[i]] as Messages;
    }
    current[parts[parts.length - 1]] = value;
  }
  return result;
}

// 使用範例
async function main() {
  const source = JSON.parse(fs.readFileSync('messages/zh-TW.json', 'utf-8'));

  const enTranslation = await translateJSON(
    source,
    '英文 (American English)',
    '這是一個網頁設計公司的官網'
  );
  fs.writeFileSync('messages/en.json', JSON.stringify(enTranslation, null, 2));

  const jaTranslation = await translateJSON(
    source,
    '日文',
    'ウェブデザイン会社のウェブサイト'
  );
  fs.writeFileSync('messages/ja.json', JSON.stringify(jaTranslation, null, 2));

  console.log('Translation complete!');
}

main();
```

### 10.4 翻譯品質檢查腳本

```typescript
// scripts/check-translations.ts
import * as fs from 'fs';
import * as path from 'path';

const MESSAGES_DIR = 'messages';
const SOURCE_LOCALE = 'zh-TW';

interface CheckResult {
  locale: string;
  missing: string[];
  extra: string[];
  emptyValues: string[];
  brokenPlaceholders: string[];
}

function getAllKeys(obj: any, prefix = ''): string[] {
  const keys: string[] = [];
  for (const [key, value] of Object.entries(obj)) {
    const fullKey = prefix ? `${prefix}.${key}` : key;
    if (typeof value === 'object' && value !== null) {
      keys.push(...getAllKeys(value, fullKey));
    } else {
      keys.push(fullKey);
    }
  }
  return keys;
}

function getNestedValue(obj: any, path: string): any {
  return path.split('.').reduce((o, k) => o?.[k], obj);
}

function checkTranslations(): void {
  const sourceFile = path.join(MESSAGES_DIR, `${SOURCE_LOCALE}.json`);
  const source = JSON.parse(fs.readFileSync(sourceFile, 'utf-8'));
  const sourceKeys = getAllKeys(source);

  const localeFiles = fs.readdirSync(MESSAGES_DIR).filter(
    (f) => f.endsWith('.json') && f !== `${SOURCE_LOCALE}.json`
  );

  const results: CheckResult[] = [];

  for (const file of localeFiles) {
    const locale = file.replace('.json', '');
    const target = JSON.parse(
      fs.readFileSync(path.join(MESSAGES_DIR, file), 'utf-8')
    );
    const targetKeys = getAllKeys(target);

    const result: CheckResult = {
      locale,
      missing: sourceKeys.filter((k) => !targetKeys.includes(k)),
      extra: targetKeys.filter((k) => !sourceKeys.includes(k)),
      emptyValues: targetKeys.filter((k) => {
        const val = getNestedValue(target, k);
        return typeof val === 'string' && val.trim() === '';
      }),
      brokenPlaceholders: [],
    };

    // 檢查佔位符是否完整
    for (const key of sourceKeys) {
      const sourceVal = String(getNestedValue(source, key) || '');
      const targetVal = String(getNestedValue(target, key) || '');

      // 找出 {{xxx}} 和 {xxx} 格式的佔位符
      const sourcePlaceholders = sourceVal.match(/\{\{?\w+\}?\}/g) || [];
      const targetPlaceholders = targetVal.match(/\{\{?\w+\}?\}/g) || [];

      for (const ph of sourcePlaceholders) {
        if (!targetPlaceholders.includes(ph)) {
          result.brokenPlaceholders.push(`${key}: missing ${ph}`);
        }
      }
    }

    results.push(result);
  }

  // 輸出報告
  console.log('\n=== Translation Check Report ===\n');

  for (const r of results) {
    const hasIssues =
      r.missing.length + r.extra.length + r.emptyValues.length + r.brokenPlaceholders.length > 0;

    console.log(`[${r.locale}] ${hasIssues ? 'ISSUES FOUND' : 'OK'}`);

    if (r.missing.length > 0) {
      console.log(`  Missing keys (${r.missing.length}):`);
      r.missing.forEach((k) => console.log(`    - ${k}`));
    }
    if (r.extra.length > 0) {
      console.log(`  Extra keys (${r.extra.length}):`);
      r.extra.forEach((k) => console.log(`    - ${k}`));
    }
    if (r.emptyValues.length > 0) {
      console.log(`  Empty values (${r.emptyValues.length}):`);
      r.emptyValues.forEach((k) => console.log(`    - ${k}`));
    }
    if (r.brokenPlaceholders.length > 0) {
      console.log(`  Broken placeholders (${r.brokenPlaceholders.length}):`);
      r.brokenPlaceholders.forEach((k) => console.log(`    - ${k}`));
    }
    console.log('');
  }

  // 統計
  const totalKeys = sourceKeys.length;
  console.log(`Source (${SOURCE_LOCALE}): ${totalKeys} keys`);
  for (const r of results) {
    const translatedCount = totalKeys - r.missing.length - r.emptyValues.length;
    const percentage = ((translatedCount / totalKeys) * 100).toFixed(1);
    console.log(`${r.locale}: ${translatedCount}/${totalKeys} (${percentage}%)`);
  }
}

checkTranslations();
```

```typescript
// 語言偵測策略
function detectLocale(request: Request): string {
  // 1. URL 路徑中的語言（最高優先）
  const urlLocale = getLocaleFromPath(request.url);
  if (urlLocale) return urlLocale;

  // 2. Cookie（使用者之前選過的語言）
  const cookieLocale = getCookie(request, 'NEXT_LOCALE');
  if (cookieLocale && isValidLocale(cookieLocale)) return cookieLocale;

  // 3. Accept-Language header（瀏覽器偏好）
  const acceptLanguage = request.headers.get('Accept-Language');
  if (acceptLanguage) {
    const matched = matchLocale(acceptLanguage);
    if (matched) return matched;
  }

  // 4. GeoIP（根據 IP 判斷地區）
  const country = request.headers.get('CF-IPCountry'); // Cloudflare
  if (country) {
    const geoLocale = getLocaleByCountry(country);
    if (geoLocale) return geoLocale;
  }

  // 5. 預設語言
  return 'en';
}

// GeoIP 到語言的對應
function getLocaleByCountry(country: string): string | null {
  const map: Record<string, string> = {
    TW: 'zh-TW',
    CN: 'zh-CN',
    HK: 'zh-TW',
    JP: 'ja',
    KR: 'ko',
    US: 'en',
    GB: 'en',
    DE: 'de',
    FR: 'fr',
    ES: 'es',
    SA: 'ar',
    AE: 'ar',
  };
  return map[country] || null;
}
```

### 11.4 繁簡中文轉換

**場景**：同時服務台灣和中國用戶

```bash
# 安裝 OpenCC（繁簡轉換工具）
npm install opencc-js
```

```typescript
// scripts/convert-zhCN.ts
import * as OpenCC from 'opencc-js';
import * as fs from 'fs';

// 繁體 → 簡體
const converter = OpenCC.Converter({ from: 'tw', to: 'cn' });

const zhTW = JSON.parse(fs.readFileSync('messages/zh-TW.json', 'utf-8'));
const zhCN = convertValues(zhTW, converter);
fs.writeFileSync('messages/zh-CN.json', JSON.stringify(zhCN, null, 2));

function convertValues(obj: any, conv: any): any {
  if (typeof obj === 'string') {
    return conv(obj);
  }
  if (typeof obj === 'object' && obj !== null) {
    const result: any = {};
    for (const [key, value] of Object.entries(obj)) {
      result[key] = convertValues(value, conv);
    }
    return result;
  }
  return obj;
}

console.log('zh-CN generated from zh-TW!');
```

**注意**：繁簡轉換不只是字形轉換，用語也不同：

| 繁體（台灣） | 簡體（中國） |
|-------------|-------------|
| 軟體 | 软件 |
| 網路 | 网络 |
| 程式 | 程序 |
| 資料庫 | 数据库 |
| 隨身碟 | U盘 |
| 滑鼠 | 鼠标 |
| 伺服器 | 服务器 |

**建議**：用 OpenCC 做初步轉換，再人工校對專有名詞。

### 11.5 接案報價參考

| 語言數量 | 建議加價 | 維護成本 |
|----------|----------|----------|
| 2 語言（中英） | 基礎價 x 1.3-1.5 | 低 |
| 3 語言（中英日） | 基礎價 x 1.5-1.8 | 中 |
| 5+ 語言 | 基礎價 x 2.0-3.0 | 高（建議用翻譯平台） |
| 含 RTL | 額外 +20-30% | 需專門測試 |

**接案注意事項**：
1. **翻譯由誰負責**？客戶自備翻譯 vs 設計師找翻譯 vs AI 翻譯
2. **翻譯維護**：新增內容時誰翻譯？建議導入 Crowdin/Lokalise
3. **SEO 多語系**：hreflang 設定要列入報價
4. **不同語言的 UI 測試**：德文比英文長 30%，日文字型大小不同
5. **合約明確寫**：支援幾種語言、新增語言的費用、翻譯責任歸屬
