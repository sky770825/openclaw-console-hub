# 27 — 專案上下文切換 SOP

> 老蔡叫你從 A 專案切到 B 專案。怎麼 10 秒進入狀況。

---

## 切換步驟

### 1. 確認專案位置
```
# 已知專案
OpenClaw: /Users/caijunchang/openclaw任務面版設計/
小蔡 repo: /Users/caijunchang/Downloads/openclaw-console-hub-main/

# 未知專案 → 找
{"action":"run_script","command":"find /Users/caijunchang/Desktop /Users/caijunchang/Downloads -maxdepth 2 -name 'package.json' -o -name 'index.html' 2>/dev/null | head -20"}
```

### 2. 看最近改動（10 秒掌握進度）
```
# git 專案 → 看最近 commit
{"action":"run_script","command":"cd /path/to/project && git log --oneline -5 2>/dev/null"}

# 非 git 專案 → 看最近修改的檔案
{"action":"run_script","command":"find /path/to/project -name '*.js' -o -name '*.ts' -o -name '*.html' | xargs ls -lt 2>/dev/null | head -10"}
```

### 3. 看結構（已知類型直接跳過）
```
{"action":"list_dir","path":"/path/to/project"}
```

### 4. 開始工作

根據老蔡的指令，直接開幹。不需要每次都完整掃描。

## 常用專案速查表

建議自己維護一份，遇到新專案就補：

```
{"action":"write_file","path":"knowledge/project-index.md","content":"# 老蔡專案索引\n\n| 專案 | 路徑 | 技術棧 | Supabase | 上次動作 |\n|------|------|--------|----------|----------|\n| OpenClaw | /Users/caijunchang/openclaw任務面版設計/ | React+Express | OpenClaw 主庫 | 每天 |\n| 餐車排班 | 待確認 | HTML+JS | 獨立庫 | 2026-03 |\n"}
```

**記住：切換專案不是重頭開始。看最近 commit + 老蔡說的需求，就夠了。**
