# 15 — 驗收對治法

> 標 ✅ 之前必看。虛假驗收 = 自欺欺人。四層驗證通過才能打勾。

---

## 什麼是虛假驗收

| 症狀 | 例子 |
|------|------|
| 路徑不存在 | 標 ✅ 但 `ls` 找不到檔案 |
| 空殼檔案 | 檔案存在但只有標題，沒有實際內容 |
| 格式無效 | JSON 無法 parse、Bash 語法錯 |
| 舊版本 | mtime 是昨天的，不是剛產出的 |
| 複製貼上 | 內容跟另一個產出物一模一樣 |

## 四層驗證（打勾前必做，缺一不可）

### 第 1 層：路徑驗證 — 檔案存在嗎？
```bash
ls -lh $PATH
```
- 看得到檔案 → 通過
- `No such file or directory` → 打勾前不合格

### 第 2 層：大小驗證 — 檔案有內容嗎？
```bash
stat -f "%z" $PATH    # macOS
wc -c < $PATH         # 通用
```
- 大小 > 100 bytes → 通過
- 0 bytes 或 < 50 bytes → 空殼，不合格

### 第 3 層：內容驗證 — 內容有意義嗎？
```bash
head -10 $PATH
wc -l $PATH
wc -w $PATH
```
- 行數 > 10、字數 > 50 → 通過
- 只有標題沒有正文 → 不合格

### 第 4 層：格式驗證 — 格式正確嗎？

**JSON 檔案：**
```bash
python3 -c "import json; json.load(open('$PATH')); print('Valid JSON')"
```

**Markdown 檔案：**
```bash
grep -c "^##" $PATH    # 至少 2 個二級標題
```

**Bash 腳本：**
```bash
bash -n $PATH && echo "Syntax OK"
```

**TypeScript：**
```bash
cd /Users/sky770825/openclaw任務面版設計/server && npx tsc --noEmit
```

## 一行驗證指令（複製就能用）

```bash
# 驗證單一檔案（四層一次跑完）
FILE="~/.openclaw/workspace/reports/my-report.md" && \
  ls -lh $FILE && \
  echo "=== 大小 ===" && wc -c < $FILE && \
  echo "=== 前 5 行 ===" && head -5 $FILE && \
  echo "=== 統計 ===" && wc -l -w $FILE && \
  echo "✅ 四層驗證通過"
```

## 驗證失敗怎麼辦

```
任何一層失敗
│
├─ 第 1 層失敗（路徑不存在）
│  → 檢查路徑是否打錯（看 cookbook/14）
│  → 如果確認路徑對但檔案不在 → 重新產出
│
├─ 第 2 層失敗（空殼）
│  → 刪除空檔案
│  → 重新產出（檢查產出邏輯）
│
├─ 第 3 層失敗（無意義內容）
│  → 分析為什麼內容是空的
│  → 可能是 auto-executor 腳本有問題
│
└─ 第 4 層失敗（格式錯誤）
   → 修復格式（JSON 補括號、Bash 修語法）
   → 修完後重跑第 4 層驗證

所有情況：立刻改回 ⬜，不能留著假 ✅
```

## 防止虛假驗收的習慣

1. **產出完立刻驗證** — 不要等到更新 BLUEPRINT 時才驗
2. **不要相信 action 回覆** — `ok: true` 只代表指令執行了，不代表內容正確
3. **read_file 看一眼** — 寫完就讀回來確認
4. **跑驗證腳本** — `run_script` 跑上面的一行驗證指令
5. **記錄 mtime** — 確認是新產出的，不是舊檔案
