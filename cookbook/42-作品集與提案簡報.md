## 1. 作品集網站設計

### 1.1 作品集的核心目的

作品集不是美術展覽，是銷售工具。訪客三秒內要知道三件事：

1. **你做什麼** — 網站設計 / UI/UX / 全端開發
2. **你做得多好** — 精選作品的視覺衝擊
3. **怎麼找你** — 清楚的聯絡方式與 CTA

### 1.2 網站結構（建議 5-7 頁）

```
首頁 (Hero)
├── 作品集 (Portfolio / Works)
│   ├── 作品 1 — Case Study 頁
│   ├── 作品 2 — Case Study 頁
│   └── 作品 3 — Case Study 頁
├── 關於我 (About)
├── 服務項目 (Services)
├── 部落格 (Blog / Insights) ← 選配但強烈建議
├── 客戶評價 (Testimonials) ← 信任建立
└── 聯絡 (Contact)
```

### 1.3 首頁 Hero 區設計

Hero 是整個作品集最重要的區域，決定訪客留不留。

**必備元素：**

| 元素 | 說明 | 範例 |
|------|------|------|
| 標題 | 一句話定位你的價值 | 「為品牌打造高轉換率的網站體驗」 |
| 副標題 | 補充說明你是誰 | 「5 年經驗 / 服務 40+ 客戶 / 專注電商與 SaaS」 |
| CTA 按鈕 | 主要行動呼籲（1-2 個） | 「看我的作品」「立即諮詢」 |
| 背景視覺 | 最強作品的動態展示或設計感背景 | 作品 Mockup 輪播、漸層背景 |

**Hero 文案公式：**

```
我幫 [目標客群] 透過 [你的服務] 達成 [具體成果]
```

範例：
- 「我幫新創公司透過品牌官網設計，讓訪客轉換率提升 40%」
- 「我幫電商品牌打造讓人想買的購物體驗」
- 「專為診所、美容、餐飲設計預約型網站」

### 1.4 作品集頁面設計

**精選原則：質勝於量**

- 剛起步：3-5 件精選作品
- 穩定接案：6-10 件精選作品
- 資深設計師：10-15 件分類展示

**作品卡片排版（Grid Layout）：**

```
┌──────────────┐  ┌──────────────┐  ┌──────────────┐
│              │  │              │  │              │
│   作品封面圖   │  │   作品封面圖   │  │   作品封面圖   │
│              │  │              │  │              │
├──────────────┤  ├──────────────┤  ├──────────────┤
│ 品牌名稱      │  │ 品牌名稱      │  │ 品牌名稱      │
│ 類型標籤      │  │ 類型標籤      │  │ 類型標籤      │
│ 年份          │  │ 年份          │  │ 年份          │
└──────────────┘  └──────────────┘  └──────────────┘
```

**封面圖規格建議：**
- 比例：16:10 或 4:3
- 解析度：1200x750px 以上
- 格式：WebP 優先（壓縮率高、畫質好）
- 載入：Lazy Loading + 低品質預覽圖（LQIP）

**篩選功能（作品超過 8 件必備）：**
- 全部 | 官網 | 電商 | Landing Page | App | UI/UX
- 用 CSS filter 或 JS 動態篩選，不要用分頁

**必備內容：**


### 1.6 服務項目頁面

**用卡片式排版，每個服務一張卡：**

| 服務 | 說明 | 起始價格 | 時程 |
|------|------|----------|------|
| 品牌官網設計 | RWD 一頁式或多頁式官網 | NT$30,000 起 | 2-4 週 |
| 電商網站 | Shopify / WooCommerce 客製化 | NT$50,000 起 | 4-6 週 |
| Landing Page | 高轉換率行銷著陸頁 | NT$15,000 起 | 1-2 週 |
| UI/UX 設計 | App 或 Web 介面設計 | NT$40,000 起 | 3-5 週 |
| 網站健檢 | SEO + 效能 + UX 審查報告 | NT$8,000 起 | 3-5 天 |

> **定價策略提醒**：列「起」價比精確價格好，讓客戶覺得有討論空間，同時篩掉預算太低的詢問。

### 1.7 CTA 策略（全站貫穿）

**每個頁面至少一個 CTA，位置如下：**

| 頁面 | CTA 文案 | 位置 |
|------|----------|------|
| 首頁 Hero | 「免費諮詢」/「看我的作品」 | Hero 區下方 |
| 作品集 | 「喜歡這個風格？來聊聊」 | 每個 Case Study 底部 |
| 關於我 | 「跟我合作」 | 頁面底部 |
| 服務項目 | 「取得報價」 | 每個服務卡片 + 頁面底部 |
| 部落格 | 「訂閱電子報」/「預約諮詢」 | 文章底部 + 側邊欄 |

**CTA 按鈕設計原則：**
- 主按鈕：實心、品牌色、大尺寸（至少 48px 高）
- 副按鈕：空心 / Ghost Button
- 文案用動詞開頭：「開始」「取得」「預約」「下載」
- 避免：「提交」「送出」這種冷冰冰的詞

### 1.8 技術實作建議

```
推薦技術棧（依經驗排序）：

入門：WordPress + Elementor / Starter Templates
      → 快速上線、成本低、客戶能自己改

進階：Next.js + Tailwind CSS + Vercel
      → 效能好、SEO 強、作品集本身就是技術展示

設計導向：Framer / Webflow
          → 設計師友善、動畫強、不用寫太多 code

極簡：Astro + Markdown
      → 超快、超輕、適合純展示型作品集
```

## 2. Case Study 撰寫法

### 2.1 為什麼 Case Study 比作品截圖重要 100 倍

客戶看截圖只會覺得「好看」或「不好看」，這是主觀判斷。
Case Study 讓客戶看到「這個設計師能解決商業問題」，這是客觀信任。

**核心公式：問題 → 方案 → 過程 → 成果 → 數據**

### 2.2 Case Study 完整模板

```markdown
# [客戶品牌名稱] — [一句話描述成果]

> 例如：「大口吃火鍋 — 重新設計官網後，線上訂位量成長 180%」

## 1. 問題（The Challenge）

大口吃火鍋是一間有 5 家分店的連鎖火鍋品牌。他們的舊官網有以下問題：

- **手機體驗差**：60% 訪客使用手機，但舊網站不是 RWD
- **找不到訂位入口**：訂位按鈕藏在第三層選單
- **品牌感不一致**：網站風格和實體店裝潢完全不搭
- **載入速度慢**：首頁載入需要 8 秒（Google PageSpeed 32 分）

> 客戶原話：「很多客人打電話來問怎麼訂位，我們才發現網站上根本找不到。」

---

## 2. 方案（The Solution）

### 設計策略

1. **Mobile-First 設計**：先做手機版，再擴展桌面版
2. **一鍵訂位**：首頁 Hero 區直接放訂位 CTA
3. **品牌視覺統一**：從店面拍攝實景照，建立一致的色彩系統
4. **效能優先**：Next.js SSG + 圖片 WebP + CDN

### 關鍵設計決策

- 選用暖色調（#E85D3A 橘紅 + #2C1810 深棕）呼應火鍋的溫暖感
- 大量使用食物實拍照（非圖庫照片）增加真實感
- 訂位流程簡化為 3 步：選分店 → 選日期時段 → 填資料

---

## 3. 過程（The Process）

### Week 1-2：研究與策略
- 分析競品（5 間同類餐飲品牌官網）
- 使用者訪談（8 位常客 + 3 位店長）
- 建立 User Persona 與 User Journey Map

### Week 3-4：設計
- Wireframe（低保真原型）→ 客戶確認流程
- UI 設計（高保真 Mockup）→ 2 輪修改
- 建立 Design System（顏色 / 字型 / 元件庫）

### Week 5-7：開發
- 前端開發（Next.js + Tailwind）
- 訂位系統串接（n8n + Google Calendar）
- SEO 基礎建設（meta tags / Schema / sitemap）

### Week 8：測試與上線
- 跨瀏覽器測試（Chrome / Safari / Firefox）
- 效能優化（Lighthouse 95+ 分）
- 客戶培訓 + 上線
```

### 2.3 Case Study 撰寫要點

**必做：**
- 用具體數字，不要用「大幅提升」「顯著改善」
- 加入客戶原話（即使只有一句也好）
- 放 Before/After 對比
- 說明你的設計決策背後的「為什麼」
- 標明你的角色與使用工具

**避免：**
- 只放截圖不解釋
- 寫成技術文件（客戶看不懂的術語）
- 太多文字不夠圖片（圖文比至少 1:1）
- 隱藏你遇到的困難（適度展示解題過程更加分）

### 2.4 沒有客戶怎麼寫 Case Study

**三種方式製造作品：**

1. **Redesign 練習**
   - 挑一個你覺得爛的真實網站，重新設計
   - 標題寫「[品牌名] Concept Redesign」
   - 一樣用完整的 Case Study 格式

2. **自己的 Side Project**
   - 幫自己做一個作品集網站（作品集的作品集）
   - 做一個 SaaS Landing Page 概念設計
   - 設計一個虛構品牌的完整視覺

3. **免費接案**
   - 幫非營利組織 / 朋友的店免費設計
   - 條件：要能公開展示、要能拿到數據回饋

## 3. Before/After 展示技巧

### 3.1 截圖對比法

**最經典也最有效的展示方式：**

```

│                 │                 │
```

### 3.3 動畫展示法

**Mockup 動畫：**


- 用 QuickTime / OBS 錄製網站操作過程

- 建議長度：15-30 秒
- 務必加速（1.5x-2x），沒人想看你慢慢滑

### 3.5 Mockup 展示最佳實踐

**一個作品至少準備以下 Mockup：**

| Mockup 類型 | 用途 | 工具 |
|-------------|------|------|
| 桌面瀏覽器 Mockup | 展示完整版面 | Figma + Browser Frame |
| 手機 Mockup | 展示 RWD 效果 | Mockup Plugin / Smartmockups |
| 多裝置組合 | Hero 展示圖 | Artboard Studio / Placeit |
| 等角透視（Isometric） | 增加立體感 | Figma 3D 外掛 |
| 實景合成 | 增加真實感 | Unsplash Mockup / Freepik |

**Mockup 注意事項：**
- 不要用太舊的裝置模型（iPhone 13 以上、MacBook Air M2 以上）
- 背景要乾淨，不要搶作品的焦點
- 統一所有作品的 Mockup 風格（角度、陰影、背景色）

### 4.2 建議策略：主站 + 副站

```
最佳策略：自建主站 + 社群平台副站

主站（必備）：自建作品集網站
  → 你的品牌基地，完全掌控
  → 所有 Case Study 的完整版在這裡
  → 客製化 CTA、SEO、追蹤

副站（曝光用）：Behance + Dribbble
  → 放精簡版作品 + 連結回主站
  → 參與社群互動、累積追蹤者
  → 利用平台搜尋流量
```

### 4.3 各平台經營重點

**自建網站：**
- 域名用 `名字.design` 或 `名字.dev`（年費約 NT$400-1200）
- 託管用 Vercel / Netlify（免費方案就夠）
- 每 2-3 個月更新一次作品
- Google Analytics + Search Console 必裝

**Behance：**
- 每個專案上傳 10-20 張圖（越長越好，演算法喜歡）
- 標題格式：「Brand Name — Project Type | Description」
- 標籤打滿（最多 50 個）
- 加入 Creative Fields 分類
- 每週至少 Appreciate 10 個別人的作品（互動提升曝光）

**Dribbble：**
- 免費帳號只能上傳 24 個 Shot
- Shot 尺寸：1600x1200px（4:3）或 800x600px
- 風格要搶眼，Dribbble 是「第一印象」平台
- Pro 帳號（$5/月）才能被搜尋、出現在搜尋結果

**Notion：**
- 適合快速需要一個作品集頁面的場合
- 用 Gallery View 排列作品卡片
- 配合 Super.so 或 Notion2Site 產出美觀的靜態站
- 非長久之計，建議當過渡方案

### 4.4 不同階段的平台選擇

| 階段 | 推薦做法 |
|------|----------|
| 剛入行（0-1 年） | Notion 快速做一個 + Behance 上傳作品 |
| 穩定接案（1-3 年） | 自建網站 + Behance/Dribbble 同步曝光 |
| 資深/工作室（3+ 年） | 自建網站為主，社群平台為輔 |

### 6.1 字型選擇

| 蒙納超剛黑 | 強烈、標題感 | 標題用 | 需授權 |

- 標題：Inter / Plus Jakarta Sans / Space Grotesk
- 內文：Inter / DM Sans / Source Sans Pro

**字型搭配原則：**

- 標題 Bold / 內文 Regular，靠粗細區分層級
- 字級系統：標題 32-40pt / 副標題 24pt / 內文 16-18pt / 註解 12pt

### 6.2 配色策略

**提案配色公式：**

```
主色（60%）— 大面積背景、基調
  → 通常是白色或淺灰

強調色（30%）— 標題、重點、圖表
  → 用客戶的品牌色，讓他覺得這份提案是為他做的

點綴色（10%）— CTA、數字、關鍵重點
  → 用品牌色的對比色或亮色
```

**配色工具：**
- Coolors.co — 隨機產生和諧配色
- Realtime Colors — 即時預覽配色在網頁上的效果
- Happy Hues — 配色靈感庫
- Color Hunt — 社群票選配色方案

### 6.3 圖表設計

**數據圖表原則：**

```
做：
✓ 一張圖表只傳達一個重點
✓ 用顏色引導視線到關鍵數字
✓ 加上數字標籤（不要讓人猜）
✓ 用「比較」而非「絕對值」（Before vs After）

不做：
✗ 3D 圓餅圖（扭曲比例感知）
✗ 多重 Y 軸（混淆視聽）
✗ 太多顏色（超過 5 色就亂了）
✗ 裝飾性圖表（沒有數據支撐的示意圖）
```

### 6.4 Less is More 原則

**每一頁檢查清單：**

```
[ ] 這一頁只講一個重點嗎？
[ ] 文字少於 40 個字嗎？（排除標題）
[ ] 有留白嗎？（內容不超過頁面 60%）
[ ] 視覺焦點明確嗎？（眼睛第一時間會看到哪裡）
[ ] 如果我是客戶，3 秒內能理解這頁在講什麼嗎？
```

**文字精簡技巧：**
- 原文：「我們擁有超過 5 年的豐富網頁設計經驗，曾經服務過超過 40 個以上的客戶」
- 精簡：「5 年經驗 | 40+ 客戶」

- 原文：「透過我們的專業設計，能夠有效提升您的網站在使用者體驗方面的表現」
- 精簡：「讓你的網站好用 3 倍」

## 8. 提案話術

### 8.1 開場白（前 30 秒決勝負）

**錯誤開場：**
> 「大家好，我是 XXX 設計工作室的負責人，我們成立於 2020 年，專注於網頁設計...」

客戶心裡想的是：跟我有什麼關係？

**正確開場（痛點開場法）：**
> 「王老闆您好，上次我們聊到，您最頭痛的是客人打電話來問怎麼訂位這件事。今天我帶來的方案，目標就是讓這通電話消失——讓客人在手機上 30 秒就能完成訂位。」

**正確開場（數據開場法）：**
> 「在我們這個產業，有 73% 的消費者會在 3 秒內判斷一個網站可不可信。而根據我們的分析，目前您的網站在手機上的體驗，可能會讓 6 成的訪客直接離開。」

**正確開場（故事開場法）：**
> 「上個月我去了您的店用餐，排了 40 分鐘的隊。回家後我試著上網訂位，花了 5 分鐘還是找不到入口。我想，如果連我這個專門做網站的人都找不到，一般客人一定更挫折。」

### 8.2 痛點共鳴

**話術結構：場景 → 痛點 → 代價 → 轉折**

```
場景：「想像一下，一個客人週五晚上想訂位......」
痛點：「他打開手機搜尋，看到你的網站，但字太小、按鈕太小，滑了半天找不到訂位......」
代價：「他放棄了，轉頭去訂隔壁那家有線上訂位的餐廳。這樣的情況，每天可能發生 10 次、20 次。」
轉折：「但如果，他打開網站的第一秒就看到訂位按鈕呢？」
```

**痛點問句（讓客戶自己說出痛點）：**
- 「目前最困擾您的客戶溝通問題是什麼？」
- 「您覺得現在的網站最想改善什麼地方？」
- 「有沒有遇過客戶因為網站而流失的經驗？」
- 「您的競爭對手在數位方面做得如何？」

### 8.3 方案介紹

**FAB 法則：Feature → Advantage → Benefit**

```
Feature（功能）：「我們會做一個 Mobile-First 的響應式設計。」
                ↓
Advantage（優勢）：「不管客人用手機、平板、電腦都有最佳體驗。」
                ↓
Benefit（好處）：「根據類似案例的經驗，這能讓手機端的跳出率降低 40%，
                  也就是說，每 10 個用手機看的客人，多留下 4 個。」
```

**方案介紹的節奏：**
1. 先講「這是什麼」（30 秒）
2. 再講「為什麼這樣做」（30 秒）
3. 最後講「做了之後你會得到什麼」（30 秒）

### 8.4 價格錨定

**三層定價策略（高 → 中 → 低）：**

```
錯誤做法：直接報一個價格
「這個案子 5 萬塊。」

正確做法：先報高錨點
「如果從零開始客製化開發，市場行情大約在 15-20 萬。」
「不過根據您的需求，我們用成熟的框架來做，可以控制在 7 萬以內。」
「而且如果選擇標準方案，只需要 4.5 萬就能解決核心問題。」

效果：客戶聽到 15 萬後，覺得 4.5 萬是「撿到便宜」。
```

**價格話術範本：**

```
報價前的鋪墊：
「在報價之前，我想先確認一件事——
 這個網站上線後，如果每個月多帶來 20 組訂位，
 以你們平均客單價 1,500 元來算，
 一個月就是多 3 萬的營收。
 所以這不是一筆費用，是一筆投資。」

報價時：
「我們準備了兩個方案，
 標準版 4.5 萬、進階版 7.2 萬。
 差異在於進階版包含訂位系統和 3 個月維護。
 如果預算允許，我推薦進階版，
 因為訂位系統是解決您核心痛點的關鍵。」

收到殺價時：
「我理解預算考量。
 不過與其降價砍功能，不如我們調整付款方式——
 改成分 3 期付款，每期只要 2.4 萬，
 而且第一期是設計定稿後才付。
 這樣您可以先看到成果再決定。」
```

### 8.5 收尾話術

**創造行動的收尾：**

```
直接收尾：
「如果方向沒問題，我們可以今天就簽約，下週一開始啟動。」

二擇一收尾：
「您比較傾向方案 A 還是方案 B？」
（不是問「要不要做」，而是問「選哪個」）

限時收尾：
「這個月我們還有 2 個名額，
 如果本週五前確認，可以保留這個時段給您。」

試用收尾：
「不確定的話，我們可以先做第一階段——
 只做首頁設計，費用 1.5 萬，
 做完您滿意再繼續後面的頁面。」

降低風險收尾：
「我們提供設計稿不滿意全額退費的保障。
 也就是說，您看到設計稿後再決定要不要往下走，
 完全沒有風險。」
```

**簡報結束後的跟進 SOP：**

```
當天    → 寄感謝信 + 提案 PDF
1 天後  → 寄補充資料（案例、參考連結）
3 天後  → 電話 / LINE 詢問是否有問題
7 天後  → 最後跟進（不要超過 3 次）
```

## 9. 競標策略

### 9.1 比稿勝出的 7 個關鍵

```
1. 客製化程度 — 提案要讓客戶覺得「這是為我量身打造的」
2. 痛點精準度 — 你比其他人更懂他的問題
3. 信任感     — 相關案例 + 數據 + 客戶推薦
4. 方案合理性 — 不只好看，還要務實可執行
5. 價格策略   — 不是最便宜，但要有「值得」的感覺
6. 人的感覺   — 客戶想跟「對的人」合作
7. 跟進速度   — 最快回覆的人，贏面最大
```

### 9.2 競標前的情報蒐集

**比稿前必做功課：**

| 項目 | 要蒐集什麼 | 怎麼找 |
|------|------------|--------|
| 客戶產業 | 產業趨勢、競爭狀況 | Google / 產業報告 |
| 客戶現狀 | 現有網站、社群、Google 評價 | 直接看他的網站和 FB |
| 競品網站 | 同業做得好的網站案例 | Google 搜同業 |
| 決策者 | 誰拍板？他的喜好？ | LinkedIn / 共同朋友 |
| 預算 | 大概願意花多少 | 問業務 / RFP 文件 |
| 對手 | 其他比稿的是誰 | 問業務 / 業界打聽 |

### 9.3 差異化策略

**當你不是最便宜的時候，靠這些贏：**

```
策略 1：免費先做一點
  → 比稿前先做一個首頁的 Wireframe 或 Concept Design
  → 讓客戶看到「你已經開始了」
  → 風險：可能做白工。但命中率大幅提升。

策略 2：展示過程不只是結果
  → 簡報時展示你的設計流程（研究 → 策略 → 設計 → 驗證）
  → 讓客戶知道「你不是隨便畫畫，是有方法論的」

策略 3：找到其他人忽略的痛點
  → 深入研究客戶的 GA 數據、熱點圖、搜尋排名
  → 提出一個連客戶自己都沒發現的問題
  → 「其實你的網站最大的問題不是設計，而是載入速度——
     你知道你的手機版首頁要 12 秒才能完全載入嗎？」

策略 4：降低決策風險
  → 「先做首頁，滿意再做其他頁」
  → 「設計稿不滿意不收錢」
  → 「提供 30 天上線後免費維護」

策略 5：建立個人連結
  → 去客戶的店實際消費，拍照、記錄體驗
  → 簡報時說「上週我去了你們的店......」
  → 這個動作 90% 的競爭對手不會做
```

### 9.4 比稿報價策略

```
情境 A：你是最便宜的
  → 強調「高 CP 值」，但不要讓客戶覺得「便宜沒好貨」
  → 展示為什麼你能用更低的價格做到同樣品質

情境 B：你是中間價位
  → 最有利的位置，強調「性價比最高」
  → 把最貴和最便宜的方案同時呈現，自己站中間

情境 C：你是最貴的
  → 強調「品質差異」和「長期價值」
  → 用數據說明為什麼多花這些錢是值得的
  → 「如果做了一個便宜但沒效的網站，三個月後還要重做，
     那其實花了兩次錢。」

情境 D：不知道別人報多少
  → 報你認為合理的價格，不要猜測
  → 用方案分層（低/中/高）覆蓋不同預算區間
```

### 9.5 輸了怎麼辦

```
比稿失敗後的 SOP：

1. 禮貌感謝 — 發一封感謝信，祝專案順利
2. 詢問原因 — 「請問是哪些方面我們可以改進？」
3. 記錄教訓 — 寫在自己的筆記裡（價格？設計？溝通？）
4. 保持聯繫 — 加 LINE 或 LinkedIn，偶爾互動
5. 等待機會 — 50% 的案子會在執行中出問題，客戶可能回來找你

重點：不要因為沒選你就斷聯繫。接案是長期關係經營。
```

## 11. 個人品牌建立

### 11.1 個人品牌的核心：定位

**定位公式：**

```
我幫 [誰] 做 [什麼] 達成 [什麼結果]
```

**定位範例：**
- 「我幫餐飲品牌做官網設計，讓線上訂位量翻倍。」
- 「我幫新創公司做 SaaS Landing Page，提升 30% 轉換率。」
- 「我幫電商品牌做 Shopify 客製化，讓商品頁轉換率提升 50%。」

**定位原則：**
- 越窄越好（「做網站的」 < 「做餐飲官網的」 < 「做連鎖餐飲訂位型官網的」）
- 定位不是限制，是行銷手段（你當然可以接其他案子）
- 先選一個打透，再擴展

### 11.2 部落格經營

**為什麼接案者需要部落格：**
1. SEO 長尾流量（客戶搜「餐飲網站設計」就找到你）
2. 建立專家權威（寫教學 = 你很專業的證明）
3. 作品展示平台（Case Study 就是最好的部落格內容）
4. 銷售漏斗入口（讀文章 → 看作品 → 聯繫你）

**部落格內容日曆（每月 4 篇）：**

| 週 | 內容類型 | 範例題目 |
|----|----------|----------|
| W1 | 教學文 | 「2026 年 RWD 設計 5 大最佳實踐」 |
| W2 | Case Study | 「XX 品牌官網改版——如何讓轉換率提升 180%」 |
| W3 | 觀點文 | 「為什麼我建議客戶不要用免費模板」 |
| W4 | 工具推薦 | 「設計師必備的 10 個 Chrome 擴充功能」 |

**部落格平台選擇：**
- 自建（Hugo / Astro / WordPress）— SEO 最強，完全掌控
- Medium — 自帶流量，但 SEO 被 Medium 吃掉
- 方格子 vocus — 台灣中文市場，可以收費
- Hashnode — 技術圈，自帶開發者流量

### 11.3 YouTube / 影片經營

**適合接案者的影片類型：**

```
1. 設計過程錄影（Design Process）
   → 錄螢幕 + 配旁白，展示 Figma 設計過程
   → 觀眾：學設計的人 + 潛在客戶

2. 案例拆解（Case Study Review）
   → 拆解知名網站的設計細節
   → 觀眾：設計同行 + 客戶（看到你的專業分析能力）

3. 工具教學
   → Figma / Webflow / CSS 技巧
   → 觀眾：設計初學者（未來可能成為你的客戶或轉介人）

4. 接案心得
   → 報價技巧、客戶溝通、踩雷經驗
   → 觀眾：同行 + 想轉職的人
```

**設備建議（低成本起步）：**
- 螢幕錄影：OBS（免費）
- 麥克風：Blue Yeti 或 Rode NT-USB Mini（NT$2,000-4,000）
- 剪輯：DaVinci Resolve（免費）或 Final Cut Pro
- 字幕：Whisper AI 自動產生中文字幕

**各平台內容策略：**

| 平台 | 內容形式 | 發文頻率 | 核心目的 |

## 附錄：一頁式快速參考

```
作品集 = 你的無聲業務員
  → 質勝於量，6-10 件精選就夠
  → 每件作品必須是 Case Study，不是截圖

Case Study = 問題 → 方案 → 過程 → 成果 → 數據
  → 沒數據的 Case Study 不如不寫

提案 = 10 頁搞定一切
  → 封面 → 痛點 → 方案 → 風格 → 時程 → 團隊 → 案例 → 報價 → CTA

報價 = 三層定價 + 錨定效應
  → 先報高的，再給客戶「划算」的選擇

競標 = 做功課 + 客製化 + 降低風險
  → 去客戶的店、研究客戶的數據、先做一點免費的

接案管道 = 口碑 > LinkedIn > FB 社團 > 接案平台
  → 長期靠口碑轉介，短期靠社群經營

個人品牌 = 定位 + 持續內容 + 社群參與
  → 越窄的定位越容易被記住
```
