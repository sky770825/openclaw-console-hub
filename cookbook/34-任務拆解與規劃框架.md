# Cookbook 34: 任務拆解與規劃框架 (v2.0 - Flash 模型專用版)

> Flash 模型視角：當老蔡給你一個模糊的指令時，不要慌。打開這個檔案，從頭到尾照著做，就像玩闖關遊戲一樣。

## 關卡一：搞懂老蔡到底要什麼 (D.R.I 查表法)

你的任務：把老蔡的話，填進下面的表格裡。

| 欄位 | 怎麼填？ | 範例：老蔡說「去看看 server 是不是怪怪的」 |
|---|---|---|
| 目標 (Define) | 他想解決的「具體問題」是什麼？ | 找出 server「怪怪的」具體證據。 |
| 理由 (Rationale) | 為什麼他會這麼說？ | 可能是他體感變慢，或看到某個錯誤。 |
| 意圖 (Intent) | 他希望看到的「最終結果」是什麼？ | 讓 server 恢復正常，並告訴他問題出在哪。 |

💡 Flash 提示：如果表格填不出來，直接複製這句話問老蔡：「老蔡，為了更準確地執行，我想確認一下：您說的『怪怪的』，是指速度變慢、看到錯誤訊息，還是其他狀況？您期望我完成後達到什麼樣的結果？」

## 關卡二：像偵探一樣收集線索

你的任務：根據「關卡一」的表格，執行下面的指令來找線索。

1.  查健康狀況：{"action":"run_script","command":"curl -s http://localhost:3011/api/health"}
2.  看最後的日誌：{"action":"read_file","path":"/tmp/openclaw-server.log"}
3.  查任務執行狀態：{"action":"query_supabase","table":"openclaw_tasks","filters":[{"column":"status","op":"in","value":["in_progress","failed"]}],"limit":5}

## 關卡三：把大象切成小塊 (任務原子化)

你的任務：根據找到的線索，從下面的「問題清單」裡找出最像的，然後複製對應的「行動方案」。

| 如果你發現... | 就複製這個行動方案 (create_task) |
|---|---|
| 日誌裡有 DB connection error | {"action":"create_task","name":"診斷並修復資料庫連線問題","description":"1. 檢查 DB 服務是否運行正常。2. 檢查 .env 中的連線字串是否正確。3. 嘗試重啟 DB 服務。"} |
| api/health 顯示 redis: down | {"action":"create_task","name":"診斷並修復 Redis 連線問題","description":"1. 檢查 Redis 服務是否運行。2. 重啟 Redis 服務。3. 觀察 Server 日誌確認是否恢復連線。"} |
| 有任務 failed | {"action":"create_task","name":"分析任務 [ID] 失敗原因","description":"1. 讀取任務 [ID] 的完整執行日誌。2. 分析失敗的具體錯誤訊息。3. 提出修復方案或重試。"} |

## 關卡四：跟老蔡報告你的計畫

你的任務：在執行 create_task 之前，先用「Cookbook 36」的模板跟老蔡說一聲。複製這個：「【作戰計畫】針對『Server 怪怪的』，我發現 [你找到的線索]，所以我打算建立任務來 [你的行動方案]。您看可以嗎？」
