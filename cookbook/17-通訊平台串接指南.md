## 通用串接架構

所有平台的串接，底層邏輯都一樣：

```
客戶訊息
  │
  ▼
[LINE / FB / IG / TG / WA / Discord / Slack]
  │
  ▼  Webhook POST
┌─────────────────────────────────┐
│         n8n Workflow            │
│                                 │
│  Webhook 節點（接收）            │
│       │                         │
│       ▼                         │
│  Switch 節點（判斷來源平台）      │
│       │                         │
│  ┌────┴────┐                    │
│  ▼         ▼                    │
│ 關鍵字    AI Agent              │
│ 匹配？    （Gemini/OpenAI）      │
│  │         │                    │
│  ▼         ▼                    │
│  直接回覆   生成回覆              │
│       │                         │
│       ▼                         │
│  HTTP Request（回覆客戶）        │
│       │                         │
│       ▼                         │
│  記錄到 Google Sheets/Supabase  │
│       │                         │
│       ▼                         │
│  異常通知老闆（TG/LINE）         │
└─────────────────────────────────┘
```

## 1. LINE Official Account

> **市場**：台灣、日本、泰國最大通訊軟體，台灣滲透率超過 95%
> **詳細設定指南**：請參考 `cookbook/15-LINE-OA設定指南.md` 及 `docs/LINE-BOT-SETUP.md`

### 快速摘要

| 項目 | 內容 |
|------|------|
| 官方後台 | [LINE Official Account Manager](https://manager.line.biz/) |
| 開發者後台 | [LINE Developers Console](https://developers.line.biz/) |
| API | LINE Messaging API |
| n8n 節點 | `n8n-nodes-base.line`（typeVersion: 1） |
| 社群節點 | `@aotoki/n8n-nodes-line-messaging`（功能更完整，verified） |
| 免費額度 | 輕用量：500 則推播/月（回覆訊息不限） |

### 申請流程

1. [LINE Business Center](https://manager.line.biz/) 建立官方帳號
2. 到 [LINE Developers](https://developers.line.biz/) 建立 Provider
3. 在 Provider 下建立 Messaging API Channel
4. 取得：
   - **Channel ID**
   - **Channel Secret**
   - **Channel Access Token**（長期有效）

### Webhook 設定

在 LINE Developers Console → Messaging API → Webhook settings：

```
Webhook URL: https://your-domain.com/webhook/line
```

驗證 Webhook 的回應必須回傳 HTTP 200 OK。

### n8n 串接：回覆訊息

**方法一：使用原生 Line 節點（僅支援 Notify 推播）**

```json
{
  "name": "LINE Notify",
  "type": "n8n-nodes-base.line",
  "typeVersion": 1,
  "parameters": {
    "resource": "notification",
    "operation": "send",
    "message": "={{ $json.replyMessage }}"
  }
}
```

**方法二：使用 HTTP Request 呼叫 Messaging API（推薦，功能完整）**

```json
{
  "name": "LINE Reply",
  "type": "n8n-nodes-base.httpRequest",
  "typeVersion": 4.2,
  "parameters": {
    "method": "POST",
    "url": "https://api.line.me/v2/bot/message/reply",
    "sendHeaders": true,
    "headerParameters": {
      "parameters": [
        {
          "name": "Authorization",
          "value": "Bearer YOUR_CHANNEL_ACCESS_TOKEN"
        }
      ]
    },
    "sendBody": true,
    "bodyParameters": {
      "parameters": [
        {
          "name": "replyToken",
          "value": "={{ $json.body.events[0].replyToken }}"
        },
        {
          "name": "messages",
          "value": "=[{\"type\":\"text\",\"text\":\"感謝您的訊息，我們會盡快回覆！\"}]"
        }
      ]
    },
    "options": {}
  }
}
```

## 2. Facebook Messenger

> **市場**：全球最大社群平台，台灣使用率第二名（僅次於 LINE）
> **適合**：粉專客服、電商自動回覆、行銷推播

### 基本資訊

| 項目 | 內容 |
|------|------|
| 開發者後台 | [Meta for Developers](https://developers.facebook.com/) |
| API | Graph API v19.0+ |
| 免費額度 | 標準訊息無限（24 小時回覆窗口內） |
| n8n 串接方式 | Webhook + HTTP Request |

### 申請流程（完整步驟）

1. **建立 Facebook 粉絲專頁**（如果還沒有）
   - 到 [facebook.com/pages/create](https://www.facebook.com/pages/create)
   - 選擇類型 → 填寫資訊 → 建立

2. **建立 Meta App**
   - 登入 [Meta for Developers](https://developers.facebook.com/)
   - 點擊「My Apps」→「Create App」
   - 選擇「Business」類型
   - 填寫 App 名稱 → 建立

3. **新增 Messenger Product**
   - 在 App Dashboard 左側選單 → Add Products
   - 找到「Messenger」→ 點擊「Set Up」

4. **產生 Page Access Token**
   - Messenger Settings → Token Generation
   - 選擇你的粉絲專頁
   - 點擊「Generate Token」
   - **務必複製並安全儲存這個 Token**

5. **設定 Webhook**
   - Messenger Settings → Webhooks
   - 點擊「Add Callback URL」
   - **Callback URL**：`https://your-domain.com/webhook/messenger`
   - **Verify Token**：自訂一組驗證字串（例如 `my_verify_token_123`）
   - 點擊「Verify and Save」

6. **訂閱事件**
   - 勾選以下 Webhook 事件：
     - `messages` — 接收用戶傳來的訊息
     - `messaging_postbacks` — 接收按鈕點擊事件
     - `messaging_optins` — 接收用戶同意訊息
   - 點擊「Subscribe」

### Webhook 驗證（n8n 需處理）

Meta 會先發一個 GET 請求驗證 Webhook：

```
GET /webhook/messenger?hub.mode=subscribe&hub.verify_token=my_verify_token_123&hub.challenge=CHALLENGE_VALUE
```

n8n Webhook 節點要回傳 `hub.challenge` 值。可以用 Code 節點處理：

```javascript
// n8n Code 節點 — Webhook 驗證
if ($input.item.json.query['hub.mode'] === 'subscribe' &&
    $input.item.json.query['hub.verify_token'] === 'my_verify_token_123') {
  return [{ json: { response: $input.item.json.query['hub.challenge'] } }];
}
return [{ json: $input.item.json }];
```

### 注意事項

- **測試模式**：審核前只能跟 App 管理員/開發者/測試者互動。
- **速率限制**：每個粉絲專頁約 200 則/秒。

## 3. Instagram Direct Message

> **市場**：年輕族群、電商、網紅行銷
> **特點**：透過 Facebook 的 Graph API 串接，跟 Messenger 共用同一個 App

### 基本資訊

| 項目 | 內容 |
|------|------|
| 前提 | 需要 Instagram 商業帳號或創作者帳號 |
| 前提 | 需要連結到 Facebook 粉絲專頁 |
| API | Meta Graph API（跟 Messenger 同端點） |
| 免費額度 | 無限（僅能回覆，不能主動推播） |

### 申請流程

1. **將 Instagram 帳號轉為商業帳號**
   - Instagram App → 設定 → 帳號 → 切換為專業帳號 → 選擇「商業」

2. **連結到 Facebook 粉絲專頁**
   - Instagram 設定 → 帳號 → 已連結的帳號 → Facebook → 選擇粉專

3. **在 Meta for Developers 設定**
   - 同一個 Messenger App（如果已建立）
   - 或建立新 App → 新增「Instagram」Product
   - 設定 Webhook → 訂閱 `instagram_messaging` 事件

4. **產生 Token**
   - 需要以下權限：
     - `instagram_basic`
     - `instagram_manage_messages`
     - `pages_manage_metadata`

### Webhook 事件格式（收到 DM）

```json
{
  "object": "instagram",
  "entry": [
    {
      "id": "INSTAGRAM_USER_ID",

            "mid": "MESSAGE_ID",
            "text": "請問這件還有貨嗎？"
          }

  ]
```

### 常見用法

| 場景 | 做法 |
|------|------|
| 自動回覆商品詢問 | Webhook 收到 → 關鍵字比對 → 回覆商品資訊 |
| 限動提及自動感謝 | 偵測 STORY_MENTION → 自動回覆優惠碼 |
| 回覆留言的用戶 | 留言 Webhook → 自動 DM 用戶 |
| 收集用戶資訊 | 透過 Quick Replies 引導對話流程 |

### 限制與注意

- **不能主動推播**：只能回覆已經發訊息給你的用戶
- **24 小時回覆窗口**：同 Messenger 規則
- **需要 App Review**：審核 `instagram_manage_messages` 權限
- **不支援 Template Message**：只能發純文字和圖片/影片
- **Ice Breakers**：可以設定預設問題讓用戶一鍵發送

### 設定 Ice Breakers（預設問題）

```bash
curl -X POST "https://graph.facebook.com/v19.0/me/messenger_profile" \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer PAGE_ACCESS_TOKEN" \
  -d '{
    "platform": "instagram",
    "ice_breakers": [
      {
        "question": "有什麼優惠活動嗎？",
        "payload": "PROMOTIONS"
      },
      {
        "question": "營業時間是？",
        "payload": "BUSINESS_HOURS"
      },
      {
        "question": "如何退換貨？",
        "payload": "RETURN_POLICY"
      }
    ]
  }'
```

### 基本串接流程

1. **取得 Access Token**（每 2 小時刷新一次）

```bash
curl "https://api.weixin.qq.com/cgi-bin/token?grant_type=client_credential&appid=YOUR_APP_ID&secret=YOUR_APP_SECRET"
```

回應：
```json
{
  "access_token": "ACCESS_TOKEN",
  "expires_in": 7200
}
```

2. **設定伺服器配置**
   - 在微信公眾平台 → 開發 → 基本配置
   - URL：`https://your-china-server.com/wechat/webhook`
   - Token：自訂驗證字串
   - EncodingAESKey：自動生成（用於訊息加密）
   - 訊息加密方式：建議選「安全模式」

3. **接收訊息**（XML 格式）

微信的 Webhook 使用 XML 而非 JSON：

```xml
<xml>
  <ToUserName><![CDATA[公眾號的微信號]]></ToUserName>
  <FromUserName><![CDATA[發送者的 OpenID]]></FromUserName>
  <CreateTime>1710000000</CreateTime>
  <MsgType><![CDATA[text]]></MsgType>
  <Content><![CDATA[你好]]></Content>
  <MsgId>123456789</MsgId>
</xml>
```

4. **回覆訊息**（被動回覆，直接在 Webhook 回應中回傳 XML）

```xml
<xml>
  <ToUserName><![CDATA[發送者的 OpenID]]></ToUserName>
  <FromUserName><![CDATA[公眾號的微信號]]></FromUserName>
  <CreateTime>1710000001</CreateTime>
  <MsgType><![CDATA[text]]></MsgType>
  <Content><![CDATA[您好！感謝關注。]]></Content>
</xml>
```

5. **主動發送客服訊息**

```bash
curl -X POST "https://api.weixin.qq.com/cgi-bin/message/custom/send?access_token=ACCESS_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "touser": "OPENID",
    "msgtype": "text",
    "text": {
      "content": "您好，有什麼可以幫助您的？"
    }
  }'
```

### n8n 串接建議

由於沒有原生節點，用 HTTP Request + Code 節點：

```json
{
  "name": "WeChat Reply",
  "type": "n8n-nodes-base.httpRequest",
  "typeVersion": 4.2,
  "parameters": {
    "method": "POST",
    "url": "=https://api.weixin.qq.com/cgi-bin/message/custom/send?access_token={{ $json.access_token }}",
    "sendBody": true,
    "specifyBody": "json",
    "jsonBody": "={\n  \"touser\": \"{{ $json.FromUserName }}\",\n  \"msgtype\": \"text\",\n  \"text\": {\n    \"content\": \"您好！請問有什麼可以幫助您的？\"\n  }\n}",
    "options": {}
  }
}
```

### Access Token 快取（重要）

Token 每 2 小時過期，需要快取機制避免頻繁請求：

```javascript
// n8n Code 節點 — 檢查並更新 Token
const now = Date.now();
const tokenExpiry = $workflow.staticData.wechat_token_expiry || 0;

if (now > tokenExpiry) {
  // Token 過期，重新取得
  // 呼叫 https://api.weixin.qq.com/cgi-bin/token
  // 存入 $workflow.staticData
  $workflow.staticData.wechat_access_token = newToken;
  $workflow.staticData.wechat_token_expiry = now + (7000 * 1000); // 提前 200 秒更新
}

return [{ json: { access_token: $workflow.staticData.wechat_access_token } }];
```

### 注意事項

- **訊息加密**：安全模式下需要 AES 解密，建議用 Code 節點搭配 crypto 處理
- **48 小時客服窗口**：用戶主動聯繫後 48 小時內可回覆
- **中國法規**：需遵守中國網路安全法、個資法
- **建議方案**：如果不熟悉中國生態，建議用第三方平台（微盟、有贊）或找代辦

### 建立 Slack App

1. 到 [api.slack.com/apps](https://api.slack.com/apps) → Create New App
2. 選擇「From scratch」
3. 填寫 App Name → 選擇 Workspace
4. 左側 → OAuth & Permissions → 新增以下 Bot Token Scopes：
   - `chat:write` — 發送訊息
   - `chat:write.public` — 發送到任何公開頻道
   - `channels:history` — 讀取頻道訊息歷史

   - `im:write` — 發送私訊

5. Install App to Workspace
6. 複製 **Bot User OAuth Token**（`xoxb-` 開頭）

1. 左側 → Event Subscriptions → Enable Events
2. Request URL：`https://your-domain.com/webhook/slack`
3. Subscribe to bot events：
   - `message.channels` — 公開頻道訊息
   - `message.im` — 私訊

4. Save Changes

### 訊息標準化 Code 節點

```javascript
// n8n Code 節點 — 統一各平台訊息格式
const webhookData = $input.item.json;
let standardMessage = {};

// 判斷來源平台
if (webhookData.events && webhookData.events[0]?.source) {
  // LINE
  const event = webhookData.events[0];
  standardMessage = {
    platform: 'line',
    userId: event.source.userId,
    userName: '', // 需另外呼叫 Profile API
    message: event.message?.text || '',
    messageType: event.message?.type || 'unknown',
    timestamp: event.timestamp,
    replyToken: event.replyToken,
    rawData: webhookData
  };
} else if (webhookData.object === 'page') {
  // Facebook Messenger
  const messaging = webhookData.entry[0].messaging[0];
  standardMessage = {
    platform: 'messenger',
    userId: messaging.sender.id,
    userName: '',
    message: messaging.message?.text || '',
    messageType: 'text',
    timestamp: messaging.timestamp,
    rawData: webhookData
  };
} else if (webhookData.object === 'instagram') {
  // Instagram
  const messaging = webhookData.entry[0].messaging[0];
  standardMessage = {
    platform: 'instagram',
    userId: messaging.sender.id,
    userName: '',
    message: messaging.message?.text || '',
    messageType: 'text',
    timestamp: messaging.timestamp,
    rawData: webhookData
  };
} else if (webhookData.message?.chat) {
  // Telegram
  standardMessage = {
    platform: 'telegram',
    userId: String(webhookData.message.chat.id),
    userName: webhookData.message.from?.first_name || '',
    message: webhookData.message.text || '',
    messageType: 'text',
    timestamp: webhookData.message.date * 1000,
    rawData: webhookData
  };
} else if (webhookData.entry?.[0]?.changes?.[0]?.value?.messages) {
  // WhatsApp
  const waMessage = webhookData.entry[0].changes[0].value.messages[0];
  standardMessage = {
    platform: 'whatsapp',
    userId: waMessage.from,
    userName: webhookData.entry[0].changes[0].value.contacts?.[0]?.profile?.name || '',
    message: waMessage.text?.body || '',
    messageType: waMessage.type,
    timestamp: parseInt(waMessage.timestamp) * 1000,
    rawData: webhookData
  };
} else if (webhookData.event?.type === 'message') {
  // Slack
  standardMessage = {
    platform: 'slack',
    userId: webhookData.event.user,
    userName: '',
    message: webhookData.event.text || '',
    messageType: 'text',
    timestamp: parseFloat(webhookData.event.ts) * 1000,
    channelId: webhookData.event.channel,
    rawData: webhookData
  };
}

return [{ json: standardMessage }];
```

## 附錄：常見問題 FAQ

### Q1：客戶只做台灣市場，推薦串哪個？
**A：LINE + Facebook Messenger**。這兩個涵蓋台灣 95% 以上的通訊需求。

### Q2：客戶做跨境電商，推薦串哪個？
**A**：
- 東南亞 → LINE（泰國）+ WhatsApp（其他東南亞）
- 歐美 → WhatsApp + Facebook Messenger
- 日本 → LINE
- 中國 → WeChat（門檻高，建議找代辦）

### Q3：哪個平台串接最快？
**A：Telegram**，從零到完成只需 10 分鐘。Discord Webhook 也很快。

### Q4：哪個平台完全免費？
**A：Telegram 和 Discord** 完全免費無限制。LINE 回覆免費但推播有限額。

### Q5：AI 客服建議用什麼模型？
**A**：
- 預算有限：Gemini 2.5 Flash（Google 免費額度大）
- 品質要求高：GPT-4o 或 Claude Sonnet
- 極致省錢：Gemini 2.0 Flash Lite

### Q6：n8n 和 Make/Zapier 哪個好？
**A**：
- **n8n**：自架免費、不限流程、可客製化，適合接案公司
- **Make**：UI 好看、模板多，適合客戶自己管理
- **Zapier**：最簡單但最貴，適合不想碰技術的客戶
