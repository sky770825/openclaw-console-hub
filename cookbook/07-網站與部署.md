## 四、版本管理

### 目前版本
```bash
grep '"version"' /Users/caijunchang/openclaw任務面版設計/package.json
grep '"version"' /Users/caijunchang/openclaw任務面版設計/server/package.json
```

### 版本號位置（更新版本要同時改這三個地方）
1. `package.json` → `"version"`
2. `server/package.json` → `"version"`
3. `server/src/index.ts` → 版本字串

### 自動遞增版本
```bash
bash /Users/caijunchang/openclaw任務面版設計/scripts/auto-bump-version.sh
```

### 版本規則
- patch +1 = 每天 00:01 自動遞增
- minor +1 = 大功能上線
- major +1 = 重大架構變更

### 🔴 定期版本推送 SOP（小蔡必會）

**為什麼要推？** 改動只留在本地，萬一 server 出事或要回滾就找不回來。

**什麼時候推？**
- 累積 3 個以上有意義的改動就該推一版
- 完成一個重大功能後立刻推
- 每天至少檢查一次是否有未推的改動

**完整步驟（照做就對）：**

```bash
# === 第 0 步：檢查有沒有需要推的東西 ===
cd /Users/caijunchang/openclaw任務面版設計
git status --short
# 如果沒有修改檔案 → 不需要推，結束

# === 第 1 步：同步老蔡最新 ===
git pull origin main
# 如果有衝突 → 先解衝突再繼續

# === 第 2 步：確認版本號 ===
grep '"version"' package.json
# 記下目前版本 X.Y.Z，新版本是 X.Y.(Z+1)

# === 第 3 步：更新三個地方的版本號 ===
# 用 edit 工具或 sed 修改以下三個檔案：
#   package.json            → "version": "X.Y.Z+1"
#   server/package.json     → "version": "X.Y.Z+1"
#   server/src/index.ts     → version: 'X.Y.Z+1'

# === 第 4 步：Build 確認沒壞 ===
cd server && npm run build
# 如果 build 失敗 → 修好再繼續，不要帶著錯誤推

# === 第 5 步：Commit ===
cd /Users/caijunchang/openclaw任務面版設計
git add <改動的檔案>   # 不要用 git add .，只加需要的
git commit -m "feat: vX.Y.Z+1 簡短描述改了什麼"

# === 第 6 步：Push（這步要謹慎） ===
git push origin main

# === 第 7 步：重啟 server 讓新版生效 ===
launchctl stop com.openclaw.taskboard && sleep 2 && launchctl start com.openclaw.taskboard

# === 第 8 步：驗證 ===
sleep 3 && curl -s http://localhost:3011/api/health | grep version
# 確認回傳的 version 是新版號

# === 第 9 步：通知老蔡 ===
bash /Users/caijunchang/openclaw任務面版設計/scripts/notify-laocai.sh "版本更新 vX.Y.Z+1" "done" "改動摘要"
```

**注意事項：**
- **不要推垃圾**：只推有實際改動的檔案，測試檔、臨時腳本不推
- **不要推密鑰**：.env、openclaw.json、任何含 API key 的檔案絕對不推
- **Build 失敗就停**：帶著錯誤推上去 = 毀掉 server
- **commit message 要有意義**：寫清楚改了什麼，方便回滾時找版本
