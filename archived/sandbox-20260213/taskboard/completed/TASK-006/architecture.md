# 市場需求監控系統架構

## 系統概述

本監控系統針對老蔡的三項業務進行市場需求趨勢監控，提供即時數據分析與趨勢預警。

## 監控業務範圍

| 業務 | 產業類別 | 主要監控面向 |
|------|----------|--------------|
| 住商不動產 | 房地產仲介 | 房價趨勢、交易量、區域熱度 |
| 飲料店 | 餐飲零售 | 品牌聲量、新品趨勢、消費者偏好 |
| 普特斯防霾紗窗 | 建材/空氣品質 | 空汙議題、競品動態、安裝需求 |

## 技術棧

| 層級 | 技術選型 |
|------|----------|
| 數據收集 | Python + Requests/Scrapy |
| 數據儲存 | JSON / SQLite |
| 數據處理 | Python + Pandas |
| 情感分析 | Jieba 分詞 + 自訂詞典 |
| 視覺化 | Python + Matplotlib/Plotly |
| 排程 | Cron / Python Schedule |

## 執行頻率

| 任務 | 頻率 | 說明 |
|------|------|------|
| Google Trends 抓取 | 每日 | 自動化趨勢數據 |
| 社群爬蟲 | 每 6 小時 | PTT/論壇話題監控 |
| 新聞監控 | 每日 | 產業相關新聞 |
| 趨勢分析 | 每週 | 產生週報 |
| 完整報告 | 每月 | 產生月報 |

## 警示機制

### 觸發條件

1. **搜尋量異常**：關鍵字搜尋量單日波動超過 ±30%
2. **負面聲量**：負面討論佔比超過 40%
3. **競品動態**：競品新品發布或促銷活動
4. **產業熱點**：相關產業話題進入熱門排行

### 警示級別

| 級別 | 顏色 | 觸發條件 | 通知方式 |
|------|------|----------|----------|
| INFO | 🟦 藍 | 日常數據更新 | 不通知 |
| WARNING | 🟨 黃 | 搜尋量波動 20-30% | 報告標註 |
| ALERT | 🟥 紅 | 搜尋量波動 >30% 或負面聲量高 | 立即通知 |
