# Taskboard 規則

## 核心原則

1. **任務處理效率**
   - 1分鐘1個任務（每個任務估計執行時間約1分鐘）
   - 支援並行執行，提高整體吞吐量
   - 1小時理論處理能力：60 個任務

2. **模型選擇策略**
   - **優先使用免費模型**：Ollama 本地模型為首要選擇
   - 僅在以下情況使用付費模型：
     - 任務需要高階推理能力
     - 本地模型無法勝任的複雜分析
     - 涉及重大決策的評估

3. **任務生命週期**

   ```
   pending/ → running/ → completed/ (成功)
                      ↘ failed/    (失敗)
   ```

4. **並行執行規則**
   - 獨立任務可同時執行
   - 相互依賴任務需按順序執行
   - 同一任務不可重複執行（避免資源衝突）

5. **任務檔案格式**
   - 每個任務為獨立 Markdown 檔案
   - 檔名格式：`{序號}-{任務名稱}.md`
   - 必需欄位：
     - `title`: 任務標題
     - `priority`: 優先級 (high/medium/low)
     - `status`: 狀態 (pending/running/completed/failed)
     - `model`: 建議使用的模型
     - `description`: 任務描述
     - `created_at`: 建立時間
     - `started_at`: 開始時間（執行時填寫）
     - `completed_at`: 完成時間（完成時填寫）

6. **監控與回報**
   - 執行中的任務需記錄於 `running/` 目錄
   - 完成後移動至 `completed/` 或 `failed/`
   - 定期回報處理進度給主會話

7. **錯誤處理**
   - 任務失敗時記錄錯誤原因
   - 可重試任務標記 `retryable: true`
   - 最多重試 3 次

8. **成本優化**
   - 優先使用本地 Ollama 模型
   - 批量處理相似任務以減少切換開銷
   - 定期檢視任務完成率與成本效率
