# 任務板自我檢查機制 (Self-Check Mechanism)

## 1. 專案進度檢查清單

### 檢查維度
```
進行中 (IN_PROGRESS)
├─ 確認所有人員已分配
├─ 檢查 taskboard/running/ 內的任務
├─ 驗證預計完成時間
└─ 檢查是否有阻塞問題

卡住 (BLOCKED)
├─ 原因分類：資訊缺失、技能缺陷、外部依賴、流程斷點
├─ 記錄在 BLOCKED_reason.txt
├─ 自動生成「解除阻塞」子任務
└─ 建議升級或轉移任務

沒人做 (UNASSIGNED)
├─ 優先級分析
├─ 檢查是否缺乏明確需求規格
├─ 檢查是否缺乏所需技能
└─ 自動分配或標記為等待
```

### 檢查執行流程
```bash
# 自檢步驟
1. 掃描 taskboard/running/
2. 掃描 taskboard/pending/
3. 掃描 taskboard/done/
4. 統計各狀態任務數
5. 識別卡住的任務
6. 識別未分配的任務
```

## 2. 缺口識別規則 (Gap Identification Rules)

### 缺口類型

#### A. 資訊不足 (INFO_GAP)
**特徵**
- 任務描述不完整（缺少背景、目標、成功標準）
- 輸入資料缺失
- 依賴項不清楚

**識別方式**
```
檢查清單：
□ 任務目標明確嗎？（SMART 原則）
□ 所需輸入/資源已備妥嗎？
□ 成功標準有定義嗎？
□ 依賴項已列舉嗎？
```

**解決規則**
→ 生成「資訊蒐集」子任務
→ 自動嘗試找資料（搜尋、爬蟲、查詢現有檔案）
→ 補充任務規格

#### B. 技能缺陷 (SKILL_GAP)
**特徵**
- 指定人員不具備所需技能
- 無人具備該領域專業知識
- 需要培訓或外包

**識別方式**
```
檢查清單：
□ 任務需要哪些技能？
□ 團隊中有人具備嗎？
□ 技能學習週期多長？
□ 是否應外包？
```

**解決規則**
→ 生成「技能培訓」或「外包」子任務
→ 調整優先級（可能延後或轉移）
→ 標記為「技能發展機會」

#### C. 流程斷點 (PROCESS_GAP)
**特徵**
- 任務間銜接不清
- 工作流程有漏洞
- 交接標準不明確

**識別方式**
```
檢查清單：
□ 前置任務完成了嗎？
□ 輸出如何流向下一步？
□ 品質驗收標準有嗎？
□ 異常情況如何處理？
```

**解決規則**
→ 生成「流程設計」任務
→ 定義清晰的輸入/輸出介面
→ 建立驗收標準

#### D. 依賴缺失 (DEPENDENCY_GAP)
**特徵**
- 外部資源無法取得
- 時間依賴無法滿足
- 人員或工具缺乏

**識別方式**
```
檢查清單：
□ 所有依賴都已分配嗎？
□ 依賴的完成時間現實嗎？
□ 有備用方案嗎？
□ 風險評估完成了嗎？
```

**解決規則**
→ 生成「依賴管理」任務
→ 建立風險應對計畫
→ 調整排程或追蹤進度

## 3. 審核機制 (Quality Assurance for Completed Tasks)

### 完成任務品質檢查

#### 檢查項目
```
功能完整性 (COMPLETENESS)
├─ 所有需求點都實現了嗎？
├─ 邊界情況處理了嗎？
├─ 文檔齊全嗎？
└─ 與相關模塊集成了嗎？

品質標準 (QUALITY)
├─ 程式碼/內容品質達標嗎？
├─ 測試涵蓋率足夠嗎？
├─ 性能/資源使用合理嗎？
└─ 安全性檢查通過了嗎？

可維護性 (MAINTAINABILITY)
├─ 代碼可讀性好嗎？
├─ 文檔說明充分嗎？
├─ 後續修改容易嗎？
└─ 知識轉移完成了嗎？

驗收 (ACCEPTANCE)
├─ 利益相關者簽核了嗎？
├─ 是否滿足成功標準？
├─ 有隱患或技術債嗎？
└─ 何時需要後續優化？
```

#### 審核流程
```
1. 自我檢查（任務執行者）
   ↓
2. 同行評審（團隊成員）
   ↓
3. 品質驗收（指定人員）
   ↓
4. 存檔並標記為 DONE
   ↓
5. 記錄經驗教訓
```

#### 品質等級
```
✅ PASS     - 符合所有標準
⚠️  PASS_WITH_NOTES - 通過但有改進空間
🔄 REWORK   - 需要改進後重新提交
❌ FAIL     - 不符合標準
```

## 4. 自檢執行規則

### 觸發時機
- ✅ 每次新對話開始
- ✅ 每天定時（如果有持續運行）
- ✅ 新增任務時
- ✅ 任務狀態變更時

### 自檢報告格式 (self-check-report.json)
```json
{
  "timestamp": "2026-02-13T07:58:00Z",
  "session_id": "agent:main:subagent:xxx",
  "summary": {
    "total_tasks": 0,
    "in_progress": 0,
    "blocked": 0,
    "unassigned": 0,
    "done": 0
  },
  "gaps_found": [
    {
      "gap_id": "GAP-001",
      "type": "INFO_GAP",
      "task_id": "TASK-001",
      "description": "...",
      "severity": "HIGH|MEDIUM|LOW",
      "suggested_action": "..."
    }
  ],
  "actions_required": [
    "spawn_subagent_for_info_collection",
    "escalate_blocked_task",
    "assign_pending_task"
  ]
}
```

### 自檢健康度指標
```
🟢 HEALTHY
- 所有任務已分配
- 無重大缺口
- 進度正常

🟡 WARNING
- 有未分配任務
- 存在中等缺口
- 進度略後

🔴 CRITICAL
- 多個任務被阻塞
- 重大信息缺失
- 無人能執行關鍵任務
```
